## Какая это мышь с точки зрения программы

Физическая реализация мыши в разных конфигурациях может отличаться, но
программная модель у Sprinter одна:

- внутри системы это **serial mouse**;
- протокол совместим с **Microsoft Mouse**;
- поверх него BIOS/DSS дают единый внутренний интерфейс.

Поэтому в документации для программиста важны не разъёмы и не конкретный
контроллер, а два уровня доступа:

1. **BIOS Mouse API** через `RST #30`;
2. **совместимый Kempston Mouse интерфейс** для прямого опроса.

> **Практическое правило:** для нативного приложения почти всегда лучше брать
> `RST #30`. Kempston-совместимый доступ полезен для ZX-совместимого кода и
> очень простых циклов опроса.

## BIOS Mouse API (`RST #30`)

Основной программный интерфейс мыши в Sprinter.

### Команды `#00`-`#0F`

| `C` | Команда | Назначение |
|-----|---------|------------|
| `#00` | `INITIALIZATION` | инициализация, проверка наличия |
| `#01` | `SHOW MOUSE CURSOR` | показать курсор |
| `#02` | `HIDE MOUSE CURSOR` | скрыть курсор |
| `#03` | `READ MOUSE STATE` | прочитать координаты и кнопки |
| `#04` | `GOTO MOUSE CURSOR` | установить позицию курсора |
| `#07` | `SET VERT. BOUNDS` | ограничить диапазон `Y` |
| `#08` | `SET HORZ. BOUNDS` | ограничить диапазон `X` |
| `#09` | `LOAD CURSOR` | загрузить графический курсор |
| `#0A` | `SET CURSOR IN TEXT MODES` | маска курсора для текстовых режимов |
| `#0B` | `RETURN CURSOR` | вернуть текущее описание курсора |
| `#0E` | `GET SENSETIVE` | получить чувствительность |
| `#0F` | `SET SENSETIVE` | установить чувствительность |

### Команды `#80`-`#83`

| `C` | Команда | Назначение |
|-----|---------|------------|
| `#80` | `MOUSE HARDWARE INTERRUPT` | внутренний обработчик аппаратного IRQ |
| `#81` | `CHANGE VIDEO MODE` | перенастройка диапазонов под видеорежим |
| `#83` | `MOUSE REFRESH` | обновление курсора и состояния |

Коды ошибок драйвера:

| `A` | Значение |
|-----|----------|
| `0` | нет ошибки |
| `1` | команда не поддержана |
| `2` | устройство отсутствует |
| `3` | курсор уже включён |
| `4` | курсор уже выключен |
| `5` | изображение курсора слишком большое |

## Базовые вызовы

### Инициализация

```asm
        ld      c, #00
        rst     #30
        jr      c, .no_mouse
```

### Показать и скрыть курсор

```asm
        ld      c, #01
        rst     #30

        ld      c, #02
        rst     #30
```

### Прочитать состояние

```asm
        ld      c, #03
        rst     #30
        ; HL = X
        ; DE = Y
        ; A  = кнопки: bit2=middle, bit1=right, bit0=left
```

### Установить позицию

```asm
        ld      hl, 160
        ld      de, 100
        ld      c, #04
        rst     #30
```

## Координаты и диапазоны

После команды `#81` драйвер сам подстраивает рабочее поле под видеорежим.
Подтверждённые диапазоны из текущего драйвера:

| Режим | `X` | `Y` |
|------|-----|-----|
| `320x256` | `0..319` | `0..255` |
| `640x256` | `0..639` | `0..255` |

Это важная поправка: в старых пересказах иногда встречается `Y=0..511`, но
текущий исходник драйвера для этих режимов ограничивает `Y` именно диапазоном
`0..255`.

### Ограничение области движения

```asm
; Y: 16..175
        ld      hl, 16
        ld      de, 175
        ld      c, #07
        rst     #30

; X: 32..287
        ld      hl, 32
        ld      de, 287
        ld      c, #08
        rst     #30
```

### Чувствительность

```asm
; Прочитать
        ld      c, #0E
        rst     #30
        ; H = vertical sensitivity
        ; L = horizontal sensitivity

; Установить
        ld      hl, #0404
        ld      c, #0F
        rst     #30
```

## Курсор

### Графический курсор

Команда `#09` загружает описание курсора:

- `IX` — адрес данных изображения;
- `H` — высота;
- `L` — ширина;
- `D` — `hot spot Y`;
- `E` — `hot spot X`;
- `B = 0`.

### Текстовый курсор

Команда `#0A` задаёт маски наложения для текстовых режимов:

- `H` — AND mask для символа;
- `L` — XOR mask для символа;
- `D` — AND mask для атрибута;
- `E` — XOR mask для атрибута;
- `B = 0`.

### Получить текущее описание

```asm
        ld      ix, CursorBuf
        ld      c, #0B
        rst     #30
        ; H/L = size, D/E = hot spot

CursorBuf:
        ds      256
```

## `MOUSE REFRESH` и игровой цикл

Команда `#83` нужна для обслуживания мыши и курсора из главного цикла или
периодического обработчика.

```asm
MainLoop:
        ld      c, #83              ; refresh
        rst     #30

        ld      c, #03              ; read state
        rst     #30
        ; HL, DE, A = текущее состояние

        halt
        jr      MainLoop
```

Для GUI и игр обычно удобно делать опрос мыши раз в кадр.

## Kempston Mouse

DCP оставляет совместимый прямой интерфейс для Kempston Mouse:

| Порт | Назначение |
|------|------------|
| `#58` | устройство мыши (`TYPE=5` в DCP) |

Для совместимого кода используются разные значения старших адресных линий,
поэтому встречаются обращения вида `xx58`. Это именно один и тот же
декодированный интерфейс, а не три независимых устройства.

### Что важно помнить

1. Kempston-доступ полезен для ZX-совместимого ПО.
2. На уровне нативного приложения он хуже BIOS API:
   - нет готовой нормализации под видеорежим;
   - нет управления курсором;
   - нет границ, чувствительности и обслуживания текстового курсора.
3. Если нужна переносимость между ревизиями прошивки и разной периферией,
   используйте `RST #30`.

## Низкоуровневое чтение serial mouse

Если нужен не BIOS API, а сырой поток пакетов мышиного контроллера, в
драйверах DSS используются порты:

| Порт | Имя | Назначение |
|------|-----|------------|
| `#1B` | `CMOUSE` | статус / управляющий порт |
| `#1A` | `DMOUSE` | байт данных |

Подтверждённый паттерн из `INTMOUSE.ASM` такой:

- состояние читается из `CMOUSE`;
- после `RRCA` флаг `C=1` означает, что байт можно забирать;
- пакет набирается чтением трёх байтов из `DMOUSE`.

Это уровень драйвера. Для обычной программы безопаснее `RST #30`, но для
низкоуровневой отладки и своих polling-циклов прямое чтение иногда удобнее.

### Пример: прочитать один сырой пакет

```asm
CMOUSE  EQU #1B
DMOUSE  EQU #1A

; выход:
;   CY=1  пакет прочитан
;   B,C,D = три сырых байта
;   CY=0  данных пока нет
ReadMouseRaw3:
        in      a, (CMOUSE)
        rrca
        ret     nc

        in      a, (DMOUSE)
        ld      b, a

.wait2: in      a, (CMOUSE)
        rrca
        jr      nc, .wait2
        in      a, (DMOUSE)
        ld      c, a

.wait3: in      a, (CMOUSE)
        rrca
        jr      nc, .wait3
        in      a, (DMOUSE)
        ld      d, a

        scf
        ret
```

В этом виде `B/C/D` содержат ещё не распакованный Microsoft Mouse-совместимый
трёхбайтовый пакет.

### Пример: распаковать `dx`, `dy` и кнопки

Ниже упрощённый перенос логики `READ_M` из DSS: он собирает относительные
смещения и выделяет кнопки.

```asm
CMOUSE  EQU #1B
DMOUSE  EQU #1A

; выход:
;   CY=1  пакет получен
;   E = dx (signed 8-bit)
;   D = dy (signed 8-bit)
;   A = buttons: bit1=right, bit0=left
;   CY=0  данных нет или пакет неполный
ReadMouseDelta:
        in      a, (CMOUSE)
        rrca
        ret     nc
        in      a, (DMOUSE)
        ld      l, a
        bit     6, a
        ccf
        ret     z

.wait2: in      a, (CMOUSE)
        rrca
        jr      nc, .wait2
        in      a, (DMOUSE)
        ld      e, a
        bit     6, a
        ccf
        ret     nz

.wait3: in      a, (CMOUSE)
        rrca
        jr      nc, .wait3
        in      a, (DMOUSE)
        ld      d, a
        bit     6, a
        ccf
        ret     nz

        ld      a, e
        and     #3F
        ld      e, a
        ld      a, l
        and     #03
        rrca
        rrca
        or      e
        ld      e, a

        ld      a, d
        and     #3F
        ld      d, a
        ld      a, l
        and     #0C
        rrca
        rrca
        rrca
        rrca
        or      d
        ld      d, a

        ld      a, l
        rlca
        rlca
        rlca
        res     6, a
        jr      nc, .btn
        set     6, a
.btn:   rlca
        rlca
        and     #03
        scf
        ret
```

Практически это означает:

1. `E` и `D` можно сразу трактовать как signed relative delta.
2. `A & 1` = левая кнопка, `A & 2` = правая.
3. Абсолютный курсор поверх этого строится накоплением `dx/dy` в собственных
   переменных с последующим clamp по рабочим границам.

## Рабочий пример опроса мыши

Ниже простой цикл, который корректно отслеживает новые нажатия кнопок.

```asm
Bios.Mouse_Refresh    EQU #83
Bios.Mouse_Read       EQU #03

MouseLoop:
        ld      c, Bios.Mouse_Refresh
        rst     #30

        ld      c, Bios.Mouse_Read
        rst     #30

        ld      (mouse_x), hl
        ld      (mouse_y), de

        ld      b, a                ; current buttons
        ld      a, (prev_buttons)
        cpl
        and     b                   ; newly pressed = current & ~previous
        bit     0, a
        call    nz, OnLeftPress
        bit     1, a
        call    nz, OnRightPress

        ld      a, b
        ld      (prev_buttons), a

        halt
        jr      MouseLoop

OnLeftPress:
        ret

OnRightPress:
        ret

prev_buttons:  db 0
mouse_x:       dw 0
mouse_y:       dw 0
```

Этот вариант исправляет типичную ошибку старых примеров, где кнопки случайно
смешиваются с регистрами координат.

## Примечание про текстовые режимы

В старой документации для текстовых режимов координаты мыши часто описывались
как "точечные". Практически это означает:

- драйвер хранит координаты в пикселях;
- для перехода к знакоместам в текстовом режиме их обычно нужно делить на 8.

Если интерфейс работает в сетке `80x32` или `64x24`, сравнивайте не `HL/DE`
напрямую с колонкой и строкой, а `X/8` и `Y/8`.

---

## Практические замечания

1. Если ваша программа сама переключает видеорежим, после этого полезно вызвать
   `C=#81`, чтобы драйвер пересчитал рабочие границы.
2. Не полагайтесь на физический тип мыши в коде: на уровне архитектуры для
   программы это Microsoft-compatible serial mouse.
3. Колесо мыши стандартным API не документировано и переносимо использовать его
   нельзя.

---

## Ключевые моменты

> - Архитектурно мышь Sprinter — это **внутренний serial mouse,
>   совместимый с Microsoft Mouse**.
> - Для нативного ПО используйте **BIOS Mouse API через `RST #30`**.
> - Поддержаны не только `init/show/hide/read`, но и границы, курсор,
>   чувствительность, refresh и реакция на смену видеорежима.
> - Для режимов `320x256` и `640x256` подтверждён диапазон `Y = 0..255`.
> - Kempston-совместимый доступ остаётся для совместимости, но является более
>   низкоуровневым и менее удобным интерфейсом.
