# Управление Flash/BIOS (ROM)

## Физическая организация

ROM-чип подключён к линиям `RA[17:14]` (4 адресных бита сверху) и `A[13:0]` (14 бит адреса внутри страницы). Итого 18 бит → 256 КБ максимальный объём Flash, разбитый на 16 страниц по 16 КБ.

Сигнал `/OE` Flash управляется через `CS_ROM` (активен когда `/SYS=0` и кэш не включён). Сигнал `/WE` — через `XACS` (активен при `ROM_RG4=1`, запись в `0x0000–0x3FFF`).

## Раскладка страниц

Согласно `DCP.MIF` (диапазон `0xE0–0xEF`) и логике `DCP.TDF` (`RA[]`):

| Страница `RA[17:14]` | Адрес в Flash | Содержимое |
|---------------------|--------------|-----------|
| `0` (`0000b`) | `0x00000–0x03FFF` | System ROM (ROM0) — основной BIOS |
| `1` (`0001b`) | `0x04000–0x07FFF` | System ROM (ROM1) — альт. BIOS |
| `2` (`0010b`) | `0x08000–0x0BFFF` | Basic 128K ROM |
| `3` (`0011b`) | `0x0C000–0x0FFFF` | (резерв) |
| `4` (`0100b`) | `0x10000–0x13FFF` | Кэш-окно 0 |
| `5` (`0101b`) | `0x14000–0x17FFF` | DOS / TR-DOS ROM |
| `6` (`0110b`) | `0x18000–0x1BFFF` | Basic 48K ROM |
| `7–15` | ... | Пользовательские страницы |

Точное распределение зависит от прошитой версии BIOS. Строки `DCP.MIF:0xE0–0xEF`:
```
E0: 2041 2041 2041 2041 2041 2041 2041 2041
    2000 2005 2002 2041 20FF 2000 2000 2041
```
- Индекс `0xE8` → PAGE=`0x000` → Basic 48K (страница 0 Flash)
- Индекс `0xE9` → PAGE=`0x005` → DOS (страница 5 Flash)
- Индекс `0xEA` → PAGE=`0x002` → Basic 128K (страница 2 Flash)
- Индекс `0xEC` → PAGE=`0x0FF` → System ROM special flags

## Регистры управления

### ROM_RG[7:0] — выбор страницы и режим записи

Порт `0x8F` (всегда). Зеркало `0x5C` (только при `/SYS=0`):

```vhdl
ROM_RG[].ena = DFF((DCPP == 0x8F), CLK42) or
               (!/SYS & DFF((A[7:0] == 01011100b), CLK42))
```

| Бит | Назначение |
|-----|-----------|
| `[3:0]` | `ROM_RG[3:0]` — номер страницы Flash (0–15 → 16 КБ × 16 = 256 КБ) |
| `4` | `ROM_RG4=1` — режим записи Flash (`XACS` активируется при `/WR` в `0x0000–0x3FFF`) |
| `[7:5]` | Зарезервированы |

### /SYS — подключение ROM к `0x0000–0x3FFF`

```vhdl
/SYS = DFFE(!A6, /IOWR, /RESET, , SYS_ENA2)
```

Декодируется по маске `A[7:0] == 0b0X111100`:
- `OUT (0x3C), 0` → A6=0 → `/SYS=0` → ROM активен
- `OUT (0x7C), 0` → A6=1 → `/SYS=1` → RAM активен (ROM отключён)

### SYS_PG — выбор ROM0/ROM1

```vhdl
SYS_PG = DFFE(D0, /IOWR, /RESET, !ROM_RG4, (SYS_ENA2 & !D1))
```

При записи `OUT (0x3C), D`:
- `D1=0, D0=0` → `SYS_PG=0` → ROM0 (страница 0)
- `D1=0, D0=1` → `SYS_PG=1` → ROM1 (страница 1)
- `D1=1` → `SYS_PG` не меняется

### XACS — строб записи Flash

```vhdl
XACS.clk = !(/MR or /WR)
XACS.d   = !ROM_RG4 or A14 or A15
XACS.prn = XACS or (!/MR & ROM_RG4)
```

`XACS=0` (активен) при одновременном выполнении:
1. `ROM_RG4=1` (режим записи разрешён)
2. Адрес в `0x0000–0x3FFF` (A14=0 и A15=0)
3. Идёт цикл записи (`/WR` активен)

## Алгоритм чтения страницы Flash

```asm
; Подключить ROM к 0x0000-0x3FFF
LD A, 0
OUT (0x3C), A       ; /SYS=0, SYS_PG=0 (ROM0)

; Выбрать страницу через ROM_RG
LD A, 5             ; страница 5 = DOS ROM
OUT (0x8F), A       ; ROM_RG[3:0]=5, ROM_RG4=0

; Читать данные
LD HL, 0x0000
LD B, 0
loop:
    LD A, (HL)      ; читается из Flash, страница 5
    ...
    INC HL
    DJNZ loop

; Восстановить RAM
LD A, 1 << 6        ; A6=1
OUT (0x7C), A       ; /SYS=1
```

## Алгоритм записи (программирование Flash)

**Важно:** последовательность программирования зависит от конкретного чипа Flash. Типичная схема для AMD/SST/Atmel (алгоритм AAA/byte program):

```asm
; 1. Подключить ROM, включить режим записи
LD A, 0
OUT (0x3C), A       ; /SYS=0 (ROM в 0x0000-0x3FFF)

LD A, страница | 0x10  ; ROM_RG4=1 + номер страницы
OUT (0x8F), A

; 2. Flash Unlock-последовательность (например, для SST39SF):
LD HL, 0x5555       ; смещение AAA в Flash = 0x555 → CPU addr 0x0AAA
LD (0x0AAA), 0xAA   ; XACS активируется → /WE на чип
LD (0x0555), 0x55
LD (0x0AAA), 0xA0   ; byte program command

; 3. Записать байт
LD HL, целевой_адрес  ; в пределах 0x0000-0x3FFF
LD (HL), байт

; 4. Ждать завершения (Data Polling)
poll:
    LD A, (HL)
    XOR байт
    AND 0x80         ; DQ7 = инверсия записанного бита 7 во время программирования
    JR NZ, poll

; 5. Отключить режим записи
LD A, страница      ; ROM_RG4=0
OUT (0x8F), A

; 6. Вернуть RAM
LD A, 1 << 6
OUT (0x7C), A
```

### Стирание сектора (Sector Erase)

```asm
; Unlock
LD (0x0AAA), 0xAA
LD (0x0555), 0x55
LD (0x0AAA), 0x80
; Second unlock
LD (0x0AAA), 0xAA
LD (0x0555), 0x55
; Sector erase command
LD (адрес_в_секторе), 0x30

; Ждать DQ7=1
poll_erase:
    LD A, (адрес_в_секторе)
    AND 0x80
    JR Z, poll_erase
```

Стирание всей микросхемы: вместо `0x30` на последнем шаге записать `0x10` по адресу `0x5555`.

## Адресация страниц через RA[17:14]

Формула из `SP2_ACEX.TDF` (`PRE_ROM=0, PRE_CASH=1`):

```
ISA_A[3:0] = (ROM_RG3 xor !SYS_PG, ROM_RG[2:0])
RA[17:14]  = ISA_A[3:0]
```

То есть при `SYS_PG=0`: `RA = ROM_RG[3:0]` (прямо).
При `SYS_PG=1`: `RA3 = ROM_RG3 xor 1` (инверсия старшего бита).

Пример: `ROM_RG=0, SYS_PG=1` → `RA = 0b1000` → страница 8 Flash.
Это сделано для разделения двух «системных» ROM без перепрограммирования `ROM_RG`.

## Адресация Flash при кэше (PRE_ROM=1, PRE_CASH=0)

```
ISA_A[3:0] = (01, ROM_RG[1:0])
RA[17:14]  = (0, 1, ROM_RG[1:0])
```

Кэш-страницы → `RA = 0100b–0111b` → Flash адреса `0x10000–0x1FFFF` (64 КБ, страницы 4–7). Программировать их можно тем же способом, предварительно переключив `CASH_ON=1` вместо `/SYS=0`.

## Защита от случайной записи

1. `ROM_RG4=0` по умолчанию (после `/RESET`) — запись невозможна.
2. `XACS` активируется только при `ROM_RG4=1` И адресе в `0x0000–0x3FFF` И `/WR`.
3. Страницы `0x4000–0xFFFF` не дают `XACS` — записать за пределами нижних 16 КБ нельзя.
4. Порт `0x5C` (зеркало `ROM_RG`) доступен **только** при `/SYS=0`: случайная запись при включённом RAM невозможна.

## Типичные сценарии использования

| Задача | Действия |
|--------|---------|
| Переключить на Basic 48K | `OUT (0x3C), 0` + `OUT (0x8F), 6` |
| Переключить на DOS | `OUT (0x3C), 0` + `OUT (0x8F), 5` |
| Переключить на Basic 128K | `OUT (0x3C), 0` + `OUT (0x8F), 2` |
| Обновить BIOS страницу 0 | Стереть сектор → записать байты (алгоритм выше) |
| Читать страницу пользователя N | `OUT (0x3C), 0` + `OUT (0x8F), N` |
| Вернуть RAM | `OUT (0x7C), 0` |
