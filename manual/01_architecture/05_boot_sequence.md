# 1.5 Последовательность загрузки

> **Навигация:** [← 1.4 Карта портов](04_port_map.md) | [Оглавление](../README.md) | [2.1 Окна памяти →](../02_memory/01_windows.md)

---

## Обзор

Для архитектуры Sprinter полезно разделять три крупные стадии:

1. **аппаратный cold start** — сброс, минимальная конфигурация логики и загрузка ACEX;
2. **основной BIOS** — переход к штатному маппингу, инициализация устройств, выбор стартового носителя;
3. **загрузчик и DSS** — чтение системного сектора, разворачивание DSS, запуск программы.

В этом разделе рассматривается именно архитектурная картина: кто и на каком
этапе получает управление. Низкоуровневые детали BIOS и DSS вынесены в
профильные разделы.

```
Сброс (RESET)
    │
    ▼
1. Аппаратный старт FPGA/CPLD
    │
    ▼
2. BIOS в ROM
    │  bootstrap, CMOS, POST, autodetect, выбор устройства
    ▼
3. Передача управления загрузчику
```

Полная цепочка DSS описана в разделе [4.1 Обзор DSS](../04_dss/01_dss_overview.md).

## Полная картина

Если смотреть на весь путь целиком, загрузка Sprinter проходит четыре фазы:

```
Сброс (RESET)
    │
    ▼
1. Аппаратный сброс FPGA (ACEX + MAX7000)
    │  минимальное стартовое состояние логики
    ▼
2. BIOS (ROM страницы 0–7, адрес 0x0000)
    │  bootstrap + инициализация, SETUP, автоопределение HDD
    ▼
3. Загрузчик с HDD / FDD / ROM
    │  Читает загрузочный сектор / EXE-файл
    ▼
4. DSS (Disk SubSystem)
    │  Загружается в защищённые страницы RAM
    ▼
   Запуск пользовательской программы (EXE)
```

---

## Фаза 1: Аппаратный cold start

При подаче питания или сигнале `RESET` система входит не сразу в обычный BIOS,
а в специальный стартовый режим, достаточный только для конфигурирования логики
и запуска Z80-кода из ROM.

Для `SP2000+` cold-start loader расположен в полном ROM-образе по смещению
`0x30000`, то есть в **16-КБ ROM-странице 12**.

Согласно `ARHITECT.TXT`, `INFO_013.TXT` и более поздним BIOS-исходникам,
при cold reset действует следующая схема:

- после `RESET` процессор видит только одну страницу ПЗУ и одну страницу
  cache SRAM;
- любая запись в память в этом режиме уходит не в обычное ОЗУ, а в тракт
  конфигурирования ПЛИС;
- стартовая программа в ПЗУ загружает конфигурацию либо из ROM, либо из cache
  SRAM, если там выставлен флаг `ACEX_30K_LOADING`.

В терминах архитектуры это выглядит так:

1. MAX7000 и ACEX входят в начальное состояние после аппаратного сброса.
2. Z80 получает минимальный ROM/RAM-мэппинг.
3. Cold-start loader программно подаёт конфигурационные данные в тракт загрузки ACEX.
   В `SP2_MAX.TDF` эта стадия привязана к линиям `10K_CLK`, `10K_D0` и
   `/CONF_X`, которыми MAX7000 управляет при конфигурировании ACEX.
4. После успешной загрузки становятся доступны штатные блоки машины: DCP,
   видеологика, контроллер памяти и стандартная маршрутизация окон.
5. Система переходит к обычному выполнению BIOS.

Начальный маппинг памяти после завершения раннего bootstrap:

```
WIN0  0x0000–0x3FFF → ROM страница 0  (начало BIOS)
WIN1  0x4000–0x7FFF → RAM страница 2
WIN2  0x8000–0xBFFF → RAM страница 10
WIN3  0xC000–0xFFFF → RAM страница 0
```

После этого Z80 начинает выполнение основного BIOS с адреса `0x0000`.

## Фаза 2: Основной BIOS

### Карта Flash / ROM

Flash (512 КБ) содержит BIOS, SETUP, bitstream FPGA и служебные
конфигурационные данные.

```
Flash               ROM-страница    Содержимое
───────────────────────────────────────────────
0x00000 ...        ...             BIOS / bitstream / служебные блоки
...                ...             ...
                   Стр. 0 (#80)    BIOS начало (0x0000)
                   Стр. 1–5        BIOS основной код
                   Стр. 6–7        BIOS дополнения
                   старшие стр.     SETUP и конфигурационные данные
0x30000            page 12          cold-start loader
```

Точное физическое размещение блоков зависит от версии прошивки.

Справочные детали BIOS вынесены в профильные разделы:

- точки входа и базовая роль BIOS: [3.1 Обзор BIOS](../03_bios/01_bios_overview.md)

- дисковые функции BIOS: [3.3 Работа с дисками](../03_bios/03_bios_disk.md)

## Фаза 3: Выбор стартового устройства и передача управления загрузчику

На этой стадии BIOS уже работает в штатной конфигурации и выполняет обычную
инициализацию: читает CMOS, определяет накопители, строит внутреннюю таблицу
устройств и выбирает стартовый носитель.

BIOS последовательно пробует:
1. **BOOTSYS** — системный диск из CMOS `#10` (биты [2:0])
2. **BOOTALT** — альтернативный диск из CMOS `#10` (биты [6:4])
3. Если оба провалились — перезагрузка или выход в ZX Basic

После выбора устройства BIOS читает начальный системный сектор и передаёт
управление его коду. С этого момента заканчивается собственно BIOS boot path
и начинается загрузчик системы.

Низкоуровневые детали этой стадии описаны в профильных разделах:

- выбор устройств, `DRV_READ`, распознавание системного сектора и входное окружение загрузчика:
  [3.3 Работа с дисками](../03_bios/03_bios_disk.md)
- дальнейшая инициализация DSS после старта загрузчика:
  [4.1 Обзор DSS](../04_dss/01_dss_overview.md)

## Фаза 4: DSS и запуск программ

После передачи управления загрузчику дальнейшая загрузка уже относится к DSS:
разворачиванию резидентных страниц, установке `RST #10`, формату `.EXE` и
рабочему окружению процесса.

## Ключевые моменты

> - После RESET Z80 начинает с cold-start loader и завершает раннюю конфигурацию логики.
> - Для `SP2000+` cold-start loader расположен в полном ROM-образе по смещению `0x30000` (`16-КБ page 12`).
> - В стадии cold start Z80 программно подаёт конфигурационные данные ACEX через специальный загрузочный тракт.
> - После раннего bootstrap BIOS читает CMOS, определяет устройства и выбирает стартовое устройство.
> - После чтения начального системного сектора BIOS передаёт управление загрузчику, а дальнейшие шаги уже относятся к DSS.
> - Низкоуровневые детали чтения сектора и запуска загрузчика вынесены в разделы BIOS и DSS.
