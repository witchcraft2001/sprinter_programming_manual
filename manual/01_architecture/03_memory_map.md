## Адресное пространство Z80

Z80 работает с 64 КБ адресного пространства (`0x0000`–`0xFFFF`). В Sprinter это пространство
разделено на **четыре окна по 16 КБ**, каждое из которых независимо переключается на
физическую страницу RAM, ROM или VRAM.

```
Адрес Z80      Окно    Порт управления    Назначение
──────────────────────────────────────────────────────
0x0000–0x3FFF   WIN0   #82 (EmmWin.P0)   RAM/ROM/Cache — гибкое
0x4000–0x7FFF   WIN1   #A2 (EmmWin.P1)   RAM — обычно страница 2
0x8000–0xBFFF   WIN2   #C2 (EmmWin.P2)   RAM — обычно страница 10
0xC000–0xFFFF   WIN3   #E2 (EmmWin.P3)   RAM — переключаемая
```

## Физическая память

### RAM (DRAM)

| Параметр | Значение |
|---------|---------|
| Объём | **4 МБ стандартно** (DIMM-модуль), расширяемо до 64 МБ на Sp2000/Game |
| Организация | Страницы по 16 КБ |
| Кол-во страниц (4 МБ) | 256 страниц (`#00`–`#FF`, без ROM/VRAM зон) |
| Адресация | 22-битная: старшие биты адреса + MA[13:0] |

Физический адрес = `page_number × 16384 + offset_within_16K`.

> **Замечание:** В исходниках FPGA и старой документации (Sp97) встречается величина 512 КБ — это объём RAM в ранних прототипах Sprinter. Серийные платы Peters Plus Sp2000 поставляются с **DIMM-слотом на 4 МБ**, что даёт 256 страниц по 16 КБ (номера `#00`–`#FF` за вычетом системных зон).

### ROM / Flash

| Параметр | Значение |
|---------|---------|
| Тип | AMD/SST Flash |
| Объём | 512 КБ |
| Страницы | 16 страниц × 32 КБ |
| Доступ | Через `ROM_RG` (порт `#8F`) |
| Запись | Только в специальном режиме (бит 4 порта `#8F`) |

Flash содержит:
- BIOS (страницы 0–7)
- SETUP утилита
- Битстрим FPGA для автозагрузки Altera

### VRAM (Video RAM)

| Параметр | Значение |
|---------|---------|
| Организация | 4 банка × 8 бит = 32-бит/такт |
| Объём | 2 × 256 КБ (итого 512 КБ) |
| Доступ | Через видеоокна (страницы `#50`–`#5F`) |
| Конфликты | При ZX-режиме: wait states при совместном доступе CPU+VIDEO |

## Страницы памяти и окна

### Как работает банкинг

Порты `#82`, `#A2`, `#C2`, `#E2` — это регистры-переключатели окон. Запись числа страницы
в порт мгновенно переключает соответствующее 16KB окно на физическую страницу.

```asm
EmmWin.P0   EQU     #82     ; окно 0: 0x0000–0x3FFF
EmmWin.P1   EQU     #A2     ; окно 1: 0x4000–0x7FFF
EmmWin.P2   EQU     #C2     ; окно 2: 0x8000–0xBFFF
EmmWin.P3   EQU     #E2     ; окно 3: 0xC000–0xFFFF

; Переключить окно 3 на страницу 5:
        ld      a, 5
        out     (EmmWin.P3), a

; Теперь 0xC000–0xFFFF указывает на физическую страницу 5
```

### Диапазоны номеров страниц

| Диапазон | Тип | Содержимое |
|---------|-----|-----------|
| `#00`–`#FF` | RAM | Основная DRAM (до 256 страниц × 16 КБ = 4 МБ) — за вычетом зон ниже |
| `#40`–`#4F` | Спецстраницы | Системные страницы DSS (зарезервированы) |
| `#50`–`#5B` | VRAM | Видеопамять (страница `#50` = VPAGE_TILES, экраны A/B) |
| `#5C`–`#5F` | VRAM | Спрайтовые банки (`VPAGE_SPRITES = #5C`) |
| `#FC`, `#FE` | Системные | Резидент DSS и системные буферы |

> VRAM-страницы (`#50`–`#5F`) и системные страницы DSS физически не являются частью DRAM — они живут в отдельных областях (VRAM чип / системные буферы), но адресуются через общий банкинг.

Конкретные числа: из `sprinter_dss/HARDWARE.INC` и `flappybird/src/include/sp_equ.asm`.

## Карта памяти при старте (после сброса)

```
Окно   Z80-адрес      Физич. стр.   Содержимое
─────────────────────────────────────────────────────
WIN0   0x0000–0x3FFF   ROM стр. 0   BIOS (начало исполнения)
WIN1   0x4000–0x7FFF   RAM стр. 2   Свободная RAM
WIN2   0x8000–0xBFFF   RAM стр. 10  Свободная RAM
WIN3   0xC000–0xFFFF   RAM стр. 0   Свободная RAM
```

После инициализации DSS карта меняется: DSS занимает системные страницы,
окно 0 переключается на RAM, код программы загружается в окно 3 по адресу `0x8100`.

## Карта памяти DSS-программы (типовая)

```
Z80-адрес       Содержимое
─────────────────────────────────────────────────────────
0x0000–0x3FFF   RAM (WIN0) — данные программы, стек
0x4000–0x7FFF   RAM (WIN1) — данные/буферы
0x8000–0xBFFF   RAM (WIN2) — данные/буферы
0x8000–0x80FF   EXE-заголовок (первые 256 байт)
0x8100–0xBFFF   Код и данные программы (ORG 0x8100)
0xC000–0xFFFF   RAM (WIN3) — VRAM или данные
```

> **Замечание:** Адрес загрузки программы `0x8100` — это стандарт DSS. Первые 512 байт файла
> занимает заголовок EXE, реальный код начинается с `0x8100`.

## ZX Spectrum-совместимый режим памяти

При включении ZX-совместимости (`SC0=1` в регистре SC) маппинг страниц следует
стандарту ZX128:

```
0x0000–0x3FFF → ROM (ZX BASIC)
0x4000–0x7FFF → RAM страница 5 (ZX экран)
0x8000–0xBFFF → RAM страница 2
0xC000–0xFFFF → RAM страница, выбранная через порт 0x7FFD
```

ZX-видеопамять всегда в окне 1 (страница 5 = физическая страница 5).

## Системные страницы DSS

DSS резервирует несколько страниц для собственных нужд. Программа не должна
их занимать самостоятельно:

| Назначение | Страница |
|-----------|---------|
| BIOS/DSS резидент | системные (определяются при загрузке) |
| Страница кода DSS | `0x40`–`0x4F` зона |
| Стек системы | в системных страницах |

Получить список страниц, выделенных вашей программе: `EMM_FN5` (BIOS `RST #08`, функция `#C5`).

## Пример: сохранение и восстановление окон

Перед переключением страниц обязательно сохраняйте текущее состояние:

```asm
EmmWin.P0   EQU     #82
EmmWin.P1   EQU     #A2
EmmWin.P2   EQU     #C2
EmmWin.P3   EQU     #E2

; Сохранить все 4 окна
SavePages:
        in      a, (EmmWin.P0)
        ld      (saved_page0), a
        in      a, (EmmWin.P1)
        ld      (saved_page1), a
        in      a, (EmmWin.P2)
        ld      (saved_page2), a
        in      a, (EmmWin.P3)
        ld      (saved_page3), a
        ret

; Восстановить все 4 окна
RestorePages:
        ld      a, (saved_page0)
        out     (EmmWin.P0), a
        ld      a, (saved_page1)
        out     (EmmWin.P1), a
        ld      a, (saved_page2)
        out     (EmmWin.P2), a
        ld      a, (saved_page3)
        out     (EmmWin.P3), a
        ret

saved_page0:    db  0
saved_page1:    db  0
saved_page2:    db  0
saved_page3:    db  0
```

---

## Ключевые моменты

> - Z80 видит 64 КБ, разделённых на 4 окна по 16 КБ.
> - Каждое окно переключается независимо через порты `#82`, `#A2`, `#C2`, `#E2`.
> - Физических страниц: до 256 (RAM + ROM + VRAM).
> - Программы DSS загружаются по адресу `0x8100` (ORG в коде).
> - Перед переключением страниц — **всегда сохраняйте** текущее состояние.
