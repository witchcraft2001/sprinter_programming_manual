# 10.1 Туториал: Hello World

> **Навигация:** [← 9.4 ZX совместимость](../09_advanced/04_zx_compat.md) | [Оглавление](../README.md) | [10.2 Файловый браузер →](02_file_browser.md)

## Цель

Написать минимальную программу для Sprinter, которая:
1. Выводит текст на экран
2. Ждёт нажатия клавиши
3. Корректно возвращается в DSS

---

## Шаг 1: подготовка

### Требования

- Установленный ассемблер **sjasmplus** ([github.com/z00m128/sjasmplus](https://github.com/z00m128/sjasmplus))
- Эмулятор `sprintem` или реальный Sprinter
- Текстовый редактор

### Структура файла

Создайте файл `hello.asm`:

```asm
;===========================================================
; hello.asm — Hello World для Sprinter DSS
;===========================================================

; Константы
DssRst          EQU #10
Dss.PChars      EQU #5C
Dss.WaitKey     EQU #30
Dss.Exit        EQU #41

;-----------------------------------------------------------
; EXE заголовок (512 байт)
;-----------------------------------------------------------
        org     #8100 - 512

        db      "EXE", 0        ; сигнатура
        dw      #0200           ; версия формата
        dw      #0200           ; смещение кода (512)
        dw      0               ; адрес загрузчика (нет)
        dw      #8100           ; адрес загрузки
        dw      main            ; точка входа
        dw      #BFFF           ; начальный SP
        ds      512 - 16        ; заполнение до 512 байт

;-----------------------------------------------------------
; Основной код программы
;-----------------------------------------------------------
        org     #8100

main:
        ; Вывести строку
        ld      hl, msg
        ld      c, Dss.PChars
        rst     DssRst

        ; Ждать клавишу
        ld      c, Dss.WaitKey
        rst     DssRst

        ; Выход в DSS
        ld      c, Dss.Exit
        rst     DssRst

;-----------------------------------------------------------
; Данные
;-----------------------------------------------------------
msg:
        db      "Hello, Sprinter!", 13, 10, 0
```

---

## Шаг 2: сборка

### Команда sjasmplus

```bash
sjasmplus --raw=hello.exe hello.asm
```

Флаг `--raw=hello.exe` заставляет sjasmplus сохранить весь собранный код от `0x7F00` до конца как бинарный файл `hello.exe` без дополнительных заголовков.

### Проверка результата

```bash
ls -l hello.exe
# Должен быть ~550 байт (512 заголовок + небольшой код + данные)
```

Первые 4 байта файла должны быть `45 58 45 00` ("EXE" + 0).

---

## Шаг 3: запуск в эмуляторе

### В sprintem

```bash
sprintem hello.exe
```

Sprintem загрузит EXE и передаст управление на `0x8100`. Вы увидите:

```
Hello, Sprinter!
```

Нажмите любую клавишу — программа завершится.

### На реальном Sprinter

1. Скопируйте `hello.exe` на диск Sprinter (через CF-карту, эмулятор HDD или DSS команды).
2. В командной строке DSS:
   ```
   HELLO.EXE
   ```

## Как это работает

### Заголовок EXE (first 512 bytes)

```asm
        org     #8100 - 512     ; #7F00 — начало заголовка

        db      "EXE", 0        ; 4 байта: сигнатура + padding
        dw      #0200           ; 2 байта: версия формата
        dw      #0200           ; 2 байта: offset кода в файле (512 = #200)
        dw      0               ; 2 байта: loader address (нет)
        dw      #8100           ; 2 байта: load address в памяти
        dw      main            ; 2 байта: точка входа (= #8100)
        dw      #BFFF           ; 2 байта: начальный SP
        ds      512 - 16        ; остаток заголовка нулями
```

DSS читает эту структуру и:
1. Проверяет сигнатуру `"EXE"`.
2. Загружает код из смещения `#0200` файла по адресу `#8100`.
3. Устанавливает `SP = #BFFF`.
4. `JP main`.

### Основной код

```asm
main:
        ld      hl, msg         ; HL = адрес строки
        ld      c, Dss.PChars   ; C = #5C (PChars функция)
        rst     DssRst          ; RST #10 → диспетчер DSS
```

`Dss.PChars #5C` печатает ASCIIZ-строку, на которую указывает `HL`.

```asm
        ld      c, Dss.WaitKey  ; #30 — ждать клавишу
        rst     #10             ; возвращается при нажатии
```

```asm
        ld      c, Dss.Exit     ; #41 — выход
        rst     #10             ; управление возвращается в родителя
```

## Типичные ошибки

### 1. Отсутствие выхода через EXIT

```asm
; ПЛОХО
main:
        ; ... код ...
        ret                     ; RET здесь не работает правильно
```

Вместо этого:

```asm
; ПРАВИЛЬНО
        ld      c, #41
        rst     #10
```

### 2. Неправильный EXE заголовок

Если заголовок собран неправильно, DSS откажется загружать файл (или загрузит не туда). Проверьте:
- Сигнатура `"EXE"` в начале
- Версия `#0200`
- Offset `#0200`
- Load address `#8100`

### 3. `org` в неправильном месте

Убедитесь, что у вас **две** директивы `org`:
- Первая `org #8100-512` — начало заголовка EXE
- Вторая `org #8100` — начало кода

Без второй `org` код будет собран сразу после заголовка, и метка `main` не совпадёт с `#8100`.

## Расширение программы

### Несколько строк текста

```asm
main:
        ld      hl, line1
        ld      c, #5C
        rst     #10

        ld      hl, line2
        ld      c, #5C
        rst     #10

        ld      hl, line3
        ld      c, #5C
        rst     #10

        ld      c, #30
        rst     #10

        ld      c, #41
        rst     #10

line1:  db "Line 1", 13, 10, 0
line2:  db "Line 2", 13, 10, 0
line3:  db "Line 3", 13, 10, 0
```

### Чтение клавиши и вывод

```asm
main:
        ld      hl, prompt
        ld      c, #5C
        rst     #10

.loop:
        ld      c, #30          ; WaitKey
        rst     #10

        cp      27              ; ESC?
        jr      z, .exit

        ; Напечатать символ
        push    af
        ld      c, #5B          ; PutChar
        rst     #10
        pop     af
        jr      .loop

.exit:
        ld      c, #41
        rst     #10

prompt: db "Type something (ESC to exit):", 13, 10, 0
```

### Ветвления

```asm
main:
        ld      c, #30          ; WaitKey
        rst     #10

        cp      'Y'
        jr      z, .yes
        cp      'y'
        jr      z, .yes

        ld      hl, msg_no
        jr      .show

.yes:
        ld      hl, msg_yes

.show:
        ld      c, #5C
        rst     #10

        ld      c, #41
        rst     #10

msg_yes: db "You said YES!", 13, 10, 0
msg_no:  db "You said NO.", 13, 10, 0
```

## Ключевые моменты

> - EXE заголовок: 512 байт с сигнатурой "EXE", версией `#0200`, адресом загрузки `#8100`.
> - Код программы начинается с `#8100` (директива `org #8100`).
> - DSS API: `RST #10`, функция в `C`: `#5C` — PChars, `#30` — WaitKey, `#41` — Exit.
> - Выход только через `Dss.Exit #41`, не через `RET`.
> - Сборка: `sjasmplus --raw=hello.exe hello.asm`.
> - Запуск: `sprintem hello.exe` или на реальном железе.
