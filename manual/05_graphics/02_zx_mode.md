# 5.2 ZX Spectrum-совместимый режим

> **Навигация:** [← 5.1 Обзор видео](01_video_overview.md) | [Оглавление](../README.md) | [5.3 Режим 320×256 →](03_mode1_320.md)

---

## Характеристики режима

ZX Spectrum-режим Sprinter аппаратно идентичен оригинальному Spectrum:

| Параметр | Значение |
|---------|---------|
| Разрешение | 256×192 пикселей |
| Пиксельный буфер | 6144 байт |
| Атрибуты | 32×24 = 768 байт |
| Цвета | 8 × 2 интенсивности (16 оттенков) |
| Привязка атрибутов | Блок 8×8 пикселей |
| Адрес в памяти | `#4000`–`#5AFF` (в WIN1 при ZX маппинге) |

---

## Включение ZX-режима

### Через DSS API

```asm
        ld      a, 0                ; 0 = ZX mode
        ld      b, 0
        ld      c, #50              ; SetVMod
        rst     #10
```

### Через прямые порты

```asm
; 1. Включить ZX-совместимость (SC0=1)
        ld      a, SYS_PORT_OFF     ; #3C
        out     (#3C), a

; 2. Установить режим видео
        ld      a, 0
        out     (#C3), a            ; ALL_MODE = ZX
```

## Организация экрана ZX

### Пиксельная область (6144 байта)

Экран разделён на **3 трети** по 64 строки каждая. Внутри трети строки идут через 8:

```
Адрес = 0x4000 + (y mod 8) * 256 + (y / 8 mod 8) * 32 + (y / 64) * 2048 + (x / 8)

Расшифровка:
  Y = Y7 Y6 Y5 Y4 Y3 Y2 Y1 Y0
  X = X7 X6 X5 X4 X3 X2 X1 X0

  Адрес (13 бит): Y7 Y6 Y2 Y1 Y0 Y5 Y4 Y3 X7 X6 X5 X4 X3
  Смещение в байте: 7 - (X2 X1 X0)
```

### Атрибутная область (768 байт)

После пиксельной области (`#5800`–`#5AFF`):
```
Атрибут(x,y) = 0x5800 + (y / 8) * 32 + (x / 8)

Формат байта атрибута:
  bit 7   — FLASH
  bit 6   — BRIGHT
  bits [5:3] — PAPER (фон)
  bits [2:0] — INK (текст)
```

### Цвета ZX (без BRIGHT)

| Код | Цвет | RGB (6-6-6) |
|-----|------|-------------|
| 0 | Чёрный | 0, 0, 0 |
| 1 | Синий | 0, 0, 42 |
| 2 | Красный | 42, 0, 0 |
| 3 | Пурпурный | 42, 0, 42 |
| 4 | Зелёный | 0, 42, 0 |
| 5 | Циан | 0, 42, 42 |
| 6 | Жёлтый | 42, 42, 0 |
| 7 | Белый | 42, 42, 42 |

С BRIGHT=1 значения = 63 вместо 42.

## Вычисление адреса пикселя

```asm
; Вычислить адрес байта в экране ZX по координатам (X, Y)
; Вход: B = X (0..255), C = Y (0..191)
; Выход: HL = адрес в #4000..#57FF, A = маска бита (1 << (7 - (X mod 8)))

CalcPixelAddr:
        ld      a, c                ; A = Y
        and     #C0                 ; Y5 Y4 Y3 бит 7:6 = треть
        or      #40                 ; базовый #4000
        ld      h, a                ; H = 01 Y7 Y6 000 000

        ld      a, c
        and     #07                 ; Y2 Y1 Y0
        rrca                        ; сдвиг на 3 вправо
        rrca
        rrca
        or      h
        ld      h, a                ; H = 01 Y7 Y6 Y2 Y1 Y0

        ld      a, c
        and     #38                 ; Y5 Y4 Y3
        rrca
        rrca
        or      h
        ld      h, a                ; HL зафиксирован в нужных 3 битах

        ld      a, b                ; A = X
        rrca
        rrca
        rrca
        and     #1F                 ; X7 X6 X5 X4 X3
        ld      l, a

        ; Маска: 1 << (7 - (X mod 8))
        ld      a, b
        and     #07                 ; X2 X1 X0
        ld      c, a                ; C = счётчик сдвигов
        ld      a, #80              ; 0x80 = 10000000
        jr      z, .done
.shift:
        rrca
        dec     c
        jr      nz, .shift
.done:
        ret
```

---

## Установка пикселя

```asm
; Поставить пиксель (X, Y)
; Вход: B = X, C = Y
SetPixelZX:
        call    CalcPixelAddr       ; HL = адрес, A = маска
        ld      c, a                ; C = маска
        ld      a, (hl)
        or      c                   ; установить бит
        ld      (hl), a
        ret

; Очистить пиксель
ClearPixelZX:
        call    CalcPixelAddr
        cpl                         ; инвертировать маску
        ld      c, a
        ld      a, (hl)
        and     c
        ld      (hl), a
        ret
```

## Установка атрибутов

```asm
; Установить атрибут блока 8×8 в точке (X, Y)
; Вход: B = X, C = Y, E = байт атрибута
SetAttr:
        ld      h, 0
        ld      a, c
        srl     a : srl a : srl a   ; Y / 8
        ld      l, a
        add     hl, hl
        add     hl, hl
        add     hl, hl
        add     hl, hl
        add     hl, hl              ; HL = (Y/8) * 32
        ld      a, b
        srl     a : srl a : srl a   ; X / 8
        add     a, l
        ld      l, a
        ld      a, h
        adc     a, #58              ; база атрибутов
        ld      h, a

        ld      (hl), e             ; записать атрибут
        ret
```

## Очистка экрана

```asm
; Очистить пиксельную область и установить атрибут #38 (чёрный фон, белый текст)
ClsZX:
        ; Очистить пиксели
        ld      hl, #4000
        ld      de, #4001
        ld      bc, 6144 - 1
        xor     a
        ld      (hl), a
        ldir

        ; Очистить атрибуты
        ld      hl, #5800
        ld      de, #5801
        ld      bc, 768 - 1
        ld      a, #38              ; белый текст на чёрном фоне
        ld      (hl), a
        ldir
        ret
```

## Подключение ZX-экрана в окно

В ZX-режиме экран автоматически доступен в `0x4000`–`0x5AFF` через WIN1 (страница 5). Это аппаратная особенность ZX-совместимости.

В нативном режиме Sprinter можно подключить ZX-видеостраницу вручную:

```asm
EmmWin.P1       EQU #A2

; Подключить страницу 5 (ZX-экран) в WIN1
        ld      a, 5
        out     (EmmWin.P1), a
```

## Чтение порта #FE (ZX клавиатура + бордюр)

Порт `#FE` в ZX-режиме выполняет несколько функций:

**Запись:**
- биты [2:0] — цвет бордюра
- бит 3 — MIC (tape out)
- бит 4 — EAR (speaker)

**Чтение:**
- биты [4:0] — клавиши в выбранной строке (по A8–A12 при чтении)
- бит 6 — EAR (tape in)

```asm
; Установить бордюр синим
        ld      a, 1
        out     (#FE), a

; Прочитать строку клавиатуры для Caps Shift, Z, X, C, V
        ld      a, #FE              ; A8–A12 = 11110 (для строки 0xFEFE)
        in      a, (#FE)
        and     #1F                 ; нижние 5 бит = клавиши
```

## Ограничения ZX-режима

- **Атрибуты 8×8**: можно использовать только 2 цвета на блок 8×8 пикселей (colour clash).
- **Только 16 цветов**: 8 базовых + BRIGHT версия.
- **Не работает double buffering** — только один экран.
- **Нет акселератора** для ZX-режима (он работает с линейным буфером 320×256).

---

## Когда использовать ZX-режим

- Совместимость со старыми играми и программами.
- Запуск кода, написанного для классического Spectrum.
- Когда нужны всего 16 цветов.

**Для новых программ Sprinter рекомендуется режим 320×256 8bpp** — он быстрее, гибче и имеет доступ к акселератору.

---

## Ключевые моменты

> - ZX-режим: 256×192, 6144 пиксельных байт + 768 атрибутов.
> - Адресация пиксельной области — нелинейная (ZX стандарт).
> - Атрибутная область линейная: `#5800 + (y/8)*32 + (x/8)`.
> - Экран автоматически в `0x4000`–`0x5AFF` при ZX-режиме.
> - Бордюр через порт `#FE` биты [2:0].
> - Ограничение: colour clash 8×8 и отсутствие double buffering.
