# Графика и текстовые режимы

## Структура экрана

Видеокарта Sprinter организует экран как сетку квадратов 8×8 (в 640‑точечном режиме — 16×8). Вся страница режима (56×39 квадратиков с запасом под бордер и синхроимпульсы) лежит в областях `24..29` блоков VRAM. Каждому квадрату соответствуют 8 байт: `Mode0..Mode2` (режим, адрес пикселей, адрес атрибутов), один зарезервирован и несколько — данные подрежимов. Порт `RGMOD/#C9` переключает страницу режима (бит 0) и таким образом позволяет держать два полноценных экрана, между которыми можно быстро переключаться [sprinter_docs/video_ram/screen_struct_and_modes.md:1-40](../sprinter_docs/video_ram/screen_struct_and_modes.md#L1).

VRAM поделена на 32 «блока» по 1 КБ. BIOS резервирует блоки 24..31: `24..29` содержат описатели квадратиков, `30..31` — палитры (`#03E0-#03FF`), область `#03A0-#03DF` отведена под служебные данные вывода. Блоки `0..23` доступны прикладному коду: ZX-экран занимает блоки 0–1, графические экраны — 2..21, а блоки 22–23 часто используются под пользовательские шрифты [docs/sp2000_man.pdf:640-705] [docs/sprinter-graphics_v3.pdf:1-40].

Квадратики адресуются через `Line1/Line2` (`1 + 2·a + #80·PM`, где `a` — координата по X, `PM` — номер страницы режима) и `LA=#0300 + 4·b` (`b` — координата по Y). Это упрощает расчёт адресов: чтобы вычислить смещение в таблице режима, умножьте X на 2, прибавьте `0x80` в случае второй страницы и умножьте Y на 4.

## Байты режима (Mode0/1/2)

- `Mode0` (младшие биты) задаёт старшие биты адреса знакогенератора, биты 4/5 говорят «текст/графика» и «320/640». Комбинация `bit3..2=11` включает Blank (квадрат гасится), а если одновременно выставить `bit0=1`, в восьмой строке квадрата генерируется `INT` — именно так BIOS подгоняет тайминги под Pentagon/Scorpion [sprinter_docs/video_ram/text_mode.md:1-25](../sprinter_docs/video_ram/text_mode.md#L1) [docs/sp2000_man.pdf:760-780].
- `Mode1` содержит младшие 8 бит адреса пикселей. В текстовом режиме он прямо равен ASCII коду символа (если предварительно «разложить» знакогенератор в одной из спектрумовских страниц) [sprinter_docs/video_ram/text_mode.md:25-40](../sprinter_docs/video_ram/text_mode.md#L25).
- `Mode2` адресует атрибуты. В ZX‑режиме равен `Mode1`, в текстовом может содержать отдельные атрибуты, что позволяет иметь до 36 независимых знакогенераторов на экране [sprinter_docs/video_ram/text_mode.md:40-70](../sprinter_docs/video_ram/text_mode.md#L40).

## Палитры

Каждый вывод пикселя считывает три байта (BGR) из концов строки (`#03E0-#03FF`). Там лежат восемь палитр по 256 цветов: четыре графические и четыре текстовые (бумага, текст, мерцание бумаги, мерцание текста). Широкие режимы (256 цветов) используют одну палитру, а 16‑цветные — первые 16 записей [sprinter_docs/video_ram/palette.md:1-25](../sprinter_docs/video_ram/palette.md#L1). Байтовый порядок — Blue, Green, Red.

## Графическая адресация

Страницы `#50..#5F` включают «графический режим» адресации. `RGADR/#89` выбирает одну из 256 строк по 1024 байта, а младшие 10 бит адреса CPU задают позицию внутри строки. Такая раскладка позволяет использовать `LDIR` для горизонтальных линий в графической адресации и вертикальных — в спектрумовской (переставляя `RGADR`). Подрежимы (биты 2 и 3 номера страницы) включают «прозрачные» заливки (#FF не записывается) и запись исключительно в VRAM (без изменения копии в основном ОЗУ) — удобно для спрайтов и курсоров [sprinter_docs/video_ram/gfx_mode.md:1-15](../sprinter_docs/video_ram/gfx_mode.md#L1) [sprinter_docs/video_ram/gfx_submodes.md:1-25](../sprinter_docs/video_ram/gfx_submodes.md#L1).

### RGADR и ZX-режим

В ZX‑режиме `RGADR` выбирает 8‑килобайтные блоки видеопамяти (`bit0` вместе с `#7FFD` определяет младший бит). Значение `>= #C0` блокирует вывод, что удобно, если вы хотите использовать страницы 5/7 как рабочие буферы без обновления экрана [sprinter_docs/video_ram/zx_mode.md:1-10](../sprinter_docs/video_ram/zx_mode.md#L1).

## Акселератор

Каманды `LD B,B`, `LD D,D`, `LD L,L`, `LD A,A`, `LD C,C`, `LD E,E` перепрофилированы ПЛИС и управляют встроенным RAM‑акселератором: включение, установка размера блока, горизонтальные/вертикальные копии, заливки, логические функции (AND/OR/XOR). Набор примеров (копирование целого экрана, перезапись блоков) приведён в [sprinter_docs/accelerator/accel.md:1-90](../sprinter_docs/accelerator/accel.md#L1). При больших операциях выключайте прерывания или используйте новую прошивку с автоматическим отключением акселератора на `RETI`.

## Практика: текстовое окно

Редактор `texteditor/routines.asm` показывает полный цикл вывода окна в текстовом режиме:

1. Перед стартом устанавливает `RGADR=#C0`, чтобы Spectrum‑режим не перехватывал записи [texteditor/routines.asm:8-20](../texteditor/routines.asm#L8).
2. Выделяет три страницы под экранные буферы через `Dss.GetMem`, подключает их в окна `#8000/#C000` (`OUT (EmmWin.P2/P3),A`) и сохраняет оригинальный экран `DSS.WinCopy` в буфер `SCR_BUF` [texteditor/routines.asm:20-70](../texteditor/routines.asm#L20).
3. Работает с окном через `Dss.Locate`, `Dss.PChars`, `Dss.WinRest`, используя «виртуальные» координаты 40×32.
4. На выходе возвращает страницы и курсор через `WinBack`/`Dss.FreeMem` [texteditor/routines.asm:116-200](../texteditor/routines.asm#L116).

Этот шаблон можно адаптировать для любых текстовых интерфейсов: главное — сохранять `RGADR` и использовать системные вызовы для копирования окон (они учитывают режимы, курсор, палитры).

## Практика: двойная буферизация в графике

Flappy Bird (`flappybird/src/fbird.asm`) демонстрирует полноценный графический пайплайн:

- Загружает ассеты через DSS (`Dss.ChDir`, `Dss.PChars`, `Dss.GetMem`) в выделенный блок памяти, затем мапит его в окно с помощью `Bios.Emm_Fn5` [flappybird/src/fbird.asm:1-60](../flappybird/src/fbird.asm#L1).
- Переключает видеорежим и палитру через `ChangeVideoMode` и прямые записи в `Palette`/`SetPalette`, после чего настраивает IM2 обработчик и двойную буферизацию (`RGMOD` XOR 1 переключает страницу режима) [flappybird/src/fbird.asm:60-110](../flappybird/src/fbird.asm#L60).
- Каждый кадр рисует фон, города и трубы в «теневом» буфере, затем ставит флаг `needChangePage`, чтобы обработчик IM2 поменял активную страницу без разрывов [flappybird/src/fbird.asm:110-190](../flappybird/src/fbird.asm#L110).

Обратите внимание: код сохраняет текущий `RGMOD`, дважды заливает экраны функциями `FillShadowScreen/DrawCity`, а перед выходом возвращает исходную страницу и палитру [flappybird/src/fbird.asm:190-230](../flappybird/src/fbird.asm#L190).

## Мышь и окна

Графический просмотрщик `gfxview` активно использует `RST #30` (мышь) и DSS окна. Сервисная таблица для `RST #30` описана в [sprinter_dss/INTMOUSE.ASM:1-80](../sprinter_dss/INTMOUSE.ASM#L1): команды `00h` — инициализация, `01h/02h` — показать/спрятать курсор, `03h` — получить координаты, `04h` — перемещение. `gfxview/gfx_view.asm` на каждой итерации опрашивает `RST #30`, сравнивает координаты, обрезает их по границам и обновляет курсор (`command 04h`). Это хороший пример того, как соединить аппаратную мышь и режимы вывода [gfxview/gfx_view.asm:1-120](../gfxview/gfx_view.asm#L1).

## Вычисление координат

- **Текст режим 40×32**: чтобы вывести символ в позицию `(x,y)`, достаточно вычислить `addr = ModeTableBase + y*4 + x*2` и записать туда `Mode0/Mode1/Mode2`. Для ZX‑совместимости можно заранее подготовить знакогенератор и просто обновлять `Mode1/Mode2` (см. пример в разделе «Практика»).
- **Графика 320×256**: строка = `RGADR`, столбец = «младшие десять бит адреса». Координаты в пикселях (`px,py`) переводятся так: `RGADR = py`, `HL = base + ((py & 0xFF) << 10) + px`. В 640‑точечном режиме каждая колонка 16 пикселей хранится в двух наборах режимов (основной и подрежим).
- **Палитра**: цвет `n` (0..255) в палитре `p` (0..3) находится по адресу `#03E0 + p*4 + n*3`. Для текстовых цветов используйте `#03F0` и далее.

## Текстовые атрибуты

BIOS следует CGA/VGA нотации: `bit7` включает мигание, `bits6..4` — цвет бумаги (0..7, RGB), `bits3..0` — цвет чернил (0..15, IRGB). Цвета 0..7 соответствуют базовой восьмёрке (чёрный/синий/зелёный/циан/красный/маджента/коричневый/белый), а 8..15 — яркие оттенки (серый/ярко-синий/…/жёлтый/ярко-белый). Меняя палитру по адресам `#03E0-#03FF`, можно переприсвоить оттенки любому из 16 текстовых кодов [docs/Sprinter_Programming.pdf:360-440].

## Пример вывода пикселя (320×256×256)

```asm
SaveP3  DB 0

        IN   A,(EmmWin.P3)
        LD   (SaveP3),A
        LD   A,50h
        OUT  (EmmWin.P3),A        ; подключить VRAM в #C000

        LD   A,(yCoord)
        OUT  (RGADR),A            ; выбрать строку
        LD   HL,(xCoord)          ; смещение по X
        LD   DE,#C000
        ADD  HL,DE
        LD   A,(color)
        LD   (HL),A               ; записать байт цвета

        LD   A,(SaveP3)
        OUT  (EmmWin.P3),A        ; вернуть страницу
```

Фрагмент повторяет рекомендации Дениса Паринова: подключаем страницу `#50` в окно `#C000`, указываем строку через `RGADR` и пишем байт цвета. После рисования обязательно верните прежнее значение `PAGE3`, иначе DSS или ISR получат неожиданные страницы [docs/Sprinter_Programming.pdf:150-220].
