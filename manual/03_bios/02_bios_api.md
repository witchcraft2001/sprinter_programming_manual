# 3.2 Справочник функций BIOS API

> **Навигация:** [← 3.1 Обзор BIOS](01_bios_overview.md) | [Оглавление](../README.md) | [3.3 Дисковые операции →](03_bios_disk.md)

---

## Соглашения вызова

- Вектор: **`RST #08`** (основной), **`RST #30`** (Mouse)
- Номер функции: в регистре **`C`**
- Параметры: в `A`, `B`, `D`, `E`, `H`, `L`, `IX`, `IY`
- Возврат: `CF=0` успех, `CF=1` ошибка

---

## Функции памяти (EMM)

### `#C0` — EMM_FN0: информация о RAM

Возвращает количество страниц RAM в системе.

**Вход:** `C = #C0`
**Выход:**
- `HL` = общее количество RAM страниц
- `BC` = количество свободных страниц

```asm
        ld      c, #C0
        rst     #08
        ; HL = total, BC = free
```

---

### `#C1` — EMM_FN1: инициализация RAM

Сбрасывает все выделения памяти. Системные страницы (0–15, VRAM 0x50–0x5F) остаются занятыми.

**Вход:** `C = #C1`
**Выход:** CF = 0

```asm
        ld      c, #C1
        rst     #08
```

---

### `#C2` — EMM_FN2: выделить блок памяти

Выделяет блок из `B` страниц RAM.

**Вход:**
- `C = #C2`
- `B` = количество 16 КБ страниц для выделения

**Выход:**
- `CF = 0` успех → `A = ID блока` (1–255)
- `CF = 1` ошибка → недостаточно памяти

```asm
        ld      c, #C2
        ld      b, 4            ; запросить 4 страницы (64 КБ)
        rst     #08
        jr      c, .no_mem
        ld      (block_id), a   ; сохранить ID для последующих операций
.no_mem:
```

---

### `#C3` — EMM_FN3: освободить блок памяти

**Вход:**
- `C = #C3`
- `A` = ID блока

**Выход:** `CF = 0` успех, `CF = 1` если блок не найден.

```asm
        ld      a, (block_id)
        ld      c, #C3
        rst     #08
```

---

### `#C4` — EMM_FN4: получить реальный индекс страницы блока

Возвращает физический номер страницы по логическому индексу внутри блока.

**Вход:**
- `C = #C4`
- `A` = ID блока
- `B` = логический индекс (0 = первая страница блока)

**Выход:**
- `CF = 0` → `A = физический номер страницы`
- `CF = 1` → не найдено

```asm
        ld      a, (block_id)
        ld      b, 0            ; первая страница блока
        ld      c, #C4
        rst     #08
        ; A = физический номер страницы для EmmWin.Px
        out     (EmmWin.P3), a  ; подключить в WIN3
```

---

### `#C5` — EMM_FN5: список всех страниц блока

Записывает в буфер все физические номера страниц блока.

**Вход:**
- `C = #C5`
- `A` = ID блока
- `HL` = адрес буфера (должен вместить до 256 байт)

**Выход:**
- `CF = 0` → `B` = количество страниц; буфер заполнен, в конце байт `#FF`
- `CF = 1` → блок не найден

```asm
        ld      a, (block_id)
        ld      hl, page_list
        ld      c, #C5
        rst     #08
        ; B = количество страниц; page_list заполнен
        ; page_list: N байт физических номеров + байт #FF

page_list:  ds  32          ; буфер под 32 страницы
```

---

### `#C6` — EMM_FN6: порт и страница окна

**Вход:**
- `C = #C6`
- `A` = номер окна (0–3)

**Выход:**
- `CF = 0` → `C` = номер порта (`#82`/`#A2`/`#C2`/`#E2`), `B` = текущая страница
- `CF = 1` → неверный номер окна

```asm
        ld      a, 2            ; WIN2
        ld      c, #C6
        rst     #08
        ; C = #C2, B = текущая физическая страница в WIN2
```

---

### `#C7` — EMM_FN7: следующая страница блока

Поиск следующей физической страницы, принадлежащей тому же блоку, что и данная.

**Вход:**
- `C = #C7`
- `A` = текущий физический номер страницы

**Выход:**
- `CF = 0` → `A` = физический номер следующей страницы блока; `A = #FF` если последняя
- `CF = 1` → страница не принадлежит ни одному блоку

---

## Функции печати текста (LP)

### `#81` — LP_PRINT_ALL: печать с атрибутами

**Вход:**
- `C = #81`
- `A` = символ
- `B` = количество повторений
- `E` = атрибут (нижние 4 бита = цвет текста, старшие 4 бита = фон)

**Выход:** `CF = 0`

---

### `#82` — LP_PRINT_SYM: печать символа

**Вход:**
- `C = #82`
- `A` = символ
- `B` = количество повторений

```asm
; Напечатать "***" (3 звёздочки)
        ld      c, #82
        ld      a, '*'
        ld      b, 3
        rst     #08
```

---

### `#84` — LP_SET_PLACE: установить позицию курсора

**Вход:**
- `C = #84`
- `D` = строка (Y)
- `E` = колонка (X)

```asm
; Переместить курсор в строку 5, колонку 10
        ld      c, #84
        ld      d, 5
        ld      e, 10
        rst     #08
```

---

### `#85` — LP_PRINT_LN: печать строки

**Вход:**
- `C = #85`
- `HL` = адрес строки
- `B` = длина строки
- `E` = атрибут

```asm
; Напечатать строку длиной 13 байт
        ld      hl, msg
        ld      b, 13
        ld      e, #0F          ; белый на чёрном
        ld      c, #85
        rst     #08
msg:    db      "Hello, world!"
```

---

### `#89` — LP_CLS_WIN: очистить окно

**Вход:**
- `C = #89`
- `D` = строка начала
- `E` = колонка начала
- `H` = высота
- `L` = ширина
- `B` = атрибут

---

### `#8A` — LP_SCROLL_UP: прокрутить

**Вход:**
- `C = #8A`
- `B` = направление (1 = вверх, 0 = вниз)
- `D` = строка начала
- `E` = конец

---

### `#8E` — LP_GET_PLACE: получить позицию

**Выход:**
- `D` = строка, `E` = колонка

---

## Функции графики (PIC)

### `#A1` — PIC_POINT: установить пиксель

**Вход:**
- `C = #A1`
- `HL` = X координата
- `DE` = Y координата
- `B` = цвет (0–255 индекс палитры)

```asm
; Поставить красный пиксель в (100, 50)
        ld      hl, 100
        ld      de, 50
        ld      b, 4            ; красный в палитре
        ld      c, #A1
        rst     #08
```

---

### `#A4` — PIC_SET_PAL: установить элемент палитры

**Вход:**
- `C = #A4`
- `HL` = индекс цвета (0–255)
- `E` = R (красный, 0–63)
- `D` = G (зелёный, 0–63)
- `A` = B (синий, 0–63)

```asm
; Установить цвет 1 на ярко-жёлтый
        ld      hl, 1           ; индекс
        ld      e, 63           ; R
        ld      d, 63           ; G
        ld      a, 0            ; B
        ld      c, #A4
        rst     #08
```

---

## Функции работы с дисками (DRV)

### `#51` — DRV_RESET: сброс диска

**Вход:** `C = #51`, `A` = номер диска.
**Выход:** `CF = 0`, `A = 0` успех; `CF = 1`, `A = #AA` — неизвестный тип диска; `A = 4` — диск не готов (FDD).

---

### `#52` — Long Read: чтение секторов с переключением страниц

**Вход:**
- `C = #52`
- `A` = номер диска
- `HL:IX` = сектор
- `DE` = адрес буфера
- `B` = количество секторов
- `A'` = ID блока памяти (EMM, для многостраничного чтения)

**Выход:**
- `CF = 0` → `HL:IX` = следующий сектор, `DE` = следующий адрес
- `CF = 1` → `A` = код ошибки, `C` = количество успешно прочитанных секторов

---

### `#53` — Long Write: запись секторов с переключением страниц

Параметры и возврат аналогичны `#52 Long Read`.

---

### `#54` — DRV_VERIFY: проверка сектора

**Вход:**
- `C = #54`
- `A` = номер диска
- `HL:IX` = сектор: `HL` = старшие 16 бит LBA, `IX` = младшие 16 бит LBA
- `B` = количество секторов

**Выход:** `CF = 0` успех, `CF = 1` ошибка.

---

### `#55` — DRV_READ: чтение сектора

**Вход:**
- `C = #55`
- `A` = номер диска (`#80` = IDE master, `#81` = IDE slave, `#00`/`#01` = FDD)
- `HL` = LBA старшие 16 бит
- `IX` = LBA младшие 16 бит
- `DE` = адрес буфера назначения
- `B` = количество секторов

**Выход:**
- `CF = 0` → буфер заполнен; `HL:IX` = следующий сектор, `DE` = следующий адрес
- `CF = 1` → `A` = код ошибки

```asm
; Прочитать сектор 100 диска #80 в буфер
        ld      a, #80
        ld      ix, 100         ; LBA (младшие 16 бит) = 100
        ld      hl, 0           ; LBA (старшие 16 бит) = 0
        ld      de, sector_buf  ; адрес буфера
        ld      b, 1            ; 1 сектор
        ld      c, #55
        rst     #08
        jr      c, .err

sector_buf: ds  512
```

---

### `#56` — DRV_WRITE: запись сектора

**Вход:** аналогично `#55 DRV_READ` (`HL:IX` = сектор, `DE` = буфер, `B` = количество).
**Выход:**
- `CF = 0` → `HL:IX` = следующий сектор, `DE` = следующий адрес
- `CF = 1` → `A` = код ошибки

---

### `#57` — DRV_DETECT: обнаружение диска

**Вход:** `C = #57`, `A` = номер диска.
**Выход:**
- FDD: `CF = 0`, `A` = флаг плотности (бит 7: `1` = HD, `0` = DD)
- Другой тип: `CF = 1`, `A = #AA` (не поддерживается)

---

### `#58` — DRV_GET_PAR: параметры диска

**Вход:** `C = #58`, `A` = номер диска.
**Выход:**
- `H` = количество головок
- `L` = секторов на дорожке
- `DE` = количество цилиндров
- `IX` = размер сектора в байтах (обычно 512)
- `B` = флаги (бит 7: 1 = HD, 0 = DD для FDD)

---

### `#59` — DRV_SET_PAR: установить параметры диска

**Вход:** `C = #59`, `A` = номер диска; остальные параметры как у `#58 DRV_GET_PAR`.
**Выход:** `CF = 0` успех; `CF = 1`, `A = #AA` — неизвестный тип диска.

---

### `#5A` — EXT_VERSION: версия BIOS

**Выход:** `DE` = версия (`VER * 256 + MOD`, например `#0229` = v2.41); `HL = 0`, `BC = 0`.

---

### `#5F` — DRV_LIST: конфигурация устройств

**Вход:** `C = #5F`, `IX` = адрес буфера (минимум 5 байт).
**Выход:** буфер заполнен структурой:

| Смещение | Значение |
|---------|---------|
| `IX+0` | `#04` (количество полей) |
| `IX+1` | количество FDD |
| `IX+2` | количество HDD |
| `IX+3` | количество CD-ROM |
| `IX+4` | `#00` (конец) |

---

## RAM-диски и назначение устройств

### `#C8` — BLK_RD_WR: чтение/запись сектора RAM-диска

> **Внимание:** функция не реализована в текущих версиях BIOS.

**Вход:**
- `C = #C8`
- `A` = ID блока памяти (ранее выделен через `#C2`)
- `A'` = команда (0 = чтение, `#FF` = запись)
- `HL` = адрес буфера
- `DE` = абсолютный номер сектора
- `B` = количество секторов

**Выход:** `CF = 0` успех; `CF = 1` — ошибка идентификатора блока.

---

### `#C9` — BLK_TO_RAMD: назначить блок как RAM-диск

Блок должен содержать данные RAM-диска в формате TR-DOS — для последующей работы с ним через другие функции.

**Вход:** `C = #C9`, `A` = номер RAM-диска (0–15, соответствует дискам `e:–t:`), `B` = ID блока.
**Выход:** `CF = 0` успех; `CF = 1` — неверный номер RAM-диска или RAM-диск уже занят.

---

### `#CA` — RAMD_CLEAR: освободить RAM-диск

Отключает блок памяти от RAM-диска. Сам блок не освобождается (не вызывает `#C3 EMM_FN3`).

**Вход:** `C = #CA`, `A` = номер RAM-диска (0–15).
**Выход:**
- `CF = 0` → `B` = ID блока, отключённого от RAM-диска
- `CF = 1` → неверный номер RAM-диска или RAM-диск не занят

---

### `#CB` — RAMD_TO_DRV: назначить RAM-диск на привод

**Вход:** `C = #CB`, `A` = номер RAM-диска (0–15), `B` = номер привода (0–3 = A:–D:).
**Выход:** `CF = 0` успех; `CF = 1` — неверный номер привода или RAM-диска.

---

### `#CC` — FDD_TO_DRV: назначить FDD на привод

**Вход:** `C = #CC`, `A` = номер физического дисковода (0–3), `B` = номер привода (0–3).
**Выход:** `CF = 0` успех; `CF = 1` — неверный номер привода или дисковода.

---

### `#CD` — HDD_TO_DRV: назначить HDD на привод

**Вход:** `C = #CD`, `A` = номер жёсткого диска (сейчас только 0; в будущем 0–15 для master/slave), `B` = номер привода (0–3).
**Выход:** `CF = 0` успех; `CF = 1` — неверный номер привода или диска.

---

### `#CE` — GET_RAMD_ST: статус RAM-диска

**Вход:** `C = #CE`, `A` = номер RAM-диска (0–15).
**Выход:**
- `CF = 0` → `A` = ID блока, назначенного на этот RAM-диск; `A = 0` если блок не назначен
- `CF = 1` → неверный номер RAM-диска

---

### `#CF` — GET_DRV_ST: статус привода

**Вход:** `C = #CF`, `A` = номер привода (0–3).
**Выход:**
- `CF = 0` → `A` = тип назначенного устройства:
  - `0–3` = реальный дисковод (A:, B:, C:, D:)
  - `4–19` = RAM-диск (A = `ram_disk + 4`)
  - `#40` = жёсткий диск
- `CF = 1` → неверный номер привода

---

## Служебные функции

### `#ED` — FN_CRIPT: Board ID

Возвращает уникальный идентификатор платы.

**Выход:**
- `A` (старший байт) + `HL` (младшие 16 бит) = 24-битный Board ID
- `DE`, `BC` = контрольные значения

---

### `#EF` — FN_VERSION: строковая версия BIOS

**Вход:** `C = #EF`, `HL` = буфер для строки
**Выход:**
- `CF = 0` → в буфере ASCIIZ-строка с версией (например `"v0.2a SprintEm"`), `HL` = тот же адрес
- `DE` = номер версии
- `BC` = идентификатор модели:

| `BC` | Модель |
|------|--------|
| `#FFFF` | не определена |
| `#FFFE` | Sprinter-1 |
| `#FFFD` | Sprinter-2 |
| `#FFFC` | Spectrum + AY8910 |
| `#FFFB` | Game-1 |
| `#FFFA` | Video-1 |

> Поле `BC` возвращается начиная с версии BIOS 1.16.

```asm
        ld      hl, verbuf
        ld      c, #EF
        rst     #08

verbuf: ds  64
```

---

### `#F0` / `#F1` — SPRINTER_1 / SPRINTER_2

Проверка модели Sprinter. `#F0` возвращает `CF=1` (не Sprinter-1), `#F1` возвращает `CF=0` (Sprinter-2 подтверждён).

## Mouse API (RST #30)

### `#00` — INITIALIZATION
**Вход:** `C = #00`
Инициализирует подсистему мыши.

### `#01` — SHOW MOUSE CURSOR
### `#02` — HIDE MOUSE CURSOR
### `#03` — READ MOUSE STATE

**Вход:** `C = #03`
**Выход:**
- `A` = кнопки (bit0 = левая, bit1 = правая)
- `HL` = X координата
- `DE` = Y координата

```asm
        ld      c, #03
        rst     #30
        bit     0, a
        jr      nz, .left_click
```

### `#83` — MOUSE REFRESH

Обновление состояния мыши (обычно вызывается из VSync).

---

## Ключевые моменты

> - `RST #08` — основной BIOS API, номер функции в `C`.
> - `RST #30` — Mouse API (`#00`–`#03`, `#83`).
> - Функции памяти `#C0`–`#C7` — выделение/освобождение RAM страниц.
> - Функции печати `#81`–`#8E` — работают с текстовой сеткой BIOS.
> - Функции дисков `#51`–`#5F` — прямой доступ к секторам; `HL:IX` = сектор (HL=старшие, IX=младшие биты LBA), `DE` = адрес буфера, `B` = кол-во секторов.
> - `CF=0` — успех, `CF=1` — ошибка в `A`.
