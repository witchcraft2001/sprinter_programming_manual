# 5.6 Аппаратный акселератор (blitter)

> **Навигация:** [← 5.5 Палитра](05_palette.md) | [Оглавление](../README.md) | [5.7 Double buffering →](07_double_buffer.md)

---

## Что такое акселератор

**Аппаратный акселератор** (blitter) — блок в FPGA ACEX 1K30, который выполняет **блочные операции с VRAM** быстрее, чем CPU через `LDIR`. Реализован в файле `ACCELER.TDF`.

### Ключевая особенность

Акселератор **перехватывает инструкции** вида `LD r,r` (где оба регистра — один и тот же), выполняемые на M1-цикле Z80. Эти инструкции в обычном Z80 — NOP (1 такт), но в Sprinter они становятся командами акселератору.

Z80 передаёт управление акселератору через FPGA, пока сам продолжает считать такты. Акселератор работает на **42 МГц независимо от turbo-режима Z80** — это важно: в turbo-режиме (21 МГц) акселератор не ускоряется.

---

## Команды акселератора

Из `ACCELER.TDF` — case `ACC_MODE[2..0]`:

| Mode | Инструкция | Опкод | Действие |
|------|-----------|-------|---------|
| 0 | `LD B,B` | `#40` | Выключить акселератор (HALT) |
| 1 | `LD C,C` | `#49` | Заполнить блок константой |
| 2 | `LD D,D` | `#52` | Установить размер блока (счётчик) |
| 3 | `LD E,E` | `#5B` | Вертикальное заполнение |
| 4 | `LD H,H` | `#64` | Двухбайтная функция |
| 5 | `LD L,L` | `#6D` | Копировать строку (горизонтально) |
| 6 | `HALT` | `#76` | Остановить (альтернатива) |
| 7 | `LD A,A` | `#7F` | Копировать вертикально |

Все эти инструкции в стандартном Z80 ничего не делают (или равны друг другу по эффекту). Acceлератор использует это свойство для "перехвата" на M1-cycle.

---

## Последовательность работы

```
┌───────────────────────────────────────┐
│  1. DI (запретить прерывания)         │
│  2. LD D,D                            │  ← установить размер блока
│  3. LD A, <size>                      │
│  4. LD <params> — HL, DE, и т.д.      │
│  5. LD L,L / LD C,C / LD A,A / ...    │  ← команда выполняется
│  6. EI (разрешить прерывания)         │
│  7. LD B,B (опционально — выкл.)      │
└───────────────────────────────────────┘
```

---

## Команда LD D,D — установка размера

```asm
; Установить длину блочной операции = 20 байт
        ld      d, d            ; команда "установить счётчик"
        ld      a, 20           ; A = размер
```

После этого все последующие команды акселератора (`LD C,C`, `LD L,L`, …) будут работать с блоками по 20 байт.

---

## Команда LD C,C — заполнить константой

```asm
; Заполнить блок в VRAM значением
; Предполагается, что размер уже установлен через LD D,D

        di
        ld      d, d            ; размер блока
        ld      a, 20

        ld      hl, dest_vram   ; HL = адрес в VRAM
        ld      a, #FF          ; A = значение заполнения
        ld      c, c            ; выполнить заполнение
        ei
```

### Полный пример: очистить строку экрана

```asm
ClearRow:
        di

        ld      d, d            ; установить размер
        ld      a, 40           ; 40 байт

        ld      hl, #C000       ; адрес в WIN3 (VRAM)
        ld      a, 0            ; цвет 0 (чёрный)
        ld      c, c            ; заполнить

        ei
        ret
```

---

## Команда LD L,L — копировать строку

Копирует `HL → DE`, длина задана предыдущим `LD D,D`.

```asm
; Скопировать 20 байт из RAM в VRAM
        di

        ld      d, d
        ld      a, 20           ; длина

        ld      hl, src_ram     ; HL = источник (RAM)
        ld      de, #C100       ; DE = назначение (VRAM)
        ld      l, l            ; копировать

        ei
```

### Пример из flappybird (`src/grx_utils.asm`)

```asm
ShowBitmapAcc:
        ex      af, af'
        ld      a, c
        ld      (.len), a       ; сохранить ширину
        IN A,(EmmWin.P1)
        push    af
        LD A,#50
        OUT (EmmWin.P1),A       ; WIN1 = VRAM

        ; ...

.loop:  push    bc
        push    af
        OUT (#89),A
        di
        ld      d, d            ; установить размер блока
        ld      a, 0            ; ??? (см. оригинал)
        ld      b, b            ; отключить (LD B,B)
        pop     af
        ; ...
```

---

## Команда LD E,E — вертикальное заполнение

Заполнить вертикальную полосу (несколько строк) константой:

```asm
        di
        ld      d, d            ; счётчик = высота
        ld      a, height

        ld      hl, start_addr
        ld      a, color
        ld      e, e            ; вертикальное заполнение
        ei
```

---

## Команда LD A,A — вертикальное копирование

Аналог `LD L,L`, но со сдвигом на **320 байт** (одна строка 320×256) между итерациями:

```asm
        di
        ld      d, d
        ld      a, height

        ld      hl, src
        ld      de, dst
        ld      a, a            ; вертикальное копирование
        ei
```

---

## Выключение акселератора и безопасное состояние

После окончания работы **обязательно** выключите акселератор и задвиньте PORT_Y в невидимую зону:

```asm
        ld      b, b                    ; LD B,B — выключить акселератор
        ld      a, $C0                  ; Y = 192 — за пределами видимой части
        out     (PORT_Y), a             ; иначе доступ к VRAM оставит «мусор»
```

Без `LD B,B` следующие `LD r,r` могут быть неправильно интерпретированы. Без записи `$C0` в `PORT_Y` следующие обращения к VRAM попадут на видимую строку и оставят видимые артефакты. См. раздел [5.1 Обзор видео / PORT_Y и «мусор» на экране](01_video_overview.md#важно-port_y-и-мусор-на-экране).

## Таблица размеров блоков

`LD D,D` устанавливает 8-битный счётчик (0–255). Это максимум **256 байт** за одну операцию. Для бо́льших объёмов нужен цикл.

---

## Прерывания и акселератор

**Критически важно:** во время операции акселератора **прерывания должны быть запрещены** (`DI`). Прерывание может повредить состояние акселератора. SDK всегда использует конструкцию:

```asm
        di
        ld      d, d
        ; ... параметры ...
        ld      l, l            ; или другая команда
        ei
```

## Специальные функции акселератора (FN_ACC)

Из `ACCELER.TDF`:

```
FN_ACC[1..0] = функция:
  0 → XMD = MD                 (просто копирование)
  1 → XMD = MD OR RAM          (логическое ИЛИ)
  2 → XMD = MD XOR RAM         (ИСКЛ. ИЛИ — XOR)
  3 → XMD = MD AND RAM         (логическое И)
```

Эти функции позволяют делать маскированное копирование (например, спрайты с прозрачностью через XOR).

Выбор функции через запись в специальный порт: из TDF — `FN_ACC[].ena = /M1M` с условием на `DI[7]=1 & DI[6]=0`, то есть опкодами `10XXXXXX` (например `LD D,(HL)` = `#56`).

---

## Производительность

| Операция | `LDIR` (Z80 7 МГц) | Акселератор |
|---------|--------------------|-------------|
| Копирование 256 байт | ~5.5 тыс. тактов = 780 мкс | ~256 тактов = 37 мкс |
| Заполнение 256 байт | аналогично | аналогично |
| Ускорение | 1× | ~20× |

В режиме 320×256 акселератор способен **полностью заполнить экран** (80 КБ) за ~10 мс, что позволяет 60+ FPS.

---

## Пример: полноэкранная очистка через акселератор

```asm
; Заполнить весь экран 320×256 цветом
; (быстрее, чем LDIR)

FastClearScreen:
        in      a, (#E2)
        push    af                  ; сохранить WIN3

        ld      b, 5                ; 5 страниц VRAM (80 КБ)
        ld      c, #50              ; начальная страница
.page:
        ld      a, c
        out     (#E2), a            ; WIN3 = VRAM страница

        ; Очистить 16 КБ через акселератор
        ld      e, 64               ; 64 итерации × 256 = 16384 байт
        ld      hl, #C000
.row:
        di
        ld      d, d                ; размер блока
        ld      a, 255              ; 255 байт (чуть меньше 256 для избежания edge case)
        ld      a, 0                ; цвет заливки
        ld      c, c                ; заполнение
        ei
        ld      a, l
        add     a, 255
        ld      l, a
        jr      nc, .no_carry
        inc     h
.no_carry:
        dec     e
        jr      nz, .row

        inc     c
        djnz    .page

        ld      b, b                ; выключить акселератор

        pop     af
        out     (#E2), a            ; восстановить WIN3
        ret
```

## Аппаратный акселератор не работает в ZX-режиме

Акселератор активен только в нативном режиме 320×256 (и частично в 640×256). В ZX Spectrum режиме инструкции `LD r,r` выполняются как обычные NOP.

---

## Ключевые моменты

> - Акселератор работает через перехват инструкций `LD r,r` на M1-цикле.
> - `LD D,D` — установить размер; `LD C,C` — заполнить; `LD L,L` — копировать; `LD E,E`/`LD A,A` — вертикальные варианты; `LD B,B` — выключить.
> - Работает на 42 МГц, **не ускоряется в turbo** (Z80 turbo — другое).
> - **Всегда `DI`** во время операции.
> - До ~20× быстрее `LDIR` для блочных операций с VRAM.
> - Функции маскирования: OR, XOR, AND (через FN_ACC).
> - Максимум **256 байт** за одну операцию (счётчик 8 бит).
> - Только в нативных режимах, не в ZX.
