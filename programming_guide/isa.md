# ISA шина и кэш-окно

Sprinter предоставляет два ISA-слота, подключённых через ПЛИС ACEX. Доступ осуществляется путём подмены окна `#C000-#FFFF` или нижних 16 КБ (кэш) на внешнее устройство. Эта логика описана в `SP2_ACEX.TDF` и задокументирована в [generated_doc/isa.md](../generated_doc/isa.md) и заметках «Работа с ISA» [docs/Programming ISA.odt](../docs/Programming%20ISA.odt).

## Последовательность включения

1. **Разрешить системное ПЗУ** (если оно отключено) и выставить карту портов стандартной последовательностью `OUT (#7C),0` → `OUT (#3C),0`.
2. **Активировать A20**: любая запись в порт `0x1B` (сигнал `ISA_A20`) разблокирует обращения к ISA/кэшу до сброса [generated_doc/isa.md].
3. **Включить карту `SC4`**. В «скопионовском» режиме `SC4` сбрасывается при RESET, поэтому перед любыми `OUT (0xF0),A` нужно записать `CNF_SC4=1` в системный порт `#3C/#7C` (см. `SPRINT00.ASZ`).
4. **Выбрать слот и режим**: запись в `ISA_PORT (0xF0..0xFF)` фиксирует биты `[2:1]` (слот: `00` — верхний, `01` — нижний) и `[1]` (0 — окно памяти, 1 — порты). Типичные значения:
   - `0xD0` — Slot 1, memory (`bit3..0 = 0000`)
   - `0xD2` — Slot 1, I/O (как в инструкции [docs/Programming ISA.odt])
   - `0xD4/0xD6` — Slot 2.
5. **Переназначить окно PAGE3**: запись в `PAGE3/#E2` значений `0xD0..0xD7` подмешивает ISA вместо RAM в `#C000-#FFFF`. Сохраните исходный `PAGE3`, чтобы восстановить карту после операций.
6. **(Опционально) включить кэш/ROM режимы**: чтение `IN A,(0xFB)` устанавливает `CASH_ON=1`, что отключает системный ROM и разрешает ISA-окно. Для обратного переключения прочитайте `0x7B` с `A7=0` [generated_doc/isa.md].

После этого любая операция чтения/записи по адресам `#C000-#FFFF` попадёт на ISA-шину. Для обращения к портам ISA используйте ту же область: «порт» с адресом `N` отображается как ячейка `#C000 + N` (см. пример ниже).

## Управление линиями RESET/AEN/A14..A19

Порт `ISA_DIR = 0x9FBD` (см. [docs/Programming ISA.odt]) контролирует дополнительные сигналы:

| Бит | Назначение |
|-----|-----------|
| 7   | RESET (1 = активный сброс устройства) |
| 6   | AEN (1 = DMA/CPU запрещён) |
| 5..0 | A19..A14 ISA-шины |

Типичный цикл инициализации:

```asm
ISA_DIR  EQU 09FBDh
SC_PORT  EQU 01FFDh
PAGE3    EQU 0E2h

OpenISA:
        LD   BC,SC_PORT
        LD   A,11h            ; разрешить системный контроллер
        OUT  (C),A
        LD   A,0D4h           ; Slot 1, I/O
        OUT  (PAGE3),A        ; подключить ISA в #C000
        XOR  A
        OUT  (ISA_DIR),A      ; снять RESET/AEN
        RET
```

После окончания работы восстановите исходный `PAGE3` и запишите `OUT (ISA_DIR),0C0h` (RESET) при необходимости.

## Доступ к памяти vs. портам

- **Память ISA**. В режиме `bit1=0` (значение `0xD0/0xD4`) чтение `LD A,(#C000+offset)` возвращает байт с адреса `offset` на ISA-шине. Чтобы использовать адреса выше `0x3FFF`, выставляйте старшие биты через `ISA_DIR` (`A14..A19`).
- **Порты ISA**. В режиме `bit1=1` каждая ячейка окна соответствует порту. Например, запись в `#C000` эквивалентна `OUT (PORT0),A` на карте. Используйте `OUTI`/`INI` для пакетной передачи, как показано в инструкциях [docs/Programming ISA.odt].

Активный ISA-окно блокирует обычную RAM и может добавить wait-state’ы (`WAIT_ROM`). После каждого обращения возвращайте исходный `PAGE3`, чтобы не ломать DSS/BIOS.

## Пример: чтение порта ISA

```asm
ReadISAPort:
        PUSH AF
        PUSH HL
        IN   A,(PAGE3)
        LD   (SaveP3),A
        LD   A,0D2h           ; slot 1, I/O
        OUT  (PAGE3),A
        LD   HL,#C000
        LD   A,(HL)           ; чтение порта #00
        LD   (PortValue),A
        LD   A,(SaveP3)
        OUT  (PAGE3),A
        POP  HL
        POP  AF
        RET
```

В режиме памяти меняйте `0xD2` на `0xD0` и используйте `LDIR` для чтения блоков. При необходимости укажите старшие адресные линии через `ISA_DIR` перед `LD (HL)`.

## Прерывания от ISA

Z84C15 предоставляет порт PIO-B (`internal port 1Eh`, внешний адрес `0x1F`) для обработки сигналов `IRQ/DRQ`. Биты соответствуют линиям слотов [sprinter_docs/z84c15/isa_int.md]:

| Бит | Сигнал |
|-----|-------|
| 0 | IRQ1 (слот 1) |
| 1 | IRQ2 (слот 2) |
| 2 | DRQ2 |
| 3 | DACK2 |
| 4 | DRQ1 |
| 5 | DACK1 |
| 6..7 | линии принтера |

Поскольку TR-DOS переназначает порт `#1F` на `#0F`, любые операции с PIO выполняйте через форму `LD BC,#001F / OUT (C),A` (или `IN A,(C)`), иначе сигнал уйдёт в джойстик. Настройка режима PIO и разрешение линий выполняется стандартными командами Z84C15 (режим 0 для входов prеrrupt).

## Связь с кэш-окном и ROM

ISA делит ресурсы со встроенным «кэшем» ROM. Сигналы `PRE_ROM`, `PRE_CASH` и `PRE_ISA` определяют, что активно на шине [generated_doc/isa.md]. Если одновременно разрешить ROM и ISA (`PRE_ROM=0`, `PRE_ISA=0`), ПЛИС вставит до четырёх тактов ожидания (`WAIT_ROM`). Поэтому при программном переключении страниц соблюдайте порядок:

1. Запретить ROM (`IN 0xFB` → `CASH_ON=1`).
2. Подключить ISA (`OUT PAGE3,0xD?`).
3. После работы вернуть ROM (`IN 0x7B` с `A7=0`).

Такой порядок гарантирует, что `CS_ISA` не пересекается с `CS_ROM`, и устройство не «повиснет».
