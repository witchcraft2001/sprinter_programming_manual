# Цветовая палитра и графические режимы

## Обзор

Sprinter поддерживает два принципиально разных способа вывода цвета:

| Режим | Регистр | Цветовой индекс | Источник |
|-------|---------|-----------------|---------|
| ZX-Spectrum | `ZX_PORT` | 4-битный (Ink/Paper + яркость) | атрибуты VRAM |
| Sprinter Graf | `DIR_PORT` | 8-битный (256 цветов) | пиксельные данные VRAM |

---

## Регистры управления цветом

### ZX_PORT[7:0] — режим ZX-Spectrum

Записывается через порты ZX-совместимых регистров страниц:

| Бит | Назначение |
|-----|-----------|
| `[5:0]` | `ZXS[5:0]` — номер страницы ZX VRAM (адрес в VRAM) |
| `6` | `ZX_SCREEN` — разрешить запись в ZX Screen (0=да) |
| `7` | `ZXA15` — инверсия A15 при ZX-адресации (для страницы 128K) |

### DIR_PORT[7:0] — режим Sprinter

| Бит | Назначение |
|-----|-----------|
| `0` | `SCR128` — выбор банка ZX Screen (0/1 для ZX128) |
| `3` | `RGMOD0` — дополнительный адресный бит (VRAM страница) |
| `4` | `RGMOD1` — `Screen Off` (1 = экран выключен, поддерживается только при `SCREEN_OFF="USE"`) |
| `[7:5]` | `BRD[2:0]` — цвет бордюра (ZX 3-бит: Ink[2:0]) |

---

## Регистры режима VIDEO (MODE0, MODE1, MODE2)

MODE-регистры **не пишутся через порты ввода-вывода**. Они захватываются из VRAM в фазе VCM=3 (цикл чтения MODE-строки). Аппаратный строб `LWR_MODE` фиксирует значения:

```
MODE0[7:0] ← VDM3[7:0]   (данные из VRAM-банка 3)
MODE1[7:0] ← VDM2[7:0]   (данные из VRAM-банка 2)
MODE2[7:0] ← VDM1[7:0]   (данные из VRAM-банка 1)
```

### MODE0[7:0] — управление режимом вывода

| Бит | Назначение |
|-----|-----------|
| `[3:0]` | `SVA[9:6]` — базовый адрес в VRAM (страница атрибутов) |
| `4` | Режим отображения: `0`=графика, `1`=ZX-атрибуты |
| `5` | Разрешение: `0`=320 пикс/строка, `1`=640 пикс/строка |
| `[7:6]` | Квадрант цвета / режим отображения (см. ниже) |
| `[7:4]==15` | `BORD=1` — фаза бордюра |

### MODE1[7:0] — адрес VRAM для Picture

| Бит | Назначение |
|-----|-----------|
| `[7:3]` | `SVA[17:13]` — старшие биты адреса VRAM пикселей |
| `[2:0]` | `SVA[12:10]` — группа строк |

### MODE2[7:0] — адрес VRAM для атрибутов / управление

| Бит | Назначение |
|-----|-----------|
| `[7:3]` | `SVA[17:13]` — адрес атрибутного банка |
| `[2:0]` | `SVA[12:10]` — адрес в атрибутном банке |
| `2` | `MODE2[2]=1` — удвоение горизонтального CTV счётчика |
| `1` | Управление CTV при двойном режиме |
| `0` | Управление CT при двойном режиме |

## Конвейер цвета: DCOL → BRVA → RBRVA → ZX_COL

Каждый пиксель проходит три регистровых ступени:

### Ступень 1: DCOL[7:0] — декодирование пикселя

```vhdl
-- В фазе BORD (MODE0[7:4]==15):
DCOL[] = (0b00, BRD[2:0], BRD[2:0])   -- цвет бордюра, оба нибла = border

-- В остальных фазах:
DCOL[] = D_PICX_[]                      -- данные из VRAM (8-битный цвет)
```

`DCOL` обнуляется при `BLANK=1` (невидимая часть кадра).

### Ступень 2: BRVA[7:0] — мультиплексирование пикселя

```vhdl
CASE (MODE0[4] or MODE0[5], CT2) IS
  WHEN "1X" => BRVA[7:0] = DCOL[]           -- графика или 640: все 8 бит
  WHEN "01" => BRVA[7:0] = (0000, DCOL[7:4])  -- ZX-атрибут, CT2=1: старший нибл
  WHEN "00" => BRVA[7:0] = (0000, DCOL[3:0])  -- ZX-атрибут, CT2=0: младший нибл
END CASE;
```

В режиме 320 пикселей (`MODE0[5]=0, MODE0[4]=0`) один байт DCOL содержит два пикселя: старший нибл выводится в первом такте (CT2=1), младший — во втором (CT2=0).

В режиме 640 пикселей (`MODE0[5]=1`) весь байт = один пиксель (8 бит цвета).

### Ступень 3: RBRVA[10:8] — расширенный цвет

```vhdl
CASE (BORD, X_MODE4) IS
  WHEN "X0" => RBRVA[10:8] = (0, X_MODE[7:6])  -- 2 бита из MODE0[7:6]
  WHEN "X1" => RBRVA[10:8] = (1, CTF4 & !BLANK, D_PIC0[7])  -- текстовый режим
END CASE;
```

`RBRVA[9:8].clrn = !BORD` и `RBRVA[10].prn = !BORD` — в фазе бордюра сбрасываются.

### Ступень 4: ZX_COL[3:0] — финальный выход на ЦАП/VGA

```vhdl
CASE (RBRVA[9:8], BRVA7) IS
  WHEN 0,1,4,7 => ZX_COL[] = (BRVA[6], BRVA[5:3])   -- верхняя группа
  WHEN 2,3,6,5 => ZX_COL[] = (BRVA[6], BRVA[2:0])   -- нижняя группа
END CASE;
```

Итого 4 бита `ZX_COL[3:0]` поступают на аналоговый ЦАП для формирования RGB-сигнала.

## Режимы пикселей

### Режим 320×200 (8-битный цвет)

- `MODE0[5]=0, MODE0[4]=0`
- 1 байт VRAM = 1 пиксель
- 256 цветов (8-битный индекс)
- Типичное разрешение: 320×200

Пиксели расположены последовательно в VRAM с базовым адресом из `MODE1`:
```
адрес_пикселя = SVA[17:0] + строка * 320 + столбец
```

### Режим 640×200 (8-битный цвет)

- `MODE0[5]=1, MODE0[4]=0`
- 1 байт VRAM = 1 пиксель
- Горизонтальное разрешение удвоено
- Типичное разрешение: 640×200

### Режим ZX-атрибутов

- `MODE0[4]=1`
- Формат как в ZX Spectrum: 8×8 символьные клетки
- BRVA содержит 4-битный ZX-цвет: `{яркость, ink2, ink1, ink0}` или `{яркость, paper2, paper1, paper0}`
- Выбор ink/paper зависит от бита пикселя в VRAM (CT2 чередует нибл)

## Цвет бордюра

Цвет бордюра задаётся через порт `0xFE` (ZX-совместимый):

```asm
LD A, border_color   ; биты [2:0] = цвет (0–7)
OUT (0xFE), A
```

В аппаратуре:
- `BORDER[2:0]` → `DIR_PORT[7:5]` → `BRD[2:0]`
- При `BORD=1`: `DCOL = (0b00, BRD[2:0], BRD[2:0])`
- `BORD` устанавливается когда `MODE0[7:4]==15` в VCM-цикле

---

## Управление гашением и прерыванием

```vhdl
BORD   = DFF((MODE0[7:4] == 15), LWR_COL)       -- фаза бордюра
BLANK  = DFF((BORD & MODE0[3] & MODE0[2]) or DIR_PORT[4], LWR_COL)  -- гашение
INTTX  = DFF((BORD & MODE0[3] & MODE0[2] & MODE0[0]), LWR_COL)      -- разрешение прерывания
INTT   = DFF(!(INTTX & (CTV[2:0] == 7)), CT5)   -- сигнал /INT в Z80
```

Прерывание от видео генерируется в конце строки `CTV[2:0]=7` при `INTTX=1`.

## Квадранты цвета MODE0[7:6]

При `X_MODE4=0` (режим графики): `RBRVA[10:8] = (0, MODE0[7], MODE0[6])`.

Значения `MODE0[7:6]` определяют «цветовой квадрант» — базовые 2 бита для финального 4-битного цветового пространства ZX_COL:

| `MODE0[7:6]` | RBRVA[9:8] | Описание |
|--------------|-----------|----------|
| `00` | `00` | Квадрант 0 (тёмные цвета) |
| `01` | `01` | Квадрант 1 |
| `10` | `10` | Квадрант 2 |
| `11` | `11` | Квадрант 3 (яркие цвета) |

Смена квадранта меняет «видимый» диапазон 256-цветной палитры.

## Двойная адресация VRAM (DOUBLE_CAS)

При `DOUBLE_CAS=1` (от акселератора) каждое обращение к VRAM читает 32-бита (4 байта одновременно). Это используется для ускоренной заливки и блитинга через акселератор.

Мультиплексор V_EN[3:0] выбирает, какой из четырёх байт записывается:
```vhdl
V_EN3 = DFF(!(!VXA1 & (!VXA0 or DOUBLE)), CLK42, F_WR)
V_EN2 = DFF(!(!VXA1 & (VXA0 or DOUBLE)), CLK42, F_WR)
V_EN1 = DFF(!(VXA1 & (!VXA0 or DOUBLE)), CLK42, F_WR)
V_EN0 = DFF(!(VXA1 & (VXA0 or DOUBLE)), CLK42, F_WR)
```

При `DOUBLE=1` все 4 байта записываются одновременно (заливка).

## Аппаратный курсор мыши

`VIDEO2.TDF` включает аппаратный курсор мыши, хранящийся в `MOUSE.MIF` (256×16 LPM_RAM):

```vhdl
MS_PNT = DFF(((MS_X == B"100000XXXX") & (MS_Y == B"100000XXXX")), CLK42)
BRVA[].clrn = !MS_POINT    -- курсор белый (сброс в 0)
BRVA[].prn  = !MS_POINT2   -- курсор инвертированный (установка в 1)
```

Курсор накладывается поверх изображения на пиксельном уровне. Данные формы курсора из `MOUSE.MIF`: `MS_DAT.q0` = белая точка, `MS_DAT.q1` = инвертированная точка.
