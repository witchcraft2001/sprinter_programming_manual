# 5.5 Палитра и порт #89

## Цветовой pipeline Sprinter

```
Индекс пикселя (8 бит, 0..255)
        │
        ▼
┌─────────────────────┐
│  Палитра (256 × 18) │   ← RAM палитры в FPGA (256 записей по 3 байта R,G,B × 6 бит)
└────────┬────────────┘
         │
         ▼
    R(6) G(6) B(6)
         │
         ▼
   ЦАП (SCART / VGA)
```

Каждый пиксель хранит **8-битный индекс** в палитре. Палитра содержит 256 записей, каждая — 18-битный цвет (6+6+6), что даёт **16 777 216 возможных цветов**.

---

## Порт #89 — универсальный регистр видео

Порт `#89` многофункционален. Его поведение зависит от контекста:

1. **Запись индекса цвета** для последующего ввода RGB.
2. **Запись компонент палитры** R, G, B последовательно.
3. **Запись Y-координаты** строки при выводе спрайтов/битмапов.
4. **Включение режимов COVOX** (для `#89` COVOX подмножество).

### Константы

```asm
PORT_Y          EQU #89     ; основной порт для палитры и Y-координаты
RGADR           EQU PORT_Y  ; алиас
```

## Установка элемента палитры

### Способ 1: через BIOS (рекомендуется)

```asm
Bios.SetPalette EQU #A4

; Установить цвет 10 на ярко-оранжевый (63, 32, 0)
        ld      hl, 10              ; индекс цвета
        ld      e, 63               ; R
        ld      d, 32               ; G
        ld      a, 0                ; B
        ld      c, Bios.SetPalette  ; #A4
        rst     #08
```

### Способ 2: прямая запись через #89 (SDK lib_startup.asm)

Анализ кода SDK показывает, что установка палитры делается через последовательность записей в порт `#89` с предварительной командой `#C0`:

```asm
; Установить весь экран на единый цвет (SDK _clear_screen)
        ld      a, #C0              ; команда "установить Y-координату"
        out     (PORT_Y), a

; Формат: после #C0 идут данные строк
; Точная последовательность зависит от режима
```

Для полноценной установки палитры через порты требуется знание точного протокола, который описан в SDK. Проще использовать BIOS `#A4`.

## Установка целой палитры

```asm
; Установить 256 цветов из таблицы в RAM
; Формат таблицы: 256 × (R, G, B)
SetFullPalette:
        ld      hl, palette_data
        ld      b, 0                ; индекс (начинаем с 0)
.loop:
        push    bc
        push    hl

        ; Вычитать R, G, B из таблицы
        ld      e, (hl) : inc hl    ; R
        ld      d, (hl) : inc hl    ; G
        ld      a, (hl)             ; B

        ; Вызвать BIOS SET_PAL
        ld      h, 0
        ld      l, b                ; HL = индекс
        push    hl
        pop     hl
        ld      c, #A4
        rst     #08

        pop     hl
        inc     hl : inc hl : inc hl    ; следующий цвет
        pop     bc
        inc     b
        jr      nz, .loop
        ret

; Палитра: 256 цветов × 3 байта (R, G, B)
palette_data:
        db  0, 0, 0         ; 0: чёрный
        db  63, 0, 0        ; 1: красный
        db  0, 63, 0        ; 2: зелёный
        db  0, 0, 63        ; 3: синий
        ; ... ещё 252 цвета ...
```

## Чтение палитры

BIOS не предоставляет функцию чтения палитры напрямую. Обычно программа **хранит копию** установленной палитры в RAM.

## Стандартные палитры

### Палитра ZX Spectrum (16 цветов)

```asm
; Первые 16 цветов палитры — совместимые с ZX Spectrum
; (для работы в ZX-режиме и эмуляции внешнего вида)

zx_palette:
;       R,  G,  B    индекс   цвет
        db  0,  0,  0       ; 0  чёрный
        db  0,  0,  42      ; 1  синий
        db  42, 0,  0       ; 2  красный
        db  42, 0,  42      ; 3  пурпурный
        db  0,  42, 0       ; 4  зелёный
        db  0,  42, 42      ; 5  циан
        db  42, 42, 0       ; 6  жёлтый
        db  42, 42, 42      ; 7  белый
        db  0,  0,  0       ; 8  чёрный bright
        db  0,  0,  63      ; 9  яркий синий
        db  63, 0,  0       ; 10 яркий красный
        db  63, 0,  63      ; 11 яркий пурпурный
        db  0,  63, 0       ; 12 яркий зелёный
        db  0,  63, 63      ; 13 яркий циан
        db  63, 63, 0       ; 14 яркий жёлтый
        db  63, 63, 63      ; 15 яркий белый
```

### Серая градация

```asm
; 256 оттенков серого (плавный градиент)
        ld      b, 0
.loop:
        ld      a, b
        srl     a : srl a           ; 0..63 (6 бит)
        ld      e, a                ; R
        ld      d, a                ; G
        ; B уже в регистре A
        ld      h, 0
        ld      l, b                ; индекс
        push    bc
        ld      c, #A4
        rst     #08
        pop     bc
        inc     b
        jr      nz, .loop
```

## Обновление палитры из SDK

SDK содержит функцию `update_pal`, которая применяет изменения палитры на VSync:

```asm
; Фрагмент lib_startup.asm
update_pal:
        ld      a, (palDirty)
        or      a
        ret     z                   ; нечего обновлять

        ; ... копирование _palette в аппаратную палитру ...

        xor     a
        ld      (palDirty), a
        ret
```

Это оптимизация — изменения накапливаются в RAM-буфере `_palette`, а применяются все сразу в начале кадра.

## Применение палитры к элементам

```asm
; Установить фон синим (цвет 1), текст белым (цвет 15)
        ; Цвет 0 — чёрный (фон)
        ld      hl, 0
        ld      e, 0 : ld d, 0 : ld a, 0
        ld      c, #A4 : rst #08

        ; Цвет 1 — синий
        ld      hl, 1
        ld      e, 0 : ld d, 0 : ld a, 63
        ld      c, #A4 : rst #08

        ; Цвет 15 — белый
        ld      hl, 15
        ld      e, 63 : ld d, 63 : ld a, 63
        ld      c, #A4 : rst #08
```

## Палитровые эффекты

### Цветовой циклинг

Быстрое изменение одного элемента палитры даёт анимацию без перерисовки пикселей:

```asm
; В цикле: менять цвет 5 по радуге
CycleColor5:
        ld      a, (cycle_counter)
        inc     a
        ld      (cycle_counter), a

        ; Простое: A → R, 63-A → G, 0 → B
        ld      e, a                ; R
        ld      d, a
        neg
        add     a, 63
        ld      d, a                ; G = 63 - A
        xor     a                   ; B = 0

        ld      hl, 5
        ld      c, #A4
        rst     #08
        ret

cycle_counter: db 0
```

### Фейдинг

Постепенное затемнение всех цветов:

```asm
; Умножить все компоненты палитры на (fade_level / 63)
; Требует сохранения оригинала в RAM
FadePalette:
        ld      hl, palette_copy
        ld      b, 0
.loop:
        push    bc

        ld      e, (hl) : inc hl    ; R_orig
        ld      a, (fade_level)
        call    MulDiv63
        ld      e, a                ; R = R_orig * fade / 63

        ; ... аналогично G, B ...

        ld      h, 0
        ld      l, b
        ld      c, #A4
        rst     #08

        pop     bc
        inc     b
        jr      nz, .loop
        ret
```

## Ключевые моменты

> - Палитра: 256 записей × 18 бит (R6, G6, B6).
> - Основной способ установки: `BIOS PIC_SET_PAL #A4` (HL = индекс, E = R, D = G, A = B).
> - Порт `#89` (`PORT_Y`) — прямая работа возможна, но требует знания протокола.
> - Значения компонент: 0–63 (6 бит каждый).
> - SDK использует технику "dirty flag" для обновления палитры на VSync.
> - Эффекты: цветовой циклинг, фейдинг, палитровая анимация.
