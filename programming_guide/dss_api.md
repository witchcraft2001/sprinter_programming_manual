# DSS API (RST #10)

Системные сервисы DSS вызываются инструкцией `RST #10h`, номер функции кладётся в регистр `C`, остальные параметры — в `A/B/DE/HL/IX`. Документация `r_dsslist.htm` показывает это на примере простейшей программы, печатающей строку и вызывающей `EXIT` [sprinter_dss/docs/r_dsslist.htm:13-35](../sprinter_dss/docs/r_dsslist.htm#L13). Возврат ошибки сигнализируется флагом переноса (`CF=1`) и кодом в `A`; успешный вызов сбрасывает CF и возвращает данные в тех же регистрах, что описаны в таблицах.

Ниже приведены все вызовы DSS v1.55, сгруппированные по категориям. Формат таблицы: код, назначение, входные и выходные параметры.

## Практические приёмы

- **Обёртка CallDss** — позволяет вызывать любую функцию, не портя видеостраницу. Сохраняет `A`, текущие страницы `PAGE1/PAGE2`, переносит стек на безопасный буфер `SafeStack`, выполняет `RST #10` и возвращает всё назад. Исходник: [texteditor/routines.asm:400-430](../texteditor/routines.asm#L400).
- **Работа с файлами и директориями** — Flappy Bird использует `Dss.AppInfo` и `Dss.ChDir`, чтобы найти каталог ресурсов, после чего вызывает `Dss.GetMem` и читает каждый файл через `LoadResource` [flappybird/src/fbird.asm:12-80](../flappybird/src/fbird.asm#L12). Ошибки печатаются через `Dss.PChars`, а на выходе освобождается память и курсор.
- **Окна и экран** — `Dss.WinCopy`/`Dss.WinRest` копируют прямоугольник 32×32 знакомест с экрана в буфер и обратно. Редактор сохраняет координаты курсора через `Dss.Cursor`, переносит окно в безопасный буфер (в том числе с учётом дополнительных страниц), и после выхода восстанавливает курсор `Dss.Locate` [texteditor/routines.asm:116-200](../texteditor/routines.asm#L116).
- **Память** — перед маппингом сторонних страниц берите блок через `Dss.GetMem` и храните дескриптор в глобальной переменной, чтобы освободить его `Dss.FreeMem` на выходе. `Dss.SetMem` можно использовать для увеличения блока без повторного выделения, если нужно загрузить больше ресурсов [sprinter_dss/docs/r_dsslist.htm:417-420](../sprinter_dss/docs/r_dsslist.htm#L417).
