# Видеотайминги и VCM-машина состояний

## Тактовая частота и счётчики

Весь видеотракт синхронизирован от `CLK42` (~42 МГц). Три счётчика формируют позицию пикселя:

| Счётчик | Разрядность | Период | Назначение |
|---------|------------|--------|-----------|
| `CT[5:0]` | 6 бит | 7 тактов CLK42 | Субцикл (состояния VCM) |
| `CTH[5:0]` | 6 бит | 56 строк (CTH=0..55) | Позиция по горизонтали |
| `CTV[8:0]` | 9 бит | 320 строк (CTV=0..319) | Позиция по вертикали |
| `CTF[6:0]` | 7 бит | счёт кадров | Кадровый счётчик |

### Горизонтальная синхронизация

```vhdl
CTH[].ena = DFF(((CT[5:2] == 15) & CT0), CLK42)
IF CTH[] == 55 THEN CTH[] = 0 ELSE CTH[] = CTH[] + 1
```

Период строки: 56 × 7 = 392 такта CLK42 ≈ 9,33 мкс при 42 МГц.

Начальное значение CTH при синхронизации: `0x32` (50).

### Вертикальная синхронизация

```vhdl
CTV[].ena = DFF(((CT[5:2] == 15) & CT0 & (CTH[] == 48)), CLK42)
IF CTV[] == 319 THEN CTV[] = 0 ELSE CTV[] = CTV[] + 1
```

Всего строк в кадре: 320. Начальное значение CTV: `0x122` (290).

Частота кадров: 42 МГц / (56 × 7 × 320) ≈ **33,6 Гц**.

### Синхронизация с внешним источником (COPY_SINC)

```vhdl
COPY_SINC_H = DFF((!SINC_1M & DFF(SINC_1M, CLK42)), CLK42)
COPY_SINC_V = DFF((!SINC_2M & DFF(SINC_2M, CLK42)), CLK42)
```

При получении импульсов SINC_1/SINC_2 от FDD-интерфейса счётчики CTH/CTV принудительно устанавливаются в начальные значения (CTH=0x32, CTV=0x122), обеспечивая синхронизацию.

## CT[2:0] — субцикл и VCM

Каждые 7 тактов CLK42 образуют один «макроцикл» памяти. `CT[2:0]` определяет субтакт:

```vhdl
CASE CT[2:0] IS
  WHEN 0 => CT = 1
  WHEN 1 => CT = 2
  WHEN 2 => CT = 4  -- состояние 3 пропускается
  WHEN 3 => CT = 4  -- недостижимо в нормальной работе
  WHEN 4 => CT = 5
  WHEN 5 => CT = 6
  WHEN 6 => CT = 0
END CASE;
```

Последовательность CT[2:0]: **0 → 1 → 2 → 4 → 5 → 6 → 0 → ...** (7 состояний, пропускается 3).

CT[5:3] инкрементируется каждые 7 тактов.

---

## VCM[2:0] — машина состояний видео

VCM — это отдельная машина состояний, синхронизируемая с CT[2:0]. Переходы:

```vhdl
CASE CT[2:0] IS
  WHEN 6 => VCM = 5   -- CT=6  → VCM=5
  WHEN 0 => VCM = 1   -- CT=0  → VCM=1
  WHEN 1 => VCM = 4   -- CT=1  → VCM=4
  WHEN 2 => VCM = 3   -- CT=2  → VCM=3
  WHEN 4 => VCM = 2   -- CT=4  → VCM=2
  WHEN 5 => VCM = 0   -- CT=5  → VCM=0
END CASE;
```

Сопоставление CT и VCM:

| CT[2:0] | VCM[2:0] | Действие |
|---------|---------|---------|
| `6` | `5` | Подготовка адреса (BRVA pipeline) |
| `0` | `1` | Загрузка адреса SVA → VLA |
| `1` | `4` | Считывание MODE-строки (если VCM2=0) |
| `2` | `3` | Фаза MODE: захват MODE0/1/2 из VRAM |
| `4` | `2` | Запись в VRAM (если MXWE=1) |
| `5` | `0` | Считывание пикселей из VRAM |

## VCM-фазы подробно

### VCM=0 (CT=5): Загрузка RAS (начало цикла DRAM)

```vhdl
WHEN VCM[1:0]==0 =>
  VLA[] = (BRVA[7:0], 1,1,1,1,1, RBRVA[10:8], 0,0)
  V_CST[] = (1,0)
  V_WE = 1; V_WEX = 1
  TSN_W3 = X_MODE_BOND
```

Устанавливается адрес строки VRAM из BRVA/RBRVA. RAS активируется в следующем цикле.

### VCM=1 (CT=0): Пиксельная строка / цвет

```vhdl
WHEN VCM[1:0]==1 =>
  WR_PIC = !VCM2        -- запись пикселей (если не fase MODE)
  WR_COL =  VCM2        -- запись цвета (если fase MODE)
  VLA[]   = SVA[]
  V_CST[] = (1,0)
```

Захватывает данные из VRAM для пикселей (WR_PIC=1 при VCM2=0) или из атрибутной памяти (WR_COL=1 при VCM2=1).

### VCM=2 (CT=4): Запись из CPU/акселератора

```vhdl
WHEN VCM[1:0]==2 =>
  VLA[]   = VXA[17:0]
  V_CST[] = (!VXA18, VXA18) or MXWE
  V_WE    = MXWE
  V_WEX   = 0
  V_WET[] = MXWE or !(byte_select_from_VXA[1:0])
```

Если `MXWE=1` (запрос записи от CPU), данные из `VXA` записываются в VRAM.

### VCM=3 (CT=2): Захват MODE-регистров

```vhdl
WHEN VCM[1:0]==3 =>
  WR_PIC  = MODE0[5]   -- разрешение 640
  VLA[]   = (DIR_PORT3, CTH[5:0], CT5, 1,1, CTV[8:3], 0,0)
  WR_MODE = !(CT[5:3] == 0b111) & !(CT4 & CT3 & !MODE0[5])
  V_CST[] = (1,0)
```

В этой фазе VRAM адресуется по координатам текущей строки для чтения «строки режима». По импульсу `LWR_MODE` регистры MODE0/1/2 захватывают данные:

```vhdl
MODE0[7:0] ← VDM3[7:0]
MODE1[7:0] ← VDM2[7:0]
MODE2[7:0] ← VDM1[7:0]
```

Адрес MODE-строки зависит от положения CTV (текущей строки кадра).

## Формирование адреса VRAM (SVA/RSVA)

Логика адресации зависит от режима отображения (`!VCM2` и `MODE0[4]`):

### Режим графики (`MODE0[4]=0, !VCM2=1`)

```vhdl
RSVA[8:0] = (M_CTV[2:0], MODE1[2:0], M_CT[5:3])
SVA[17:13] = MODE1[7:3]
SVA[9:6]   = MODE0[3:0]
```

Где:
```vhdl
M_CTV[2:0] = (!MODE2[2] & CTV[2:0]) or (MODE2[2] & (MODE2[1], CTV[2:1]))
M_CT[5:3]  = (!MODE2[2] & (CT5, !CT[4:3])) or (MODE2[2] & (MODE2[0], !CT[5:4]))
```

`MODE2[2]=1` активирует «двойной» режим (каждая строка CTV используется дважды).

### Режим ZX-атрибутов (`MODE0[4]=1, VCM2=1`, фаза атрибут)

```vhdl
RSVA[8:0] = (MODE2[2:0], SCR128, 1,1, 0, !MODE0[7:6])
SVA[17:13] = MODE2[7:3]
```

### Режим ZX-пикселей (`MODE0[4]=1, !VCM2=1`, фаза пиксель)

```vhdl
RSVA[8:0] = (MODE1[2:0], SCR128, MODE0[7:6], CTV2, !CTV1, !CTV0)
SVA[17:13] = MODE1[7:3]
```

## BLANK и BORD

```vhdl
BORD  = DFF((MODE0[7:4] == 15), LWR_COL)
BLANK = DFF((BORD & MODE0[3] & MODE0[2]) or DIR_PORT[4], LWR_COL)
```

- `BORD=1`: текущий пиксель — бордюр (DCOL = цвет бордюра)
- `BLANK=1`: гашение (DCOL обнуляется через `DCOL[].clrn = !BLANK`)
- `DIR_PORT[4]=1` (RGMOD1): принудительное гашение экрана

---

## Видеопрерывание (INTT)

```vhdl
INTTX = DFF((BORD & MODE0[3] & MODE0[2] & MODE0[0]), LWR_COL)
INTT  = DFF(!(INTTX & (CTV[2:0] == 7)), CT5)
```

Прерывание генерируется в конце строки `CTV[2:0]=7` при условии `INTTX=1`.

`INTT` — активный низкий сигнал: `INTT=0` → `/INT` активен на Z80.

---

## Сигналы VGA VRAM CS

```vhdl
V_CS1 = VGA_SW      -- выбор VRAM-банка (для VGA второго монитора)
V_CS0 = GND         -- основной VRAM всегда активен
```

Переключение `V_CS1` через `VGA_SW` (F12+Shift) позволяет направить вывод на второй монитор.

## Временна́я диаграмма (упрощённо)

```
CLK42: __|‾|_|‾|_|‾|_|‾|_|‾|_|‾|_|‾|_|‾|_|‾|_|‾|_|‾|_|‾|_|‾|_
CT[2:0]:  0   1   2   4   5   6   0   1   2   4   5   6
VCM[2:0]: 1   4   3   2   0   5   1   4   3   2   0   5
          ↑   ↑   ↑   ↑   ↑   ↑
          PIC COL MOD WR  RAS PRE

CTH:      меняется каждые 7 CLK42 (392 такта = 1 строка)
CTV:      меняется каждые 56 CTH (= каждые 392×56 CLK42 = 1 кадр)
```

Обозначения фаз:
- **PIC** — загрузка пикселей из VRAM
- **COL** — загрузка цвета/атрибутов
- **MOD** — захват MODE-регистров
- **WR**  — запись от CPU/акселератора
- **RAS** — выставление адреса строки DRAM
- **PRE** — подготовка (precharge)
