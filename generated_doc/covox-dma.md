# Covox / DMA-аудио (CBL)

## Обзор

Аудиоподсистема Sprinter основана на двухканальном 8-битном ЦАП с DMA-буфером (`CBL`). Данные из программы или акселератора накапливаются в кольцевом буфере, считываются аппаратно с заданной частотой и выводятся на цифровой аудиовыход I²S (сигналы `DAC_DATA`, `DAC_WS`, `DAC_BCK`).

---

## Порты управления

| Порт | Направление | Назначение |
|------|------------|-----------|
| `0x89` | Запись | `CBL_XX[7:0]` — регистр управления Covox |
| `0x88` | Запись | Данные в CBL-буфер (один байт) |

### Регистр CBL_XX (порт 0x89)

```vhdl
CBL_XX[].clk = /IOWR;
CBL_XX[].ena = DFF((DCPP[] == 0x89), CLK42)
CBL_XX[]     = D[]
```

| Бит | Имя | Назначение |
|-----|-----|-----------|
| `7` | `CBL_MODE` | `1`=DMA-режим (буферный); `0`=прямая подача D[7:0] |
| `6` | `CBL_STEREO` | `1`=стерео (указатель `CBL_CNT` шагает по 2); `0`=моно |
| `5` | `CBL_MODE16` | `1`=16-битный режим (2 байта на выборку) |
| `4` | `CBL_INT_ENA` | `1`=разрешить прерывание при заполнении буфера |
| `[3:0]` | `CBL_XX[3:0]` | Коэффициент делителя частоты — индекс в `CBL_TAB` |

---

## Таблица частот (CBL_TAB)

| `CBL_XX[3:0]` | Делитель (`CBL_TAB`) | Частота воспроизведения |
|---------------|---------------------|------------------------|
| `0` | 13 | ~16 кГц (моно/стерео) |
| `1` | 9 | ~22 кГц (моно/стерео) |
| `2–7` | 0 | зарезервировано |
| `8` | 27 | 7,8125 кГц |
| `9` | 19 | 10,9375 кГц |
| `10` | 13 | 15,625 кГц |
| `11` | 9 | 21,875 кГц |
| `12` | 6 | 31,25 кГц |
| `13` | 4 | 43,75 кГц |
| `14` | 3 | 54,6875 кГц |
| `15` | 1 | 109,375 кГц |

Базовая тактовая частота делителя = `CTH1` (производная от CLK42).

## Структура кольцевого буфера

CBL — это `LPM_RAM` 256×16 бит (256 16-битных слов = 512 байт):

```
CBL[255:0] — кольцевой буфер (256 позиций × 16 бит)
CBL_WA[7:0] — указатель записи (write address)
CBL_CNT[7:0] — указатель чтения (read counter)
CBL_IND = CBL_CNT7 XOR CBL_WA7  — индикатор полноты буфера
```

### Запись в буфер

```vhdl
-- Через порт 0x88:
CBL_WR = DFF((DCPP == 0x88), CLK42) & !/IOWR

-- Или через акселератор (DMA):
CBL_WR = DFF((DECODE.PAGE[7:0] == 0xFD), CLK42, CBL_INT_ENA & ACC.ACC_DIR1)
       & !DFF((DECODE.MC_WRITE or DECODE.CAS), CLK42, /RESET)
```

При записи (`CBL_WR=1`, `CBL_WAE=0`):
```vhdl
CBL.wraddress[] = ((!A[15:8]) & !CBL_INT_ENA) XOR CBL_WA[]
CBL.data[]      = (ACC.MDO15 XOR CBL_MODE16, ACC.MDO[14:8], CBD[])
```

**Источник данных для записи:** `ACC.MDO[15:0]` (выход акселератора/шины данных).

### Режим 8 бит (обычный, `CBL_MODE16=0`)

- `CBD[7:0]` не используется
- Слово в буфере: `{D[7] XOR 0, D[6:0], 00000000b}`
- Старший байт = данные; младший байт = 0

### Режим 16 бит (`CBL_MODE16=1`)

- `CBD[7:0]` — первый байт 16-битной пары, захваченный при чётном `CBL_WR`
- `CBL_WAE` — flip-flop для чётных/нечётных байт
- Слово в буфере: `{D[7] XOR 1, D[6:0], CBD[7:0]}`
- `CBL_WA` продвигается только при `CBL_WAE=0` (второй байт пары)

```vhdl
CBL_WAE = CBL_MODE16 & DFF(!CBL_WAE, !CBL_WR,, CBL_INT)
CBD[].clk = !CBL_WR; CBD[].ena = CBL_WAE; CBD[].d = ACC.MDO[7:0]
```

## Стерео

В стерео-режиме (`CBL_STEREO=1`) буфер интерлевируется:

- Чётные индексы → левый канал (L)
- Нечётные индексы → правый канал (R)
- `CBL_CNT` шагает по **2** за каждый период частоты

```vhdl
CASE (CBL_STEREO, (CBL_CTX == 0)) IS
  WHEN 0,2 => CBL_CNT = CBL_CNT         -- счётчик не меняется
  WHEN 1   => CBL_CNT = CBL_CNT + 1     -- моно: шаг 1
  WHEN 3   => CBL_CNT = CBL_CNT + 2     -- стерео: шаг 2
END CASE;
```

Адрес чтения:
```vhdl
CBL.rdaddress[] = (CBL_CNT[7:1], (CBL_CNT0 & !CBL_STEREO) or (AUDIO_CH & CBL_STEREO))
```

`AUDIO_CH` — сигнал чередования каналов (формируется из CTH0/CT[5:3]).

## Прерывание

```vhdl
CBL_INT = DFF(GND, !CBL_CNT6,, (CBL_INT_ENA & (/IO or /M1)))
```

Прерывание устанавливается при переполнении `CBL_CNT[6]` (т.е. каждые 64 выборки в моно, 128 — в стерео). Очищается по `/IO` или `/M1`.

Флаг полуполноты: `CBL_IND = CBL_CNT7 XOR CBL_WA7` — если `1`, то указатель записи «опережает» указатель чтения на более чем половину буфера (128 позиций).

---

## DMA из акселератора

Когда `CBL_INT_ENA=1` и `ACC.ACC_DIR1=1`, акселератор может автоматически записывать данные в CBL-буфер без участия CPU:

```vhdl
CBL_WR = ... or (DFF((DECODE.PAGE == 0xFD), CLK42, CBL_INT_ENA & ACC.ACC_DIR1)
              & !DFF((DECODE.MC_WRITE or DECODE.CAS), CLK42, /RESET))
```

Страница памяти `0xFD` работает как «DMA-окно»: чтение из RAM автоматически триггерит запись в CBL.

## Путь данных к ЦАП (I²S выход)

```vhdl
AY_CHS[15:0] = (GND, AY_FULL[]) + (VCC, CBL_R[15:5])   -- микс AY + Covox
AUDIO_R[15:0] = AY_CHS[15:0]                             -- выходной регистр
DAC_DATA = DFF(AUDIO_R15, CT2)                           -- I²S данные (MSB first)
DAC_WS   = DFF(CTH1, CLK42)                              -- I²S word select (L/R)
DAC_BCK  = DFF(CT2, CLK42)                               -- I²S bit clock
```

`CBL_R[15:0]` — зарегистрированные данные из CBL (при `CBL_MODE=1`) или прямые D[7:0] (при `CBL_MODE=0`):

```vhdl
CASE CBL_MODE IS
  WHEN 0 => CBL_R[] = (D[7:0], 0x00)   -- прямая подача
  WHEN 1 => CBL_R[] = CBL.q[]          -- из DMA-буфера
END CASE;
```

## Примеры программирования

### Прямой вывод (без DMA)

```asm
; Включить прямой режим (CBL_MODE=0)
LD A, 0
OUT (0x89), A

; Постоянная подача значений в реальном времени:
loop:
    LD A, (sample_ptr)     ; взять сэмпл
    OUT (0x88), A          ; передать на ЦАП
    ; ... небольшая задержка ...
    JR loop
```

### DMA-режим (предпочтительный)

```asm
; Настройка: 22 кГц, стерео, DMA, разрешить прерывание
; CBL_MODE=1, CBL_STEREO=1, CBL_MODE16=0, CBL_INT_ENA=1, частота=1 (22 кГц)
LD A, 0b11010001    ; бит7=1 (DMA), бит6=1 (стерео), бит5=0, бит4=1 (int), [3:0]=1
OUT (0x89), A

; Записать блок сэмплов в буфер (стерео: L, R, L, R...)
LD HL, samples
LD B, 128
fill_loop:
    LD A, (HL)
    OUT (0x88), A    ; L-сэмпл
    INC HL
    LD A, (HL)
    OUT (0x88), A    ; R-сэмпл
    INC HL
    DJNZ fill_loop

; Обработчик прерывания: подгружать следующие 64 сэмпла
```

### 16-битный режим

```asm
; CBL_MODE=1, CBL_MODE16=1
LD A, 0b10100001    ; бит7=1, бит5=1 (16-bit), частота=1
OUT (0x89), A

; Запись 16-битных сэмплов (LSB, MSB):
LD A, sample_lo
OUT (0x88), A       ; первый байт (CBD)
LD A, sample_hi
OUT (0x88), A       ; второй байт → запись в буфер, CBL_WA++
```
