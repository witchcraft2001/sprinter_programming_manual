# Видеопамять и режимы вывода

## Архитектура видеоподсистемы

Видеоконтроллер `VIDEO2.TDF` работает параллельно с CPU, непрерывно читая DRAM через 4 независимые 8-битные шины (`VD0–VD3`). CPU и ускоритель обращаются к той же DRAM через сигналы `V_WR[3:0]` и `V_WEN[3:0]`.

```
     Z80 / ACCELER
          │
     DCP (MA[], RAS/CAS)
          │
    ┌─────┴──────────────────────────────┐
    │           DRAM 512 КБ              │
    │  Bank 0    Bank 1    Bank 2    Bank 3│
    └──┬──────────┬──────────┬──────────┬─┘
       │VD0       │VD1       │VD2       │VD3
       └──────────┴──────────┴──────────┘
                       │
                  VIDEO2 (VA[15:0])
                       │
                 DAC → R/G/B → монитор
```

Адресная шина видеоконтроллера: `VA[15:0]` (выход `SVIDEO.vao[]`), стробы `V_CS[1:0]`.

## Формирование адреса VAI[19:0]

`VAI` (Video Address In) — 20-битный адрес, по которому видеоконтроллер обращается к DRAM:

```vhdl
SVIDEO.vai[] = (DECODE.GA[9:0], ACC.AO[9:0])
```

Верхние 10 бит приходят из `DCP.GA[]`, нижние 10 — из ускорителя (`AO[]`, который обычно совпадает с адресом Z80).

**Формирование GA[] в DCP:**

```vhdl
CASE GRAF IS
    WHEN 0 => GA[] = (0, 0, MEM.q[3:0], A[13:10])  -- обычная страница
    WHEN 1 => GA[] = (1, G_LINE[8:0])               -- графический режим
END CASE;
```

- `GRAF=0`: верхние биты из таблицы DCP (страничная адресация)
- `GRAF=1`: `GA9=1` — это признак графического адреса, `GA[8:0] = G_LINE[8:0]`

**Ключевой бит VAI19:** если `VAI[19]=1`, видеоконтроллер работает в линейном режиме Sprinter. Если `VAI[19]=0` — в ZX-совместимом режиме.

## Счётчики синхронизации

Видеоконтроллер ведёт четыре счётчика:

| Счётчик | Разрядность | Назначение |
|---------|------------|-----------|
| `CT[5:0]` | 6 бит | Суб-такт (CLK42 → субпиксельное время) |
| `CTH[5:0]` | 6 бит | Горизонтальная позиция (пиксели) |
| `CTV[8:0]` | 9 бит | Вертикальная строка |
| `CTF[6:0]` | 7 бит | Номер кадра |

Счётчики синхронизируются внешними сигналами `COPY_SINC_H` и `COPY_SINC_V` (приходят от FDD/синхронизаторов через шлейф `SINC_1/SINC_2`). При запуске сбрасываются через `START_UP`.

Высота кадра: запись в порты `0x2C/0x2D` (`DCP.MIF` индексы `0x2C/0x2D`, значения `0x7120/0x7124`) переключает между 320 и 312 строками.

## ZX-совместимый режим (VAI19=0)

При `VAI19=0` контроллер адресует DRAM так же, как ZX Spectrum.

### ZX_PORT[7:0]

```vhdl
SVIDEO.ZX_PORT[5:0] = ACC.G_LINE[5:0]       -- текущая строка (от ускорителя)
SVIDEO.ZX_PORT[7:6] = (DECODE.SP_SA,
    LCELL(DECODE.SP_SCR & !(A13 & !ACC.G_LINE7) & !ACC.G_LINE6 & !ALL_MODE0))
```

| Биты | Назначение |
|------|-----------|
| `[5:0]` | Текущая строка внутри страницы экрана (ZXS[5:0]) |
| `6` | SP_SCR: разрешение ZX-адресации (зависит от A15:A14 и GRAF) |
| `7` | SP_SA: выбор второй страницы ZX 128K (`PG3[1]`) |

### Формула ZX-адреса

В ZX-совместимом режиме видеоконтроллер строит адрес по правилу ZX Spectrum:

- `ZXS[4:1]` — базовый 16-КБ блок
- `ZXS[0]` вместе с `VAI13` и `ZXA15` реализуют XOR по `/7FFD`/`/DFFD`
- Атрибуты — по смещению +6144 байт от начала страницы

При `ALL_MODE0=1` (ускоритель включён) `SP_SCR` принудительно обнуляется (строка `SP2_ACEX.TDF:1030`), что отключает ZX-адресацию — ускоритель работает напрямую с линейными адресами.

### SP_SCR и SP_SA (из DCP)

```vhdl
CASE A[15:14] IS
    WHEN 0 => SP_SCR = 0;       SP_SA = 0;
    WHEN 1 => SP_SCR = !GRAF;   SP_SA = 0;
    WHEN 2 => SP_SCR = 0;       SP_SA = PG3[1];
    WHEN 3 => SP_SCR = !GRAF & (PG3[] == 110101b); SP_SA = PG3[1];
END CASE;
```

`SP_SCR=1` при обращении к `0x4000–0x7FFF` когда `GRAF=0` (не графическая страница). `SP_SA` отражает страницу 128K, хранящуюся в `PG3[1]`.

### SCR128

```vhdl
SVIDEO.DIR_PORT[0] = DECODE.SCR128  -- = PN3
```

При `PN3=1` видеоконтроллер переключается на «теневой» экран 128K (страница 7 вместо страницы 5 по умолчанию).

## Линейный режим Sprinter (VAI19=1)

При `VAI19=1` используется прямая адресация VRAM. Адрес: `VXA = VAI[19:0]` — 20-битный линейный, до 1 МБ (реально 512 КБ физической DRAM).

### Регистры MODE0–MODE2

Записываются через порты видеоконтроллера (`DCP.MIF` записи `0x7120` и соседние), когда `WR_MODE` активен.

**MODE0[7:0]:**

| Биты | Назначение |
|------|-----------|
| `[3:0]` | Базовый адрес слоя `SVA[9:6]` (старшие биты) |
| `4` | `0` = атрибутный режим; `1` = прямографический режим |
| `5` | `0` = 320 пикселей/строка; `1` = 640 пикселей/строка |
| `[7:6]` | Цветовая четверть (2 бита палитры) + выбор ZX-атрибутов |

**MODE1[7:0]:**

Стартовая точка пиксельных данных. В чисто-графическом режиме — база пиксельного буфера. В гибридном ZX+Sprinter — отдельная база для пикселей.

**MODE2[7:0]:**

| Биты | Назначение |
|------|-----------|
| `[2:0]` | Шаг по строкам атрибутов (log2 ширины, в байтах) |
| `[7:3]` | Страница в DRAM для атрибутов/текстовых ячеек |

### Работа счётчиков при выводе

Адрес пикселя вычисляется из счётчиков:
```
pix_addr = BASE_PIX + CTV * LINE_WIDTH + CTH
attr_addr = BASE_ATTR + (CTV >> 3) * ATTR_WIDTH + CTH >> 3
```

где `BASE_PIX = MODE1` и `BASE_ATTR = MODE2`, а `LINE_WIDTH` зависит от `MODE0[5]` (320 или 640).

## DIR_PORT[7:0] — управление режимами

```vhdl
SVIDEO.DIR_PORT[0]   = DECODE.SCR128        -- бит 0: 128K режим экрана
SVIDEO.DIR_PORT[2:1] = 0                    -- биты 1-2: зарезервированы
SVIDEO.DIR_PORT[3]   = RGMOD0               -- бит 3: режим мультиплексора
SVIDEO.DIR_PORT[4]   = RGMOD1               -- бит 4: отключение экрана
SVIDEO.DIR_PORT[7:5] = BORDER[2:0]          -- биты 7-5: цвет бордера (3 бита)
```

| Бит | Источник | Назначение |
|-----|---------|-----------|
| 0 | PN3 | ZX 128K режим (SCR128) |
| 1 | 0 | — |
| 2 | 0 | — |
| 3 | RGMOD0 | Режим адресации видеостраниц |
| 4 | RGMOD1 | Отключение экрана (если SCREEN_OFF="USE") |
| 7:5 | BORDER[2:0] | Цвет бордера: бит5=Blue, бит6=Red, бит7=Green |

## Буферы VDM0–VDM3

Четыре 8-битных видеобуфера принимают данные с DRAM-шин:

```vhdl
SVIDEO.VDM0[] = VD0[]
SVIDEO.VDM1[] = VD1[]
SVIDEO.VDM2[] = VD2[]
SVIDEO.VDM3[] = VD3[]
```

Данные захватываются синхронно по сигналам `V_WEN[]` (от видеоконтроллера). В ZX-режиме:
- `VDM0` — пиксели
- `VDM1` — атрибуты

В Sprinter-режиме все четыре канала используются параллельно для 32-битного доступа.

## Запись в видеопамять CPU

```vhdl
VIDEO_PG = (DECODE.PAGE[7:4] == 0101b)     -- признак видеостраницы
BLK_MR   = VIDEO_PG & (PAGE[2] or (PAGE[3] & ACC.GLISSER))
SVIDEO.wr = DFF((DECODE.MC_WRITE or GLISSER or DECODE.CAS), !CLK42, , /RESET)
```

При `VIDEO_PG=1` (страница помечена как видео в DCP.MIF):
1. `SVIDEO.wr=1` — видеоконтроллер знает о записи
2. `BLK_MR` блокирует обычное чтение из DRAM (чтобы не конфликтовать с видеодоступом)
3. Строб `V_WR[]` (от `V_WEN` видеоконтроллера) активирует запись в нужный банк

Для записи в ZX-экран необходимо, чтобы `ZX_SCREEN=1` (бит `DIR_PORT[2]` или соответствующий порт `OUT (0x7FFD),A`).

## DOUBLE_CAS — удвоенный CAS для DMA

```vhdl
SVIDEO.DOUBLE_CAS = ACC.DOUBLE_CAS

-- В SP2_ACEX.TDF:
CASXE0 = (MCA0==0) or ((MCA0==1) & DOUBLE_CAS)
CASXE1 = (MCA0==1) or ((MCA0==0) & DOUBLE_CAS)
```

При `DOUBLE_CAS=1` каждый такт DRAM выдаёт по 2 байта вместо 1. Видеоконтроллер обрабатывает оба байта параллельно. Используется при копировании кадра ускорителем.

## Мышиный курсор

Видеоконтроллер накладывает аппаратный курсор мыши:

```vhdl
SVIDEO.MOUSE_X[] = MS.OUT_X[]
SVIDEO.MOUSE_Y[] = MS.OUT_Y[]
```

Позиция курсора приходит из модуля `MOUSE.TDF`. Паттерн курсора хранится в `MOUSE.MIF` (256×2 бит).

## WR_COL — синхронизация цвета

```vhdl
WR_COL = SVIDEO.WR_PIX
```

Сигнал `WR_COL` выводится на внешний разъём. В режиме SPECTRUM он управляет записью цветовых данных, в режиме SPRINTER — синхронизирует обновление палитры.

## Режимы вывода

### 1. ZX 48K

- `VAI19=0`, `SCR128=0`, `MODE0[4:5]=0`
- Экран: 256×192 пикселей + атрибуты 8×8
- Страница пикселей: `0x4000–0x57FF` (6144 байт)
- Страница атрибутов: `0x5800–0x5AFF` (768 байт)
- Частота CPU: 3.5 МГц, с ULA-задержками по `V_RAM`

### 2. ZX 128K

- `VAI19=0`, `SCR128=PN3`
- Дополнительно: доступна «теневая» страница экрана (страница 7)
- `OUT (0x7FFD), A` — бит 3 (shadow screen)

### 3. Sprinter 320×256

- `VAI19=1`, `MODE0[4]=1`, `MODE0[5]=0`
- 320 пикселей × 256 строк, 256 цветов (8 бит/пиксель)
- Линейный буфер: `BASE_PIX = MODE1 × 256`
- Полный кадр: 81920 байт

### 4. Sprinter 640×256

- `VAI19=1`, `MODE0[4]=1`, `MODE0[5]=1`
- 640 пикселей × 256 строк, требует DOUBLE_CAS для заполнения
- Скорость вывода удвоена

### 5. Гибридный ZX+Sprinter

- `VAI19=1`, `MODE0[4]=0`
- ZX-атрибуты из `MODE2`-страницы
- Пиксели из `MODE1`-страницы
- Позволяет разместить ZX-совместимый экран в любом месте VRAM

## Программирование видеорежима

```asm
; Включить Sprinter 320×256
LD A, %00010000      ; MODE0: графика, 320px
OUT (0xC5), A        ; RGMOD — или через соответствующий порт DCP

; Установить BASE_PIX = страница 2 (адрес 0x20000)
LD A, 2
OUT (MODE1_PORT), A

; Установить BASE_ATTR = страница 4
LD A, 4
OUT (MODE2_PORT), A

; Обновить бордер
LD A, %00000111      ; белый бордер
OUT (0xC2), A        ; BORDER port
```

## Связь с ускорителем

При скринкопии (режим `LD L,L`, `LD A,A`):
1. Ускоритель подаёт `DOUBLE_CAS=1` → DRAM выдаёт 16 бит за такт
2. `G_LINE[]` обновляется после каждой строки
3. `GLISSER` сигнализирует о байтах `0xFF`
4. `VIDEO_PG` + `BLK_MR` обеспечивают синхронизацию с видеопотоком

Таймингово: ускоритель «вклинивается» между видеовыборками DRAM, используя свободные такты цикла. Конкуренция разрешается через `MC_BEGIN`/`MC_END`.
