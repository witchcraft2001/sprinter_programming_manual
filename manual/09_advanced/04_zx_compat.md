# 9.4 ZX Spectrum-совместимость

> **Навигация:** [← 9.3 Скрытые возможности](03_hidden_features.md) | [Оглавление](../README.md) | [10.1 Hello World →](../10_tutorials/01_hello_world.md)

---

## Обзор

Sprinter может запускать оригинальное ПО ZX Spectrum — игры, программы, музыкальные плееры. Совместимость реализована на нескольких уровнях:

1. **Аппаратная** — FPGA эмулирует ZX, Pentagon, Scorpion.
2. **Программная** — BIOS/DSS позволяют загружать TRD, TAP, SNA файлы.
3. **Поведенческая** — тайминги, порты, прерывания совпадают.

---

## Поддерживаемые модели

| Модель | Статус | Особенности |
|--------|--------|-------------|
| ZX Spectrum 48K | Полная | Режим SC0=1 |
| ZX Spectrum 128K | Полная | Порт `#7FFD` |
| Pentagon 128K | Полная | Расширенные страницы (PN бит 5) |
| Pentagon 512K | Поддерживается | CNF7 = 1, PN[7:5] |
| Scorpion ZS 256 | Частично | CNF3, расширенные порты |
| Спецификация Sp2000 | Родная | SC0 = 0 |

## Включение ZX режима

### Через CMOS (при старте)

```asm
; CMOS регистр #1B, биты [4:3] — тип ПК:
; 0 = Sprinter, 1 = Spectrum 256, 2 = Pentagon 128, 3 = Other

        ld      bc, CMOS_AWR
        ld      a, #1B
        out     (c), a
        ld      bc, CMOS_DRD
        in      a, (c)
        and     #E7                 ; clear bits 3-4
        or      #10                 ; set bit 4 (Pentagon 128)
        push    af
        ld      bc, CMOS_AWR
        ld      a, #1B
        out     (c), a
        pop     af
        ld      bc, CMOS_DWR
        out     (c), a
```

### Программно (в runtime)

```asm
; Включить ZX-режим
        ld      a, #01              ; SC0 = 1
        out     (#C0), a

; Выключить (нативный)
        xor     a
        out     (#C0), a
```

## Порты ZX Spectrum

В ZX-режиме работают стандартные порты:

| Порт | Назначение |
|------|-----------|
| `#FE` | Клавиатура + бордюр + tape |
| `#7FFD` | ZX 128 страничный переключатель |
| `#BFFD` | AY данные (ZX 128) |
| `#FFFD` | AY адрес (ZX 128) |
| `#1FFD` | Scorpion расширения |
| `#DFFD` | Pentagon-512 расширения |

### Порт #7FFD (ZX 128)

| Бит | Назначение |
|-----|-----------|
| [2:0] | Номер RAM страницы (0-7) в C000-FFFF |
| 3 | Screen select: 0 = страница 5, 1 = страница 7 |
| 4 | ROM select: 0 = ROM0 (128 BASIC), 1 = ROM1 (48 BASIC) |
| 5 | Disable port (1 = игнорировать дальнейшие записи) |

```asm
; Переключить RAM страницу в C000-FFFF на 6 (ZX 128)
        ld      a, 6
        ld      bc, #7FFD
        out     (c), a
```

---

## Карта памяти ZX 128

```
0x0000–0x3FFF   ROM (128 BASIC или 48 BASIC)
0x4000–0x7FFF   RAM страница 5 (содержит ZX экран)
0x8000–0xBFFF   RAM страница 2
0xC000–0xFFFF   RAM переключаемая страница (0..7 через #7FFD)
```

---

## ZX экран

### Пиксельная область

Адрес: `#4000`–`#57FF` (6144 байт)

Адресация: сложная (см. раздел [5.2 ZX режим](../05_graphics/02_zx_mode.md))

### Атрибутная область

Адрес: `#5800`–`#5AFF` (768 байт)
Формат: `FLASH[7] BRIGHT[6] PAPER[5:3] INK[2:0]`

---

## AY-3-8910 в ZX режиме

Доступен через стандартные порты:

```asm
; ZX 128 совместимо: выбрать регистр
        ld      bc, #FFFD
        ld      a, reg_num
        out     (c), a

; Записать данные
        ld      bc, #BFFD
        ld      a, value
        out     (c), a
```

В нативном режиме Sprinter параллельно работают порты `#8D`/`#8E`.

---

## Прерывания ZX

В ZX-режиме прерывания генерируются в конце кадра:
- **50 Гц** PAL (стандарт ZX)
- **IM 1**: переход на `RST #38`
- **IM 2**: используется игры (редко)

В Sprinter поведение то же. Таймер прерываний не зависит от turbo.

---

## TRD — файлы

TRD — образ дискеты TR-DOS. BIOS умеет монтировать TRD как виртуальную дискету. Для загрузки игры:

1. Скопировать TRD в любой каталог DSS.
2. Использовать специальную утилиту (GAME.EXE, LOADER.EXE) для подключения TRD как FDD.
3. Загрузить через TR-DOS `RANDOMIZE USR 15616 : REM : LOAD "file"`.

---

## SNA — snapshot

SNA — снимок состояния ZX Spectrum 48K (27 КБ). Можно загружать через утилиту из DSS:

```asm
; Загрузка SNA: открыть файл, скопировать регистры и память,
; установить I, затем RET в точку прерывания
```

Детали — в утилитах SDK (`zx-sprinter-sdk/sdk/src/loader/`).

---

## Тайминги и мультицвет

Многие демо-эффекты ZX Spectrum полагаются на точные тайминги кадра:

- **Contended memory** (порт `#4000`-`#7FFF` + `#C000`-`#FFFF` на странице 7) — медленная при одновременном доступе CPU и видео.
- **Multicolour** эффекты — синхронизация инструкций с лучом.
- **Floating bus** — чтение из `#FF` даёт значение текущего атрибута.

Sprinter эмулирует это **в ZX-режиме** (`SC0=1`), но не в нативном.

## Частые проблемы ZX совместимости

### Проблема 1: игра работает слишком быстро

Решение: убедиться, что Sprinter в 7 МГц (не turbo). Для автоматического отключения turbo при запуске ZX-программы:

```asm
; При старте загрузчика ZX-игры
; (не стандартизировано — зависит от утилиты загрузки)
```

### Проблема 2: неправильные цвета

Решение: проверить, включён ли ZX-режим видео (`ALL_MODE = 0`). В режиме 320×256 палитра другая, ZX-атрибуты не работают.

### Проблема 3: "чёрный экран"

Часто — проблема с маппингом страниц. Проверить, что страница 5 в WIN1 (0x4000).

## Переключение между режимами

```asm
; Сохранить текущее состояние и перейти в ZX режим
SaveAndEnterZX:
        ; Сохранить режим и регистры
        in      a, (#C3)            ; ALL_MODE
        ld      (save_mode), a

        ; Установить ZX
        ld      a, 0
        out     (#C3), a
        ld      a, 1
        out     (#C0), a            ; SC0 = 1

        ; Инициализировать ZX: страница 5 в WIN1, и т.п.
        ; ...
        ret

RestoreFromZX:
        xor     a
        out     (#C0), a            ; SC0 = 0

        ld      a, (save_mode)
        out     (#C3), a
        ret

save_mode:  db 0
```

## Программа-запускатор (пример)

```asm
; Примитивный запускатор SNA для ZX 48K

        org     #8100

main:
        ; Открыть файл
        ld      hl, sna_name
        ld      a, 1
        ld      c, #11 : rst #10

        ; Прочитать заголовок SNA (27 байт)
        ld      hl, sna_header
        ld      de, 27
        ld      c, #13 : rst #10

        ; Прочитать 48K RAM в страницы 5, 2, 0
        ; ... (сложная процедура — распределение по страницам)

        ; Установить ZX режим
        ld      a, 1
        out     (#C0), a

        ; Восстановить регистры из заголовка
        ; ... pop AF, BC, DE, HL, ix, iy ...

        ; Установить SP
        ld      sp, (sna_sp)

        ; JP в адрес
        ld      a, (sna_border)
        out     (#FE), a
        ; RETN возвращает из NMI на адрес из стека
        ; (или JP через извлечённый PC)

sna_name:   db "GAME.SNA", 0
sna_header: ds 27
sna_sp:     dw 0
sna_border: db 0
```

---

## Ключевые моменты

> - Sprinter совместим с ZX 48K/128K/Pentagon/Scorpion через FPGA.
> - Включение ZX: `SC0 = 1` (порт `#C0` бит 0) + режим видео `ALL_MODE = 0`.
> - Порты: `#FE` (клавиатура + бордюр), `#7FFD` (банкинг), `#FFFD`/`#BFFD` (AY).
> - ZX экран: `0x4000`–`0x5AFF` (пиксели + атрибуты) при WIN1 = страница 5.
> - Формат программ: TRD (диск), TAP (лента), SNA (снапшот).
> - Для запуска старых игр — выключать turbo (CMOS регистр `#1B` бит 0).
