## Структура EXE-файла

`.EXE` файл DSS состоит из **заголовка** (512 байт) и **тела программы** (произвольный размер).

### Формат заголовка

```
Смещение  Размер  Поле          Описание
──────────────────────────────────────────────────
0x000     3       id            "EXE" (сигнатура)
0x003     1       exe_ver       Версия EXE: `1` — актуальная, `0` — deprecated
0x004     2       hdr_size      Размер заголовка (обычно `#0200` = 512)
0x006     2       offset        Смещение начала кода в файле (обычно 512)
0x008     2       loader        Адрес загрузчика (обычно 0)
0x00A     2       load          Адрес загрузки в памяти (обычно #8100)
0x00C     2       start         Точка входа (обычно = load)
0x00E     2       stack         Начальный SP (обычно #BFFF)
0x010     ...     резерв         Заполнено нулями/#FF до 0x200
0x200     ...     код           Тело программы
```

### Эквивалент в C

```c
struct SprintEXE {
    char     id[3];     // "EXE"
    uint8_t  exe_ver;   // 1 = актуальная версия, 0 = deprecated
    uint16_t hdr_size;  // 0x0200 (512)
    uint16_t offset;    // file offset of code
    uint16_t loader;    // loader addr (0 = no loader)
    uint16_t load;      // load address in memory
    uint16_t start;     // entry point
    uint16_t stack;     // initial SP
    uint8_t  reserved[512 - 16];
};
```

## Шаблон EXE в sjasmplus

### Вариант 1: минимальный (из flappybird)

```asm
        org     #8100-512           ; #7F00

        db      "EXE", 1
        dw      #0200               ; hdr_size = 512
        dw      0, 0, 0, 0, 0       ; padding (5 слов)
        dw      begin               ; entry duplicate
        dw      begin               ; entry
        dw      #BFFF               ; stack
        ds      490                 ; padding до 512 байт

begin:
        ; ... код программы ...
```

### Вариант 2: явный

```asm
        org     #8100-512

        db      "EXE", 1            ; id + exe_ver (актуальная)
        dw      #0200               ; hdr_size = 512
        dw      #0200               ; offset = 512
        dw      0                   ; loader
        dw      #8100               ; load address
        dw      main                ; start (точка входа)
        dw      #BFFF               ; stack
        ds      512 - 16            ; резерв до конца заголовка

        org     #8100
main:
        ; ... код программы начинается здесь ...
```

## Запуск программы

DSS запускает EXE в несколько шагов:

1. **Чтение заголовка** файла.
2. **Проверка сигнатуры** `"EXE"`.
3. **Выделение памяти** — `GetMem(3)` → 3 дополнительные страницы.
4. **Сохранение контекста** родителя (страницы, SP, PC).
5. **Установка окон**: страницы нового блока подключаются в WIN1, WIN2, WIN3.
6. **Чтение тела программы** из файла в память по адресу `exe.load`.
7. **Сохранение аргументов** командной строки по адресу `IX-0x80` (обычно `#8080`).
8. **Установка SP** = `exe.stack`.
9. **Передача управления** `JP exe.start`.

### Версия EXE и current dir

- `exe_ver = 1` (актуальная версия): при передаче управления программа получает **текущий каталог вызывающей стороны**.
- В версии `1` операции с файлами сами по себе **не меняют** current dir.
- Текущий путь сохраняется до явной смены диска через API (`CHDISK`).
- После `CHDISK` current dir на новом диске начинается **с корня**.
- Открытие файла по полному пути на другом диске не сбрасывает текущий каталог текущего диска.
- `exe_ver = 0` — deprecated (оставлен только для обратной совместимости).

### EXEC из программы

```asm
Dss.Exec        EQU #40

; Запустить вложенную программу
        ld      hl, child_cmd
        ld      c, Dss.Exec
        rst     #10
        jr      c, .error
        ; Программа завершилась, управление вернулось

child_cmd:  db  "CHILD.EXE arg1 arg2", 0
```

После возврата из дочерней программы:
- Страницы окон восстановлены.
- SP восстановлен.
- Управление возвращается после `RST #10`.

## Вызовы после старта программы

После передачи управления на `exe.start` программа работает через стандартные
системные входы:

- `RST #10` — DSS API: файлы, каталоги, память, процессы, видео, клавиатура;
- `RST #08` — BIOS API: низкоуровневые сервисы устройств и памяти;
- `RST #30` — BIOS mouse API;
- `RST #18` — дисковый драйверный вход, обычно нужен только собственным драйверам.

Типичное окружение на старте:

- код программы загружен по адресу `exe.load` (обычно `#8100`);
- `SP` установлен в `exe.stack` (обычно `#BFFF`);
- аргументы доступны по `#8080`;
- DSS уже резидентен, `RST #10` активен;
- системные страницы заняты и не используются как обычная память программы.

## Командная строка

### Доступ к аргументам

Аргументы командной строки записываются DSS в память программы по адресу **`#8080`** (256 байт):

```
#8080  байт 0   : длина строки аргументов
#8081  байт 1   ┐
...             ├─ текст аргументов (разделённый пробелами, заканчивается 0)
#817F  байт 255 ┘
```

### Пример: печать аргументов

```asm
PrintArgs:
        ld      hl, #8080
        ld      a, (hl)             ; длина
        or      a
        ret     z                   ; нет аргументов
        inc     hl

        ld      b, a                ; B = длина
.loop:
        ld      a, (hl)
        or      a                   ; 0 → конец
        ret     z
        push    hl
        push    bc
        ld      c, #5B              ; DSS PUTCHAR
        rst     #10
        pop     bc
        pop     hl
        inc     hl
        djnz    .loop
        ret
```

## Разбор пути — EX_PATH (`#45`)

Функция разбирает строку пути на компоненты (диск, каталог, имя, расширение). Подфункция выбирается регистром `B`:

```
B   Описание                  Буфер DE               Размер буфера
─────────────────────────────────────────────────────────────────────
0   Разобрать строку          —                       —
1   Извлечь имя диска         "A:\0"                  13 байт
2   Извлечь путь              "DIR1\DIR2\\0"          256 байт
3   Извлечь имя файла         "FILE.EXT\0"            13 байт
4   Извлечь расширение        "TXT\0"                 4 байта
5   Извлечь всё сразу         структура 8 байт        см. ниже
```

### Вход / выход

```
Вход:
  C  = #45
  B  = подфункция (0–5)
  HL = указатель на строку пути (ASCIIZ)
  DE = указатель на буфер результата

Выход (CF=0, успех):
  A  = битовая маска найденных компонентов:
       бит 7 — символы ? или * (wildcard)
       бит 3 — указан диск
       бит 2 — указан путь
       бит 1 — указано расширение
       бит 0 — указано имя файла
  Буфер DE заполнен результатом (ASCIIZ)

Выход (CF=1, ошибка):
  A  = код ошибки
```

### Подфункция 5: извлечь все компоненты

Буфер `DE` содержит 4 указателя (8 байт):

```
DE+0  → буфер имени диска     (13 байт)
DE+2  → буфер пути            (256 байт)
DE+4  → буфер имени файла     (13 байт)
DE+6  → буфер расширения      (4 байта)
```

### Пример

```asm
Dss.EX_Path     EQU #45

; Извлечь путь (подфункция 2)
        ld      hl, full_path
        ld      de, path_buf
        ld      b, 2                ; путь
        ld      c, Dss.EX_Path
        rst     #10
        jr      c, .error
        ; path_buf = "PROGS\TOOLS\"

; Извлечь все компоненты (подфункция 5)
        ld      hl, full_path
        ld      de, ptrs
        ld      b, 5
        ld      c, Dss.EX_Path
        rst     #10

full_path:  db  "C:\PROGS\TOOLS\TEST.EXE", 0
path_buf:   ds  256
ptrs:       dw  drv_buf, dir_buf, name_buf, ext_buf
drv_buf:    ds  13
dir_buf:    ds  256
name_buf:   ds  13
ext_buf:    ds  4
```

## Информация о приложении — APPINFO (`#47`)

Возвращает информацию о текущей запущенной программе. Подфункция выбирается регистром `B`:

```
B   Описание                        Результат в буфере DE
─────────────────────────────────────────────────────────────
0   Параметры командной строки      "arg1 arg2\0"
1   Каталог приложения              "C:\PROGS\TOOLS\\0"
2   Полный путь к EXE               "C:\PROGS\TOOLS\APP.EXE\0"
```

### Вход / выход

```
Вход:
  C  = #47
  B  = подфункция (0–2)
  DE = указатель на буфер результата (ASCIIZ)

Выход (CF=0):
  Буфер DE заполнен строкой (ASCIIZ)

Выход (CF=1):
  A  = код ошибки (B > 2 → ERR_INVALID_FUNCTION)
```

### Пример

```asm
Dss.AppInfo     EQU #47

; Получить каталог приложения
        ld      de, dir_buf
        ld      b, 1                ; APP_PATH
        ld      c, Dss.AppInfo
        rst     #10
        ; dir_buf = "C:\PROGS\"

; Получить параметры командной строки
        ld      de, param_buf
        ld      b, 0                ; APP_PARAM
        ld      c, Dss.AppInfo
        rst     #10
        ; param_buf = "file.txt /v"

dir_buf:    ds  256
param_buf:  ds  128
```

## Возврат из программы

**Единственный корректный способ** завершить DSS-программу — вызов `EXIT`:

```asm
Dss.Exit        EQU #41

        ld      b, 0                ; код возврата 0
        ld      c, Dss.Exit
        rst     #10
        ; управление возвращается в родительскую программу
        ; программа больше не получит управление
```

При вызове `EXIT`:
1. Код возврата из регистра `B` сохраняется в переменную `ERLEVEL`.
2. DSS освобождает блок памяти программы (если был выделен автоматически).
3. Восстанавливает страницы родителя.
4. Восстанавливает SP родителя.
5. `RET` в родителя (в точку после его `EXEC`).

---

## Вложенные вызовы EXEC

DSS поддерживает **многоуровневый вызов**. Родитель может запустить потомка, тот — ещё один, и т.д.

```
Shell.EXE          ← уровень 0
   └── EXEC
       App1.EXE    ← уровень 1
          └── EXEC
              Tool.EXE    ← уровень 2
                 └── EXIT
              (возврат в App1.EXE)
          └── EXIT
       (возврат в Shell.EXE)
```

DSS хранит массив `SavePages[Level]` и `SaveSP[Level]` для каждого уровня.

## Передача результата между программами

### Exit code (стандартный механизм)

DSS поддерживает стандартный механизм кода возврата. При вызове `EXIT (#41)` программа передаёт код возврата в регистре `B`. Родительская программа получает его в регистре `A` после возврата из `EXEC (#40)`. Код также сохраняется в глобальной переменной `ERLEVEL` и может быть получен позже через функцию `WAIT (#42)`:

```asm
; Дочерняя программа: завершение с кодом 1
        ld      b, 1                ; exit code = 1
        ld      c, #41              ; EXIT
        rst     #10

; Родительская программа: получение кода возврата
        ld      hl, child_cmd
        ld      c, #40              ; EXEC
        rst     #10
        jr      c, .exec_error      ; CF=1 — ошибка запуска
        ; A = код возврата дочерней программы
        or      a
        jr      nz, .child_failed

; Или позже через WAIT:
        ld      c, #42              ; WAIT
        rst     #10
        ; A = последний exit code (ERLEVEL)
```

### WAIT (`#42`) — получение кода завершения

Функция возвращает код завершения последней программы, выполненной через `EXEC`. Внутренне читает глобальную переменную `ERLEVEL`. **Не является паузой** — возвращает результат немедленно.

```
Вход:
  C = #42

Выход:
  A = код завершения (0–255)
  ZF установлен если A = 0
```

### Другие способы обмена данными

1. **Через файл** — временный файл на диске.
2. **Через память** — если родитель знает физический номер страницы потомка.
3. **Через переменные окружения** — функция `ENVIRON #46` (см. ниже).

## Переменные окружения — ENVIRON (`#46`)

Управление переменными окружения. Подфункция выбирается регистром `B`:

```
B     Описание                    Вход
──────────────────────────────────────────────────────
#FF   Инициализация (сброс)       —
#00   Чтение всего блока          DE = буфер назначения
#01   Получить переменную         HL = "NAME\0", DE = буфер значения
#02   Установить/удалить          HL = "NAME=VALUE\0" или "NAME=\0"
```

Переменные хранятся в отдельной странице памяти (PAGE 2, адрес `#E400`) в формате `VAR1=value1\0VAR2=value2\0\0`. Имена автоматически приводятся к **верхнему регистру**.

### Вход / выход

**GETENV (B = #01):**

```
Вход:
  C  = #46, B = #01
  HL = имя переменной (ASCIIZ, без "=")
  DE = буфер для значения

Выход:
  A  = #FF — переменная найдена, значение скопировано в DE
  A  = #00 — переменная не найдена, DE[0] = 0
```

**SETENV (B = #02):**

```
Вход:
  C  = #46, B = #02
  HL = строка "NAME=VALUE\0" (установить) или "NAME=\0" (удалить)

Выход:
  A  = 0, CF = 0
```

### Ограничения

- Имя переменной: до 32 символов.
- Значение: до 255 символов.
- Общий объём окружения ограничен страницей памяти (~16 КБ).

### Пример

```asm
Dss.Environ     EQU #46

; Установить переменную
        ld      hl, set_str
        ld      b, #02              ; SETENV
        ld      c, Dss.Environ
        rst     #10

; Прочитать переменную
        ld      hl, var_name
        ld      de, val_buf
        ld      b, #01              ; GETENV
        ld      c, Dss.Environ
        rst     #10
        ; A = #FF → найдена, val_buf = "C:\BIN"
        ; A = #00 → не найдена

; Удалить переменную
        ld      hl, del_str
        ld      b, #02              ; SETENV с пустым значением
        ld      c, Dss.Environ
        rst     #10

set_str:    db  "PATH=C:\BIN", 0
var_name:   db  "PATH", 0
del_str:    db  "PATH=", 0
val_buf:    ds  256
```

## Пример полной программы с EXEC

```asm
        org     #8100-512

        db      "EXE", 1
        dw      #0200
        dw      #0200
        dw      0
        dw      #8100
        dw      main
        dw      #BFFF
        ds      512 - 16

        org     #8100

main:
        ; Показать приветствие
        ld      hl, welcome
        ld      c, #5C              ; PCHARS
        rst     #10

        ; Запустить дочернюю программу
        ld      hl, cmd
        ld      c, #40              ; EXEC
        rst     #10
        jr      c, .no_child

        ; Показать сообщение о завершении
        ld      hl, done
        ld      c, #5C
        rst     #10
        jr      .exit

.no_child:
        ld      hl, err_msg
        ld      c, #5C
        rst     #10

.exit:
        ld      b, 0                ; exit code 0
        ld      c, #41              ; EXIT
        rst     #10

welcome:    db  "Starting child...", 13, 10, 0
done:       db  "Child finished.", 13, 10, 0
err_msg:    db  "Failed to run child!", 13, 10, 0
cmd:        db  "CHILD.EXE", 0
```

## Ограничения формата EXE

1. **Максимальный размер кода** — ограничен окнами: WIN2 + 3 выделенных страницы = до 64 КБ непрерывного кода.
2. **Для программ > 64 КБ** нужны **оверлеи**: код грузится частями в разные страницы.
3. **Стек** обычно в `#BFFF`, но можно установить где угодно в адресуемой памяти.
4. **Адрес загрузки** берётся из заголовка EXE (поле `load`, смещение `#0A`). DSS определяет окно по старшим двум битам адреса и подключает страницы автоматически:
   - `#4xxx` → загрузка в WIN1 (`#4000–#7FFF`)
   - `#8xxx` → загрузка в WIN2 (`#8000–#BFFF`)
   - `#Cxxx` → загрузка в WIN3 (`#C000–#FFFF`)

   Наиболее распространённые адреса: **`#8100`** (системные утилиты, компактные программы, основные загрузчики) и **`#4100`** (SolidC, программы побольше — используют WIN1+WIN2).

---

## Ключевые моменты

> - EXE = 512-байтный заголовок + код.
> - Заголовок: сигнатура "EXE", `exe_ver` (`1` актуальная, `0` deprecated), размер заголовка `#0200`, адрес загрузки `#8100`, точка входа, стек.
> - При EXEC DSS выделяет 3 дополнительные страницы и подключает их в WIN1/2/3.
> - Аргументы командной строки — по адресу `#8080` (длина + текст).
> - Выход через `EXIT #41` с кодом возврата в `B`; никогда не через `RET`.
> - `EXEC` возвращает exit code в `A`; `WAIT (#42)` читает последний exit code.
> - Поддержаны вложенные EXEC с сохранением контекста.
