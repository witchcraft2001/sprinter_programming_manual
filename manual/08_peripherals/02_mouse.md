## Два интерфейса

Sprinter поддерживает два способа работы с мышью:

1. **PS/2 мышь** — через BIOS API (`RST #30`)
2. **Kempston Mouse** — прямые порты (`#58`, `#158`, `#358`)

Обычно они указывают на один и тот же физический PS/2 порт, но программа может использовать любой из API.

---

## BIOS Mouse API (RST #30)

Рекомендуемый способ для нативных программ.

### Инициализация

```asm
        ld      c, #00              ; INITIALIZATION
        rst     #30
```

### Показать/скрыть курсор

```asm
        ld      c, #01              ; SHOW MOUSE CURSOR
        rst     #30

        ld      c, #02              ; HIDE MOUSE CURSOR
        rst     #30
```

### Прочитать состояние

```asm
        ld      c, #03              ; READ MOUSE STATE
        rst     #30
        ; A = кнопки (bit 0 = левая, bit 1 = правая)
        ; HL = X координата
        ; DE = Y координата
```

### Обновление состояния (для VSync)

```asm
; Вызывать из прерывания 50 Гц
        ld      c, #83              ; MOUSE REFRESH
        rst     #30
```

## Пример: курсор как спрайт

```asm
; Главный цикл: рисовать курсор в (X, Y), перемещение по мыши

MouseLoop:
.loop:
        ; Прочитать мышь
        ld      c, #03
        rst     #30
        ld      (mouse_buttons), a
        ld      (mouse_x), hl
        ld      (mouse_y), de

        ; Проверить выход (обе кнопки)
        ld      a, (mouse_buttons)
        cp      3
        ret     z

        ; Очистить предыдущую позицию (упрощённо)
        ; ... восстановить фон ...

        ; Нарисовать курсор в новой позиции
        ld      hl, (mouse_x)
        ld      de, (mouse_y)
        ld      b, 15               ; цвет курсора
        ld      c, #A1              ; PIC_POINT
        rst     #08

        halt                        ; ждать кадр
        jr      .loop

mouse_buttons:  db 0
mouse_x:        dw 0
mouse_y:        dw 0
```

## Kempston Mouse (прямые порты)

Классический интерфейс мыши ZX Spectrum, использует порт `#58` с разными значениями адресных линий:

| Условие | Функция |
|---------|---------|
| A10=0, A8=0 | Кнопки (`#058`) |
| A10=0, A8=1 | X координата (`#158`) |
| A10=1, A8=1 | Y координата (`#358`) |

### Чтение кнопок

```asm
        ld      bc, #005C           ; A10=0, A8=0 (нижняя часть = 58 + 4 = 5C)
        in      a, (c)
        ; A = биты: 0 = L, 1 = R (обычно)
        and     #03
```

### Чтение X

```asm
        ld      bc, #01DF           ; A10=0, A8=1
        in      a, (c)
        ; A = X координата
```

### Чтение Y

```asm
        ld      bc, #05DF           ; A10=1, A8=1
        in      a, (c)
        neg                         ; Y инвертирован
```

> **Замечание:** точные адреса портов зависят от прошивки FPGA. Используйте BIOS API для переносимости.

## Координатная система

В BIOS Mouse API:
- X: 0..319 (или 0..639 в режиме 640×256)
- Y: 0..255 (или 0..511)
- (0, 0) в левом верхнем углу

В Kempston Mouse:
- X, Y — 8-битные счётчики, измеряют **относительные** изменения.
- Программа сама интегрирует их в абсолютные координаты.

---

## Кнопки

Kempston и BIOS возвращают одинаково:
- bit 0 = левая
- bit 1 = правая
- bit 2 = средняя (если поддерживается)

## Горячие клавиши vs мышь

В старых играх ZX иногда удобно использовать мышь как "Kempston Joystick":
- Горизонтальное движение → влево/вправо
- Вертикальное → вверх/вниз
- Левая кнопка → fire

```asm
; Преобразовать Kempston Mouse в Kempston Joystick эмуляцию
        ld      bc, #01DF
        in      a, (c)              ; X
        cp      128
        jr      c, .left
        jr      z, .center
        ; X > 128 → правый
        ld      a, #01              ; Kempston Right
        jr      .done
.left:
        ld      a, #02              ; Kempston Left
        jr      .done
.center:
        xor     a
.done:
        ret
```

## Использование в GUI приложениях

SDK из `flexnavigator` содержит полноценный GUI с поддержкой мыши:

```asm
; Типичный обработчик в GUI
GuiLoop:
        ld      c, #03              ; READ MOUSE
        rst     #30

        ; Проверить нажатие на кнопку
        ld      a, (prev_buttons)
        cpl
        and     e                   ; новое нажатие = текущие & ~предыдущие
        bit     0, a
        jr      z, .no_click

        ; Клик по (HL, DE)
        call    HandleClick

.no_click:
        ld      a, e
        ld      (prev_buttons), a

        halt
        jr      GuiLoop
```

## Ограничения

- **PS/2 специфика**: если в Sprinter нет подключенной PS/2 мыши, BIOS API возвращает CF=1 или статические данные.
- **Задержка опроса**: обычно 50 Гц (совпадает с кадром) — достаточно для большинства приложений.
- **Колесо мыши**: не поддерживается стандартным API (может быть добавлено прошивкой).

---

## Ключевые моменты

> - BIOS Mouse API через `RST #30`: `#00` init, `#01` show, `#02` hide, `#03` read state.
> - `READ MOUSE` возвращает: A = кнопки, HL = X, DE = Y.
> - Kempston Mouse — прямые порты `#58`/`#158`/`#358`.
> - BIOS API удобнее и портируемее.
> - Для GUI — опрашивайте мышь каждый кадр (в HALT/VSync цикле).
