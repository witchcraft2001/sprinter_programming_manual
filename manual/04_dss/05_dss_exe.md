## Структура EXE-файла

`.EXE` файл DSS состоит из **заголовка** (512 байт) и **тела программы** (произвольный размер).

### Формат заголовка

```
Смещение  Размер  Поле          Описание
──────────────────────────────────────────────────
0x000     3       id            "EXE" (сигнатура)
0x003     1       pad           0
0x004     2       version       Версия формата (#0200 = 2.0)
0x006     2       offset        Смещение начала кода в файле (обычно 512)
0x008     2       loader        Адрес загрузчика (обычно 0)
0x00A     2       load          Адрес загрузки в памяти (обычно #8100)
0x00C     2       start         Точка входа (обычно = load)
0x00E     2       stack         Начальный SP (обычно #BFFF)
0x010     ...     резерв         Заполнено нулями/#FF до 0x200
0x200     ...     код           Тело программы
```

### Эквивалент в C

```c
struct SprintEXE {
    char     id[3];     // "EXE"
    uint8_t  pad;
    uint16_t version;   // 0x0200
    uint16_t offset;    // file offset of code
    uint16_t loader;    // loader addr (0 = no loader)
    uint16_t load;      // load address in memory
    uint16_t start;     // entry point
    uint16_t stack;     // initial SP
    uint8_t  reserved[512 - 16];
};
```

## Шаблон EXE в sjasmplus

### Вариант 1: минимальный (из flappybird)

```asm
        org     #8100-512           ; #7F00

        db      "EXE", 0
        dw      #0200               ; version
        dw      0, 0, 0, 0, 0       ; padding (5 слов)
        dw      begin               ; entry duplicate
        dw      begin               ; entry
        dw      #BFFF               ; stack
        ds      490                 ; padding до 512 байт

begin:
        ; ... код программы ...
```

### Вариант 2: явный

```asm
        org     #8100-512

        db      "EXE", 0            ; id
        dw      #0200               ; version
        dw      #0200               ; offset = 512
        dw      0                   ; loader
        dw      #8100               ; load address
        dw      main                ; start (точка входа)
        dw      #BFFF               ; stack
        ds      512 - 16            ; резерв до конца заголовка

        org     #8100
main:
        ; ... код программы начинается здесь ...
```

## Запуск программы

DSS запускает EXE в несколько шагов:

1. **Чтение заголовка** файла.
2. **Проверка сигнатуры** `"EXE"`.
3. **Выделение памяти** — `GetMem(3)` → 3 дополнительные страницы.
4. **Сохранение контекста** родителя (страницы, SP, PC).
5. **Установка окон**: страницы нового блока подключаются в WIN1, WIN2, WIN3.
6. **Чтение тела программы** из файла в память по адресу `exe.load`.
7. **Сохранение аргументов** командной строки по адресу `IX-0x80` (обычно `#8080`).
8. **Установка SP** = `exe.stack`.
9. **Передача управления** `JP exe.start`.

### EXEC из программы

```asm
Dss.Exec        EQU #40

; Запустить вложенную программу
        ld      hl, child_cmd
        ld      c, Dss.Exec
        rst     #10
        jr      c, .error
        ; Программа завершилась, управление вернулось

child_cmd:  db  "CHILD.EXE arg1 arg2", 0
```

После возврата из дочерней программы:
- Страницы окон восстановлены.
- SP восстановлен.
- Управление возвращается после `RST #10`.

## Командная строка

### Доступ к аргументам

Аргументы командной строки записываются DSS в память программы по адресу **`#8080`** (256 байт):

```
#8080  байт 0   : длина строки аргументов
#8081  байт 1   ┐
...             ├─ текст аргументов (разделённый пробелами, заканчивается 0)
#817F  байт 255 ┘
```

### Пример: печать аргументов

```asm
PrintArgs:
        ld      hl, #8080
        ld      a, (hl)             ; длина
        or      a
        ret     z                   ; нет аргументов
        inc     hl

        ld      b, a                ; B = длина
.loop:
        ld      a, (hl)
        or      a                   ; 0 → конец
        ret     z
        push    hl
        push    bc
        ld      c, #5B              ; DSS PUTCHAR
        rst     #10
        pop     bc
        pop     hl
        inc     hl
        djnz    .loop
        ret
```

        ; path_buf содержит путь, например "\PROGS\TEST.EXE"

## Возврат из программы

**Единственный корректный способ** завершить DSS-программу — вызов `EXIT`:

```asm
Dss.Exit        EQU #41

        ld      b, 0                ; код возврата 0
        ld      c, Dss.Exit
        rst     #10
        ; управление возвращается в родительскую программу
        ; программа больше не получит управление
```

При вызове `EXIT`:
1. Код возврата из регистра `B` сохраняется в переменную `ERLEVEL`.
2. DSS освобождает блок памяти программы (если был выделен автоматически).
3. Восстанавливает страницы родителя.
4. Восстанавливает SP родителя.
5. `RET` в родителя (в точку после его `EXEC`).

## Вложенные вызовы EXEC

DSS поддерживает **многоуровневый вызов**. Родитель может запустить потомка, тот — ещё один, и т.д.

```
Shell.EXE          ← уровень 0
   └── EXEC
       App1.EXE    ← уровень 1
          └── EXEC
              Tool.EXE    ← уровень 2
                 └── EXIT
              (возврат в App1.EXE)
          └── EXIT
       (возврат в Shell.EXE)
```

DSS хранит массив `SavePages[Level]` и `SaveSP[Level]` для каждого уровня.

## Передача результата между программами

### Exit code (стандартный механизм)

DSS поддерживает стандартный механизм кода возврата. При вызове `EXIT (#41)` программа передаёт код возврата в регистре `B`. Родительская программа получает его в регистре `A` после возврата из `EXEC (#40)`. Код также сохраняется в глобальной переменной `ERLEVEL` и может быть получен позже через функцию `WAIT (#42)`:

```asm
; Дочерняя программа: завершение с кодом 1
        ld      b, 1                ; exit code = 1
        ld      c, #41              ; EXIT
        rst     #10

; Родительская программа: получение кода возврата
        ld      hl, child_cmd
        ld      c, #40              ; EXEC
        rst     #10
        jr      c, .exec_error      ; CF=1 — ошибка запуска
        ; A = код возврата дочерней программы
        or      a
        jr      nz, .child_failed

; Или позже через WAIT:
        ld      c, #42              ; WAIT
        rst     #10
        ; A = последний exit code (ERLEVEL)
```

### Другие способы обмена данными

1. **Через файл** — временный файл на диске.
2. **Через память** — если родитель знает физический номер страницы потомка.
3. **Через переменные окружения** — функция `ENVIRON #46`.

```asm
Dss.Environ     EQU #46

; Прочитать/записать переменную окружения
        ld      hl, var_name        ; "PATH", "TEMP" и т.п.
        ld      de, var_value
        ld      c, Dss.Environ
        rst     #10
```

## Пример полной программы с EXEC

```asm
        org     #8100-512

        db      "EXE", 0
        dw      #0200
        dw      #0200
        dw      0
        dw      #8100
        dw      main
        dw      #BFFF
        ds      512 - 16

        org     #8100

main:
        ; Показать приветствие
        ld      hl, welcome
        ld      c, #5C              ; PCHARS
        rst     #10

        ; Запустить дочернюю программу
        ld      hl, cmd
        ld      c, #40              ; EXEC
        rst     #10
        jr      c, .no_child

        ; Показать сообщение о завершении
        ld      hl, done
        ld      c, #5C
        rst     #10
        jr      .exit

.no_child:
        ld      hl, err_msg
        ld      c, #5C
        rst     #10

.exit:
        ld      b, 0                ; exit code 0
        ld      c, #41              ; EXIT
        rst     #10

welcome:    db  "Starting child...", 13, 10, 0
done:       db  "Child finished.", 13, 10, 0
err_msg:    db  "Failed to run child!", 13, 10, 0
cmd:        db  "CHILD.EXE", 0
```

## Ограничения формата EXE

1. **Максимальный размер кода** — ограничен окнами: WIN2 + 3 выделенных страницы = до 64 КБ непрерывного кода.
2. **Для программ > 64 КБ** нужны **оверлеи**: код грузится частями в разные страницы.
3. **Стек** обычно в `#BFFF`, но можно установить где угодно в адресуемой памяти.
4. **Точка входа** — всегда `#8100` в большинстве программ. Другие адреса приводят к непредсказуемым результатам, если не использовать кастомный загрузчик.

---

## Ключевые моменты

> - EXE = 512-байтный заголовок + код.
> - Заголовок: сигнатура "EXE", версия `#0200`, адрес загрузки `#8100`, точка входа, стек.
> - При EXEC DSS выделяет 3 дополнительные страницы и подключает их в WIN1/2/3.
> - Аргументы командной строки — по адресу `#8080` (длина + текст).
> - Выход через `EXIT #41` с кодом возврата в `B`; никогда не через `RET`.
> - `EXEC` возвращает exit code в `A`; `WAIT (#42)` читает последний exit code.
> - Поддержаны вложенные EXEC с сохранением контекста.
