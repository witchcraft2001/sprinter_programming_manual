# DCP: портовый маппер и арбитр памяти

## Роль маппера

`DCP.TDF` — программируемая таблица из 256 слов × 16 бит (`DCP.MIF`). Это центральный декодер всего адресного и портового пространства Z80. DCP отвечает за:

- Формирование адресов DRAM (`MA[11:0]`, `MCA[1:0]`) и стробов `RAS`/`CAS`
- Выбор страницы памяти или устройства (`PAGE[11:0]`)
- Определение типа доступа (`TYPE[3:0]`) для верхнего уровня
- Генерацию управляющих сигналов периферии
- Вставку WAIT-состояний для медленной периферии
- Формирование тактовой частоты Z80

## Структура слова DCP.MIF

```
Биты [15:12] = TYPE   — тип доступа / источник данных при чтении
Биты [11:0]  = PAGE   — детальная конфигурация доступа

Разбивка PAGE:
  [11:8] — подтип / режим регистра (0 для IDE-данных, другие значения — регистры)
  [7:4]  — блочные флаги (IDE, видео, ISA, Beta, CMOS...)
  [3:0]  — адресные биты или флаги команды
```

### Поле TYPE[3:0]

| TYPE[2:1] | TYPE0 | Источник данных | Кто использует |
|-----------|-------|----------------|---------------|
| `11` | x | `MDP[]` — RAM/IDE/FDD | Все обычные устройства |
| `10` | `0` | ZX-клавиатура + TAPE_IN + CBL_IND | Порт 0xFE |
| `10` | `1` | Kempston/мышь (`KEMPS[]`) | Порт 0x58 |
| `01` | x | AY3.DO[] — регистр AY | Порты 0x52/0x8C/0xFFFD |
| `00` | x | `0xFF` (шина в Z-состоянии) | Write-only порты |

`TYPE=7` (т.е. `0111b`) — TYPE[2:1]=11 + TYPE0=1 — формирует дополнительный сигнал `RD_KMPS=0` при чтении (низкий = активен для TM9/Beta/Kempston).

### Поле PAGE[7:4] — блочные флаги

| PAGE[7:4] | Назначение |
|-----------|-----------|
| `0000` | Обычная RAM |
| `0101` | Видеостраница (`GRAF_X=1`) |
| `0110`–`0111` | Beta/TM9 дисковый контроллер |
| `1010`–`1011` | Beta/TM9 (доп. режимы) |
| `1100` | Системные регистры (DCP конфигурация) |
| `1101` | Тип CMOS/RTC |
| `1110` | ROM |
| `1111` | Кэш/ISA |

### Поле PAGE[11:8] — подтип устройства

При `PAGE[7:5]=101` (IDE/HDD):
- `PAGE[11:8]=0` → регистр данных IDE (без задержки, `HDD_W=4`)
- `PAGE[11:8]≠0` → регистры статуса IDE (с задержкой, `HDD_W=10`)

## Полная таблица портов DCP.MIF

Ниже приведены все значимые записи из `DCP.MIF`. Значения в формате `TTPP`, где `TT`=TYPE, `PP`=PAGE.

### Порты FDD/Beta/TM9 (индексы 0x10–0x17)

| Индекс DCP | Порт Z80 | Значение | Назначение |
|-----------|---------|---------|-----------|
| `0x10` | `0x1F` | `0x7018` | WG93 статус / команда (CS1) |
| `0x11` | `0x3F` | `0x7019` | WG93 дорожка |
| `0x12` | `0x5F` | `0x701A` | WG93 сектор |
| `0x13` | `0x7F` | `0x701B` | WG93 данные |
| `0x14` | `0xFF` | `0x7017` | WR_PDOS (PDOS-команда) |
| `0x15` | `0xDF` | `0x701F` | Чтение клавиш / запись A20 |
| `0x16` | `0x23` | `0x7023` | Переключение 720К |
| `0x17` | `0x27` | `0x7027` | Переключение 1440К |

### Порты CMOS/RTC (индексы 0x1C–0x1E)

| Индекс DCP | Порт Z80 | Значение | Назначение |
|-----------|---------|---------|-----------|
| `0x1C` | `0x71` | `0x71D8` | Чтение данных CMOS |
| `0x1D` | `0x70` | `0x70D4` | Запись адреса CMOS |
| `0x1E` | `0x71` | `0x71D4` | Запись данных CMOS |

Детально: `PAGE[7:6]=11` → CMOS, `PAGE[5]=1` → строб, `PAGE[8]` = CS CMOS data/adr.

### Порты IDE/ATA (индексы 0x20–0x2F)

| Индекс DCP | Порт Z80 | Значение | Назначение |
|-----------|---------|---------|-----------|
| `0x20` | `0x60` | `0x60A8` | ATA данные (16-бит, CS1, reg 0) |
| `0x21` | `0x62` | `0x62A8` | ATA ошибки/функция |
| `0x22` | `0x64` | `0x64A8` | ATA счётчик секторов |
| `0x23` | `0x66` | `0x66A8` | ATA номер сектора |
| `0x24` | `0x68` | `0x68A8` | ATA цилиндр (младший) |
| `0x25` | `0x6A` | `0x6AA8` | ATA цилиндр (старший) |
| `0x26` | `0x6C` | `0x6CA8` | ATA голова/диск |
| `0x27` | `0x6E` | `0x6EA8` | ATA команда/статус |
| `0x28` | `0x3F6`→`0x6D` | `0x6DA8` | ATA альт. статус (CS3) |
| `0x29` | `0x3F7`→`0x6F` | `0x6FA8` | ATA управление (CS3) |
| `0x2A` | служебный | `0x7060` | Переключение на HDD1 |
| `0x2B` | служебный | `0x7064` | Переключение на HDD2 |
| `0x2C` | видео | `0x7120` | Высота кадра = 320 строк |
| `0x2D` | видео | `0x7124` | Высота кадра = 312 строк |
| `0x2E` | система | `0x7160` | Программный сброс |
| `0x2F` | система | `0x7164` | Зарезервировано |

Признак IDE: `PAGE[7:5]=101`, `PAGE[6:5]=01` (CS1) или `10` (CS3). `PAGE[4]=0` — строб включён, `MA[5]=1`. Детальный состав `MA[]` при IDE:

```
MA[11:9] = HDD_A[2:0]   (регистр ATA, см. таблицу ниже)
MA[8]    = HDD CS1/CS3
MA[7:6]  = HDD1/HDD2
MA[5]    = строб
MA[3]    = /WR
MA[2]    = MEM.q[2] (доп. бит)
MA[1:0]  = A[6:5]
```

**Таблица HDD_A[3:0] из A[14] и A[2:0]:**

| A[14]:A[2:0] | HDD_A | Регистр ATA |
|-------------|-------|------------|
| X:000 | 0 | Data |
| X:001 | 1 | Error/Features |
| X:010 | 2 | Sector Count |
| X:011 | 3 | LBA bits 0–7 |
| X:100 | 4 | LBA bits 8–15 |
| X:101 | 5 | LBA bits 16–23 |
| X:110 | 0 | — |
| X:111 | 0 | — |
| 1:010 | 6 | Cylinder Hi / LBA 24–27 / DH |
| 1:011 | 7 | Command/Status |
| 1:100 | 14 | Alt Status (CS3) |
| 1:101 | 15 | Device Addr (CS3) |

### Порты ISA слотов (индексы 0x30–0x33)

| Индекс DCP | Значение | Назначение |
|-----------|---------|-----------|
| `0x30` | `0x7000` | Слот 1: порты |
| `0x31` | `0x7001` | Слот 2: порты |
| `0x32` | `0x7002` | Слот 1: окно памяти |
| `0x33` | `0x7003` | Слот 2: окно памяти |

### Порты клавиатуры и Kempston (индексы 0x40, 0x58)

| Индекс DCP | Порт Z80 | Значение | Назначение |
|-----------|---------|---------|-----------|
| `0x40` | `0xFE` | `0x4000` | ZX-клавиатура (TYPE=4 → `KEY/KEMS` с TYPE0=0) |
| `0x58` | `0xFADF`/`0x7ADF` | `0x5000` | Kempston/мышь (TYPE=5 → `KEY/KEMS` с TYPE0=1) |

### Порты AY-3-8910 (индексы 0x52, 0x8C–0x8E)

| Индекс DCP | Порт Z80 | Значение | Назначение |
|-----------|---------|---------|-----------|
| `0x52` | `0xFFFD` | `0x3000` | Чтение регистра AY (TYPE=3 → AY3.DO[]) |
| `0x8C` | `0xFFFD`(зеркало) | `0x3000` | Чтение регистра AY (дубль) |
| `0x8D` | `0xBFFD` | `0x2000` | Запись адреса регистра AY (write-only) |
| `0x8E` | `0xBFFD` | `0x2000` | Запись данных в регистр AY (write-only) |

Примечание: реальные порты AY — `OUT (0x90),A` (адрес) и `OUT (0x91),A` (данные). Индексы `0x8D/0x8E` — стандартные ZX 128 адреса.

### Порты Covox и ROM (индексы 0x88–0x8F)

| Индекс DCP | Порт Z80 | Значение | Назначение |
|-----------|---------|---------|-----------|
| `0x88` | `0xFB??` | `0x2000` | Covox DAC данные |
| `0x89` | `0xFA??` | `0x2000` | Регистр режима Covox (CBL_XX) |
| `0x8F` | `0x7FFD` | `0x2000` | Управление ROM_RG |

### Системные порты (диапазон 0xC0–0xCF)

| Индекс DCP | Значение | Назначение |
|-----------|---------|-----------|
| `0xC0–0xCF` | `0x2000` | Все системные регистры (SC, PN, CNF, RGMOD, ALL_MODE и пр.) |

Запись определяется конкретным младшим нибблом адреса порта:
- `0xC0/0xC8` → SC
- `0xC1/0xC9` → PN
- `0xC2/0xCA` → BORDER (бордер + TAPE_OUT)
- `0xC3` → ALL_MODE
- `0xC5/0xCD` → RGMOD
- `0xC6/0xCE` → CNF (только при SYS_ENA)
- `0xCB` → HOLD

### Страницы RAM (диапазоны 0x90, 0xB0, 0xD0, 0xF0)

Индексы `0x90–0x9F`, `0xB0–0xBF`, `0xD0–0xDF`, `0xF0–0xFF` — страницы RAM 0–63. Значения:

- `0x90/0x91` = `0x3030/0x3031` (тип 3 — для AY/специальных устройств, страницы 0x30/0x31)
- `0x92–0x9F` = `0x2032–0x203F` (обычные RAM-страницы 2–15 банка 0)
- `0xB0–0xBF` = `0x2020–0x202F` (RAM-страницы 16–31)
- `0xD0–0xDF` = `0x2010–0x201F` (RAM-страницы 32–47)
- `0xF0–0xFF` = `0x2000–0x200F` (RAM-страницы 48–63)

### Страницы ROM (диапазон 0xE0)

Индексы `0xE0–0xEF` — ROM страницы. В `DCP.MIF`:
```
0xE0: 2041 2041 2041 2041 2041 2041 2041 2041
      2000 2005 2002 2041 20FF 2000 2000 2041
```
- `0xE8` = `0x2000` — Basic 48K ROM
- `0xE9` = `0x2005` — DOS/TR-DOS
- `0xEA` = `0x2002` — Basic 128K ROM
- `0xEB` = `0x2041` — System ROM A
- `0xEC` = `0x20FF` — System ROM B (спец. флаги)

## Формирование адреса MA[11:0]

`MA[]` содержит RAS-адрес для DRAM (строка). Логика зависит от `IO_RW` и `MA_CT1` (два такта мультиплексирования):

**Такт MA_CT=0 (I/O=0, обращение к памяти):**
```
MA[11:0] = X_ADR[11:0]
X_ADR = (0, CNF4, PN5, DOS, /WR, A15, A14, A[6:5], A13, A7, A[2])
```

**Такт MA_CT=0 (I/O=1, I/O цикл):**
```
MA[11:0] = X_ADR[11:0]
X_ADR = (0, 0, CNF[4:3], 01000000b)
```

**Такт MA_CT=1 (IDE цикл):**
```
MA[11:0] = (HDD_A[2:0], MEM.q[8:6],
            (MEM.q[5:4] & (!/IOMZ, !/IOMY)),
            /WR, MEM.q[2], A[6:5])
```

**Такт MA_CT=1 (I/O, не IDE):**
```
MA[11:0] = (HDD_A[2:0], 0,0,0,0,0, /WR, 1, A[6:5])
```

`MCA[1:0]` — адрес CAS (столбец), всегда = `A[1:0]`.

`MA[14:12]` в верхнем уровне: `(!TURBO, INT_X, NMI_X)` — несут не адрес, а служебные сигналы на шлейф.

## Тайминги и WAIT-состояния

### Цикл памяти

Диаграмма (CT = 3-битный счётчик CLK42):

```
CLK42:     ▔▔▔▔____▔▔▔▔____▔▔▔▔____▔▔▔▔
CT:         0    1    2    0    1    2
MC_BEGIN:  ▔▔▔▔▔▔▔▔____▔▔▔▔▔▔▔▔▔▔▔▔
MC_END:    ________▔▔▔▔▔▔____________
RAS:       ▔▔▔▔________▔▔▔▔▔▔▔▔▔▔▔▔
CAS:       ▔▔▔▔▔▔▔▔____▔▔▔▔▔▔▔▔▔▔▔▔
```

RAS активен при `CT=0`, CAS — при `CT=1`. MC_BEGIN ждёт `CT1 & CT2`. MC_END — при `CT0 & CT2 & CONTINUE`.

### Таблица WAIT W_TAB

Количество дополнительных тактов ожидания зависит от `TURBO` и кода `MEM.q[14:12]`:

| MEM.q[14:12] | TURBO=0 | TURBO=1 | Устройство |
|-------------|---------|---------|-----------|
| 0 | 2 | 2 | Обычный доступ |
| 1 | 2 | 2 | Обычный доступ |
| 2 | 1 | 4 | Быстрый / Beta CS |
| 3 | 1 | 4 | Beta данные |
| 4 | 1 | 1 | RAM fast |
| 5 | 2 | 2 | |
| 6 | 7 | 7 | IDE регистры / CMOS |
| 7 | 10 | 10 | IDE команды (медленно) |

Счётчик `WT_CT[3:0]` загружается из `W_TAB[]` при входе в цикл ввода/вывода, затем декрементируется каждый CLK21.

### WAIT для IDE (HDD_W)

```
PAGE[11:8] == 0  →  HDD_W = 4   (данные, быстро)
PAGE[11:8] != 0  →  HDD_W = 10  (регистры статуса, медленно)
```

### HDD_FLIP — 16-битный IDE

`HDD_FLIP` переключает между младшим и старшим байтом 16-битного IDE-слова. Активируется на сигнале `/IOM`, при `HDD_ENA=1` и `DCPP == 0010XXXX`:

```vhdl
HDD_FLIP.clk  = /IOM
HDD_FLIP.ena  = HDD_ENA & DFF((DCPP == 0010XXXX), CLK42)
HDD_FLIP.d    = !HDD_FLIP & (PAGE[11:8] == 0)
HDD_FLIP.clrn = /RESET & DFF(GND, !DOUBLE_CAS, , HDD_FLIP)
```

Первый цикл чтения IDE данных: `HDD_FLIP=0` → читается `MD[7:0]`. Второй: `HDD_FLIP=1` → читается `HDDR[]` (буфер старшего байта).

## Сигналы синхронизации периферии

`/IOM`, `/IOMM`, `/IOMX`, `/IOMY`, `/IOMZ` — последовательные задержки сигнала активности I/O цикла:

```
/IOM  = активен весь I/O цикл
/IOMM = активен, пока WT_CT != 0
/IOMX = /IOMM с задержкой 1 такт
/IOMY = /IOMX with дополнительным условием по WT_CT == 000X
/IOMZ = /IOMX с учётом A8 xor /RD (для дифференциации чтения/записи IDE)
```

Эти сигналы управляют внешними шлейфами и используются для формирования строба `MA[5]` (HDD/CMOS strobe).

## Практические советы

- При добавлении порта: назначьте свободный индекс в `DCP.MIF`, заполните TYPE и PAGE. Не забудьте `MEM.q[14:12]` (W_TAB) — от него зависит количество WAIT.
- Все порты `0xC?` зеркалируются (маска `1100XXXX`), поэтому `0xC0` и `0xC8` попадают в один регистр `SC`.
- `PORTS_X = (DCPP[7:4] == 2) or (DCPP[7:4] == 1)` — признак «порты первого уровня» (FDD, Beta). При `PORTS_X=1` сигнал `/IOMY.prn = PORTS_X` сбрасывает задержку.
- Ни один из конфигурационных регистров (`SC`, `PN`, `CNF`, `ALL_MODE`, `RGMOD`, `ROM_RG`, `ISA_PORT`) не сбрасывается сам по себе — все требуют явного `/RESET`.
- Параметр `UPDATE=1` переносит логику кэша/ISA в `SP2_ACEX.TDF`. В DCP при `UPDATE=1`: `BLK_R=SC4`, `CASH_ON=0`, `/CASH=0` — всё кэш-управление в верхнем модуле.
