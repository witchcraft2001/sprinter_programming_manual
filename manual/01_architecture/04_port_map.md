## Обзор системы декодирования портов

В Sprinter декодирование портов I/O реализовано в FPGA **ACEX 1K30** через таблицу **DCP** (Dynamic Configuration Port) — 256-элементную LUT-память шириной 16 бит (`DCP.MIF`).

При каждой операции I/O:
1. Адрес порта (8-битный A7–A0) подаётся на вход LUT.
2. LUT возвращает 16-битный код `DCPP[15..0]`.
3. Старший нибл `TYPE[3..0]` (биты 15–12) определяет тип операции.
4. Младший байт `MA[11..0]` (биты 11–0) управляет мультиплексором MAX7000 и HDD/CMOS сигналами.

### Типы портов (`TYPE[3..0]`)

| TYPE | Код | Описание |
|------|-----|----------|
| 0 | `0000` | Нет функции (заглушка) |
| 1 | `0001` | HDD данные (INIR/OTIR через Z80) |
| 2 | `0010` | Запись в порт (системный) |
| 3 | `0011` | Чтение AY / COVOX-read |
| 4 | `0100` | Чтение клавиатуры |
| 5 | `0101` | Kempston Mouse |
| 6 | `0110` | Чтение/запись видео (RGMOD/RGADR) |
| 7 | `0111` | MAX7000: FDD/HDD/CMOS |
| C | `1100` | Системные порты (страницы, режимы) |

## Полная карта портов

### Управление памятью (банкинг)

| Порт | Имя | Чтение / Запись | Описание |
|------|-----|-----------------|----------|
| `#82` | `EmmWin.P0` | RW | Номер страницы в окне 0 (`0x0000–0x3FFF`) |
| `#A2` | `EmmWin.P1` | RW | Номер страницы в окне 1 (`0x4000–0x7FFF`) |
| `#C2` | `EmmWin.P2` | RW | Номер страницы в окне 2 (`0x8000–0xBFFF`) |
| `#E2` | `EmmWin.P3` | RW | Номер страницы в окне 3 (`0xC000–0xFFFF`) |

Диапазон `#90`–`#9F` и `#B0`–`#BF`, `#D0`–`#DF`, `#F0`–`#FF` — это зеркала (алиасы) тех же четырёх регистров банкинга (по 16 адресов на каждый). Для совместимости с Pentagon/Scorpion используйте точные адреса выше.

```asm
EmmWin.P0   EQU     #82
EmmWin.P1   EQU     #A2
EmmWin.P2   EQU     #C2
EmmWin.P3   EQU     #E2

; Переключить окно 1 на страницу 7 (RAM)
        ld      a, 7
        out     (EmmWin.P1), a
```

### Видео и графика

| Порт | Имя | Чтение / Запись | Описание |
|------|-----|-----------------|----------|
| `#89` | `PORT_Y` / `RGADR` | W | Запись адреса палитры / номера цвета |
| `#C9` | `RGMOD` | W | Режим видео и управление экраном |
| `#C2`/`#CA` | `BORDER` (DCP B"1100X010") | W | Бордюр (биты [2:0] = цвет) |
| `#C3` | `ALL_MODE` | W | Графический режим (bit[1:0]: 00=ZX, 01=320×256, 10=640×512) |
| `#CB` | `HOLD` | W | Управление акселератором (HOLD) |
| `#C5`/`#CD` | `RGMOD` alias | W | Дублирование RGMOD |

**Порт `#89` (PORT_Y / RGADR)** — адрес/данные палитры:

Запись числа от 0 до 255 выбирает индекс палитры. Следующая запись задаёт цвет в формате BGR (6-6-6).

**Порт `#C9` (RGMOD)**:

| Бит | Назначение |
|-----|-----------|
| 0 | Double buffer flip: 0 = экран A активен, 1 = экран B активен |
| 1 | Screen off: 1 = экран выключен |
| 4 | Screen on/off |

```asm
RGADR           EQU     #89
RGMOD           EQU     #C9

; Установить палитровую запись 0 в белый (R=63, G=63, B=63)
        ld      a, 0            ; индекс цвета
        out     (RGADR), a
        ld      a, 63           ; R
        out     (RGADR), a
        ld      a, 63           ; G
        out     (RGADR), a
        ld      a, 63           ; B
        out     (RGADR), a

; Переключить отображаемый буфер (double buffering)
        in      a, (RGMOD)
        xor     1
        out     (RGMOD), a
```

**Порт `#C3` (ALL_MODE)** — выбор графического режима:

| Значение | Режим |
|---------|-------|
| `#00` | ZX Spectrum (256×192, атрибуты) |
| `#01` | 320×256, 8bpp (256 цветов из палитры) |
| `#02` | 640×512, 4bpp (16 цветов из палитры) |
| `#80` | Pentagon 128 совместимость |

```asm
ALL_MODE        EQU     #C3

; Включить режим 320×256
        ld      a, #01
        out     (ALL_MODE), a
```

### COVOX (PCM DAC)

| Порт | Имя | Описание |
|------|-----|----------|
| `#88` | `CBL_WR` | Запись 8-битного аудиосэмпла |
| `#89` | `CBL_XX` | Управление режимом COVOX |

**Порт `#89` (режим COVOX)**:

| Бит | Назначение |
|-----|-----------|
| 7 | 1 = COVOX включён |
| 6 | 1 = стерео режим (16-битный) |
| 5 | 1 = 16-битные сэмплы |
| 4 | 1 = прерывания от COVOX |
| [3:0] | Частота дискретизации |

Таблица частот (биты [3:0]):

| Код | Частота |
|-----|---------|
| `#0` | 7 813 Гц |
| `#1` | 10 417 Гц |
| `#2` | 13 889 Гц |
| `#3` | 15 625 Гц |
| `#4` | 20 833 Гц |
| `#5` | 31 250 Гц |
| `#6` | 41 667 Гц |
| `#7` | 62 500 Гц |
| `#8` | 83 333 Гц |
| `#9` | 109 375 Гц |

```asm
COVOX_DAT       EQU     #88
COVOX_MOD       EQU     #89

; Настроить COVOX: моно, 8-bit, 31.25 кГц (код 5)
        ld      a, #85          ; bit7=on, bits[3:0]=5
        out     (COVOX_MOD), a

; Вывести сэмпл
        ld      a, (sample_ptr)
        out     (COVOX_DAT), a
```

### AY-3-8910 (звуковой синтезатор)

AY эмулируется внутри FPGA ACEX 1K30.

| Порт | Имя | Описание |
|------|-----|----------|
| `#8C` | `AY_D_RD` | Чтение данных из текущего регистра AY |
| `#8D` | `AY_A_WR` | Запись номера регистра AY (0–15) |
| `#8E` | `AY_D_WR` | Запись данных в текущий регистр AY |
| `#52` | `AY_D_RD2` | Дополнительный порт чтения AY |

> **Примечание:** В некоторых источниках встречаются порты `#90`/`#91` (AY адрес/данные) — это DCP-алиасы, актуальные для SDK. `#8D`/`#8E` — стандарт для прямого доступа.

```asm
AY_REG          EQU     #8D     ; выбор регистра
AY_DAT_WR       EQU     #8E     ; запись данных
AY_DAT_RD       EQU     #8C     ; чтение данных

; Записать значение в регистр AY: REG → VALUE
; Пример: установить громкость канала A (регистр 8) = 15
        ld      a, 8
        out     (AY_REG), a
        ld      a, 15
        out     (AY_DAT_WR), a

; Прочитать текущее значение
        in      a, (AY_DAT_RD)
```

### Flash / ROM

| Порт | Имя | Описание |
|------|-----|----------|
| `#8F` | `ROM_RG` | Управление Flash: режим записи (бит 4 = разрешение записи), выбор страницы |

```asm
ROM_RG          EQU     #8F

; Включить запись во Flash (бит 4)
        ld      a, #10
        out     (ROM_RG), a

; Выключить запись
        xor     a
        out     (ROM_RG), a
```

### Клавиатура ZX и Kempston

| Порт | Имя | Описание |
|------|-----|----------|
| `#FE` | ZX_KBD | Чтение клавиш ZX (A8–A12 = строка клавиатуры) |
| `#40` | KBD_READ | Чтение расширенной клавиатуры (DCP type 4) |
| `#F8` | `P_KBD_OUT` | Запись команды в контроллер PS/2 |
| `#FE` | `P_KBD_IN` | Чтение ответа от PS/2 контроллера |

Порт `#FE` — стандартный ZX-порт: биты [4:0] = состояние 5 клавиш строки, бит 6 = tape in, бит 7 = tape out.

### PS/2 клавиатура (через SIO Z84C15)

Основной способ приёма PS/2 сканкодов — через встроенный **SIO канал A** процессора Z84C15:

| Порт | Имя | Описание |
|------|-----|----------|
| `#18` | `SIO_DATA_REG_A` | Принятый байт от PS/2 клавиатуры |
| `#19` | `SIO_CONTROL_A` | Статус SIO; бит 0 = 1 когда байт готов |
| `#1A` | `DAT_B` | SIO канал B данные (KBD_DAT) |
| `#1B` | `COM_B` | SIO канал B управление (KBD_COM) |

### Kempston Mouse

| Порт | Описание |
|------|----------|
| `#58` (A10=0, A8=0) | Кнопки мыши (биты [2:0]) |
| `#158` (A10=0, A8=1) | X-координата мыши |
| `#358` (A10=1, A8=1) | Y-координата мыши (инверсная) |

Тип `5` в DCP MIF — это Kempston Mouse.

```asm
MOUSE_BTN       EQU     #58     ; кнопки (при A10=0, A8=0)

; Прочитать кнопки мыши
        in      a, (MOUSE_BTN)
        and     #07             ; 3 кнопки
```

### CTC (счётчики/таймеры Z84C15)

| Порт | Имя | Описание |
|------|-----|----------|
| `#10` | `CTC_CH0` | Канал 0 |
| `#11` | `CTC_CH1` | Канал 1 |
| `#12` | `CTC_CH2` | Канал 2 (системный таймер ~50 Гц) |
| `#13` | `CTC_CH3` | Канал 3 (дополнительный таймер) |

### FDD (дисковод) — через MAX7000

| Порт | Назначение |
|------|-----------|
| `#1F`/`#0F` | WG93 статус (RD) |
| `#3F` | WG93 трек |
| `#5F` | WG93 сектор |
| `#7F` | WG93 данные |
| `#FF` | WG93 команда (WR_PDOS) |
| Бит ISA_A20 | А20 управление (порт `#1B`) |

Доступ через MAX7000 с паттерном `TYPE=7` в DCP.MIF. FDD контроллер совместим с WD1793.

### IDE (жёсткий диск)

IDE порты декодируются через внешнюю шину — адрес на старших битах A15–A8 + порт.

**Запись (WR):**

| Порт | Имя | ATA-эквивалент |
|------|-----|----------------|
| `#0150` | `HDW_DAT` | 1F0h Data |
| `#0151` | `HDW_ERR` | 1F1h Features/Error |
| `#0152` | `HDW_CNT` | 1F2h Sector Count |
| `#0153` | `HDW_SEC` | 1F3h Sector Number |
| `#0154` | `HDW_CLL` | 1F4h Cylinder Low |
| `#0155` | `HDW_CLH` | 1F5h Cylinder High |
| `#4152` | `HDW_DRV` | 1F6h Drive/Head |
| `#4153` | `HDW_COM` | 1F7h Command |
| `#4154` | `P_HD3F6` | 3F6h Device Control |

**Чтение (RD):**

| Порт | Имя | ATA-эквивалент |
|------|-----|----------------|
| `#0050` | `HDR_DAT` | 1F0h Data |
| `#0051` | `HDR_ERR` | 1F1h Error |
| `#0052` | `HDR_CNT` | 1F2h Sector Count |
| `#0053` | `HDR_SEC` | 1F3h Sector Number |
| `#0054` | `HDR_CLL` | 1F4h Cylinder Low |
| `#0055` | `HDR_CLH` | 1F5h Cylinder High |
| `#4052` | `HDR_DRV` | 1F6h Drive/Head |
| `#4053` | `HDR_CTL` | 1F7h Status |
| `#4055` | `P_HD3F7` | 3F7h Alt Status |

Биты статуса (`HDR_CTL`):

| Бит | Имя | Описание |
|-----|-----|----------|
| 7 | BSY | Устройство занято |
| 6 | RDY | Устройство готово |
| 3 | DRQ | Данные готовы к передаче |
| 0 | ERR | Ошибка |

Данные передаются через `HDW_DAT`/`HDR_DAT` с командами `OTIR`/`INIR` (блочный режим).

### CMOS / RTC

CMOS доступен через ISA-шину с особыми адресами:

| Порт | Имя | Описание |
|------|-----|----------|
| `#FFBDh` | `CMOS_DRD` | Чтение данных из CMOS |
| `#BFBDh` | `CMOS_DWR` | Запись данных в CMOS |
| `#DFBDh` | `CMOS_AWR` | Запись адреса регистра CMOS |

Прямые порты через MAX7000 (DCP MA битовые поля CMOS):

| DCP-порт | Функция |
|----------|---------|
| `#1C` | CMOS_DAT_RD |
| `#1D` | CMOS_ADR_WR |
| `#1E` | CMOS_DAT_WR |

### ISA слот

| Значение | Описание |
|----------|----------|
| `#9FBDh` | `ISA_PORT` — базовый порт ISA |
| `#C000` | `isa_adr_base` — базовый адрес ISA в пространстве памяти |

ISA адрес формируется через регистр `ISA_PORT[7..0]`, биты A[9:2] из регистра ISA_PORT.

```asm
Port.ISA        EQU     #9FBDh
isa_adr_base    EQU     #C000

; Вывести байт на ISA порт (например, COM1: #3F8)
        ld      bc, Port.ISA
        ld      a, #F8          ; младший байт ISA-адреса
        out     (c), a
```

## Ключевые моменты

> - Декодирование портов в FPGA ACEX 1K30 через 256×16-бит LUT (`DCP.MIF`).
> - Поле `TYPE[3..0]` (биты 15–12) определяет функцию: 0=нет, 1=HDD, 2=write, 3=read AY, 4=keyboard, 5=mouse, 7=MAX7000, C=системный.
> - Память переключается через `#82`/`#A2`/`#C2`/`#E2` (EmmWin.P0–P3).
> - Видео: `#89` (палитра), `#C9` (RGMOD, double buffer), `#C3` (режим).
> - IDE использует нестандартные Z80-порты с старшими битами адреса.
> - AY через `#8D`/`#8E`/`#8C`; ZX-совместимый вариант через `#FFFD`/`#BFFD`.
