# 6.3 COVOX DMA режим

> **Навигация:** [← 6.2 COVOX](02_covox.md) | [Оглавление](../README.md) | [6.4 Примеры звука →](04_examples.md)

## Что такое DMA-режим COVOX

В простом режиме COVOX программа должна записывать каждый сэмпл в порт `#88` с точным таймингом. Это блокирует CPU и нестабильно при прерываниях.

**DMA-режим** решает проблему: FPGA сама читает сэмплы из памяти (обычно со страницы `#FD`) и подаёт их на DAC. CPU только **подготавливает буфер** и обрабатывает прерывания при необходимости дозаполнения.

---

## Механизм DMA в CBL

Анализ `SP2_ACEX.TDF` показывает:

```
CBL_WR = (DFF((DCPP[] == B"10001000"),CLK42,,) & !/IOWR) or
         (DFF((DECODE.PAGE[7..0] == B"11111101"),CLK42,(CBL_INT_ENA & ACC.ACC_DIR1),) &
          !DFF((DECODE.MC_WRITE or DECODE.CAS),CLK42,,/RESET));
```

Это означает, что CBL может писать сэмплы **автоматически при обращениях к странице `#FD`** — запись в аккумулятор через акселератор с активной DMA.

Детали механизма сложны и выходят за рамки простого примера. На практике SDK содержит готовые функции DMA-воспроизведения, и эта документация описывает их использование.

---

## Страница #FD — DMA буфер

При включённом DMA страница `#FD` трактуется как **буфер сэмплов**:
- CBL читает байты из `#FD` с частотой, заданной в регистре `#89`.
- Программа пишет новые сэмплы в эту же страницу.

Типичный flow:
1. Выделить страницу памяти для DMA буфера.
2. Подключить её как `#FD` (может требовать специальных команд DSS).
3. Заполнить начальные сэмплы.
4. Включить CBL с флагом `CBL_INT_ENA`.
5. В обработчике прерывания дозаписывать новые сэмплы.

---

## Инициализация DMA

Упрощённая схема (требует доработки в зависимости от прошивки):

```asm
; Включить COVOX в DMA режиме
InitCovoxDMA:
        ; Настроить режим: моно 8 бит, 31.25 кГц, с прерываниями
        ld      a, #9C          ; 10011100 = on + INT + 31.25 kHz
        out     (#89), a
        ret
```

Для полноценной работы DMA требуется знание внутренней архитектуры FPGA 3.x, которая несколько отличается от 2.17.

## Альтернатива: стриминг через прерывания

Более переносимый подход — использовать прерывания COVOX для чтения/записи буфера в обычных страницах RAM:

```asm
; Двойной буфер 256 байт каждый
; Прерывание срабатывает, когда аппаратный FIFO опустошается наполовину

AudioBuffer:    ds 256 * 2  ; 2 × 256 = 512 байт

cur_buf:        db 0        ; текущий буфер (0 или 1)

InitStreamingAudio:
        ; Установить обработчик прерываний COVOX (через IM2)
        ld      hl, CovoxISR
        ld      (#80FF), hl  ; IM2 вектор (при I = #80)
        ld      a, #80
        ld      i, a
        im      2
        ei

        ; Включить COVOX с прерываниями
        ld      a, #9C
        out     (#89), a
        ret

CovoxISR:
        push    af
        push    bc
        push    hl

        ; Определить текущий буфер
        ld      a, (cur_buf)
        or      a
        jr      z, .buf_a

.buf_b:
        ld      hl, AudioBuffer + 256
        jr      .write

.buf_a:
        ld      hl, AudioBuffer

.write:
        ; Записать 256 байт в COVOX
        ld      b, 0            ; 256 итераций
.loop:
        ld      a, (hl)
        out     (#88), a
        inc     hl
        djnz    .loop

        ; Переключить буфер
        ld      a, (cur_buf)
        xor     1
        ld      (cur_buf), a

        ; Поднять флаг "буфер готов" для основного потока
        ld      a, 1
        ld      (buf_ready), a

        pop     hl
        pop     bc
        pop     af
        ei
        reti

buf_ready:  db 0
```

## Главный цикл: заполнение буфера

```asm
MainLoop:
        ; Проверить, нужно ли заполнить буфер
        ld      a, (buf_ready)
        or      a
        jr      z, .do_other

        ; Заполнить неактивный буфер новыми данными
        xor     a
        ld      (buf_ready), a

        ld      a, (cur_buf)    ; текущий активный
        ; Заполнить *другой* буфер
        xor     1
        jr      z, .fill_a

.fill_b:
        ld      de, AudioBuffer + 256
        jr      .generate

.fill_a:
        ld      de, AudioBuffer

.generate:
        ; Сгенерировать 256 новых сэмплов
        call    GenerateSamples

.do_other:
        ; ... другая работа ...
        jr      MainLoop

; Пример: простая синусоида (заглушка)
GenerateSamples:
        ld      b, 0            ; 256 байт
        ld      a, (phase)
.loop:
        ld      (de), a         ; очень простой "треугольник"
        inc     de
        add     a, 2
        djnz    .loop
        ld      (phase), a
        ret

phase:  db 0
```

## Использование SDK

SDK содержит готовые модули:
- `ayfxplay.asm` — плеер для WYZ/AYFX звуковых эффектов (использует AY)
- COVOX для фоновой музыки / эффектов

Пример интеграции:
```asm
; В обработчике VSync прерывания
Handler50Hz:
        call    PlayMusicAY     ; AY плеер
        call    UpdateCovoxBuf  ; заполнить COVOX буфер
        ret
```

## Частоты и размер буфера

| Частота | Сэмплов/50 Гц кадр | Байт/кадр |
|---------|-------------------|-----------|
| 7.8 кГц | 156 | 156 |
| 15.6 кГц | 312 | 312 |
| 31.25 кГц | 625 | 625 |
| 44 кГц | 880 | 880 |
| 109 кГц | 2188 | 2188 |

Для частоты 31.25 кГц моно 8 бит оптимально генерировать **625 байт каждые 20 мс** (по VSync).

## Заливка буфера через акселератор

Для статичного воспроизведения можно заполнить буфер синусоидой через `LDIR` / акселератор:

```asm
; Заполнить буфер 256 байт повторяющейся синусоидой из таблицы
FillBufFromTable:
        ld      hl, sine_table
        ld      de, AudioBuffer
        ld      bc, 256
        ldir
        ret

sine_table:
        ; 256 байт синусоиды, предвычисленной
        ds  256
```

## Стерео стриминг

Для стерео нужно `CBL_STEREO=1` и писать пары сэмплов:

```asm
        ld      a, #DC          ; 11011100 = on + stereo + 31.25 kHz
        out     (#89), a

; В буфере: L0, R0, L1, R1, ..., L127, R127 (256 байт = 128 стерео сэмплов)
```

## Ключевые моменты

> - DMA режим CBL позволяет автоматическое чтение сэмплов из страницы `#FD` (сложная настройка).
> - Практический подход: использовать прерывания CBL для стриминга двойного буфера в обычной RAM.
> - Частота задаётся битами [3:0] порта `#89`; прерывания — битом 4.
> - Буфер 256 байт требует дозаполнения каждые ~8 мс (при 31.25 кГц).
> - Типичная схема: двойной буфер, prcomputing в главном цикле, запись в `#88` из обработчика прерывания.
> - Используйте готовые решения из SDK для музыкальных форматов.
