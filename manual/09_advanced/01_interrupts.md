## Режимы прерываний Z80

Z80 поддерживает 3 режима прерываний:

| Режим | Инструкция | Поведение |
|-------|-----------|-----------|
| IM 0 | `IM 0` | Устройство кладёт опкод на шину, CPU исполняет |
| IM 1 | `IM 1` | При прерывании → `RST #38` |
| IM 2 | `IM 2` | Векторный режим: I × 256 + vector |

В Sprinter чаще всего используется **IM 2** — он позволяет разделить прерывания от разных источников (VSync, CTC, SIO).

---

## Источники прерываний

| Источник | Частота | Описание |
|----------|---------|----------|
| VBlank (видео) | 50 Гц (PAL) / 60 Гц (NTSC) | Конец кадра — основной таймер |
| CTC каналы | настраивается | Пользовательские таймеры |
| SIO Rx | по событию | Получен байт PS/2 клавиатуры |
| CBL (COVOX) | настраивается | Буфер аудио опустошён |

---

## IM 1 — простой режим

При IM 1 все прерывания идут на `RST #38` (адрес `0x0038`).

```asm
        im      1
        ei

; Обработчик по адресу 0x0038
        org     #0038
        push    af
        ; ... обработка ...
        pop     af
        ei
        reti
```

Проблема IM 1: невозможно различить источники — одна точка входа для всех прерываний.

---

## IM 2 — векторный режим

В IM 2 Z80 вычисляет адрес обработчика по формуле:
```
Vector_addr = (I << 8) | Vector_byte
```

где `I` — регистр I (старший байт), `Vector_byte` — значение, которое устройство кладёт на шину данных при подтверждении прерывания.

По этому адресу лежит **16-битный указатель** на обработчик. Z80 читает его и переходит туда.

### Проблема в Sprinter

Регистр I можно установить произвольно (обычно `#80`), но устройства (особенно CTC) кладут разные векторы. Чтобы не готовить отдельный обработчик на каждый вектор, используется **трюк с таблицей из 257 одинаковых байт**.

---

## Стандартная настройка IM 2 в Sprinter

### Таблица векторов на 0xFE00

Создаётся таблица в 257 одинаковых байт (обычно `#FD`). Адрес `I × 256 + vector` всегда попадает в эту таблицу независимо от значения vector. По адресу `#FDFD` (0xFDFD) лежит указатель на единый обработчик.

```asm
        ; Заполнить таблицу #FE00 .. #FF00 значением #FD
        ld      hl, im2_table
        ld      de, im2_table + 1
        ld      bc, 256
        ld      (hl), #FD
        ldir

        ; Указатель: #FDFD → адрес обработчика
        ld      hl, int_handler
        ld      (#FDFD), hl

        ; Установить I = #FE
        ld      a, #FE
        ld      i, a

        im      2
        ei

        org     #FE00
im2_table:
        ds      257, #FD
```

### Обработчик

```asm
int_handler:
        push    af
        push    bc
        push    de
        push    hl
        push    ix
        push    iy
        ex      af, af'
        push    af
        exx
        push    bc
        push    de
        push    hl

        ; ... основная обработка ...
        call    VSyncHandler

        pop     hl
        pop     de
        pop     bc
        exx
        pop     af
        ex      af, af'
        pop     iy
        pop     ix
        pop     hl
        pop     de
        pop     bc
        pop     af
        ei
        reti
```

## Пример из SDK (lib_startup.asm)

SDK использует `I = #80` и таблицу на `#8000`–`#80FF`:

```asm
; set_im2: установить обработчик IM2
; DE = адрес обработчика
set_im2:
        ld      hl, #80FF
        ld      (hl), e
        inc     hl
        ld      (hl), d

        ld      a, #80
        di
        ld      i, a
        im      2
        ei
        ret
```

Таблица `#8000`–`#80FF` — 257 байт `#FF`, все указывают на адрес `(#80FF, #8100)`. При прерывании Z80 читает слово по `I × 256 + vector = #80FF`, что даёт адрес `#DE`, запомненный через `set_im2`.

---

## Пример из flappybird

```asm
; Установка IM2 из flappybird/src/im2_utils.asm
set_im2:
        ld      hl, #80FF       ; адрес в таблице векторов
        ld      (hl), e         ; lo-байт адреса
        inc     hl
        ld      (hl), d         ; hi-байт адреса

        ld      a, i
        ld      (save_i), a
        ld      a, #80          ; таблица с 0x8000
        di
        ld      i, a            ; I = #80
        im      2
        ei
        ret

save_i: db 0
```

## VSync прерывание

Основное прерывание Sprinter — конец кадра видео, 50 Гц (PAL). В обработчике обычно:

```asm
VSyncHandler:
        ; 1. Обработать аудио AY (обновить регистры из плеера)
        call    PlayMusic

        ; 2. Обновить ввод (клавиатура, мышь)
        call    UpdateInput

        ; 3. Установить флаг "новый кадр" для главного цикла
        ld      a, 1
        ld      (frame_ready), a

        ret
```

## Подсчёт кадров

```asm
; Счётчик кадров в обработчике
VSyncHandler:
        push    hl
        ld      hl, (frame_count)
        inc     hl
        ld      (frame_count), hl
        pop     hl
        ret

frame_count:    dw 0
```

## Ожидание VSync (halt)

```asm
; Главный цикл: рисовать и ждать кадр
GameLoop:
        call    DrawFrame
        halt                        ; ждать следующего VSync
        jr      GameLoop
```

`HALT` работает только при разрешённых прерываниях (`EI`). Z80 останавливается и возобновляется на следующем прерывании.

---

## Приоритеты прерываний

Z80 не имеет аппаратного приоритета — прерывания обрабатываются в порядке поступления, одно за раз. Если во время обработки приходит новое — оно ждёт `EI` + `RETI`.

**Вложенные прерывания** (reentrant) возможны только если в обработчике явно сделать `EI`, но это требует тщательного управления стеком.

## Пример: полный каркас программы с IM2

```asm
        org     #8100

main:
        di

        ; Заполнить таблицу IM2
        ld      hl, #FE00
        ld      de, #FE01
        ld      bc, 256
        ld      (hl), #FD
        ldir

        ; Указатель обработчика на #FDFD
        ld      a, #C3              ; JP opcode
        ld      (#FDFD), a
        ld      hl, int_handler
        ld      (#FDFE), hl

        ld      a, #FE
        ld      i, a
        im      2
        ei

        ; Главный цикл
.loop:
        halt                        ; ждать VSync
        call    GameLogic
        jr      .loop

int_handler:
        push    af
        push    hl

        ld      hl, (tick_count)
        inc     hl
        ld      (tick_count), hl

        pop     hl
        pop     af
        ei
        reti

tick_count:     dw 0

GameLogic:
        ; ... игровая логика ...
        ret
```

## Ключевые моменты

> - Z80 имеет 3 режима: IM 0, IM 1 (RST #38), IM 2 (векторный).
> - В Sprinter используется в основном **IM 2**.
> - Стандартный трюк: таблица из 257 одинаковых байт на выровненной странице (например, `#FE00`–`#FF00` = `#FD`), указатель на обработчик по `#FDFD`.
> - SDK использует `I = #80`, таблица `#8000`–`#80FF`.
> - Основной источник прерываний — VBlank (50 Гц).
> - CTC каналы — настраиваемые таймеры.
> - Сохраняйте все регистры в начале обработчика, восстанавливайте в конце, используйте `EI + RETI`.
