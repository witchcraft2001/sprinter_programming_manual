## Архитектура видеосистемы

Видеоконтроллер Sprinter реализован в FPGA **ACEX 1K30** (файл `VIDEO2.TDF`) и имеет следующие ключевые характеристики:

| Параметр | Значение |
|---------|---------|
| VRAM | 2 × 256 КБ (512 КБ всего) |
| Организация VRAM | 4 независимых банка по 8 бит (32-битный доступ/такт) |
| Частота кадров | ~50 Гц (PAL) или 60 Гц (NTSC) |
| Режимы | ZX (256×192), 320×256 8bpp, 640×256 4bpp |
| Палитра | 256 цветов из 16 777 216 (6-6-6 RGB) |
| Double buffering | Поддерживается через RGMOD |

### Физическая структура VRAM

```
VRAM 512 КБ
├── Банк 0 (8 бит) — 128 КБ
├── Банк 1 (8 бит) — 128 КБ
├── Банк 2 (8 бит) — 128 КБ
└── Банк 3 (8 бит) — 128 КБ
```

Все 4 банка работают параллельно — за один такт видеоконтроллер читает 32 бита (4 пикселя в режиме 320×256 8bpp).

---

## Страницы VRAM

VRAM доступна через банкинг как страницы `#50`–`#5F` (16 × 16 КБ = 256 КБ). Вторые 256 КБ используются для скрытого буфера (second screen) и спрайтов.

| Страница | Константа | Назначение |
|---------|----------|-----------|
| `#50` | `VPAGE_TILES` | Начало видеобуфера (экран A) |
| `#51`–`#54` | — | Продолжение экрана A |
| `#55`–`#58` | — | Экран B (для double buffering) |
| `#5C` | `VPAGE_SPRITES` | Спрайты |
| `#5D`–`#5F` | — | Продолжение спрайтов |

## Видеорежимы

### ZX Spectrum (256×192, 8-цветные атрибуты)

Полностью совместим с ZX Spectrum 48K/128K:
- 6912 байт экрана (256×192 пикселей / 8 = 6144 + 768 атрибутов)
- Адресация: нелинейная (стандартная ZX)
- Маппинг: WIN1 (`0x4000`) при `SC0=1`

### 320×256, 8bpp (256 цветов)

Линейный буфер, **1 байт = 1 пиксель**:
- Адрес пикселя `(x,y)` = `BASE + y*320 + x`
- Объём одного экрана: 320 × 256 = **81 920 байт** (~5 страниц по 16 КБ)
- Double buffering: 2 экрана = 163 840 байт

### 640×256, 4bpp (16 цветов из 16М)

Линейный буфер, **2 пикселя = 1 байт** (старший ниббл = левый пиксель). 16 одновременных цветов выбираются из палитры 16 777 216:
- Адрес: `BASE + (y*640 + x) / 2`
- Объём: 640 × 256 / 2 = **81 920 байт** (~5 страниц по 16 КБ)
- Double buffering возможен (2 экрана помещаются в 512 КБ VRAM)

### Текстовый режим 80×32

Видеоконтроллер имеет **аппаратный текстовый режим** 80×32 символа (шрифт 8×8, разрешение 640×256). Программа записывает коды символов и атрибуты в VRAM, видеоконтроллер растеризует их аппаратно через знакогенератор. BIOS загружает шрифт при старте; вывод возможен как через DSS API (`#5B`, `#5C`), так и прямой записью в VRAM. См. раздел [5.8 Текстовый режим 80×32](08_text_mode.md).

## Ключевые порты видеоконтроллера

| Порт | Имя | Назначение |
|------|-----|-----------|
| `#89` | `PORT_Y` / `RGADR` | Адрес/данные палитры, Y-координата строки |
| `#C3` | `ALL_MODE` | Режим видео (ZX / 320 / 640) |
| `#C9` | `RGMOD` | Double buffer flip, screen on/off |
| `#C2`/`#CA` | `BORDER` | Бордюр (биты [2:0]) |
| `#FE` | ZX compat | Бордюр в ZX-режиме (биты [2:0]) |

---

## Установка видеорежима

Самый простой способ — через DSS API:

```asm
Dss.SetVMod     EQU #50

; Установить режим 320×256, страница 0
        ld      a, 1            ; 0=ZX, 1=320×256, 2=640×256
        ld      b, 0            ; видеостраница
        ld      c, Dss.SetVMod
        rst     #10
```

Прямой доступ через порты:

```asm
; Установить режим 320×256 напрямую
        ld      a, #01
        out     (#C3), a        ; ALL_MODE = 320×256

; Включить экран (бит 4 RGMOD = 0 → экран ON)
        xor     a
        out     (#C9), a        ; RGMOD = 0
```

## Бордюр

### В ZX-режиме — через порт #FE (нижние 3 бита)

```asm
        ld      a, 1            ; синий
        out     (#FE), a        ; ZX-бордюр
```

### В нативных режимах — через #C2/#CA (биты [2:0] BORDER)

В документации FPGA регистр `BORDER[7..0]` содержит биты [2:0] цвета бордюра.

## Double Buffering

Техника, позволяющая рисовать следующий кадр во **втором буфере**, пока первый отображается. Переключение мгновенное (на следующем VSync).

### Механизм

- Экран A — страницы VRAM `#50`–`#54`
- Экран B — страницы VRAM `#55`–`#59`
- Порт `#C9` (RGMOD) бит 0 определяет, какой экран отображается:
  - `RGMOD bit 0 = 0` → отображается A, рисуем в B
  - `RGMOD bit 0 = 1` → отображается B, рисуем в A

### Переключение

```asm
; Flip (переключить активный буфер)
        in      a, (#C9)        ; RGMOD
        xor     1
        out     (#C9), a
```

Пример из flappybird:
```asm
ChangeVideoPage:
        in      a, (RGMOD)
        xor     1
        out     (RGMOD), a
        ret
```

## Аппаратный акселератор (blitter)

Это специальный блок в ACEX, который **перехватывает инструкции `LD r,r`** с одинаковыми регистрами на M1-цикле и выполняет блочные операции с VRAM быстрее, чем CPU.

| Инструкция | Действие | Параметры |
|-----------|---------|----------|
| `LD B,B` | Выключить акселератор | — |
| `LD D,D` | Установить размер блока | `A` = размер |
| `LD C,C` | Заполнить блок | `A` = значение, `HL` = адрес VRAM |
| `LD E,E` | Вертикальное заполнение | `A` = значение, `HL` = адрес |
| `LD H,H` | Двухбайтная функция | — |
| `LD L,L` | Копировать строку | `HL` = src, `DE` = dst, `A` = размер |
| `LD A,A` | Копировать вертикально | аналогично |

Акселератор работает на частоте **42 МГц** и не зависит от turbo-режима Z80.

> Подробнее см. раздел [5.6 Акселератор](06_accelerator.md).

## Важно: PORT_Y и «мусор» на экране

**Порт `#89` (`PORT_Y`)** — это не обычный регистр, а индекс строки для доступа к VRAM. Аппаратура использует его значение как Y-координату, по которой идёт запись/чтение в видеопамять. Следствия:

1. Если программа записывает в VRAM (`LD (HL), A` при WIN* → VRAM-страница) и `PORT_Y` указывает на **видимую строку** (0–191/255), следующая инструкция, которую выполняет Z80 — особенно если адрес кода или данных попадает в ту же VRAM — может **исказить пиксели** этой строки. На экране это проявляется как «мусор» в случайных местах.
2. Даже обычные чтения/записи переменных могут «просвечивать» в VRAM, если забыть снять маппинг страницы.

### Правило №1: после работы с VRAM задвигать PORT_Y за пределы видимой области

SDK (см. `zx-sprinter-sdk/sdk/src/sprinter/lib_startup.asm`, `_clear_screen`, `_swap_screen`) всегда завершает работу с PORT_Y командой:

```asm
        ld      a, #C0                  ; 192 — за нижним краем
        out     (PORT_Y), a
```

Это **обязательный** приём: загнать Y-координату в область 192–255, где любой доступ к VRAM не отразится на видимом изображении. Без этого после каждого `RET`/прерывания на экране появляется мусор.

```asm
; Правильный паттерн работы с VRAM
        di
        ld      a, <row>
        out     (PORT_Y), a             ; установить строку
        ld      d, d                    ; акселератор
        ld      a, <size>
        ld      l, l                    ; копирование
        ei

        ; ОБЯЗАТЕЛЬНО: увести PORT_Y в невидимую зону
        ld      a, #C0
        out     (PORT_Y), a
```

### Правило №2: не держать код и стек в тех же страницах, что активная VRAM

Если программа подключает VRAM-страницу (`#50`…) в одно из окон, убедитесь что:
- Код программы (`WIN2`, `#8100`) не пересекается с этой же страницей
- Стек (`#BFFF`) не оказывается в VRAM-странице
- Обработчики прерываний находятся в не-VRAM области

Если Z80 начнёт выполнять инструкцию из страницы, на которую сейчас смотрит видеоконтроллер, или запишет туда значение со стека — на экране появится мусор.

### Правило №3: SDK-идиома для очистки экрана

Посмотрите на `_clear_screen` из SDK как на канонический пример:

```asm
_clear_screen:
        push    af
        ld      a, VPAGE_TILES : out (PAGE1), a

        ld      hl, #4160               ; начальный адрес в VRAM
        ld      a, (_screenActive)
        and     1
        jr      nz, .m1
        ld      hl, #4020
.m1:    ld      a, 32
        out     (PORT_Y), a             ; стартовая строка

        di
        ld      d, d : ld a, 0 : ld b, b ; настройка акселератора
        pop     af
        ld      b, 192
.lp1:   di
        ld      c, c                    ; заполнить строку
        ld      (hl), a
        ld      b, b
        ei
        exa : in a, (PORT_Y) : inc a : out (PORT_Y), a : exa
        djnz    .lp1

        ld      a, #C0 : out (PORT_Y), a    ; <-- увести PORT_Y в безопасную зону
        MRestoreMemMap12
        ret
```

Обратите внимание на `ld a, #C0 : out (PORT_Y), a` в конце — это и есть тот самый «задвигать повыше».

## Интервал VSync

Кадровые прерывания генерируются в конце обратного хода (аналогично ZX Spectrum) с частотой 50 Гц (PAL) или 60 Гц (NTSC). Программа может использовать прерывание для:
- Синхронизации отрисовки (double buffer flip)
- Обновления анимации
- Чтения ввода
- Воспроизведения звука (COVOX DMA)

Подробности — в разделе [9.1 Прерывания](../09_advanced/01_interrupts.md).

---

## Ключевые моменты

> - VRAM = 512 КБ, 4 параллельных банка по 8 бит.
> - Режимы: ZX (256×192), 320×256 8bpp, 640×256 4bpp.
> - VRAM страницы: `#50`–`#5F` через стандартный банкинг.
> - Порты: `#C3` (режим), `#89` (палитра), `#C9` (RGMOD, double buffer), `#C2/#CA` (бордюр).
> - Акселератор работает на 42 МГц через перехват `LD r,r`.
> - Установка режима через `Dss.SetVMod #50` проще, чем прямая запись в порты.
