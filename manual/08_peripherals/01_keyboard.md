## Что видит программа

В Sprinter физическая клавиатура может быть подключена по-разному, но для
программиста это один и тот же класс устройства: **AT-совместимая
клавиатура**, дающая поток make/break кодов PC-клавиатуры.

Внутри логики Sprinter этот поток:

1. принимается контроллером клавиатуры;
2. преобразуется в нативные коды DSS;
3. при необходимости дублируется в **ZX-матрицу** для Spectrum-совместимости.

То есть нативная программа обычно работает либо через **DSS keyboard API**,
либо через низкоуровневый поток скан-кодов. ZX-матрица нужна в основном для
совместимости со старым ПО.

> **Практическое правило:** для прикладной программы используйте `RST #10`
> (`WaitKey`, `ScanKey`, `TestKey`, `K_SETUP`). Прямое чтение скан-кодов нужно
> только драйверам, играм с собственным вводом или средствам отладки.

## Три уровня доступа

### 1. DSS API

Основной способ для нативного ПО:

- символьный код уже декодирован;
- возвращаются модификаторы и состояние клавиатурного режима;
- не нужно разбирать `F0`/`E0` вручную.

### 2. Низкоуровневый поток скан-кодов

Используется, когда программе нужен "сырой" поток make/break-кодов клавиатуры.
В существующих исходниках он читается через регистры контроллера:

| Порт | Имя | Назначение |
|------|-----|-----------|
| `#18` | `SIO_DATA_REG_A` | принятый байт |
| `#19` | `SIO_CONTROL_A` | статус приёмника |

Бит 0 порта `#19`: `1` означает, что байт готов к чтению.

### 3. ZX-матрица

Классический совместимый интерфейс через порт `#FE`, как у ZX Spectrum. Он
полезен для ZX-режима и совместимого кода, но не даёт полной PC-клавиатуры.

---

## ZX-матрица через порт `#FE`

При чтении порта `#FE` адресные линии `A8..A15` выбирают строку матрицы, а
младшие биты данных возвращают состояние пяти клавиш этой строки (`0 = нажата`).

### Строки матрицы

```
Адрес       Клавиши
#FEFE       Caps Shift, Z, X, C, V
#FDFE       A, S, D, F, G
#FBFE       Q, W, E, R, T
#F7FE       1, 2, 3, 4, 5
#EFFE       0, 9, 8, 7, 6
#DFFE       P, O, I, U, Y
#BFFE       Enter, L, K, J, H
#7FFE       Space, Symbol Shift, M, N, B
```

### Чтение одной строки

```asm
; Прочитать строку 1..5
        ld      bc, #F7FE
        in      a, (c)
        and     #1F
        cpl                         ; 1 = клавиша нажата
```

### Проверка пробела

```asm
CheckSpace:
        ld      bc, #7FFE
        in      a, (c)
        and     #01                 ; bit0 = Space
        ret                         ; Z=1 если нажата
```

> Преобразование PC-клавиатуры в ZX-матрицу делает совместимость возможной, но
> не заменяет нативный API: модификаторы, расширенные клавиши и раскладки
> полноценно доступны именно через DSS.

## Низкоуровневые скан-коды

Sprinter использует AT-совместимый поток скан-кодов. Для программы это обычные
PC make/break-коды:

- `#F0` означает отпускание клавиши;
- `#E0` означает расширенную клавишу;
- обычные make-коды приходят без префикса.

### Неблокирующее чтение байта

```asm
SIO_DATA_REG_A  EQU #18
SIO_CONTROL_A   EQU #19

ScanKbdByte:
        in      a, (SIO_CONTROL_A)
        bit     0, a
        ret     z                   ; байта нет

        in      a, (SIO_DATA_REG_A) ; A = очередной код
        or      1                   ; сбросить Z
        ret
```

### Часто встречающиеся make-коды

| Клавиша | Код |
|--------|-----|
| `A` | `#1C` |
| `B` | `#32` |
| `Enter` | `#5A` |
| `Space` | `#29` |
| `Esc` | `#76` |
| `Backspace` | `#66` |
| `Tab` | `#0D` |
| `Left Shift` | `#12` |
| `Ctrl` | `#14` |
| `F1` | `#05` |
| `F12` | `#07` |
| `Up` | `#E0 #75` |
| `Down` | `#E0 #72` |
| `Left` | `#E0 #6B` |
| `Right` | `#E0 #74` |

## DSS keyboard API

Клавиатурные функции DSS вызываются через `RST #10`.

### Общий контракт возврата

Для функций `#30`, `#31`, `#32`, `#37`:

- `A` — готовый символьный код;
- `D` — код физической клавиши;
- `B` — состояние модификаторов;
- `C` — режим клавиатуры.

Внутри драйвера тот же символьный байт также лежит в `E`, но для прикладного
кода ориентироваться следует на `A`.

### Поле `B` — модификаторы

| Бит | Значение |
|-----|----------|
| 7 | Left Shift |
| 6 | Right Shift |
| 5 | Ctrl |
| 4 | Alt |
| 3 | Left Ctrl |
| 2 | Left Alt |
| 1 | Right Ctrl |
| 0 | Right Alt |

### Поле `C` — режим клавиатуры

| Бит | Значение |
|-----|----------|
| 7 | альтернативная раскладка / RUS-LAT |
| 3 | Num Lock |
| 2 | Scroll Lock |
| 1 | Insert |
| 0 | Caps Lock |

## Функции `#30`-`#37`

### `#30` — `WaitKey`

Блокирующее ожидание символа.

```asm
        ld      c, #30
        rst     #10
        ; A = символ, D = keycode, B/C = состояние режима
```

### `#31` — `ScanKey`

Неблокирующая проверка буфера.

```asm
        ld      c, #31
        rst     #10
        jr      z, .no_key
        ; A = символ
.no_key:
```

### `#32` — `EchoKey`

Как `WaitKey`, но введённый символ сразу выводится на экран консоли.

```asm
        ld      c, #32
        rst     #10
```

### `#33` — `CtrlKey`

Возвращает текущее состояние модификаторов и режима клавиатуры без извлечения
символа из буфера.

```asm
        ld      c, #33
        rst     #10
        ; B = модификаторы, C = режим, A = 0 если буфер пуст
```

### `#35` — `K_CLEAR`

Очищает буфер клавиатуры и затем может сразу выполнить одну из функций
`#30..#34`, номер которой передаётся в `B`.

```asm
; Очистить буфер и сразу перейти к обычному ожиданию клавиши
        ld      b, #30
        ld      c, #35
        rst     #10
```

> По текущему исходнику драйвера допустимы подфункции `#30..#34`. Если нужен
> только flush без дополнительной логики, на практике удобно вызывать `#35`
> перед следующим `WaitKey`/`ScanKey`.

### `#36` — `K_SETUP`

Управление раскладкой и звуковыми сигналами клавиатуры.

Подфункции:

- `B = 0` — установить или прочитать таблицу раскладки;
- `B = 1` — прочитать флаги key-sound;
- `B = 2` — записать флаги key-sound.

#### Выбор раскладки

При `B = 0` регистр `A` задаёт таблицу:

| `A` | Таблица |
|-----|---------|
| `0` | normal |
| `1` | shift |
| `2` | caps |
| `3` | caps+shift |
| `4` | normal alternative |
| `5` | shift alternative |
| `6` | caps alternative |
| `7` | caps+shift alternative |

```asm
; Включить альтернативную "обычную" раскладку
        ld      a, 4
        ld      b, 0
        ld      c, #36
        rst     #10
```

В исходнике также поддержано чтение самой таблицы по `bit7=1` в `A`, но это уже
сервисная возможность для утилит настройки раскладки, а не для обычного
прикладного кода.

#### Флаги key-sound

При `B = 1` или `B = 2` в `A` используется битовая маска:

| Бит | Назначение |
|-----|------------|
| 0 | сигнал переполнения буфера |
| 1 | сигнал переключения раскладки |

```asm
; Включить оба сигнала
        ld      a, %00000011
        ld      b, 2
        ld      c, #36
        rst     #10
```

### `#37` — `TestKey`

Проверка текущего содержимого буфера без ожидания.

```asm
        ld      c, #37
        rst     #10
        jr      z, .empty
        ; A = символ, D = keycode
.empty:
```

## Пример: ввод строки

Ниже рабочий пример ввода строки с поддержкой `Enter`, `Esc` и `Backspace`.

```asm
; Вход:
;   HL = адрес буфера
; Выход:
;   CF=0 - строка готова, 0-terminated
;   CF=1 - отмена по Esc

ReadLine:
        ld      b, 0
.loop:
        ld      c, #30              ; WaitKey
        rst     #10

        cp      27                  ; Esc
        jr      z, .esc
        cp      13                  ; Enter
        jr      z, .done
        cp      8                   ; Backspace
        jr      z, .backspace
        cp      32
        jr      c, .loop            ; пропустить управляющие

        ld      (hl), a
        inc     hl
        inc     b

        push    bc
        push    hl
        ld      c, #5B              ; PutChar
        rst     #10
        pop     hl
        pop     bc
        jr      .loop

.backspace:
        ld      a, b
        or      a
        jr      z, .loop

        dec     hl
        dec     b

        push    bc
        ld      a, 8
        ld      c, #5B
        rst     #10
        ld      a, ' '
        ld      c, #5B
        rst     #10
        ld      a, 8
        ld      c, #5B
        rst     #10
        pop     bc
        jr      .loop

.done:
        ld      (hl), 0
        or      a
        ret

.esc:
        scf
        ret
```

## Практические замечания

1. Для большинства программ хватает пары `ScanKey` + `CtrlKey`.
2. Если вы работаете с собственным игровым циклом, удобнее опрашивать клавиатуру
   каждый кадр через `ScanKey`, а не висеть в `WaitKey`.
3. Если нужен точный физический keycode, берите `D`, а не `A`: `A` уже является
   результатом раскладки и преобразований.
4. ZX-порт `#FE` хорош для совместимости, но не подходит как основной интерфейс
   нативного редактора, оболочки или GUI.

---

## Ключевые моменты

> - Для программирования клавиатуры на Sprinter правильная абстракция —
>   **AT-совместимый поток + DSS API**, а не конкретный тип разъёма.
> - ZX-матрица через `#FE` остаётся для совместимости, но не заменяет нативный
>   интерфейс.
> - DSS-функции `#30`, `#31`, `#32`, `#33`, `#35`, `#36`, `#37` дают не только
>   символ, но и keycode, модификаторы и флаги режима.
> - Внутренняя логика Sprinter умеет транслировать PC-клавиатуру в ZX-матрицу,
>   но подробно документировать эту совместимость здесь не требуется.
