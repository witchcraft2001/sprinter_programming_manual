# 6.2 COVOX DAC и PCM-вывод

> **Навигация:** [← 6.1 AY-3-8910](01_ay_3_8910.md) | [Оглавление](../README.md) | [6.3 COVOX DMA →](03_covox_dma.md)

## Что такое COVOX

**COVOX** — простой 8-битный DAC (Digital-to-Analog Converter) для воспроизведения оцифрованного звука (PCM). В оригинальном Spectrum это был внешний адаптер на LPT-порту. В Sprinter COVOX **встроен в FPGA** (ACEX 1K30) и расширен до полноценного стереофонического PCM-блока.

Внутри FPGA этот блок называется **CBL** (COVOX Blaster) — ссылка на внешнее имя.

---

## Характеристики CBL

| Параметр | Значение |
|---------|---------|
| Разрядность | 8 бит (моно) или 16 бит (стерео) |
| Каналы | Моно / Стерео |
| Частоты | 7.8 кГц – 109.4 кГц (см. таблицу) |
| Буфер | 256 байт FIFO в FPGA |
| Прерывания | По опустошению половины буфера |
| Выход | I²S DAC на плате |

---

## Порты COVOX

| Порт | Имя | Назначение |
|------|-----|-----------|
| `#88` | `CBL_WR` | Запись аудио-сэмпла |
| `#89` | `CBL_MODE` / `CBL_XX` | Регистр режима COVOX |

**Внимание:** порт `#89` используется ещё и для установки Y-координаты палитры/строки в видео. Это не конфликт: FPGA различает их по значениям.

---

## Регистр #89 (режим COVOX)

| Бит | Имя | Назначение |
|-----|-----|-----------|
| 7 | `CBL_MODE` | 1 = COVOX включён |
| 6 | `CBL_STEREO` | 1 = стерео (16 бит L/R), 0 = моно (8 бит) |
| 5 | `CBL_MODE16` | 1 = 16-битные сэмплы |
| 4 | `CBL_INT_ENA` | 1 = разрешить прерывания от CBL |
| [3:0] | — | Код частоты дискретизации |

### Таблица частот

Из `SP2_ACEX.TDF` — CBL_TAB:

| Бит [3:0] | Частота | Примечание |
|----------|---------|-----------|
| `#0` | 16 кГц | |
| `#1` | 22 кГц | |
| `#2`–`#7` | зарезервировано | |
| `#8` | 7.8125 кГц | Низкое качество |
| `#9` | 10.9375 кГц | |
| `#A` | 15.625 кГц | |
| `#B` | 21.875 кГц | |
| `#C` | 31.25 кГц | CD-подобное |
| `#D` | 43.75 кГц | |
| `#E` | 54.6875 кГц | |
| `#F` | 109.375 кГц | Максимум |

### Пример конфигурации

```asm
; Моно, 8 бит, 22 кГц, без прерываний
        ld      a, #81          ; 10000001 = on + 22kHz
        out     (#89), a

; Моно, 8 бит, 31.25 кГц
        ld      a, #8C          ; 10001100
        out     (#89), a

; Стерео, 16 бит, 44 кГц, с прерываниями
        ld      a, #FD          ; 11111101 = on + stereo + 16bit + INT + 44kHz
        out     (#89), a

; Выключить COVOX
        xor     a
        out     (#89), a
```

## Прямой вывод сэмпла

Самый простой способ — писать байты в порт `#88` в нужном темпе:

```asm
; Моно 8 бит: каждый OUT (#88), A = один сэмпл
PlaySampleDirect:
        ; A = сэмпл (0–255)
        out     (#88), a
        ret
```

### Воспроизведение буфера через задержки (неэффективно)

```asm
PlaySamples:
        ; Включить COVOX
        ld      a, #8C          ; моно, 31.25 кГц
        out     (#89), a

        ld      hl, sample_data
        ld      bc, sample_len
.loop:
        ld      a, (hl)
        out     (#88), a
        inc     hl

        ; Задержка между сэмплами (подобрать под частоту)
        ld      e, 8
.wait:
        dec     e
        jr      nz, .wait

        dec     bc
        ld      a, b : or c
        jr      nz, .loop

        xor     a
        out     (#89), a        ; выключить
        ret

sample_data:    ds 1024
sample_len      equ $-sample_data
```

> **Проблема:** этот подход требует точного таймирования и не работает при прерываниях. Для серьёзного применения используйте **DMA режим** — см. [6.3 COVOX DMA](03_covox_dma.md).

## Прерывания от COVOX

Когда `CBL_INT_ENA = 1` (бит 4 порта `#89`), CBL генерирует прерывание Z80 при опустошении буфера:

```asm
; Включить COVOX с прерываниями
        ld      a, #9C          ; режим + INT + 31.25 кГц
        out     (#89), a

; В обработчике прерываний IM2:
; - определить, что это CBL (другие источники возможны)
; - записать новые сэмплы в #88
```

### Механизм работы

CBL имеет внутренний **FIFO буфер 256 байт**. Когда буфер заполнен наполовину, генерируется прерывание. Z80 должен успеть записать новую порцию сэмплов до полного опустошения.

На частоте 31.25 кГц:
- 256 сэмплов буфера = ~8 мс
- Прерывание каждые ~4 мс

Это даёт ЦП время на вычисления между блоками.

---

## Стерео режим

В стерео режиме порт `#88` принимает **пары сэмплов** L/R:

```asm
; Стерео 8 бит: порядок L, R, L, R, ...
        ld      a, #C0          ; стерео, моно 8-бит
        out     (#89), a

.loop:
        ld      a, (hl)         ; L
        out     (#88), a
        inc     hl
        ld      a, (hl)         ; R
        out     (#88), a
        inc     hl
        ; ...
```

### 16-битный стерео

16-битные сэмплы передаются байтами (LSB, MSB). Всего 4 байта на стерео сэмпл (L_LO, L_HI, R_LO, R_HI):

```asm
        ld      a, #E0          ; стерео + 16 бит
        out     (#89), a
```

## Пример: проигрывание WAV-файла (упрощённо)

```asm
; Открыть WAV-файл, пропустить заголовок 44 байта, играть моно 8-бит

PlayWAV:
        ld      hl, wav_name
        ld      a, 1
        ld      c, #11          ; Open
        rst     #10
        jr      c, .err
        ld      (handle), a

        ; Пропустить заголовок WAV (44 байта)
        ld      a, (handle)
        ld      b, 0            ; SEEK_SET
        ld      hl, 0
        ld      ix, 44
        ld      c, #15          ; Move_FP
        rst     #10

        ; Включить COVOX
        ld      a, #8C          ; моно 31.25 кГц
        out     (#89), a

.read_chunk:
        ld      a, (handle)
        ld      hl, buf
        ld      de, 1024
        ld      c, #13          ; Read
        rst     #10
        jr      c, .done

        ld      a, d
        or      e
        jr      z, .done

        ; Проиграть chunk
        ld      hl, buf
        ld      bc, 1024
.play:
        ld      a, (hl)
        out     (#88), a
        inc     hl

        ; Задержка ~32 мкс для 31.25 кГц (настройте под свой turbo)
        push    bc : ld b, 20
.wait:  djnz    .wait
        pop     bc

        dec     bc
        ld      a, b : or c
        jr      nz, .play

        jr      .read_chunk

.done:
        xor     a
        out     (#89), a        ; выключить COVOX

        ld      a, (handle)
        ld      c, #12
        rst     #10

.err:
        ld      c, #41
        rst     #10

wav_name:   db "TEST.WAV", 0
handle:     db 0
buf:        ds 1024
```

## Уровень громкости

COVOX — аппаратный 8-битный DAC. Громкость регулируется **только через значения сэмплов**: чем дальше от центра (128), тем громче. Программный регулятор громкости реализуется умножением сэмплов на коэффициент.

```asm
; Уменьшить громкость вдвое
; A = исходный сэмпл (0–255)
HalfVolume:
        sub     128             ; центрировать
        sra     a               ; /2 со знаком
        add     a, 128
        ret
```

---

## Ключевые моменты

> - COVOX (CBL) — 8-битный DAC в FPGA.
> - Порт `#88` — данные, порт `#89` — режим.
> - Частота: 7.8 кГц – 109 кГц (бит [3:0] регистра режима).
> - Стерео и 16-бит поддерживаются через биты [6:5] регистра режима.
> - FIFO 256 байт генерирует прерывания при опустошении половины.
> - Для синхронного вывода — задержки между `OUT`.
> - Для надёжного вывода — DMA режим с прерываниями (см. 6.3).
> - Громкость — программная (умножение сэмплов).
