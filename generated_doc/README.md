## Быстрый справочник по портам

| Порт | Регистр | Назначение |
|------|---------|-----------|
| `0xC0/0xC8` | SC | Системный ROM, запрет DOS, BLK_R |
| `0xC1/0xC9` | PN | Страницы RAM верхнего окна, 128K экран |
| `0xC3` | ALL_MODE | Ускоритель, WAIT, прерывания клавиатуры |
| `0xC5/0xCD` | RGMOD | Режим видео (ZX/прямой), отключение экрана |
| `0xCB` | HOLD | Начальные значения счётчиков синхронизации |
| `0x8F/0x5C` | ROM_RG | Страница ROM при записи, режим записи Flash |
| `0x3C/0x7C` | /SYS | Включение системного ROM, выбор ROM0/ROM1 |
| `IN (0xFB/0x7B)` | CASH_ON | Включение кэша/ISA |
| `0x1B` | ISA_A20 | Линия A20 для ISA |
| `0xF0–0xFF` | ISA_PORT | Выбор слота/окна ISA |
| `0x88` | COVOX data | Данные Covox DAC |
| `0x89` | CBL_XX | Режим/частота Covox |
| `0x90` | AY addr | Адрес регистра AY-3-8910 |
| `0x91` | AY data | Данные AY-3-8910 |
| `0xFE` | KBD | ZX-клавиатура/бордер |
| `0x60–0x6F` | IDE CS1 | ATA-порты данных/регистров |
| `0x3F6/0x3F7` | IDE CS3 | ATA альтернативный статус/управление |
| `0x1C–0x1E` | CMOS | Чтение/запись NVRAM |

## Обзор архитектуры

Sprinter K30 — это ZX-совместимый компьютер на базе Z80, реализованный в ПЛИС Altera ACEX. Главные компоненты:

```
Z80 CPU
  │
  ├── DCP (port-mapper/decoder) ── DCP.MIF (256×16 бит, LUT)
  │     ├── DRAM-контроллер (RAS/CAS, MA[])
  │     ├── Декодер портов (TYPE, PAGE)
  │     ├── Генератор CLK_Z80 (3.5 / 7 МГц)
  │     └── WAIT-логика
  │
  ├── ACCELER (ускоритель) ── 256×16 внутренний RAM
  │     ├── Перехват LDIR/LDDR/LDI/LDD и спец.кодов LD r,r
  │     ├── DMA для Covox
  │     └── Прямое чтение IDE (HDD_FLIP)
  │
  ├── VIDEO2 (видеоконтроллер)
  │     ├── ZX 256×192 с атрибутами (VAI19=0)
  │     ├── Sprinter 320/640 линейный (VAI19=1)
  │     └── 4×VRAM (VD0–VD3, 8 бит каждый)
  │
  ├── AY (AY-3-8910 эмулятор)
  │     └── I²S DAC (DAC_DATA/WS/BCK)
  │
  ├── KBD (клавиатура ZX + PS/2)
  └── MOUSE (PS/2 мышь)
```

Центральный элемент — таблица `DCP.MIF` (256 × 16 бит). По индексу, производному от адреса Z80 и флагов `/IO`, из неё берутся поля `TYPE[3:0]` и `PAGE[11:0]`, которые управляют всем: выбором источника данных, адресом DRAM, генерацией WAIT и сигналами периферии.

## Принципы разработки

1. Все файлы написаны на основе исходников `SP2_ACEX.TDF`, `DCP.TDF`, `ACCELER.TDF`, `VIDEO2.TDF`, `AY.TDF`, `MOUSE.TDF`, `KBD.TDF` и таблицы `DCP.MIF`.
2. При добавлении нового порта или устройства — сначала правьте `DCP.MIF`, затем верхний уровень `SP2_ACEX.TDF`.
3. Параметр `UPDATE=1` в `DCP.TDF` означает, что логика кэша/ISA/ROM вынесена в `SP2_ACEX.TDF`. При `UPDATE=0` эта логика жила внутри DCP (старые проекты).
4. Сборка: Altera Max+II / Quartus, скрипт `altera/acex/make.cmd`. Выход: `SP2_ACEX.sof`.
