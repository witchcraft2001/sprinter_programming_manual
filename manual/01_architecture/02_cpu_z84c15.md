## Z84C15 vs классический Z80

Sprinter использует **Zilog Z84C15** — CMOS-версию Z80, разработанную для встраиваемых систем.
Для программиста это почти идентичный Z80, но с рядом важных отличий.

### Полная совместимость

- Все инструкции Z80 поддерживаются без исключений
- Те же флаги, регистры, режимы адресации
- Прерывания IM0/IM1/IM2 работают идентично
- Коды опкодов совпадают полностью

### Дополнительные периферийные блоки Z84C15

Z84C15 содержит **встроенные контроллеры** прямо на кристалле:

| Блок | Назначение | Порты |
|------|-----------|-------|
| **SIO** (два канала) | Последовательный ввод-вывод (RS-232, PS/2) | `#18`/`#19` (канал A) |
| **CTC** (4 канала) | Счётчики/таймеры | `#10`–`#13` |
| **PIO** (два порта) | Параллельный ввод-вывод | (не используется напрямую) |
| **DART** | Дуплексный асинхронный приёмопередатчик | (опц.) |

В Sprinter активно используются:
- **SIO канал A** — приём PS/2 кодов клавиатуры
- **CTC** — системный таймер, генерация прерываний

## Система тактирования

Всё тактирование формируется в чипе **ACEX 1K30** из одного источника — кварца **42 МГц**.

```
TG42 (42 МГц)
     │
     ├─► CLK84  (84 МГц) — внутренние нужды FPGA
     │
     ├─► CLK42  (42 МГц) — главный клок FPGA
     │
     ├─► CLK21  (21 МГц) — Z80 turbo-режим
     │
     ├─► CLK_Z80 (7 МГц) — Z80 нормальный режим
     │         42 ÷ 6, паттерн: !CT2 & CT1
     │
     └─► CLK_K  (15 кГц)  — PS/2 clock sampling
```

Источник: `DCP.TDF` и `SP2_ACEX.TDF` из [`sp-altera-src-2025-11-30`](#ссылки).

### Нормальный режим: 7 МГц

По умолчанию Z80 работает на **7 МГц** — вдвое быстрее оригинального Spectrum (3.5 МГц).
Это достигается делением 42 МГц на 6 внутри DCP:

```vhdl
-- из DCP.TDF
CLK_Z80 = !CT2 & CT1  -- счётчик CTZ, период 6 тактов CLK42
```

Wait-состояния при доступе к памяти (без turbo):

| Тип памяти | Wait-тактов | Эффективная скорость |
|-----------|------------|---------------------|
| RAM (обычная) | 2 | ~4.7 МГц эффективно |
| ROM | 2 | ~4.7 МГц |
| VRAM (ZX-экран) | до 10 | замедление при видео-конфликтах |
| IDE порты | 10 (данные: 4) | — |

### Turbo-режим: 21 МГц

Включается аппаратно комбинацией **F12 + Ctrl + Shift** (или F12 + Alt + Shift — зависит от прошивки).
Переключение мгновенное, без сброса. Z80 переходит на CLK21.

Wait-состояния в turbo:

| Тип памяти | Wait-тактов |
|-----------|------------|
| RAM | 2 |
| ROM | 4 |
| Видео | 7 |

> **Важно:** Аппаратный ускоритель (blitter) работает на CLK42 и **не зависит от режима Z80**.
> Включение turbo ускоряет только CPU-инструкции, но не операции ускорителя.

### Проверка режима из программы

```asm
; Программно определить turbo-режим нельзя напрямую.
; Обходной путь — измерить длительность известного цикла через CTC:

        ld      a, 0x57         ; CTC: таймерный режим, делитель/256
        out     (CTC_CH2), a
        ld      a, 255          ; начальное значение
        out     (CTC_CH2), a
        ; ... выполнить фиксированное число инструкций ...
        in      a, (CTC_CH2)   ; прочитать остаток
        ; если остаток мал — turbo включён
```

## SIO: приём PS/2 клавиатуры

SIO канал A настроен для приёма PS/2 сканкодов от клавиатуры (через MAX7000).

Порты SIO канала A:

| Порт | Назначение |
|------|-----------|
| `#18` (`SIO_DATA_REG_A`) | Регистр данных (принятый байт) |
| `#19` (`SIO_CONTROL_A`) | Регистр управления/статуса |

**Бит 0 порта `#19`**: `1` = принят байт, готов к чтению.

Типичный паттерн опроса:

```asm
SIO_CONTROL_A   EQU     #19
SIO_DATA_REG_A  EQU     #18

; Проверить наличие байта от клавиатуры
KeysHandler:
        in      a, (SIO_CONTROL_A)
        bit     0, a                    ; бит 0: байт готов?
        ret     z                       ; нет — выход

        in      a, (SIO_DATA_REG_A)    ; читаем PS/2 скан-код
        cp      #F0
        jr      nz, .key_down
        ; #F0 = префикс "клавиша отпущена"
        ld      a, 1
        ld      (.skip_flag), a
        jr      KeysHandler             ; читаем следующий байт
.key_down:
        ; a = скан-код нажатой клавиши
        ; ...
```

Источник: [`flappybird/src/sys_utils.asm`](#ссылки), [`zx-sprinter-sdk/sdk/src/sprinter/lib_startup.asm`](#ссылки).

## CTC: системный таймер

CTC (Counter/Timer Circuit) имеет 4 независимых канала. В SDK используются каналы 2 и 3
для генерации прерываний (VSync handler).

Порты CTC:

| Порт | Канал |
|------|-------|
| `#10` (`CTC_CH0`) | Канал 0 |
| `#11` (`CTC_CH1`) | Канал 1 |
| `#12` (`CTC_CH2`) | Канал 2 |
| `#13` (`CTC_CH3`) | Канал 3 |

Инициализация из SDK (таймер 50 Гц при 7 МГц):

```asm
CTC_CH0         equ     #10
CTC_CH1         equ     #11
CTC_CH2         equ     #12
CTC_CH3         equ     #13

; Настройка CTC для генерации прерываний ~50 Гц
        ld      a, 0x57         ; Timer mode, prescaler=256, INT enable
        out     (CTC_CH2), a
        ld      a, 112          ; константа счёта
        out     (CTC_CH2), a

        ld      a, 0xD7         ; Timer mode, prescaler=256, INT enable
        out     (CTC_CH3), a
        ld      a, 160
        out     (CTC_CH3), a
```

Источник: [`zx-sprinter-sdk/sdk/src/sprinter/lib_startup.asm`](#ссылки).

## Управление прерываниями

### IM1 (стандартный режим Spectrum)

В IM1 при прерывании CPU выполняет `RST #38`. Sprinter генерирует прерывание в конце
обратного хода луча (аналогично Spectrum). Частота ~50 Гц (60 Гц в NTSC-варианте).

```asm
        im      1               ; режим IM1
        ei
        ; прерывания уходят на RST #38
```

### IM2 (векторное прерывание)

IM2 — правильный способ организации прерываний в Sprinter. Позволяет разделить
прерывания от VSYNC, клавиатуры (SIO) и CTC.

Типичная настройка (из flappybird):

```asm
; Установка IM2
; DE = адрес обработчика
set_im2:
        ld      hl, #80FF       ; адрес в таблице векторов: I==#80, вектор==#FF
        ld      (hl), e         ; lo-байт адреса
        inc     hl
        ld      (hl), d         ; hi-байт адреса

        ld      a, i            ; сохранить старый I
        ld      (save_i), a
        ld      a, #80          ; таблица с 0x8000
        di
        ld      i, a            ; I = #80
        im      2
        ei
        ret
```

SDK организует полную таблицу на `#FE00` (257 байт `#FD`), обработчик на `#FDFD`:

```asm
        org     #FE00
im2vector:
        ds      257, #FD        ; все векторы → #FDFD

        org     #FDFD
        jp      im2VShandler    ; диспетчер прерываний
```

Источники: [`flappybird/src/im2_utils.asm`](#ссылки), [`zx-sprinter-sdk/sdk/src/sprinter/lib_startup.asm`](#ссылки).

## Ключевые отличия от стандартного Z80

| Аспект | Классический Z80 | Z84C15 в Sprinter |
|--------|-----------------|-------------------|
| Тактовая | 3.5 МГц | 7 / 21 МГц |
| Wait-состояния | нет | 2–10 тактов (через FPGA) |
| Встроенный SIO | нет | SIO/0 (#18, #19) — клавиатура |
| Встроенный CTC | нет | 4 канала (#10–#13) |
| Встроенный PIO | нет | есть (не задействован) |
| BUSRQ/BUSACK | стандартно | используется ускорителем |

> **Практический вывод:** Для программиста Z84C15 — это Z80 + SIO + CTC + CTC.
> Знание этих периферийных блоков открывает правильную работу с клавиатурой
> и прерываниями без костылей.

---

## Ключевые моменты

> - Z84C15 — полностью совместим с Z80 на уровне инструкций.
> - Нормальная частота: **7 МГц**; turbo: **21 МГц** (F12+Ctrl+Shift).
> - Turbo не ускоряет аппаратный ускоритель — только CPU.
> - SIO (`#18`/`#19`) используется для чтения PS/2 клавиатуры.
> - CTC (`#10`–`#13`) используется для системных таймеров и VSync.
> - IM2 — правильный режим прерываний для Sprinter-программ.
