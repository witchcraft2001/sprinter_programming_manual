## Что такое Turbo

**Turbo-режим** в Sprinter — аппаратное увеличение тактовой частоты Z80 с 7 МГц до **21 МГц** (в 3 раза). Вся остальная аппаратура (видео, звук, ускоритель) продолжает работать на своих частотах.

### Активация

Turbo включается **аппаратно** комбинацией клавиш:
- **F12 + Ctrl + Shift** (или F12 + Alt + Shift — зависит от прошивки)

Переключение мгновенное, без сброса. Состояние turbo сохраняется до следующего выключения или повторного нажатия комбинации.

### Начальное состояние

Начальное состояние turbo при загрузке задаётся в CMOS регистре `#1B`, бит 0:
- `0` → 7 МГц при старте
- `1` → 21 МГц при старте

## Как определить режим из программы

Z80 **не имеет регистра**, показывающего текущий режим. Программа может измерить длительность цикла через CTC:

```asm
; Определить turbo через таймер CTC
DetectTurbo:
        ; Установить CTC канал 0 на максимальный счёт
        ld      a, #47              ; Timer mode, prescaler 256
        out     (#10), a
        ld      a, 255              ; максимальное значение
        out     (#10), a

        ; Выполнить фиксированное количество инструкций
        ld      b, 100
.loop:  nop
        nop
        nop
        nop
        djnz    .loop

        ; Прочитать оставшееся значение
        in      a, (#10)

        ; Turbo если остаток > threshold
        cp      200                 ; пороговое значение
        jr      c, .normal

        ; Turbo режим
        or      1
        ret

.normal:
        xor     a
        ret
```

> Точное пороговое значение зависит от констант и требует калибровки.

## Wait states

Даже в turbo не все обращения к памяти ускоряются — FPGA вставляет **wait states** для медленных типов памяти:

### Нормальный режим (7 МГц)

| Тип памяти | Wait states |
|-----------|-------------|
| RAM (DRAM) | 2 |
| ROM/Flash | 2 |
| VRAM | 7 |
| FDD / MAX7000 | 10 |
| IDE регистры | 10 |
| IDE данные | 4 |

### Turbo режим (21 МГц)

| Тип памяти | Wait states |
|-----------|-------------|
| RAM | 2 |
| ROM/Flash | **4** (удвоилось!) |
| VRAM | 7 |
| FDD / MAX7000 | 10 |
| IDE регистры | 10 |
| IDE данные | 4 |

### Эффективная скорость

В turbo-режиме:
- **Код из RAM** ускоряется в 3 раза.
- **Доступ к ROM** ускоряется в ~1.5 раза (из-за удвоенных waits).
- **VRAM** почти не ускоряется.
- **Акселератор** не зависит от turbo (работает на 42 МГц).

Практический прирост от turbo для игр: ~2× при правильной оптимизации.

## Оптимизация для turbo

### 1. Держать горячий код в RAM

```asm
; ПЛОХО: вызов BIOS в цикле
.loop:
        ld      c, #82
        rst     #08                 ; BIOS в ROM — медленно
        djnz    .loop

; ЛУЧШЕ: своя функция в RAM
.loop:
        call    MyPrintChar         ; всё в RAM — быстро
        djnz    .loop
```

### 2. Минимизировать обращения к VRAM

Собирайте кадр в RAM-буфере, затем копируйте в VRAM через акселератор:

```asm
; Рисуем в RAM
        call    DrawToRAM           ; работа с буфером в RAM (быстро)

        ; Копируем в VRAM через акселератор
        di
        ld      d, d
        ld      a, 255
        ld      hl, ram_buffer
        ld      de, #C000           ; VRAM через WIN3
        ld      l, l
        ei
```

### 3. Использовать регистровые операции

```asm
; ПЛОХО: операции с памятью
        ld      a, (counter)
        inc     a
        ld      (counter), a

; ЛУЧШЕ: регистровые операции, память в конце
        inc     b                   ; счётчик в регистре
        ; ... использовать B ...
        ld      (counter), b        ; один раз в конце
```

### 4. Избегать инструкций, долго работающих с памятью

```asm
; LDIR копирует 1 байт за ~21 такта (при 2 wait states)
; Это ~3 мкс/байт при 7 МГц, ~1.8 мкс/байт при 21 МГц (не 3× ускорение!)

; Для больших блоков используйте акселератор
```

## Тайминги инструкций

В turbo wait states относительно меньше, поэтому инструкции без обращений к памяти получают максимальный выигрыш:

| Инструкция | 7 МГц (циклов) | 7 МГц (мкс) | 21 МГц (мкс) |
|-----------|----------------|-------------|--------------|
| `NOP` | 4 | 0.57 | 0.19 |
| `LD r, r` | 4 | 0.57 | 0.19 |
| `LD r, (HL)` | 7 + 2 wait = 9 | 1.29 | 0.43 |
| `LD r, n` | 7 + 2 = 9 | 1.29 | 0.43 |
| `JP` | 10 | 1.43 | 0.48 |
| `CALL` | 17 | 2.43 | 0.81 |
| `LDIR` (байт) | 21 | 3.0 | 1.0 |

## Turbo и периферия

**Важное свойство:** FPGA-периферия работает на своих собственных частотах:

- **Видеоконтроллер** — 42 МГц (не зависит от turbo)
- **Ускоритель** — 42 МГц (не зависит)
- **CTC** — получает клок 7 МГц всегда
- **SIO** — фиксированная частота для PS/2 (~15 кГц)
- **COVOX** — настраивается отдельно

Это означает, что **прерывания 50 Гц VBlank** приходят с одной и той же частотой независимо от turbo. Но **между прерываниями CPU выполнит в 3 раза больше инструкций** в turbo.

## Проблемы совместимости

Многие старые программы, особенно игры ZX Spectrum, полагаются на точные тайминги (эффект "multicolour", цветовая полоска). В turbo они могут:
- Работать слишком быстро
- Терять синхронизацию
- Показывать глюки

### Решение: выключить turbo для старых программ

```asm
; Принудительное выключение turbo через CMOS
DisableTurboInCMOS:
        ld      bc, CMOS_AWR
        ld      a, #1B
        out     (c), a
        ld      bc, CMOS_DRD
        in      a, (c)
        and     #FE                 ; сбросить бит 0
        push    af
        ld      bc, CMOS_AWR
        ld      a, #1B
        out     (c), a
        pop     af
        ld      bc, CMOS_DWR
        out     (c), a
        ; Для применения — перезагрузка
        ret
```

Но это применяется только при следующем старте. Программное переключение turbo во время работы **не поддерживается стандартным API** — только аппаратная клавиатурная комбинация.

## Измерение времени

Для профилирования можно использовать CTC как счётчик:

```asm
; Измерить время выполнения MyFunc в тактах CTC
MeasureTime:
        ; Сбросить счётчик
        ld      a, #47              ; Timer, prescaler 256
        out     (#10), a
        ld      a, 255
        out     (#10), a

        call    MyFunc              ; измеряемая функция

        in      a, (#10)
        ld      (elapsed), a
        ret

elapsed:    db 0
```

При prescaler 256, на 7 МГц каждый тик CTC = 256 × 142 нс ≈ 36 мкс.

---

## Ключевые моменты

> - Turbo: 7 МГц → 21 МГц (3×), включается F12+Ctrl+Shift.
> - Начальное состояние в CMOS регистре `#1B` бит 0.
> - Не все операции ускоряются одинаково: RAM 3×, ROM 1.5×, VRAM почти нет.
> - Acceлератор не зависит от turbo (работает на 42 МГц).
> - Для максимальной пользы: горячий код в RAM, минимум обращений к VRAM, акселератор для блочных операций.
> - CTC и прерывания не зависят от turbo — VSync остаётся 50 Гц.
> - Программное переключение turbo невозможно (только CMOS при старте).
