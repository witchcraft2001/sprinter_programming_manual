## Роль BIOS в Sprinter

**BIOS** (Basic Input/Output System) — низкоуровневый слой прошивки Sprinter, хранящийся во Flash. Он предоставляет:

1. **Инициализацию аппаратуры** при включении: RAM, IDE, FDD, клавиатура, мышь, CMOS.
2. **Загрузчик** — читает системный диск и запускает DSS.
3. **SETUP** утилиту — конфигурация системы через CMOS.
4. **API** для программ: функции работы с дисками, памятью, видеовыводом текста, мышью.

BIOS Sprinter — потомок BIOS обычного IBM PC по духу, но с другой структурой вызовов. Вместо прерываний `INT` используются **RST-вектора** Z80.

---

## Точки входа BIOS

BIOS отвечает на три Z80 RST-вектора:

| Вектор | Адрес | Назначение |
|--------|-------|-----------|
| `RST #08` | `0x0008` | Основной BIOS API |
| `RST #18` | `0x0018` | Альтернативный (внутренний) BIOS вызов |
| `RST #30` | `0x0030` | Mouse API (работа с мышью) |

Номер функции передаётся в регистре **C** (или **A**, в зависимости от функции).

### Вызов BIOS — общая схема

```asm
; Стандартный вызов BIOS
        ld      c, <function_number>    ; номер функции в C
        ; ... установить входные параметры в A, B, D, E, H, L, IX, IY ...
        rst     #08
        ; после возврата: CF = 0 успех / CF = 1 ошибка
        ; выходные значения в A/B/C/D/E/H/L
```

### Mouse API через RST #30

```asm
; Чтение состояния мыши
        ld      c, #03                  ; READ MOUSE STATE
        rst     #30
        ; A = кнопки (bit0=левая, bit1=правая)
        ; HL = X координата
        ; DE = Y координата
```

## Флаг результата

BIOS возвращает **CF** (Carry Flag) Z80 как признак ошибки:

- `CF = 0` → операция успешна, выходные данные в регистрах.
- `CF = 1` → ошибка, в `A` обычно код ошибки.

```asm
        ld      c, Bios.Drv_Read
        ; ... параметры ...
        rst     #08
        jr      c, .error               ; CF=1 → ошибка
        ; успех
        jr      .done
.error:
        ; обработка ошибки, A = код
```

## Группы функций BIOS

### 1. Работа с памятью (EMM — Extended Memory Manager) — `#C0`–`#C7`

Управление выделением и освобождением RAM-страниц.

| Код | Имя | Назначение |
|-----|-----|-----------|
| `#C0` | `Emm_Fn0` | Получить информацию о RAM (всего/свободно) |
| `#C1` | `Emm_Fn1` | Инициализация RAM (сброс всех выделений) |
| `#C2` | `Emm_Fn2` | Выделить блок памяти (B страниц → A=ID) |
| `#C3` | `Emm_Fn3` | Освободить блок (A=ID) |
| `#C4` | `Emm_Fn4` | Получить физический индекс страницы блока |
| `#C5` | `Emm_Fn5` | Получить список физических страниц блока |
| `#C6` | `Emm_Fn6` | Получить порт окна и текущую страницу |
| `#C7` | `Emm_Fn7` | Получить индекс следующей страницы блока |

### 1b. RAM-диски и устройства — `#C8`–`#CF`

| Код | Имя | Назначение |
|-----|-----|-----------|
| `#C8` | `Blk_Rd_Wr` | Чтение/запись сектора RAM-диска |
| `#C9` | `Blk_To_Ramd` | Назначить блок памяти как RAM-диск |
| `#CA` | `Ramd_Clear` | Очистить RAM-диск |
| `#CB` | `Ramd_To_Drv` | Назначить RAM-диск на букву привода |
| `#CC` | `Fdd_To_Drv` | Назначить FDD на букву привода |
| `#CD` | `Hdd_To_Drv` | Назначить HDD на букву привода |
| `#CE` | `Get_Ramd_St` | Статус RAM-диска |
| `#CF` | `Get_Drv_St` | Статус привода (тип устройства) |

### 2. Вывод текста на экран — `#81`–`#8E`

Стандартные функции печати символов и строк в видеопамяти.

| Код | Имя | Назначение |
|-----|-----|-----------|
| `#81` | `Lp_Print_All` | Печать с атрибутами |
| `#82` | `Lp_Print_Sym` | Печать символа |
| `#83` | `Lp_Print_Atr` | Установка атрибутов |
| `#84` | `Lp_Set_Place` | Установить позицию курсора (D=row, E=col) |
| `#85`–`#88` | `Lp_Print_Ln*` | Печать строк (разные варианты) |
| `#89` | `Lp_Cls_Win` | Очистить окно |
| `#8A` | `Lp_Scroll_Up` | Прокрутить окно |
| `#8B`–`#8C` | `Lp_Print_Ln5/6` | Печать с терминатором |
| `#8D` | `Lp_Cls_Win2` | Очистить окно с цветом |
| `#8E` | `Lp_Get_Place` | Получить позицию курсора |

### 3. Графика — `#A0`–`#A6`

| Код | Имя | Назначение |
|-----|-----|-----------|
| `#A1` | `Pic_Point` | Установить пиксель (HL=X, DE=Y, B=цвет) |
| `#A4` | `Pic_Set_Pal` | Установить цвет палитры (HL=индекс, E=R, D=G, A=B) |
| `#A6` | `Set_Pal_Init` | Установить палитру по умолчанию |

### 4. Окна — `#B0`–`#B8`

| Код | Имя | Назначение |
|-----|-----|-----------|
| `#B0` | `Win_Open` | Открыть окно |
| `#B1` | `Win_Close` | Закрыть окно |
| `#B2` | `Win_Copy_Win` | Скопировать окно |
| `#B3` | `Win_Restore_Win` | Восстановить окно |
| `#B4` | `Win_Get_Sym` | Получить символ |
| `#B5` | `Win_Put_Sym` | Установить символ |
| `#B6` | `Win_Set_ZG` | Загрузить пользовательский знакогенератор (шрифт) в VRAM |
| `#B7` | `Win_Move_Win` | Переместить окно |
| `#B8` | `Win_Get_ZG` | Получить текущий знакогенератор из VRAM |

### 5. Диски (IDE/FDD) — `#51`–`#5F`

| Код | Имя | Назначение |
|-----|-----|-----------|
| `#51` | `Drv_Reset` | Сброс диска |
| `#54` | `Drv_Verify` | Проверка сектора |
| `#55` | `Drv_Read` | Чтение сектора |
| `#56` | `Drv_Write` | Запись сектора |
| `#57` | `Drv_Detect` | Обнаружение дисков |
| `#58` | `Drv_Get_Par` | Получить параметры диска |
| `#59` | `Drv_Set_Par` | Установить параметры диска |
| `#5A` | `Ext_Version` | Версия BIOS |
| `#5F` | `Drv_List` | Список доступных дисков |

### 6. Служебные — `#ED`–`#F1`

| Код | Имя | Назначение |
|-----|-----|-----------|
| `#ED` | `Fn_Cript` | Получить Board ID (уникальный идентификатор) |
| `#EF` | `Fn_Version` | Получить версию BIOS (строка) |
| `#F0`/`#F1` | `Sprinter_1/2` | Проверка модели Sprinter |

## Пример: получение версии BIOS

```asm
BiosRst         EQU     #08
Bios.Ext_Version EQU    #5A
Bios.Fn_Version EQU     #EF

; Получить версию BIOS как строку в буфер
        ld      hl, version_buf
        ld      c, Bios.Fn_Version
        rst     BiosRst
        ; HL указывает на начало строки "v0.2a ..."
        ; DE = версия в hex
        ; A = major версии

version_buf:    ds  64
```

---

## Пример: информация о памяти

```asm
Bios.Emm_Fn0    EQU     #C0

; Узнать сколько RAM страниц всего и сколько свободно
GetMemInfo:
        ld      c, Bios.Emm_Fn0
        rst     #08
        ; HL = всего RAM страниц
        ; BC = свободных страниц
        ld      (total_pages), hl
        ld      (free_pages), bc
        ret

total_pages:    dw  0
free_pages:     dw  0
```

## Пример: работа с мышью

```asm
; Инициализация и чтение позиции мыши
InitMouse:
        ld      c, #00                  ; INITIALIZATION
        rst     #30
        ret

ReadMouse:
        ld      c, #03                  ; READ MOUSE STATE
        rst     #30
        ; A  = кнопки (bit0=L, bit1=R)
        ; HL = X координата
        ; DE = Y координата
        ld      (mouse_buttons), a
        ld      (mouse_x), hl
        ld      (mouse_y), de
        ret

mouse_buttons:  db  0
mouse_x:        dw  0
mouse_y:        dw  0
```

## BIOS и DSS — различия

BIOS (`RST #08`) — низкий уровень, работает напрямую с железом.
DSS (`RST #10`) — высокий уровень, файловая система и управление процессами.

| Задача | Вызвать BIOS (RST #08) | Вызвать DSS (RST #10) |
|-------|----------------------|-----------------------|
| Чтение сектора по LBA | `Drv_Read #55` | — (нет прямого) |
| Открыть файл по имени | — | `OPEN #11` |
| Выделить RAM | `Emm_Fn2 #C2` | `GETMEM #3D` (обёртка) |
| Печать символа на экране | `Lp_Print_Sym #82` | `PUTCHAR #5B` |
| Установить видеорежим | — | `SETVMOD #50` |
| Выйти из программы | — | `EXIT #41` |

Для большинства задач используется **DSS API** — он проще и работает через файловую систему. BIOS API нужен, когда:
- Требуется прямой доступ к диску (форматирование, boot-sector).
- Нужно управление памятью страниц (EMM_FN).
- Программа работает без файловой системы (загрузчик).

---

## Ключевые моменты

> - BIOS API: `RST #08`, номер функции в `C`.
> - Mouse API: `RST #30`, номер функции в `C`.
> - `CF=0` успех, `CF=1` ошибка (код в `A`).
> - Группы: EMM (память) `#C0–#C7`, печать `#81–#8E`, графика `#A0–#A6`, окна `#B0–#B8`, диски `#51–#5F`, служебные `#ED–#F1`.
> - Для высокоуровневых операций используйте DSS (`RST #10`), не BIOS.
