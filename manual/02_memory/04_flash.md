# 2.4 Flash / ROM — чтение и запись

## Физические характеристики Flash

| Параметр | Значение |
|---------|---------|
| Тип | AMD/SST Flash (JEDEC-совместимый) |
| Объём | 512 КБ |
| Организация ROM | 16 страниц × 32 КБ |
| Организация через банкинг Z80 | Страницы `#80`–`#FF` по 16 КБ |
| Номер стр. ROM0 (начало BIOS) | `#80` |
| Порт управления | `#8F` (`ROM_RG`) |

Flash разделена на области:
```
Flash offset    ROM-страница    Содержимое
──────────────────────────────────────────
0x00000         Стр. #80        BIOS начало (0x0000 при WIN0→ROM)
0x04000         Стр. #81        BIOS продолжение
...             ...             ...
0x38000         Стр. #8E–#8F   FPGA битстрим / SETUP
```

## Регистр ROM_RG (порт #8F)

Порт `#8F` управляет:
1. **Выбором страницы ROM** при обычном доступе через банкинг.
2. **Включением режима записи во Flash** (бит 4).

| Бит | Назначение |
|-----|-----------|
| [3:0] | Адрес страницы ROM (0–15, для 32 КБ страниц) |
| 4 | ROM_WRITE_MODE: 1 = режим записи во Flash |
| [7:5] | Зарезервировано |

```asm
ROM_RG          EQU     #8F

; Выбрать страницу ROM 2 для чтения
        ld      a, 2
        out     (ROM_RG), a

; Включить режим записи во Flash
        ld      a, #10          ; бит 4 = ROM_WRITE_MODE
        out     (ROM_RG), a

; Выключить режим записи
        xor     a
        out     (ROM_RG), a
```

## Доступ к ROM через банкинг

ROM доступна как страницы `#80`–`#FF` в регистрах окон `EmmWin.P0`–`P3`.

```asm
EmmWin.P0   EQU     #82

; Подключить первую страницу ROM (BIOS начало) в WIN0
        ld      a, #80          ; ROM страница 0 (#80)
        out     (EmmWin.P0), a
; Теперь 0x0000–0x3FFF = ROM (первые 16 КБ BIOS)

; Читать данные из ROM как из обычной памяти
        ld      hl, #0100       ; адрес в ROM
        ld      a, (hl)         ; чтение
```

## Запись во Flash (программирование)

Flash — это **JEDEC Flash** (AMD/SST 29F040 или аналог). Протокол записи требует **командной последовательности** перед записью и операции стирания сектора.

### Структура команд AMD Flash (JEDEC)

Flash имеет собственный контроллер команд. Все команды подаются через стандартные записи в определённые адреса при включённом `ROM_WRITE_MODE`.

#### Включение режима записи

```asm
; Шаг 1: Включить ROM_WRITE_MODE через порт #8F
        ld      a, #10          ; бит 4 = включить запись
        out     (ROM_RG), a
```

#### Разблокировка Flash (Unlock Sequence)

Перед любой командой (стирание, запись) нужна unlock-последовательность:

```asm
; JEDEC Unlock: записать #AA в адрес #555, #55 в адрес #2AA
; Адреса указаны в пространстве Flash (не Z80)
; При ROM в WIN0 (0x0000–0x3FFF): Flash #555 = Z80 #1555 (сдвиг 1 бит для 16-бит addr)

; Стандартная последовательность:
        ld      a, #AA
        ld      (#5555), a      ; ADR #555 в Flash → Z80 зависит от маппинга
        ld      a, #55
        ld      (#2AAA), a      ; ADR #2AA в Flash
```

> **Замечание:** Точные Z80-адреса для Flash-команд зависят от того, в какое окно подключена ROM и по какому смещению. В Sprinter этим занимается BIOS — прямое программирование Flash из пользовательского кода не рекомендуется. Используйте BIOS API для обновления прошивки.

#### Стирание сектора (Sector Erase)

```asm
; Последовательность стирания сектора (AMD Erase):
; 1. Unlock (AA→555, 55→2AA)
; 2. Erase setup: 80→555
; 3. Unlock again (AA→555, 55→2AA)
; 4. Sector erase: 30→<sector_addr>

        ; Unlock
        ld      a, #AA : ld      (#5555), a
        ld      a, #55 : ld      (#2AAA), a
        ; Erase setup
        ld      a, #80 : ld      (#5555), a
        ; Second unlock
        ld      a, #AA : ld      (#5555), a
        ld      a, #55 : ld      (#2AAA), a
        ; Sector erase
        ld      a, #30
        ld      (sector_addr), a
        ; Ждать завершения (~1 сек для 64 КБ сектора)
```

#### Запись байта (Byte Program)

```asm
; AMD Byte Program:
; 1. Unlock (AA→555, 55→2AA)
; 2. Program setup: A0→555
; 3. Записать байт по нужному адресу

        ld      a, #AA : ld      (#5555), a
        ld      a, #55 : ld      (#2AAA), a
        ld      a, #A0 : ld      (#5555), a
        ld      a, (byte_to_write)
        ld      (target_addr), a
        ; Ждать завершения (~50 мкс)
```

#### Проверка завершения программирования

Флаг `DQ7` (бит 7) возвращает инверсию записанного бита 7 во время программирования:

```asm
WaitFlash:
        ld      a, (target_addr)
        xor     (byte_to_write)  ; если бит 7 совпал — готово
        bit     7, a
        jr      nz, WaitFlash   ; ждать
```

## Страницы ROM в системе

Диапазон страниц `#80`–`#FF` покрывает всю Flash. Конкретные BIOS-страницы:

| Страница (EmmWin) | Содержимое |
|------------------|-----------|
| `#80` | BIOS страница 0 (вектора, начало) |
| `#81` | BIOS страница 1 |
| `#82`–`#87` | BIOS основной код |
| `#88`–`#8B` | SETUP утилита |
| `#8C`–`#8F` | FPGA битстрим для автозагрузки |
| `#90`–`#FF` | Свободные (или расширенный BIOS) |

> Точное распределение зависит от версии прошивки. Не записывайте в страницы BIOS без необходимости — повреждение BIOS требует внешнего программатора для восстановления.

## Пример: чтение строки из ROM

```asm
; Прочитать 256 байт из ROM страницы #81 по смещению #0200
; в буфер по адресу ROM_BUF

ReadRomPage:
        in      a, (EmmWin.P1)
        push    af                  ; сохранить WIN1

        ld      a, #81
        out     (EmmWin.P1), a      ; WIN1 = ROM страница #81

        ; ROM страница #81 теперь в 0x4000–0x7FFF
        ; смещение #0200 внутри страницы → Z80 адрес 0x4200
        ld      hl, #4200
        ld      de, ROM_BUF
        ld      bc, 256
        ldir

        pop     af
        out     (EmmWin.P1), a      ; восстановить WIN1
        ret

ROM_BUF:    ds  256
```

## Защита Flash от случайной записи

FPGA обеспечивает аппаратную защиту:
- `ROM_WRITE_MODE` = 0 по умолчанию (после сброса).
- Запись в ROM-страницы при `ROM_WRITE_MODE=0` игнорируется аппаратурой.
- Только установка бита 4 порта `#8F` открывает физический доступ к Flash для записи.

```asm
; После окончания программирования ВСЕГДА выключить запись
        xor     a
        out     (ROM_RG), a     ; бит 4 = 0, запись запрещена
```

---

## Ключевые моменты

> - Flash (512 КБ) доступна как страницы `#80`–`#FF` через стандартный банкинг.
> - Порт `#8F` (`ROM_RG`) управляет выбором ROM-страницы и режимом записи.
> - Бит 4 порта `#8F` = 1 → включён режим физической записи во Flash.
> - Для записи/стирания Flash используется JEDEC-протокол (AMD Unlock → Command → Write/Erase).
> - Стирание сектора занимает ~1 секунду, запись байта ~50 мкс.
> - Не записывайте в страницы BIOS из пользовательского кода — используйте BIOS API.
