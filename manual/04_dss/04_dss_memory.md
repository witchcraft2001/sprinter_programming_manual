# 4.4 Управление памятью в DSS

## Модель памяти DSS

DSS управляет физическими страницами RAM как **пул блоков**. Программа запрашивает нужное количество страниц, получает **ID блока**, и затем может подключать отдельные страницы блока в свои окна через `SetWin`.

### Концепция

```
┌─────────────────────────────────────┐
│  Физический пул RAM (например 32 стр.) │
│                                       │
│  [#00][#01][#02][#03][#04][#05]...   │
│   SYS  SYS  PRG  PRG  FRE  FRE       │
│                                       │
│  SYS = занято системой               │
│  PRG = выделено программе (блок ID=2)│
│  FRE = свободно                      │
└─────────────────────────────────────┘
         │
         ▼ программа видит
┌─────────────────────────────────────┐
│  Логический блок ID=2 (2 страницы)  │
│  Индексы: 0, 1                       │
│                                       │
│  index 0 → физическая #02            │
│  index 1 → физическая #03            │
└─────────────────────────────────────┘
```

---

## Автоматическое выделение при EXEC

Когда DSS загружает программу через `EXEC #40`:

1. Вызывает `GetMem #3D` с `B=3` — выделяет **3 дополнительные страницы** для программы.
2. Подключает их в WIN1, WIN2, WIN3 (WIN0 остаётся для системных данных).
3. Копирует код в WIN2 (по адресу `#8100`).

После запуска программа уже имеет:
- WIN0 — системные данные, стек
- WIN1 — дополнительная страница (буфер)
- WIN2 — код программы
- WIN3 — дополнительная страница (свободная)

Программа может запросить **дополнительные** блоки через `GetMem`.

---

## Карта памяти при запуске DSS-программы

С точки зрения прикладной программы начальное состояние памяти после `EXEC`
выглядит так:

| Окно | Адреса | Типичное содержимое после старта |
|------|--------|----------------------------------|
| `WIN0` | `#0000..#3FFF` | системная страница, служебные структуры DSS и BIOS |
| `WIN1` | `#4000..#7FFF` | дополнительная страница программы / рабочий буфер |
| `WIN2` | `#8000..#BFFF` | код программы, аргументы, рабочий стек |
| `WIN3` | `#C000..#FFFF` | дополнительная страница программы / свободное окно |

Практически важные области внутри адресного пространства процесса:

| Адрес | Назначение |
|-------|------------|
| `#8000..#80FF` | служебная область процесса |
| `#8080..#817F` | командная строка |
| `#8100...` | обычная точка загрузки кода программы |
| `#BFFF` | типичное начальное значение `SP` |

При вызовах `RST #10` и `RST #08` стек должен оставаться в диапазоне
`#8000..#BFFF`.

## Информация о памяти

### `INFOMEM #3C`

```asm
Dss.InfoMem     EQU #3C

        ld      c, Dss.InfoMem
        rst     #10
        ; HL = всего RAM страниц
        ; BC = свободных страниц
```

---

## Выделение блока

### `GETMEM #3D`

**Вход:** `B` = количество страниц.
**Выход:** `A` = ID блока (CF=0) или ошибка (CF=1).

```asm
Dss.GetMem      EQU #3D

; Запросить 8 страниц (128 КБ) под буфер
AllocBuffer:
        ld      b, 8
        ld      c, Dss.GetMem
        rst     #10
        jr      c, .out_of_memory
        ld      (buffer_block), a
        ret

.out_of_memory:
        ; Ошибка: недостаточно памяти
        ret

buffer_block:   db 0
```

---

## Подключение страниц блока в окно

### `SETWIN #38` — общий вариант

**Вход:**
- `A` = ID блока
- `B` = логический индекс страницы внутри блока (0 = первая)
- `H` = старший байт адреса окна × 64:
  - `#00` → WIN0
  - `#40` → WIN1
  - `#80` → WIN2
  - `#C0` → WIN3

```asm
Dss.SetWin      EQU #38

; Подключить 2-ю страницу блока в WIN3
        ld      a, (buffer_block)
        ld      b, 1                ; индекс 1 = вторая страница
        ld      h, #C0              ; WIN3
        ld      c, Dss.SetWin
        rst     #10
```

### `SETWIN1/2/3 #39/#3A/#3B` — сокращения

```asm
Dss.SetWin1     EQU #39
Dss.SetWin2     EQU #3A
Dss.SetWin3     EQU #3B

; Подключить страницу блока в WIN3 (короткая форма)
        ld      a, (buffer_block)
        ld      b, 0                ; первая страница блока
        ld      c, Dss.SetWin3
        rst     #10
```

## Освобождение блока

### `FREEMEM #3E`

**Вход:** `A` = ID блока.

```asm
Dss.FreeMem     EQU #3E

        ld      a, (buffer_block)
        ld      c, Dss.FreeMem
        rst     #10
```

> **Важно:** DSS автоматически освобождает блоки программы при выходе через `EXIT #41`. Явное освобождение нужно только для долгоживущих программ, которые выделяют/освобождают память многократно.

---

## Пример: работа с большим буфером (128 КБ)

```asm
; Выделить 128 КБ и заполнить их значением 0xAA

        org     #8100

main:
        ; Выделить 8 страниц = 128 КБ
        ld      b, 8
        ld      c, Dss.GetMem
        rst     #10
        jr      c, .error
        ld      (block_id), a

        ; Для каждой страницы: подключить в WIN3 и заполнить
        ld      b, 0                ; индекс страницы
.fill_loop:
        push    bc

        ld      a, (block_id)
        ; B уже = индекс страницы
        ld      c, Dss.SetWin3
        rst     #10

        ; Заполнить WIN3 значением #AA
        ld      hl, #C000
        ld      de, #C001
        ld      bc, #3FFF
        ld      (hl), #AA
        ldir

        pop     bc
        inc     b
        ld      a, b
        cp      8
        jr      c, .fill_loop       ; пока не все 8 страниц

        ; Освободить блок (не обязательно, EXIT освободит)
        ld      a, (block_id)
        ld      c, Dss.FreeMem
        rst     #10

.error:
        ld      c, Dss.Exit
        rst     #10

block_id:   db 0

Dss.GetMem      EQU #3D
Dss.SetWin3     EQU #3B
Dss.FreeMem     EQU #3E
Dss.Exit        EQU #41
```

## Переход между окнами блока (EMM_FN7 через BIOS)

DSS не имеет прямой функции "получить следующую страницу блока", но можно использовать BIOS API:

```asm
Bios.Emm_Fn7    EQU #C7

; Получить физический номер следующей страницы блока
; A = текущая физическая страница → возвращает следующую
NextPageOfBlock:
        ld      c, Bios.Emm_Fn7
        rst     #08
        ; A = следующая физическая страница
        ; A = #FF если последняя
        ret
```

---

## Получение списка всех страниц блока (EMM_FN5)

```asm
Bios.Emm_Fn5    EQU #C5

; Получить полный список физических страниц блока
        ld      a, (block_id)
        ld      hl, page_list
        ld      c, Bios.Emm_Fn5
        rst     #08
        ; B = количество страниц
        ; page_list заполнен физическими номерами, в конце #FF

page_list:  ds 16
```

## Прямой доступ к страницам блока

Зная физический номер, можно подключить страницу через стандартные порты `#82`/`#A2`/`#C2`/`#E2`:

```asm
; Быстрое подключение первой страницы блока без DSS API
        ld      a, (page_list)      ; физический номер первой страницы
        out     (#E2), a            ; WIN3
```

Это быстрее, чем `SetWin`, но требует предварительного получения списка страниц через `EMM_FN5`.

---

## Ограничения

1. **Минимальная единица** — 16 КБ (одна страница).
2. **Максимальное количество страниц на блок** — ограничено свободной RAM.
3. **Системные страницы** (`#00`–`#0F`, `#50`–`#5F`) никогда не выделяются программе.
4. **Фрагментация** — блок может состоять из **несоседних** физических страниц. Нельзя предполагать линейность.
5. **Пересечение окон**: если в двух окнах подключена одна и та же страница, изменения видны в обоих.

---

## Стратегии работы с памятью

### Малые программы (< 48 КБ)

Используйте только **стандартные 3 страницы**, выделенные при EXEC. Не нужно вызывать GetMem.

### Средние программы (до ~512 КБ)

Выделите 1 большой блок под данные, подключайте его страницы в WIN3 по мере необходимости.

```asm
; Выделить блок под "большой массив" и работать со страницами последовательно
        ld      b, 16               ; 256 КБ
        ld      c, Dss.GetMem
        rst     #10

        ; Итерация по всем страницам блока
        ld      b, 0
.each_page:
        ld      a, (block_id)
        ld      c, Dss.SetWin3
        rst     #10
        ; ... работа с WIN3 ...
        inc     b
        ld      a, b
        cp      16
        jr      c, .each_page
```

### Большие программы (> 512 КБ)

Используйте несколько блоков, каждый под отдельную задачу (код оверлеев, данные уровня, графические ассеты).

---

## Ключевые моменты

> - DSS даёт программе 3 "бесплатных" страницы (WIN1/2/3) при загрузке.
> - Дополнительная память — `GetMem #3D` (B = страниц → A = ID блока).
> - Подключение страниц блока — `SetWin3 #3B` (A = ID, B = индекс).
> - Блок — это набор **несоседних** физических страниц; линейный доступ через последовательные индексы.
> - `EXIT` автоматически освобождает все блоки программы.
> - Для быстрого доступа используйте `EMM_FN5` (BIOS) для получения списка страниц, затем прямой `OUT` на порт окна.
