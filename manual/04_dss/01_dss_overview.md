## Что такое DSS

**DSS** (Disk SubSystem) — резидентная операционная система Sprinter, логический наследник CP/M и MS-DOS. Версия на момент текущих исходников — **1.60**. Основные особенности:

- **Резидентная** — постоянно находится в защищённых страницах RAM после загрузки.
- **Однозадачная** — одна программа в каждый момент времени, но поддерживает вложенные вызовы EXEC.
- **FAT16** — работает с FAT16-разделами на IDE HDD.
- **Поддержка каталогов** — полный набор функций `MkDir`, `RmDir`, `ChDir`, `F_First`/`F_Next`.
- **API через `RST #10`** — 60+ функций, покрывающих файлы, память, клавиатуру, видео, запуск программ.

---

## Как DSS появляется после BIOS

С точки зрения архитектуры DSS начинается там, где BIOS уже выбрал устройство
загрузки и начал читать первый сектор.

Дальше работает уже загрузочная цепочка DSS:

1. загрузчик получает управление после чтения стартового сектора;
2. подгружает системные компоненты DSS с носителя;
3. разворачивает резидентный код в защищённые страницы RAM;
4. устанавливает вход `RST #10`;
5. подготавливает системные структуры, память и текущий контекст процесса;
6. передаёт управление оболочке или `Exec`-программе.

Для прикладного кода важно именно это состояние:

- DSS уже резидентен;
- `RST #10` доступен;
- системные страницы заняты и не возвращаются через менеджер памяти;
- BIOS остаётся нижним слоем для устройств и низкоуровневых сервисов.

## Архитектура DSS

```
┌─────────────────────────────────────────────────────┐
│                  Пользовательская программа         │
│                  (ORG 0x8100, EXE формат)           │
└─────────────────────┬───────────────────────────────┘
                      │  RST #10
                      ▼
┌─────────────────────────────────────────────────────┐
│                      DSS резидент                   │
│    (FAT16 драйвер, управление процессами,          │
│     диспетчер вызовов, файловые дескрипторы)        │
└─────────────────────┬───────────────────────────────┘
                      │  RST #08 (BIOS API)
                      ▼
┌─────────────────────────────────────────────────────┐
│                        BIOS                         │
│   (драйверы IDE, FDD, клавиатура, видео, мышь)      │
└─────────────────────────────────────────────────────┘
```

## Размещение DSS в памяти

После загрузки DSS занимает следующие страницы RAM:

| Страница | Назначение |
|---------|-----------|
| `#FE` | Системная страница DSS (резидент, переменные, буферы) |
| `#FC` | Дополнительная системная страница |
| `#40`–`#4F` | Зона системных страниц DSS (при необходимости) |

Эти страницы **недоступны** для пользовательской программы — их номера никогда не возвращаются в ответ на `GetMem`.

### Системные данные на странице #FE

| Адрес | Имя | Назначение |
|-------|-----|-----------|
| `#C020` | `SYSTEM_ID` | Идентификатор системы |
| `#C0FE` | `SYS_SP` | Дополнительный системный стек |
| `#C100` | `DISK_TYPE` | Маршрутизация дисков |
| `#C104`–`#C107` | `COPY_PAGE0–3` | Копии текущих значений окон |
| `#C108` | `RAMD_VARS` | Переменные RAM-дисков |
| `#C11A` | `SP_SAVE` | Сохранённый SP программы |
| `#C124`–`#C127` | INT | Расширенный обработчик прерываний |
| `#C1C0`, `#C1C8` | IDE0, IDE1 | Дескрипторы IDE дисков |
| `#C1E0`, `#C1E8` | FDD0, FDD1 | Дескрипторы FDD |

## Файловая система

DSS работает с **FAT16** разделами:

| Параметр | Значение |
|---------|---------|
| Типы томов | HDD (IDE), FDD (3.5"), RAM-диск |
| Имена файлов | 8.3 (8 символов + 3 расширение) |
| Атрибуты | Normal, RdOnly, Hidden, System, Label, Dir, Arch |
| Максимальный размер файла | 2 ГБ (FAT16 ограничение) |
| Иерархия каталогов | Поддерживается |

### Константы атрибутов

```asm
FileAttrib.Normal   EQU #00     ; обычный файл
FileAttrib.RDOnly   EQU #01     ; только для чтения
FileAttrib.Hidden   EQU #02     ; скрытый
FileAttrib.System   EQU #04     ; системный
FileAttrib.Label    EQU #08     ; метка тома
FileAttrib.Direc    EQU #10     ; каталог
FileAttrib.Arch     EQU #20     ; архивный
```

### Режимы открытия файла

```asm
FileMode.RW     EQU #00     ; чтение + запись
FileMode.Read   EQU #01     ; только чтение
FileMode.Write  EQU #02     ; только запись
```

## Формат EXE-файла

Программы DSS — это файлы с расширением `.EXE` и собственным заголовком. Первые 512 байт файла — структура заголовка, далее — код.

### Структура заголовка

```c
struct SprintEXE {
    char  id[3];        // "EXE"
    uint8_t exe_ver;    // 1 = актуальная версия, 0 = deprecated
    uint16_t hdr_size;  // #0200 (512 байт)
    uint16_t offset;    // смещение кода в файле (обычно 512)
    uint16_t loader;    // адрес загрузчика (если есть) или 0
    uint16_t load;      // адрес загрузки (обычно #8100)
    uint16_t start;     // точка входа (обычно совпадает с load)
    uint16_t stack;     // начальный SP (обычно #BFFF)
    // ... padding до 512 байт ...
};
```

### Шаблон EXE в sjasmplus

```asm
        org     #8100-512       ; = 0x7F00 (начало заголовка)

        db      "EXE", 1        ; сигнатура + актуальная версия EXE
        dw      #0200           ; размер заголовка (512)
        dw      0               ; смещение кода (будет заменено на 512 при сборке)
        dw      0               ; адрес загрузчика (нет)
        dw      #8100           ; адрес загрузки
        dw      begin           ; точка входа
        dw      #BFFF           ; начальный SP
        ds      490             ; padding до 512 байт

begin:  org     #8100
        ; ... код программы ...
```

> **Примечание:** реальные значения полей смещения и размера заполняются сборщиком или ручной калькуляцией. Шаблон выше показывает структуру — конкретные значения `offset` зависят от компоновки.

> **По версии EXE:** используйте `exe_ver = 1` как актуальный вариант. Значение `0` считается deprecated. Для `exe_ver = 1` при запуске наследуется current dir вызывающей программы; операции с файлами не меняют его автоматически. Исключение: после `CHDISK` текущий каталог нового диска начинается с корня.

## Запуск программы DSS

Когда DSS исполняет `Dss.Exec` (функция `#40`):

1. Читает файл, проверяет сигнатуру `"EXE"`.
2. Выделяет 3 дополнительные страницы памяти под переменные программы.
3. Сохраняет текущий контекст (страницы, SP).
4. Копирует командную строку в буфер по адресу `IX-0x80` (256 байт аргументов).
5. Копирует код программы из файла в память по адресу `exe.load`.
6. Устанавливает `SP = exe.stack`.
7. Передаёт управление на `exe.start` (обычно `#8100`).

### Аргументы командной строки

Аргументы командной строки доступны программе по адресу **`#8080`** (256 байт):
- Байт 0: длина аргументов
- Байты 1–N: текст аргументов, разделённый пробелами
- Завершается `0`

```asm
; Прочитать командную строку
        ld      hl, #8080
        ld      a, (hl)             ; длина аргументов
        inc     hl
        ; HL указывает на текст, A = длина
```

---

## Возврат из программы

Программа **не должна** завершаться через `RET`. Для корректного выхода:

```asm
        ld      c, Dss.Exit         ; #41
        rst     #10
        ; управление вернётся в родительскую программу (оболочку)
```

## Ограничения при вызове DSS/BIOS

### Стек

При вызове `RST #10` (DSS) и `RST #08` (BIOS) стек **обязан** находиться в диапазоне `#8000`–`#BFFF`. За пределами этого диапазона вызовы могут привести к краху, потому что DSS переключает страницы окон и стек должен быть в «безопасном» окне (WIN2).

При работе собственного кода (без обращений к DSS/BIOS) стек может быть где угодно.

### Альтернативные регистры (EXX)

Регистры `HL'`, `DE'`, `BC'` **зарезервированы DSS и BIOS** — они используются внутри обработчиков для экономии на `PUSH`/`POP`. **Нельзя** использовать `EXX` для сохранения регистров перед вызовом `RST #10` / `RST #08`:

```asm
; НЕПРАВИЛЬНО — альтернативные регистры будут затёрты DSS:
        exx
        ld      c, #13              ; DSS Read
        rst     #10
        exx                         ; HL'/DE'/BC' уже не те!

; ПРАВИЛЬНО — сохранять через стек:
        push    hl
        push    de
        ld      c, #13
        rst     #10
        pop     de
        pop     hl
```

### RST #18 — дисковый API

Помимо `RST #08` (BIOS) и `RST #10` (DSS), существует **`RST #18`** — вход для дисковых драйверов (DISK INT). Используется для прямого доступа к устройствам (OPEN, CLOSE, MEDIA_CHECK, GET_BPB, INPUT, OUTPUT). Вызывается при подключённом системном ПЗУ. Для большинства задач используйте `RST #10` (DSS) — `RST #18` нужен только для написания собственных драйверов.

## Группы функций DSS API

| Группа | Функции | Описание |
|--------|---------|----------|
| Система | `#00`–`#09` | Версия, дисковая инфо |
| Файлы | `#0A`–`#18` | Create, Open, Read, Write, Delete |
| Каталоги | `#19`–`#1E` | F_First, F_Next, MkDir, ChDir, CurDir |
| Время | `#21`–`#22` | SysTime, SetTime |
| Клавиатура | `#30`–`#37` | WaitKey, ScanKey, TestKey |
| Память | `#38`–`#3F` | SetWin, GetMem, FreeMem |
| Процессы | `#40`–`#47` | Exec, Exit, GSwitch, DosName, Environ |
| Видео | `#50`–`#5C` | SetVMod, Locate, PutChar, Scroll |
| Печать | `#5F` | Print (высокоуровневая) |

## Пример: минимальная программа DSS

```asm
        org     #8100-512       ; EXE header

        db      "EXE", 0
        dw      #0200
        dw      #0200           ; offset (512)
        dw      0               ; loader
        dw      #8100           ; load address
        dw      main            ; entry point
        dw      #BFFF           ; stack
        ds      490

        org     #8100
main:
        ; Вывести строку через DSS PCHARS
        ld      hl, msg
        ld      c, #5C          ; DSS PCHARS
        rst     #10

        ; Ждать любую клавишу
        ld      c, #30          ; WAITKEY
        rst     #10

        ; Выход
        ld      c, #41          ; EXIT
        rst     #10

msg:    db      "Hello, DSS!", 13, 10, 0

DssRst          EQU #10
Dss.PChars      EQU #5C
Dss.WaitKey     EQU #30
Dss.Exit        EQU #41
```

---

## Ключевые моменты

> - DSS — резидентная ОС с API через `RST #10`.
> - Программы в формате `.EXE`: заголовок 512 байт + код с адреса `#8100`.
> - DSS занимает системные страницы `#FE`, `#FC`, `#40`–`#4F`.
> - FAT16 файловая система, 8.3 имена, иерархия каталогов.
> - Выход из программы — только через `Dss.Exit #41`.
> - Командная строка доступна по адресу `#8080`.
> - **Стек при вызове DSS/BIOS** обязан быть в `#8000`–`#BFFF`.
> - **EXX запрещён** перед `RST #10`/`RST #08` — альтернативные регистры зарезервированы DSS.
