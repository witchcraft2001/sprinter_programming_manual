# BIOS API (RST #08)

BIOS обслуживает низкоуровневые операции: EMS/оконные переключатели, вывод текста, доступ к дискам, мышь и системные функции (RESET, загрузка новой конфигурации). Вызовы выполняются через `RST #08h`, номер функции кладётся в `C`, параметры — в `A/B/DE/HL/IX`. Модуль `bios_equ.asm` разбивает ID на группы [texteditor/bios_equ.asm:1-40](../texteditor/bios_equ.asm#L1). Для удобства можно подключить файл `programming_guide/include/bios_equ.inc`, где эти константы уже определены (`INCLUDE "bios_equ.inc"`).

- `C0h..C9h` — **EMS функции** (`Bios.Emm_Fn0..9`): получить описатели страниц, подключать окна, копировать списки mаpping’а. Их используют редактор и игра Flappy Bird сразу после `Dss.GetMem`, чтобы «подписать» новые страницы и подключить их в окна [texteditor/routines.asm:20-70](../texteditor/routines.asm#L20) [flappybird/src/fbird.asm:30-70](../flappybird/src/fbird.asm#L30).
- `B0h..B8h` — **BIOS Windows**: открыть/закрыть окно, копировать/восстанавливать содержимое, управлять знакогенератором, двигать окно. Редактор `texteditor` использует `Bios.Win_Copy_Win`/`Win_Restore_Win` для сохранения экранной области без участия DSS [texteditor/routines.asm:120-160](../texteditor/routines.asm#L120).
- `81h..8Eh` — **Line Printer / текстовый вывод**: посимвольный вывод, строки, CLS, прокрутка, позиционирование курсора. Эти вызовы полезны, если вы работаете в BIOS‑оконном режиме или хотите быстро вывести диагностический текст до старта DSS.
- `51h..59h` — **Диски**: сброс контроллера, чтение/запись, проверка наличия, получение параметров. Вместо прямого обращения к VG93/IDE портам (см. раздел «Порты») используйте эти вызовы — BIOS обеспечит правильные тайминги и маршрутизирует операции к RAM/ROM.

## Другие BIOS сервисы

- **RST #30h (мышь)** — часть BIOS и обрабатывается модулем `INTMOUSE`. Команды `00h-0Fh` — базовые операции курсора (инициализация, показать/убрать, получить координаты, ограничить границы, загрузить изображение), `80h` — аппаратное прерывание мыши, `81h` — переключение видеорежима, `83h` — принудительное обновление [sprinter_dss/INTMOUSE.ASM:1-80](../sprinter_dss/INTMOUSE.ASM#L1). `gfxview/gfx_view.asm` демонстрирует цикл `RST #30` → проверка кнопок → `RST #10` (`Dss.ScanKey`) для клавиатурных событий [gfxview/gfx_view.asm:1-120](../gfxview/gfx_view.asm#L1).
- **Системный RESET / конфигурации** — BIOS позволяет загрузить новый битстрим в ПЛИС: записать конфигурацию в кэш‑ОЗУ, установить флаг `ACEX_30K_LOADING`, а затем инициировать `RESET` записью в страницу `#A0`. После перезапуска ПЗУ-прошивка подхватывает новый битстрим [sprinter_docs/intro/architecture.md:50-75](../sprinter_docs/intro/architecture.md#L50) [sprinter_docs/reset.md:1-15](../sprinter_docs/reset.md#L1).
- **Flash/ROM управление** — порты `ROM_RG` (`#8F/#5C`) и сигнал `XACS` управляются BIOS. Полное описание алгоритмов прошивки (стирание, запись, защита от повторного RESET) находится в [generated_doc/flash.md:1-60](../generated_doc/flash.md#L1). Пользовательские программы должны обращаться к BIOS процедурам, чтобы не нарушить работу системных страниц.
- **CMOS/RTC** — BIOS оркестрирует работу с портами `#FFBD/#BFBD/#DFBD`, поэтому прямой доступ допускать только для обновления времени. В остальном используйте функции `Dss.SysTime/SetTime` [sprinter_docs/periferal/cmos.md:1-8](../sprinter_docs/periferal/cmos.md#L1).

## Примеры

Перед примерами важно помнить, что `Bios.Emm_Fn5` работает в два шага. В `HL` передаётся адрес структуры из трёх байтов (`Pg0`, `Pg1`, `Pg2`) — это снимок будущих содержимых окон `#8000-#FFFF`. BIOS лишь сохраняет таблицу в собственные регистры и подготавливает системные сигналы; само подключение производится уже обычными `OUT (EmmWin.P2)`/`OUT (EmmWin.P3)`. Поэтому фраза «загрузить таблицу страниц в окна #8000/#C000» означает «считать массив страниц и позволить BIOS использовать его при последующих переключениях».

1. **Переключение окон** — Flappy Bird мапит набор страниц с ресурсами на лету:

```asm
        LD  HL,MemoryBuffer
        LD  C,Bios.Emm_Fn5 ; загрузить таблицу страниц в окна #8000/#C000
        RST 08h
        LD  A,(EditorPages.Pg0)
        OUT (EmmWin.P2),A
```

(адаптировано по [flappybird/src/fbird.asm:35-70](../flappybird/src/fbird.asm#L35)).

2. **Работа с окнами в текстовом режиме** — `texteditor` вызывает `Bios.Win_Copy_Win` с координатами `DE=#0000`, `HL=#2050` и указателем на буфер в `IX`, чтобы сохранить всю область экрана. После выхода возвращает окно `Bios.Win_Restore_Win` и позицию курсора через `Dss.Locate` [texteditor/routines.asm:120-190](../texteditor/routines.asm#L120).

3. **ISA и ROM порты** — `espprobe/isa.asm` показывает, как BIOS ожидает, что пользователи будут работать с ISA: переключить `PAGE3`, записать код `0x11` в системный порт, выдать `0xD4` в `PAGE3` (доступ к ISA-окну), а затем использовать `Port.ISA` для чтения/записи [espprobe/isa.asm:20-80](../espprobe/isa.asm#L20). Это гарантирует, что BIOS сможет вернуть конфигурацию при выходе.

## Рекомендации

- Всегда **сохраняйте текущие страницы** (`PAGE0..PAGE3`, `RGADR`, `RGMOD`) перед вызовом BIOS — особенно это критично при работе с графикой и ISA.
- **Сочетайте DSS и BIOS**: DSS удобно работать с файлами и окнами, а BIOS — с железом и прямыми окнами. `RST #10` → `RST #08` — стандартный тандем (см. CallDss в текстовом редакторе).
- **Используйте BIOS для FDD/IDE**, а не прямые порты `VG93/IDE`, если не пишете собственную ОС: BIOS уже решает вопросы ожидания, DMA и переключения страниц [sprinter_docs/periferal/fdd.md:1-15](../sprinter_docs/periferal/fdd.md#L1).
