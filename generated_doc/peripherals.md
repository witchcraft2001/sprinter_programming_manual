# Устройства ввода/вывода Sprinter (ACEX K30)

## Клавиатура (KBD.TDF)

### Аппаратная реализация

Модуль `KBD.TDF` реализует ZX-совместимый сканер 8×5 матрицы. Физический интерфейс — последовательная шина через шлейф (`XA0`=KBD_CC, `XA1`=KBD_DD), синхронизируется счётчиком `CTH5`.

Матрица клавиш хранится в `KBD_INI2.MIF` (256×8 бит). Сканирование происходит автоматически — результат доступен через порт `0xFE`.

### Порт 0xFE — ZX-клавиатура/бордер

| Операция | Назначение |
|---------|-----------|
| `IN A,(0xFE)` | Читать колонку матрицы, выбранную битами A8–A15 |
| `OUT (0xFE),A` | Bits [2:0] → бордер, bit 3 → TAPE_OUT, bit 4 → не используется |

При чтении: разряды D[4:0] содержат 5 битов выбранной колонки (0 = нажата), D5 = TAPE_IN, D6 = CBL_MODE status при активном Covox DMA.

Декодирование в DCP.MIF: индекс `0x40` = `0x4000` (TYPE=4, TYPE[2:1]=10, TYPE0=0 → KEY/KEMS с ZX-клавиатурой).

```vhdl
KEY/KEMS[] = (KEYS.kbo5 & !CBL_MODE) or (CBL_IND & CBL_MODE),
              TAPE_IN,
              CBL_MODE & CTV8,   -- бит статуса DMA Covox
              KEYS.kbo[4:0]
```

### Строки клавиатурной матрицы (порт 0xFE, A8–A15)

| A15:A8 (0=выбран) | Клавиши |
|------------------|---------|
| `FEh` (A8=0) | Shift, Z, X, C, V |
| `FDh` (A9=0) | A, S, D, F, G |
| `FBh` (A10=0) | Q, W, E, R, T |
| `F7h` (A11=0) | 1, 2, 3, 4, 5 |
| `EFh` (A12=0) | 0, 9, 8, 7, 6 |
| `DFh` (A13=0) | P, O, I, U, Y |
| `BFh` (A14=0) | Enter, L, K, J, H |
| `7Fh` (A15=0) | Space, Sym, M, N, B |

### Специальные клавиши

| Клавиша | Сигнал | Действие |
|---------|--------|---------|
| F12 | `KEYS.kb_F12` | Триггер переключения режима / NMI |
| Ctrl | `KEYS.kb_ctrl` | Тестовый режим (F12+Ctrl) |
| Alt | `KEYS.kb_alt` | NMI (F12+Alt) при `NMI_ON="ON"` |
| Shift | `KEYS.kb_sh` | VGA-переключение (F12+Shift) |
| Del | `KEYS.kb_reset` | Аппаратный сброс |

Комбинации:
- `F12+Alt` → `NMI_X` активен → `/NMI` процессору
- `F12+Shift` → `VGA_SWITCH` переключает вывод (TFF)
- `F12+Ctrl+Shift` → тестовый генератор в SPECTRUM режиме

### Прерывания клавиатуры

```vhdl
KEYS.int_ena = ALL_MODE0 & ALL_MODE3
```

- `ALL_MODE0=0` → прерывания от клавиатуры всегда отключены
- `ALL_MODE0=1, ALL_MODE3=0` → прерывания отключены при активном ускорителе
- `ALL_MODE0=1, ALL_MODE3=1` → прерывания разрешены всегда

Прерывание: `INT_X = !DFF(GND, (SVIDEO.INTT & KEYS.int), , ...)` — объединяет видеопрерывание с клавиатурным.

### BORDER — бордер и TAPE_OUT

```vhdl
BORDER[].clk = /IOWR
BORDER[].ena = DFF((DCPP == 1100X010), CLK42)  -- порт 0xC2/0xCA
BORDER[].d   = D[]

TAPE_OUT = BORDER[3]
SVIDEO.DIR_PORT[7:5] = BORDER[2:0]  -- биты цвета бордера
```

Бордер: `D[2:0]` → биты BGR видеоконтроллера (`DIR_PORT[7:5]`). `D[3]=TAPE_OUT` — магнитофонный выход.

## Мышь PS/2 (MOUSE.TDF)

### Аппаратная реализация

`MOUSE.TDF` — PS/2 протокол-декодер. Физическая линия `MOUSE_D` приходит через `XA3` шлейфа (при `SXA=0`). Тактирование внутреннее, синхронизировано с `CTH5`.

### Декодер пакетов

```vhdl
STATE[1:0] машина:
  0 → idle (ожидание START-бита)
  1 → X-пакет (8 бит)
  2 → Y-пакет (8 бит) → INT при завершении

SUM_X[9:0] = SUM_X + (DDX7, RG[])  -- знаковое суммирование X
SUM_Y[9:0] = SUM_Y + (DDY7, RG[])  -- знаковое суммирование Y
```

Знаковое расширение: `DDX7/DDY7` — знаковый бит из пакета мыши (отрицательное перемещение).

### Порт мыши (0x58 / 0x7ADF / 0xFADF)

Декодируется из `DCP.MIF` индекс `0x58` = `0x5000` (TYPE=5 → KEY/KEMS с TYPE0=1).

Выбор данных по битам A10 и A8:

```vhdl
CASE (A10, A8) IS
    WHEN 0,2 => KEMPS[] = (0b111111, !MS.OUT_K0, !MS.OUT_K1)  -- кнопки
    WHEN 1   => KEMPS[] = MS.OUT_X[7:0]                         -- X (знаковый)
    WHEN 3   => KEMPS[] = !MS.OUT_Y[7:0]                        -- Y (инвертированный)
END CASE;
```

| A10:A8 | Данные |
|--------|-------|
| `00`, `10` | Биты [1:0] = кнопки (0=нажата): [0]=правая, [1]=левая; [7:2]=1 |
| `01` | X-координата, 8 бит знаковый |
| `11` | Y-координата, инвертированная (возрастает вниз) |

Адреса чтения: `IN A,(0xFADF)` = кнопки (A10=1, A8=1 при адресе `0xFADF`), `IN A,(0xFBDF)` — нестандартно. Обычно используют стандартный Kempston-адрес `0xFADF` или работают через прямое чтение координат.

### Видеокурсор

Координаты мыши автоматически передаются в видеоконтроллер:
```vhdl
SVIDEO.MOUSE_X[] = MS.OUT_X[]
SVIDEO.MOUSE_Y[] = MS.OUT_Y[]
```

Видеоконтроллер накладывает курсор из `MOUSE.MIF` на кадр без участия CPU.

## Аудиоподсистема

### AY-3-8910 (AY.TDF)

Полная программная эмуляция AY-3-8910. Регистровый файл реализован через LPM_RAM_DQ (`AY.MIF`, 256×8).

**Порты:**

| Порт | Адрес DCP | Назначение |
|------|----------|-----------|
| `0x90` | индекс `0x90` | Запись адреса регистра AY (`AY_A_WR`) |
| `0x91` | индекс `0x91` | Запись данных в регистр AY (`AY_D_WR`) |
| `0xFFFD` | индекс `0x52`/`0x8C` | Чтение текущего регистра AY |
| `0xBFFD` | индекс `0x8D`/`0x8E` | Запись в AY (стандартные ZX 128K адреса) |

```vhdl
AY3.AY_D_WR = DFF((DECODE./IOM or /WR) or !DFF((DCPP == 0x91), CLK42), CLK42)
AY3.AY_A_WR = DFF((DECODE./IOM or /WR) or !DFF((DCPP == 0x90), CLK42), CLK42)
```

**Регистры AY-3-8910:**

| Регистр | Назначение | Биты |
|---------|-----------|------|
| 0–1 | Период канала A | 12 бит (R1[3:0]:R0) |
| 2–3 | Период канала B | 12 бит |
| 4–5 | Период канала C | 12 бит |
| 6 | Период шума | [4:0] = 5 бит |
| 7 | Разрешение: | [0]=тон A, [1]=тон B, [2]=тон C, [3]=шум A, [4]=шум B, [5]=шум C |
| 8 | Амплитуда A | [3:0] = уровень, [4] = огибающая |
| 9 | Амплитуда B | аналогично |
| 10 | Амплитуда C | аналогично |
| 11–12 | Период огибающей | 16 бит |
| 13 | Форма огибающей | [3]=Continue, [2]=Attack, [1]=Alternate, [0]=Hold |
| 14–15 | I/O порты | (обычно не используются) |
| 16 (ext) | Панорамирование | расширение Sprinter |
| 17 (ext) | AY_CH_DIR | направления каналов L/R |

**Стерео вывод:**

```vhdl
CASE DFF(CTH0, CLK42) IS
    WHEN 0 => AY_FULL[] = (AY3.AY_CH_L[], 0)   -- левый канал
    WHEN 1 => AY_FULL[] = (AY3.AY_CH_R[], 0)   -- правый канал
END CASE;
AY_CHS[].d = ((0, AY_FULL[]) + (1, CBL_R[15:5])), CBL_R[4:1]
```

AY и Covox суммируются в `AY_CHS[15:0]` — 16-битный сигнал на DAC.

**Бипер:**
```vhdl
AY3.BEEPER = BORDER[4]
```

Бит 4 регистра `BORDER` (порт `0xFE`) управляет динамиком-бипером через AY-модуль.

### I²S DAC

```vhdl
DAC_DATA = DFFE(DFF(AUDIO_R15, CT2), !CT2)  -- MSB первым
DAC_WS   = DFF(CTH1, CLK42)               -- Word Select (L/R)
DAC_BCK  = DFF(CT2, CLK42)                -- Bit Clock
```

- `DAC_BCK` ≈ 10.5 МГц (CLK42/4)
- `DAC_WS` — переключение L/R канала (≈ 43 кГц)
- Частота дискретизации: ≈ 43 кГц

### Covox DMA (CBL)

Аппаратный ЦАП с DMA-буфером (256×16 LPM_RAM_DP).

**Порты:**

| Порт | DCP индекс | Назначение |
|------|-----------|-----------|
| `0x88` | `0x88` | Запись сэмпла Covox (немедленно) |
| `0x89` | `0x89` | Запись регистра режима `CBL_XX` |

**Регистр CBL_XX (порт 0x89):**

| Биты | Название | Назначение |
|------|---------|-----------|
| 7 | CBL_MODE | `1` = DMA-режим из буфера; `0` = прямой DAC |
| 6 | CBL_STEREO | `1` = стерео (2 байта на отсчёт) |
| 5 | CBL_MODE16 | `1` = 16-битные сэмплы |
| 4 | CBL_INT_ENA | `1` = прерывание при опустошении буфера |
| 3:0 | делитель | частота воспроизведения (см. таблицу) |

**Таблица частот (CBL_XX[3:0]):**

| Код | Частота | Режим |
|-----|---------|-------|
| 0 | 16 кГц | моно/стерео 8-бит |
| 1 | 22 кГц | моно/стерео 8-бит |
| 2–7 | — | зарезервировано |
| 8 | 7.8125 кГц | моно/стерео 8/16-бит |
| 9 | 10.9375 кГц | |
| 10 | 15.625 кГц | |
| 11 | 21.875 кГц | |
| 12 | 31.25 кГц | |
| 13 | 43.75 кГц | |
| 14 | 54.6875 кГц | |
| 15 | 109.375 кГц | |

**DMA-запись в буфер Covox:**

Есть два пути записи:

1. **Прямой** (`CBL_WR` от порта `0x88`): запись в `D[]` → в буфер по адресу `CBL_WA`.
2. **DMA от ускорителя**: при `DECODE.PAGE == 0xFD` и `CBL_INT_ENA=1` и `ACC_DIR1=1`:
   ```vhdl
   CBL_WR = ... or (DFF((DECODE.PAGE[7:0] == 0xFD), CLK42, (CBL_INT_ENA & ACC.ACC_DIR1)) &
            !DFF((DECODE.MC_WRITE or DECODE.CAS), CLK42, , /RESET))
   ```
   Данные берутся из `ACC.MDO[7:0]`.

**Прерывание Covox:**

```vhdl
CBL_INT = DFF(GND, !CBL_CNT6, , (CBL_INT_ENA & (/IO or /M1)))
```

`CBL_INT` активируется (лог. 1) когда `CBL_CNT[6]` обнуляется — буфер воспроизведения наполовину опустел. Прерывание объединяется с видеопрерыванием: `INT_X = ... or !CBL_INT`.

**Алгоритм DMA-воспроизведения:**

```asm
; 1. Настроить режим
LD A, 0b10010000    ; CBL_MODE=1, CBL_INT_ENA=1, freq=0 (16kHz)
OUT (0x89), A

; 2. Заполнить буфер (256 сэмплов)
LD HL, sound_data
LD BC, 256
LD DE, 0            ; адрес записи в Covox-буфер (внутренний)
loop:
    LD A, (HL)
    OUT (0x88), A   ; прямая запись в DAC/буфер
    INC HL
    DEC BC
    JR NZ, loop

; 3. В обработчике INT:
;    заполнить следующие 128 сэмплов
;    CBL_INT сбрасывается автоматически при EI или чтении флага
```

**Адресация буфера Covox:**

```vhdl
CBL.rdaddress[] = (CBL_CNT[7:1], (CBL_CNT0 & !CBL_STEREO) or (AUDIO_CH & CBL_STEREO))
CBL.wraddress[] = ((!A[15:8]) & !CBL_INT_ENA) xor CBL_WA[]
```

В стерео-режиме четные адреса — левый канал, нечётные — правый.

## Beta/TM9 (WG1793 / WD1793)

### Порты

| Порт Z80 | DCP индекс | MA[]-биты | Назначение |
|---------|-----------|----------|-----------|
| `0x1F` | `0x10` | `7018` | Регистр статуса (чтение) / команды (запись) |
| `0x3F` | `0x11` | `7019` | Регистр дорожки |
| `0x5F` | `0x12` | `701A` | Регистр сектора |
| `0x7F` | `0x13` | `701B` | Регистр данных |
| `0xFF` | `0x14` | `7017` | PDOS команда |
| `0xDF` | `0x15` | `701F` | Чтение клавиш / WR A20 |
| `0x23` | `0x16` | `7023` | Переключение на 720К |
| `0x27` | `0x17` | `7027` | Переключение на 1440К |

### Сигналы управления WG93

```vhdl
WR_TM9   = (!MA_CT1 or (!/IO & !PORTS_X))    -- строб WG93
WR_AWG   = WR_AWGX                            -- выбор WG93/TM9
RD_KP11  = !(MA_CT[] == 0)                   -- чтение Kempston/порт
KP11_MIX = TFF(VCC, RD_KP11)                 -- мультиплекс
```

`PORTS_X = (DCPP[7:4] == 2) or (DCPP[7:4] == 1)` — признак портов FDD (диапазоны `0x1x` и `0x2x` в DCP).

Сигналы идут через шлейф при `SXA=1`:
- `XA0` = FDD_W (запись в FDD)
- `XA1` = FDD_CH (сигнал смены дискеты)
- `XA2` = SINC_1 (синхронизатор 1)
- `XA3` = SINC_2 (синхронизатор 2)

## IDE/ATA (HDD)

### Порты

Стандартный ATA: `0x60–0x6E` (CS1), `0x3F6–0x3F7` (CS3):

| Порт Z80 | Регистр ATA |
|---------|------------|
| `0x60` | Data (16-бит) |
| `0x62` | Error/Features |
| `0x64` | Sector Count |
| `0x66` | LBA 0–7 |
| `0x68` | LBA 8–15 |
| `0x6A` | LBA 16–23 |
| `0x6C` | Drive/Head |
| `0x6E` | Command/Status |
| `0x3F6` | Alt Status |
| `0x3F7` | Device Control |

Выбор диска: `OUT (0x7060/0x7064), A` — DCP индексы `0x2A/0x2B`.

### 16-битный доступ (HDD_FLIP)

Регистр данных IDE — 16-битный. `HDD_FLIP` чередует младший и старший байт:

```vhdl
HDD_FLIP.clk = /IOM
HDD_FLIP.d   = !HDD_FLIP & (PAGE[11:8] == 0)
```

Первый `IN A,(0x60)` → читается `MD[7:0]`, `MD[15:8]` сохраняется в `HDDR[]`.
Второй `IN A,(0x60)` → читается `HDDR[]` (буфер).

При `DOUBLE_CAS=1` (ускоритель в IDE-DMA режиме) оба байта читаются за один цикл.

### HDDR буфер

```vhdl
HDDR[].clk = /WR & (/RD or DECODE./IOMM)
HDDR[].ena = DECODE.HDD_DATA & !DECODE./IOM

CASE DFF(/WR, DECODE.RAS) IS
    WHEN 0 => HDDR[].d = D[]         -- запись: буферизуем байт от CPU
    WHEN 1 => HDDR[].d = MD[15:8]    -- чтение: сохраняем старший байт IDE
END CASE;
```

## CMOS/NVRAM (RTC)

### Порты

| Порт Z80 | DCP индекс | Значение | Операция |
|---------|-----------|---------|---------|
| `0x71` | `0x1C` | `0x71D8` | Чтение данных CMOS |
| `0x70` | `0x1D` | `0x70D4` | Запись адреса CMOS |
| `0x71` | `0x1E` | `0x71D4` | Запись данных CMOS |

### Алгоритм доступа к CMOS

```asm
; Запись в CMOS:
LD A, адрес         ; 0–127: регистры RTC, 128–255: пользовательская NVRAM
OUT (0x70), A       ; записать адрес
LD A, данные
OUT (0x71), A       ; записать данные

; Чтение из CMOS:
LD A, адрес
OUT (0x70), A
IN A, (0x71)        ; прочитать данные
```

Физический интерфейс: `MA[7:6]=11` → CMOS, `MA[8]` = data/addr, `MA[5]` = строб, `/WR` = направление.

## Прерывания

### INT_X (видео + Covox)

```vhdl
INT_X = !DFF(GND, (SVIDEO.INTT & KEYS.int),
    , ((/IO or /M1) & DFF(DFF(!INT_X, CTH2), CTH2)))
    or !CBL_INT
```

- `SVIDEO.INTT` — вертикальный ретрейс (≈50 раз/сек)
- `KEYS.int` — разрешение прерываний клавиатуры (= `int_ena`)
- `CBL_INT` — прерывание опустошения буфера Covox
- `INT_X` выводится на `/INT` процессора Z80

Прерывание маскируемое. После обработки автоматически сбрасывается при `EI` + `(M1 or IO)`.

### NMI_X (немаскируемое)

```vhdl
IF NMI_ON == "ON" GENERATE
    NMI_X = (!KEYS.kb_f12 & KEYS.kb_alt)
```

Активируется нажатием F12+Alt. Параметр `NMI_ON="OFF"` (по умолчанию) отключает аппаратный NMI (`NMI_X = 0`).

### Программный сброс (SOFT_RESET)

```vhdl
SOFT_RES[].clk = !CT4
CASE SOFT_RES[] IS
    WHEN 0,1 => SOFT_RES[] = 0
    WHEN 2   => SOFT_RES[] = 1
    WHEN 3   => SOFT_RES[] = 2
END CASE;
SOFT_RESET = (SOFT_RES[] == 0)

/RESET = TRI(GND, !DFF((KEYS.kb_reset & SOFT_RESET), CLK42))
```

Сброс формируется:
- Клавишей Del (`KEYS.kb_reset`)
- Записью в порт `0x2E` (DCP индекс `0x2E` = `0x7160`)

`SOFT_RESET` — счётчик защиты от случайного сброса: только при двух последовательных подтверждениях.

## Синхронизаторы и шлейф XA

### Мультиплексирование XA[3:0]

Шлейф `XA[0:3]` используется попеременно для двух функций, управляемых сигналом `SXA`:

```vhdl
SXA = DFF((CT3 xor CT2), CLK42)

-- При SXA=0 (клавиатура/мышь):
XA0 = TRI(KBD_CX, WR_TM9)   -- выход: данные клавиатуры
XA1 = TRI(KBD_DX, WR_TM9)
XA2 = TRI(TAPE_OUT, WR_TM9)
XA3 = TRI(GND, GND)

KBD_CC   = DFFE(XA0, T_RDXA, , , !SXA)  -- чтение при SXA=0
KBD_DD   = DFFE(XA1, T_RDXA, , , !SXA)
TAPE_IN  = DFFE(XA2, T_RDXA, , , !SXA)
MOUSE_D  = DFFE(XA3, T_RDXA, , , !SXA)

-- При SXA=1 (FDD/синхро):
FDD_CH   = DFFE(XA1, T_RDXA, , , SXA)  -- чтение при SXA=1
FDD_W    = DFFE(XA0, T_RDXA, , , SXA)
SINC_1   = DFFE(XA2, T_RDXA, DECODE./RES, , SXA)
SINC_2   = DFFE(XA3, T_RDXA, DECODE./RES, , SXA)
```

`RDXA` — строб чтения шлейфа:
```vhdl
RDXA = DFF(!((CT[2:0] == 110b) or (CT[2:0] == 0)), CLK42)
```

### SINC — внешняя синхронизация видео

`SINC_1` и `SINC_2` — входы внешних синхросигналов (например, от PAL-декодера или второго монитора):

```vhdl
CASE !SINC_1 IS
    WHEN 0 => SINC_HOLD[3:0] = HOLD[3:0]        -- загрузка начального значения
    WHEN 1 => SINC_HOLD[3:0] = (SINC_HOLD[3:0]+1) or !SINC_1M
END CASE;
COPY_SINC_H = DFF((!SINC_1M & DFF(SINC_1M, CLK42)), CLK42)  -- фронт
```

- `SINC_1` → горизонтальная синхронизация (4-битный счётчик `SINC_HOLD[3:0]`)
- `SINC_2` → вертикальная синхронизация (5-битный счётчик `SINC_HOLD[8:4]`)
- `COPY_SINC_H/V` → передаются в `VIDEO2` для синхронизации счётчиков кадра

Регистр `HOLD[7:0]` (порт `0xCB`) задаёт начальные значения счётчиков при переинициализации.

## Управляющие порты 0xC? — сводная таблица

| Порт | Маска DCPP | Регистр | Примечание |
|------|----------|---------|-----------|
| `0xC0/0xC8` | `1100X000` | SC | Системная конфигурация |
| `0xC1/0xC9` | `1100X001` | PN | Страничник |
| `0xC2/0xCA` | `1100X010` | BORDER | Бордер + TAPE_OUT |
| `0xC3` | `11000011` | ALL_MODE | Ускоритель, WAIT, IRQ |
| `0xC4` | `1100X100` | AGR (через ACC) | Строчный адрес |
| `0xC5/0xCD` | `1100X101` | RGMOD | Режим видео |
| `0xC6/0xCE` | `1100X110` | CNF | Системная конфигурация 2 |
| `0xC7` | `11000111` | WR_C7 (ALT_ACC) | Альтернативный адрес ускорителя |
| `0xCB` | `11001011` | HOLD | Счётчики синхронизации |

Все порты `0xCx` дублируются по маске `1100XXXX` — запись в `0xC0` идентична записи в `0xC8`, `0xCE`, и т.д. (при совпадении младших 3 бит).
