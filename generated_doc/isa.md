# Подсистема ISA и кэширование ROM

## Назначение

ACEX K30 обеспечивает два аппаратных ISA-слота и механизм «кэша» — подстановки внешней ROM/RAM вместо системного ROM или произвольной страницы нижних 16 КБ. Логика сосредоточена в блоке `SP2_ACEX.TDF` (строки 552–717) и активна только при параметре `UPDATE=1`.

Ключевая идея: ISA и кэш не имеют собственного адресного пространства — они подмешиваются в окно `0x0000–0x3FFF`. Z80 не знает, что читает не из RAM, а из внешнего устройства.

## Регистры управления

### /SYS (порты `0x3C`/`0x7C`)

Декодируется по маске `A[7:0] == 0b0X111100`. Записывается по `/IOWR`.

```vhdl
/SYS   = DFFE(!A6, /IOWR, /RESET, , SYS_ENA2)
SYS_PG = DFFE(D0,  /IOWR, /RESET, !ROM_RG4, (SYS_ENA2 & !D1))
```

- `A6=0` → `/SYS=0` → системный ROM активен в `0x0000–0x3FFF`
- `A6=1` → `/SYS=1` → обычная RAM
- `D0` при `/SYS=0` и `D1=0` записывается в `SYS_PG` (выбор ROM0/ROM1)

### ISA_PORT (порты `0xF0–0xFF`)

```vhdl
ISA_PORT[].clk = /IOWR
ISA_PORT[].ena = DECODE.BLK_R & DFF((DCPP == 1111XXXX), CLK42)
ISA_PORT[].d   = D[]
```

Записывается только при `BLK_R=1` (SC4=1). Биты:

| Биты | Назначение |
|------|-----------|
| `[2:1]` | Номер слота: `00`=слот 1, `01`=слот 2 |
| `[7:4]` | Если `= 0xD` (1101b) → `PRE_ISA=1` (ISA заблокирована) |

Использование: `OUT (0xF0), N` — выбрать слот N.

### ISA_A20 (порт `0x1B`)

```vhdl
ISA_A20 = (/IOWR or !DFF((DCPP == 00011011), CLK42))
```

`OUT (0x1B), X` — запись любого байта. `ISA_A20=0` → кэш/ISA полностью заблокированы. Должен быть установлен (записью любого значения без сброса) перед работой с ISA.

Примечание: `ISA_A20` — это сигнал низкого уровня (write-strobe): он равен `0` ровно в момент записи в порт `0x1B`, иначе `1`. Т.е. «установить A20» значит просто записать что-то в `0x1B` — сигнал уйдёт в `0` на один такт и вернётся.

На практике для постоянного A20 используют: повторную запись перед каждым ISA-доступом, или фиксацию внешней логикой.

### CASH_ON (чтение портов `0xFB`/`0x7B`)

```vhdl
CASH_ON = DFFE(A7, /IORD, /RESET, , DFF((A[7:0] == X1111011), CLK42))
```

`IN A,(0xFBh)` — читает произвольный байт, но фиксирует `CASH_ON = A7`. Поскольку внутри DCP нет внятного источника данных для этого порта, на шине будет `0xFF` (что даёт `A7=1`). Для сброса кэша: `IN A,(0x7Bh)` при `A7=0` (т.е. не читать `0xFB`, а читать порт с A7=0 вручную выставленным).

## Сигналы предразрешения (PRE_*)

Три сигнала определяют, что активно при чтении `0x0000–0x3FFF`:

```vhdl
PRE_ISA  = !( ISA_PORT == 1101XXXX ) or !BLK_R or !A14 or !A15
PRE_ROM  = /SYS or A14 or A15 or CASH_ON
PRE_CASH = A14 or A15 or !CASH_ON
```

Значение `0` = «разрешено», `1` = «запрещено»:

| PRE_ROM | PRE_CASH | PRE_ISA | Что активно |
|---------|---------|---------|------------|
| 0 | 0 | x | Ошибка — оба одновременно (даёт `ISA_A = 1000`) |
| 0 | 1 | x | Системный ROM (`RA = ROM_RG адрес`) |
| 1 | 0 | x | Кэш (`RA = 01, ROM_RG[1:0]`) |
| 1 | 1 | 0 | ISA (`RA = !PRE_ISA, 0, ISA_PORT[2:1]`) |
| 1 | 1 | 1 | Обычная RAM |

Когда `PRE_ROM=0` и `PRE_CASH=0` одновременно — это недопустимое состояние, `ISA_A = 1000` (признак ошибки, некоторые ISA-платы используют как «acknowledge»).

## Формирование CS сигналов

```vhdl
CS_ISA   = DFF(!/RF or PRE_ISA,   !/MR, , CS_ISA or !/MR)
CS_ROMT  = DFF(!/RF or PRE_ROM  or !PRE_CASH, !/MR, , CS_ROMT  or !/MR)
CS_CASHT = DFF(!/RF or !PRE_ROM or  PRE_CASH, !/MR, , CS_CASHT or !/MR)

CS_ROM  = CS_ROMT
CS_CASH = CS_CASHT & CS_ISA & ISA_A20
```

Все три регистра защёлкиваются по `/MR` (начало машинного цикла памяти). Сигнал `CS_CASH` требует одновременного выполнения трёх условий: кэш разрешён, ISA разрешён, A20 активен.

Важно: `CS_CASH` выводится на вывод ПЛИС и управляет внешним сигналом `/OE` или `/CE` ISA-слота.

## Мультиплекс RA[3:0]

Адресные линии ROM/ISA (`RA[17:14]`):

```vhdl
CASE (PRE_ROM, PRE_CASH) IS
    WHEN 00 => ISA_A = 1000                              -- ошибка
    WHEN 01 => ISA_A = (ROM_RG3 xor !SYS_PG, ROM_RG[2:0])  -- ROM
    WHEN 10 => ISA_A = (01, ROM_RG[1:0])                    -- кэш
    WHEN 11 => ISA_A = (!PRE_ISA, 0, ISA_PORT[2:1])         -- ISA
END CASE;
RA[17:14] = ISA_A[3:0]
```

Таким образом, `RA[17:14]` несут:
- Для ROM: полный 4-битный адрес страницы (16 × 16 КБ = 256 КБ Flash)
- Для кэша: `01XX` — страница 4–7 (кэш-окно в пределах 128 КБ)
- Для ISA: старший бит = «слот активен», 2 бита = номер слота

## Сигнал XACS (ROM Write Enable)

```vhdl
XACS.clk = !(/MR or /WR)
XACS.d   = !ROM_RG4 or A14 or A15
XACS.prn = XACS or (!/MR & ROM_RG4)
```

`XACS` — выход `/WE` для внешней Flash-памяти. Активен (лог. 0) при:
- `ROM_RG4=1` (режим записи Flash)
- Обращение к `0x0000–0x3FFF` (A14=0 и A15=0)
- Цикл записи (`/WR` активен)

Прерывается (устанавливается в 1) как только `/MR` отпускается.

## BLK_MEM — блокировка одновременного доступа

```vhdl
BLK_MEM = !PRE_ROM or !PRE_CASH or !PRE_ISA
```

При `BLK_MEM=1` (ROM, кэш или ISA активны):
- `RD_RAM = !(/MR or /RD or BLK_MEM)` → CPU не читает из RAM
- `BLK_MWR2 = DECODE.RAM or BLK_MEM` → запрет записи в RAM
- `DECODE.BLK_MEM = BLK_MEM` → DCP добавляет задержки

## WAIT_ROM — ожидание при одновременном ROM+ISA

При одновременно активных `CS_ROM` и `CS_ISA` (переходное состояние, когда ROM ещё не отключился, а ISA ещё не включился), вставляются 4 такта ожидания:

```vhdl
WAIT_ROMX = CS_ROM & CS_ISA
WAIT_ROM  = WAIT_ROMX or DFF((WT_R[] == 0), CLK42, !WAIT_ROMX)

CASE (WAIT_ROMX, WT_R[]) IS
    WHEN 1XXX => WT_R[] = 4    -- запуск таймера
    WHEN 0001..01XX => WT_R[] = WT_R[] - 1  -- декремент
    WHEN 0000 => WT_R[] = 0
END CASE;
```

Общий WAIT: `/WAIT_ALL = DECODE./WAIT & WAIT_ROM & WAIT_ORIG`.

## Порты ISA-слотов (0x30–0x33 из DCP.MIF)

`DCP.MIF` содержит записи:
- `0x30` = `0x7000` — слот 1, I/O порты
- `0x31` = `0x7001` — слот 2, I/O порты
- `0x32` = `0x7002` — слот 1, окно памяти
- `0x33` = `0x7003` — слот 2, окно памяти

При обращении к этим индексам TYPE=7 → `MDP` активен, а `RD_KMPS=0` — признак чтения внешнего устройства. Реальные адреса на ISA-шине определяются через `RA[]` и `CS_CASH`.

## Алгоритм работы с ISA-слотом (пошагово)

```
; Инициализация ISA-слота 1
LD A, 0          ; номер слота 0 (биты 2:1 = 00)
OUT (0xF0), A    ; записать ISA_PORT (нужен BLK_R=1, т.е. SC4=1)

OUT (0x1B), A    ; активировать ISA_A20 (любое значение)

IN A, (0xFB)     ; CASH_ON = A7 = 1 (шина выдаст 0xFF → A7=1)

; Теперь при обращении к 0x0000–0x3FFF будет активен CS_CASH
; PRE_ROM=1 (CASH_ON=1), PRE_CASH=0, PRE_ISA=0
; RA[17:14] = (!PRE_ISA, 0, ISA_PORT[2:1]) = (1, 0, 0, 0)

LD HL, 0x0000
LD A, (HL)       ; чтение из ISA-слота 1 (адрес 0 в окне)

; Отключение ISA
IN A, (0x7B)     ; при A7=0 — CASH_ON=0 (если шина выдала байт с A7=0)
                 ; или: выставить A=0 через резистор и IN
```

**Важно:** при `CNF6=0` (`Scorpion-совместимость`) `SC` сбрасывается по `/RESET`, а значит `SC4=0` → `BLK_R=0` → записать `ISA_PORT` невозможно. Нужно сначала выставить `SC4=1` через порт `0xC0`.

## Связь с Beta/TM9 дисковым контроллером

Сигналы `WR_AWG`, `WR_TM9`, `RD_KP11`, `KP11_MIX` идут на тот же шлейф (XA0–XA3), что и ISA. Разделение осуществляется через `SXA`:

- `SXA=0` → читаем/пишем клавиатуру/мышь (KBD_CC/DD, TAPE_IN, MOUSE_D)
- `SXA=1` → читаем/пишем FDD/синхро (FDD_CH, FDD_W, SINC_1, SINC_2)

`BLK_MEM` запрещает одновременный доступ ISA и WG93: при `BLK_MEM=1` логика WG93 видит неактивный `MC_BEGIN` и ждёт.

## Практические заметки

1. **Инициализация при старте:** все регистры (`ISA_PORT`, `ISA_A20`, `CASH_ON`, `ROM_RG`, `/SYS`) сбрасываются в `0` при `/RESET`. Порядок инициализации: SC4=1 → ISA_PORT → ISA_A20 → CASH_ON.

2. **Блокировка ISA во время Flash-записи:** записать в `ISA_PORT` значение `0xD?` (любой байт с верхней тетрадой `D`). Это установит `PRE_ISA=1` и `CS_ISA=0` независимо от `CASH_ON`.

3. **Диапазоны `DCP.MIF` для ISA/кэша:** `PAGE[7:4]=1111` зарезервирован для ISA/кэш-операций. Не используйте его для других устройств.

4. **WAIT при переключении:** между отключением ROM (`/SYS=1`) и первым ISA-доступом может возникнуть `WAIT_ROMX`. `WT_R[]` автоматически вставит 4 такта — дополнительного кода не требуется.

5. **Кэш ROM:** кэш (`PRE_CASH=0`, `PRE_ROM=1`) использует `RA = (01, ROM_RG[1:0])`, т.е. 4 страницы по 16 КБ в диапазоне Flash 64–128 КБ. Это отдельный регион от системного ROM.
