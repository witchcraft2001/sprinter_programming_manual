# 7.1 Обзор дисковой подсистемы

> **Навигация:** [← 6.4 Примеры звука](../06_audio/04_examples.md) | [Оглавление](../README.md) | [7.2 IDE →](02_ide.md)

---

## Типы дисков

Sprinter поддерживает три типа устройств хранения:

| Тип | Номера | Протокол | Максимум |
|-----|--------|---------|---------|
| **FDD** (дискета 3.5") | `#00`, `#01` | WD1793-совместимый | 1.44 МБ |
| **IDE HDD** | `#80`, `#81`, `#82`, `#83` | ATA-2 | 2 ГБ (FAT16 лимит) |
| **IDE CD-ROM** | те же | ATAPI | CD до 700 МБ |
| **RAM-диск** | виртуальный | — | зависит от RAM |

---

## Уровни доступа

### 1. DSS API (высокий уровень)

- Работа с файлами: `Open`, `Read`, `Write`, `Close`
- Каталоги: `F_First`, `F_Next`, `ChDir`
- Абстрагирует от типа устройства
- См. раздел [4.3 Файловая система DSS](../04_dss/03_dss_filesystem.md)

### 2. BIOS API (средний уровень)

- Прямой доступ к секторам: `Drv_Read #55`, `Drv_Write #56`
- Работает с номером диска и LBA
- См. раздел [3.3 BIOS: работа с дисками](../03_bios/03_bios_disk.md)

### 3. Прямой доступ к портам (низкий уровень)

- IDE: порты `#0150`–`#4153`
- FDD: WG93 через MAX7000 (порты `#1F`–`#FF`)
- Используется только в загрузчиках и оптимизированном коде

---

## Структура раздела

Sprinter использует **MBR** (Master Boot Record) с несколькими разделами **FAT16**:

```
Диск HDD:
├── MBR (LBA 0)
├── Boot sector раздела 1 (FAT16)
│   ├── FAT 1
│   ├── FAT 2
│   ├── Root directory
│   └── Data area
├── Boot sector раздела 2
└── ...
```

## Дескрипторы устройств

Параметры устройств хранятся в системной странице `#FE` по фиксированным адресам:

| Адрес | Дескриптор | Размер |
|-------|-----------|--------|
| `#C1C0` | IDE 0 (master) | 8 байт |
| `#C1C8` | IDE 1 (slave) | 8 байт |
| `#C1E0` | FDD 0 (A:) | 8 байт |
| `#C1E8` | FDD 1 (B:) | 8 байт |

### Структура дескриптора IDE

```
+0  DRVHD_H   — Drive/Head байт (старший бит = slave)
+1  SC_PT_H   — секторов на трек
+2  HEADS_H   — количество головок
+3  CYL_L_H   — цилиндры (младший)
+4  CYL_H_H   — цилиндры (старший)
+5  SPCLL_H   — write precomp (младший)
+6  SPCLH_H   — write precomp (старший)
+7  TYPE_H    — тип: 0=нет, 1=HDD, 2=CD-ROM
```

## Обнаружение устройств

При старте BIOS вызывает `DRV_DETECT #57`, который:

1. Пытается опросить IDE 0 master (ATA IDENTIFY)
2. Определяет тип (HDD / CD-ROM / отсутствует)
3. Записывает параметры в дескриптор `#C1C0`
4. Повторяет для IDE 0 slave (`#C1C8`)
5. Проверяет FDD (`#C1E0`, `#C1E8`)

Программа может вызвать `DRV_DETECT` повторно, если вставлен новый диск.

## Маршрутизация дисков (letter → device)

DSS использует таблицу маршрутизации, определяющую, какая буква диска (`A:`, `B:`, `C:`, …) какому физическому устройству соответствует. Конфигурация хранится в CMOS регистре `#1E` (TR-DOS маршрутизация) и системной переменной `#C100` (DISK_TYPE).

Стандартная схема:
- `A:` → FDD 0
- `B:` → FDD 1
- `C:` → IDE 0 master (первый раздел)
- `D:` → IDE 0 master (второй раздел)
- `R:` → RAM-диск

---

## Статус диска (BSY, RDY, DRQ, ERR)

IDE-статус читается из порта `#4053` (`HDR_CTL`):

| Бит | Имя | Описание |
|-----|-----|---------|
| 7 | BSY | Устройство занято, другие биты недостоверны |
| 6 | RDY | Устройство готово к командам |
| 5 | DWF | Device Write Fault |
| 4 | DSC | Drive Seek Complete |
| 3 | DRQ | Данные готовы к передаче |
| 2 | CORR | Corrected data |
| 1 | IDX | Index |
| 0 | ERR | Ошибка (детали в HDR_ERR) |

---

## Типичный алгоритм чтения сектора

```
1. Проверить BSY → ждать
2. Установить номер диска (DRV_HEAD)
3. Установить LBA или CHS
4. Установить Sector Count
5. Послать команду READ (0x20)
6. Ждать DRQ
7. Читать 256 слов из порта данных
8. Проверить ERR
```

Всё это делает BIOS `DRV_READ #55`. Напрямую работать с портами стоит только в загрузчиках.

---

## Совместимость FAT16

DSS читает/пишет стандартные FAT16 тома:
- Sector size: 512 байт
- Cluster size: 1–64 сектора (зависит от объёма раздела)
- Имена файлов: 8.3 (long filenames не поддерживаются)
- Метка тома: поддерживается
- Hidden/System/ReadOnly атрибуты: поддерживаются

Тома, отформатированные в MS-DOS или Windows 95/98 (FAT16), работают в Sprinter без изменений.

---

## Геометрия CHS vs LBA

- **CHS** (Cylinder-Head-Sector) — старый адрес диска (3D).
- **LBA** (Logical Block Addressing) — линейный номер сектора (0..N).

BIOS Sprinter работает в основном с **LBA** через пары регистров HL (младшие 16 бит) + DE (старшие 16 бит).

```
Для IDE с геометрией CHS (C=1024, H=16, S=63):
  LBA = (C * HEADS + H) * SECTORS + S - 1

Для современных дисков всегда используйте LBA.
```

---

## Ограничения

- **2 ГБ на раздел** (FAT16 лимит при кластере 32 КБ)
- **Максимум 4 основных раздела** (MBR)
- **Без LBA48** — диски > 137 ГБ не поддерживаются полностью
- **Один активный диск** в системе BIOS одновременно

---

## Ключевые моменты

> - 3 уровня доступа: DSS (файлы), BIOS (секторы), порты (прямой).
> - Устройства: FDD `#00`/`#01`, IDE `#80`–`#83`.
> - Дескрипторы в системной странице `#FE` по адресам `#C1C0`, `#C1C8`, `#C1E0`, `#C1E8`.
> - Статусы IDE: BSY=7, RDY=6, DRQ=3, ERR=0.
> - FAT16 на всех типах носителей.
> - Для большинства задач используйте DSS API; BIOS — только для прямого доступа к секторам.
