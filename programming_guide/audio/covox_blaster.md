# Covox-Blaster

Covox-Blaster (CBL) добавляет буфер на 256 байт и аппаратный счётчик (15/22 кГц). Процессор просто заполняет половины буфера, а ПЛИС самостоятельно выдаёт семплы на ЦАП. Статус текущей половины отображается в бите 7 порта `#FE`.

## Управляющий порт `#4E`

`CBL.Ctrl/#4E` — 16‑битный порт (пишите через `OUT (C),A`). Биты:

| Бит | Назначение |
|-----|------------|
| 7   | 1 — включить CBL, 0 — обычный Covox. |
| 6   | 1 — стерео (чередование L/R), 0 — моно. |
| 5   | 1 — 16‑битные семплы (256 байт на половину буфера). |
| 4   | 1 — разрешить прерывания запросов данных. |
| 3..0 | Частота дискретизации. `0/1` — старые режимы (16/22 кГц), значения `8..F` — 7.8125..109.375 кГц, перечислены ниже. |

Частотные коды (`clock/256`): `8=7.8125`, `9=10.9375`, `A=15.625`, `B=21.875`, `C=31.25`, `D=43.75`, `E=54.6875`, `F=109.375` кГц. Режимы `0..7` зарезервированы для совместимости [docs/Программирование_Спринтера_Денис_Паринов.doc:200-260].

## Запись блоков

Счётчик автоматически переключает банки по 128 байт. Бит 7 порта `#FE` показывает активный банк (0 — нижняя половина, 1 — верхняя). Заполняем половину тогда, когда отображается другая.

```asm
; HL = адрес PCM, A = номер страницы (для PAGE3),
; каждая половина содержит 128 байт
FillCBLHalf:
        IN   A,(#FE)
        XOR  80h            ; выяснить, где проигрывается звук сейчас
        BIT  7,A
        JR   Z,FillCBLHalf  ; ждём переключение

        PUSH AF
        LD   A,(saved_page3)
        OUT  (#E2),A        ; восстановить, если нужно
        POP  AF

        LD   A,(pcm_page)
        OUT  (#E2),A        ; подключить страницу PCM в окно #C000

        LD   BC,#00FB
        LD   D,128
loop:
        OUTI
        DEC  D
        JR   NZ,loop

        LD   A,(saved_page3)
        OUT  (#E2),A
        RET
```

В ISR достаточно вызвать `FillCBLHalf` дважды за период (по переходам бита 7). При переполнении адреса страницы переключите `pcm_page` и `HL` на следующую страницу данных.

## Обработка запросов и прерываний

- `Port #FE`, bit7 — запрос данных от CBL (поднимается каждые 128 байт в 8‑битном режиме или 256 байт в 16‑битном). `bit5` можно использовать как кадровый импульс (4 мс / 20 мс период).
- В режиме 16 бит счётчик хранит 256 байт, поэтому в ISR проверяйте `bit7` и узнавайте, какую половину заполнять.
- Если включили прерывания CBL (бит4), используйте IM2: DOS‑обработчик иначе будет вызываться слишком часто и звук начнёт «тормозить» на больших частотах.

## Минимальный пример

```asm
        LD  BC,#004E
        LD  A,8Bh          ; CBL on, mono, 8 бит, 22 кГц
        OUT (C),A

        LD  HL,Sample
        LD  BC,#004F
        OTIR               ; отправить 256 байт в буфер
        RET

Sample: DB 80h,81h,7Fh,... ; смещение нуля = 80h, в стерео L/R чередуются
```

Для 16‑битного режима нуль должен быть `0000h`, а данные записываются парами (младший байт — младшие 8 бит, старший байт — старшие) [docs/Программирование_Спринтера_Денис_Паринов.doc:260-330].

## Советы

- Если звук берётся из DSS страницы, заранее запросите её через `Dss.GetMem` и переключайте `PAGE3` только на время `OUTI`.
- Для стерео (когда появятся новые прошивки) используйте дополнительные биты порта `#4E`. В текущей версии всегда выводится моно.
- Чтобы «заглушить» CBL, заполните весь буфер значением `#80` (середина шкалы) и сбросьте страницу `SND_P` в ноль.
