# Аппаратный акселератор RAM/VRAM

Акселератор реализован внутри ПЛИС и ускоряет массовые операции с памятью: копирование горизонтальных/вертикальных линий, заливки, и логические функции AND/OR/XOR. Управление строится на переопределении «пустых» команд Z80: вместо выполнения `LD B,B` (NOP) ПЛИС переключает режим акселератора [sprinter_docs/accelerator/accel.md:1-80](../sprinter_docs/accelerator/accel.md#L1).

## Команды

| Инструкция | Назначение |
|------------|-----------|
| `LD B,B`   | Выключить акселератор (вернуться к обычному доступу).
| `LD D,D`   | Включить акселератор и принять следующий байт как длину блока (1..256). В новых прошивках `LD H,H` используется для установки старших бит длины (>256) [sprinter_docs/accelerator/accel.md:20-60](../sprinter_docs/accelerator/accel.md#L20).
| `LD L,L`   | Копирование горизонтального блока. Следующая операция чтения (`LD A,(HL)` или `LDI/LDIR`) помещает данные в буфер акселератора, запись (`LD (DE),A`) воспроизводит блок.
| `LD A,A`   | Копирование вертикального блока (взаимодействует с видеостроками, удобно для VRAM).
| `LD C,C`   | Заливка горизонтальной линии: запись следующего байта (`LD (HL),A`) размножает его `N` раз.
| `LD E,E`   | Заливка вертикальной линии (например, спрайтов на VRAM).
| `XOR (HL)`/`OR (HL)`/`AND (HL)` | Логические функции: после `LD L,L` буфер можно обработать логически перед записью.

Любая операция выполняется над количеством байт, установленным в `LD D,D` (и, при необходимости, `LD H,H`). Для минимизации артефактов рекомендуется отключать прерывания (`DI`) до завершения цикла, хотя в последних прошивках акселератор автоматически приостанавливается на `RETI` [sprinter_docs/accelerator/accel.md:60-90](../sprinter_docs/accelerator/accel.md#L60).

## Примеры использования

### Копирование всего экрана

```asm
        LD  HL,#C040      ; источник (верхняя строка экрана 0)
        LD  DE,#C180      ; назначение (экран 1)
        LD  BC,#0140      ; ширина в байтах (320 пикселей)
        DI
        LD  D,D           ; включить акселератор, длина = 256
        LD  A,0
        LD  A,A           ; режим вертикальных линий
        LDIR              ; CPU выполняет 320 копий, акселератор разворачивает блоки
        LD  B,B           ; выключить
        EI
```

Этот фрагмент из официального описания копирует весь экран менее чем за 1.2 кадра [sprinter_docs/accelerator/accel.md:70-90](../sprinter_docs/accelerator/accel.md#L70).

### Заливка окна (gfxview)

`gfxview` использует процедуру `FillWndAcc`: она временно переключает `PAGE1` на страницу режима (`50h`), выставляет `RGADR` на нужную строку и для каждой строки вызывает `LD D,D`/`LD C,C`/`LD (HL),C` для заполнения прямоугольника, затем `LD B,B` выключает акселератор [gfxview/gfx_wind.asm:150-220](../gfxview/gfx_wind.asm#L150). Ключевые моменты:

1. Сохранить и восстановить порты (`PAGE1`, `RGADR`).
2. Запретить прерывания на время цикла (`DI`/`EI`).
3. Использовать `DJNZ` для копирования N строк — акселератор сам размножает байты по горизонтали.

### Логические операции

Чтобы кодировать блок (например, XOR‑шифрование), достаточно подключить акселератор и заменить `LD (DE),A` на `XOR (HL)` (или `OR/AND`). Буфер в акселераторе обновляется, и запись в память выполняется с уже изменёнными данными [sprinter_docs/accelerator/accel.md:90-130](../sprinter_docs/accelerator/accel.md#L90).

## Использование из C

Хотя акселератор управляется инструкциями Z80, SDCC и z88dk позволяют вставлять inline‑asm. Шаблон `__naked` функции для SDCC, заполняющей строку:

```c
void accel_fill(uint8_t* dst, uint8_t value, uint8_t len) __naked {
__asm
        pop bc          ; вернуться куда надо
        pop hl          ; dst
        pop de          ; value/len в DE (пасём сами)
        push de
        push hl
        push bc
        ld a,e          ; len
        ld d,d          ; включить акселератор и принять длину (A)
        ld c,c          ; режим заливки
        ld a,d          ; value из D (или загрузить отдельно)
        ld (hl),a
        ld b,b          ; выключить
        ret
__endasm;
}
```

В SDK можно оформить набор таких обёрток (копирование, вертикальная заливка) и вызывать их из C кода. Главное — сохранить/восстановить флаги и регистры, которые ожидает вызывающий код.

## Практические рекомендации

- **Сохраняйте `RGADR` и окна**. Перед работой с VRAM (страницы `#50..#5F`) убедитесь, что нужная страница подключена, а по завершении восстановите исходные значения.
- **Отключайте акселератор** (`LD B,B`), даже если операция закончилась, — иначе последующие инструкции могут неожиданно перехватить управление.
- **Синхронизация с прерываниями**. Если ISR тоже использует акселератор (например, эффект IM2), следите за приоритетами: выключайте акселератор в ISR или используйте новую прошивку, где он автоматически приостанавливается на `RETI`.
- **Документируйте «магические» опкоды**. Команды `LD D,D`, `LD C,C` и т.п. трудно заметить при чтении кода, поэтому комментируйте каждое использование (как делает `gfxview`).
- **Оценивайте длительность**. ПЛИС пересылает байты со скоростью ≈7 МБ/с, поэтому время операции = «обычная команда» + `размер/7_000_000`. На старых прошивках прерывания нужно запрещать вручную (`DI`/`EI`), иначе ISR получит «сломанный» набор инструкций [docs/sp2000_man.pdf:1230-1360].
