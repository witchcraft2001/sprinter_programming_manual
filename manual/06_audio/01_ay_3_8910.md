## О чипе AY-3-8910

**AY-3-8910** — 3-канальный программируемый звуковой генератор, стандарт ZX Spectrum 128K и многих других 8-битных компьютеров. В Sprinter AY **эмулируется внутри FPGA ACEX 1K30** (файлы `AY.TDF` / `AY.MIF`).

Эмуляция полная: регистры, огибающие, шум, 3 голоса — идентичные оригиналу.

---

## Возможности

| Параметр | Значение |
|---------|---------|
| Голоса | 3 независимых тональных канала + 1 шумовой |
| Тон диапазон | 12-битный делитель (1–4095) |
| Амплитуда | 4 бита (0–15, логарифмическая шкала) |
| Огибающая | 32-битный счётчик с 10 формами |
| Количество регистров | 16 (R0–R15) |
| Тактовая частота | 1.75 МГц (стандарт ZX128) |

---

## Порты AY в Sprinter

### Нативные порты Sprinter

| Порт | Имя | Назначение |
|------|-----|-----------|
| `#8D` | `AY_REG` | Запись номера регистра (0–15) |
| `#8E` | `AY_DAT` | Запись/чтение данных выбранного регистра |
| `#8C` | `AY_DAT_RD` | Чтение данных (альтернатива) |

### ZX Spectrum-совместимые порты

| Порт | Назначение |
|------|-----------|
| `#FFFD` | Запись номера регистра (как в ZX 128) |
| `#BFFD` | Запись данных регистра |

> Используйте нативные порты `#8D`/`#8E` в нативных программах — они работают быстрее и независимо от ZX-режима.

## Регистры AY

| Регистр | Назначение | Диапазон |
|---------|-----------|---------|
| R0 | Тон канала A, младший байт | 0–255 |
| R1 | Тон канала A, старший | 0–15 (4 бита) |
| R2 | Тон канала B, младший | 0–255 |
| R3 | Тон канала B, старший | 0–15 |
| R4 | Тон канала C, младший | 0–255 |
| R5 | Тон канала C, старший | 0–15 |
| R6 | Период шума | 0–31 (5 бит) |
| R7 | Микшер (Enable) | см. ниже |
| R8 | Амплитуда канала A | 0–15 или 16 (envelope) |
| R9 | Амплитуда канала B | — |
| R10 | Амплитуда канала C | — |
| R11 | Период огибающей, младший | 0–255 |
| R12 | Период огибающей, старший | 0–255 |
| R13 | Форма огибающей | 0–15 |
| R14 | I/O порт A (не используется) | — |
| R15 | I/O порт B (не используется) | — |

### R7 — регистр микшера

| Бит | Значение |
|-----|---------|
| 0 | Тон A (0 = вкл) |
| 1 | Тон B (0 = вкл) |
| 2 | Тон C (0 = вкл) |
| 3 | Шум A (0 = вкл) |
| 4 | Шум B (0 = вкл) |
| 5 | Шум C (0 = вкл) |
| 6 | I/O A направление |
| 7 | I/O B направление |

**Обратите внимание:** биты микшера **инвертированы** — 0 означает "включено".

Типичное значение для звука без шума: `#F8` = `11111000` (тон A, B, C включены; шум выключен).

### R8–R10 — амплитуда

| Значение | Эффект |
|---------|--------|
| 0–15 | Фиксированная громкость |
| 16 (`#10`) | Использовать огибающую (envelope) |

### R13 — форма огибающей

10 уникальных форм (0–15, некоторые дублируются):

```
0-3:   \___     одно падение до 0
4-7:   /___     одно повышение до пика, затем 0
8:     \\\\     пила вниз (повторяется)
9:     \___     одно падение
10:    \/\/     треугольник (повторяется)
11:    \‾‾‾     падение, потом max
12:    ////     пила вверх
13:    /‾‾‾     одно повышение, потом max
14:    /\/\     обратный треугольник
15:    /___     одно повышение
```

## Вычисление периода тона

```
Frequency = MasterClock / (16 × TonePeriod)

где MasterClock = 1 750 000 Гц (1.75 МГц)
    TonePeriod  = 12-битное число из R0 (младший) + R1 (старший 4 бита)
```

Для частоты 440 Гц (нота A4):
```
TonePeriod = 1 750 000 / (16 × 440) ≈ 248.6 ≈ 249 (#F9)
R0 = 0xF9, R1 = 0x00
```

## Базовые примеры

### Простой тон

```asm
; Воспроизвести чистый тон 440 Гц на канале A
PlayTone440:
        ; R0 = #F9, R1 = #00 (период тона)
        ld      a, 0            ; регистр 0
        out     (#8D), a
        ld      a, #F9
        out     (#8E), a

        ld      a, 1            ; регистр 1
        out     (#8D), a
        ld      a, 0
        out     (#8E), a

        ; Микшер: только тон A включён
        ld      a, 7            ; регистр 7
        out     (#8D), a
        ld      a, #FE          ; 11111110 — только тон A on
        out     (#8E), a

        ; Амплитуда канала A = 15 (максимум)
        ld      a, 8
        out     (#8D), a
        ld      a, 15
        out     (#8E), a
        ret
```

### Остановка всех каналов

```asm
StopAY:
        ; Отключить всё в микшере
        ld      a, 7
        out     (#8D), a
        ld      a, #FF          ; все биты = 1 → всё выключено
        out     (#8E), a

        ; Нулевая амплитуда
        xor     a               ; в A остаётся 0, пишем в каждый канал
        ld      b, 3            ; 3 канала
        ld      c, 8            ; начальный регистр амплитуды
.loop:
        ld      a, c
        out     (#8D), a
        xor     a
        out     (#8E), a
        inc     c
        djnz    .loop
        ret
```

### Помощник: запись в регистр AY

```asm
; Записать значение A в регистр B чипа AY
; Сохраняет регистры C, DE, HL
WriteAY:
        push    af
        ld      a, b
        out     (#8D), a        ; выбрать регистр
        pop     af
        out     (#8E), a        ; записать данные
        ret
```

## Воспроизведение ноты

```asm
; Таблица периодов для 5 октав (60 нот)
; Период = 1750000 / (16 × freq) для каждой MIDI-ноты
note_periods:
        dw  0xD5D   ; C0 (16.35 Гц)  — пример, высчитайте по формуле
        ; ... до C6 ...

; Сыграть MIDI-ноту на канале A
; Вход: A = номер ноты (0..63)
PlayNote:
        add     a, a            ; index в word
        ld      h, 0
        ld      l, a
        ld      bc, note_periods
        add     hl, bc
        ld      e, (hl)         ; lo
        inc     hl
        ld      d, (hl)         ; hi

        ld      b, 0            ; регистр 0 (A lo)
        ld      a, e
        call    WriteAY

        ld      b, 1            ; регистр 1 (A hi)
        ld      a, d
        call    WriteAY

        ld      b, 7            ; mixer
        ld      a, #FE
        call    WriteAY

        ld      b, 8            ; амплитуда A
        ld      a, 15
        call    WriteAY
        ret
```

## Огибающая

```asm
; Использовать треугольную огибающую для канала A
PlayWithEnvelope:
        ; Период огибающей
        ld      a, 11
        out     (#8D), a
        ld      a, #FF
        out     (#8E), a

        ld      a, 12
        out     (#8D), a
        ld      a, #0F
        out     (#8E), a

        ; Форма огибающей: треугольник
        ld      a, 13
        out     (#8D), a
        ld      a, 10           ; форма "треугольник"
        out     (#8E), a

        ; Установить амплитуду канала A в режим "envelope"
        ld      a, 8
        out     (#8D), a
        ld      a, #10          ; бит 4 = использовать envelope
        out     (#8E), a
        ret
```

## Воспроизведение музыки

Стандартные форматы музыки для AY:
- **PT3** (Pro Tracker 3) — очень популярный для ZX Spectrum
- **AY** — стандартный бинарный формат
- **WYZ** — используется в SDK

SDK содержит готовый плеер в `zx-sprinter-sdk/sdk/src/wyzplayer/ayfxplay.asm`.

### Запуск плеера на прерывании

```asm
; В обработчике прерывания 50 Гц вызывать плеер
InterruptHandler:
        push    af
        push    bc
        push    de
        push    hl

        call    PT3_Play        ; или AYFX_Play — зависит от плеера

        pop     hl
        pop     de
        pop     bc
        pop     af
        ei
        reti
```

## Ключевые моменты

> - AY в Sprinter эмулируется FPGA — полная совместимость с ZX 128K AY.
> - Нативные порты: `#8D` (регистр), `#8E` (данные), `#8C` (чтение).
> - 16 регистров: R0–R5 тон, R6 шум, R7 микшер, R8–R10 амплитуда, R11–R13 огибающая.
> - Период тона = `1750000 / (16 × freq)`.
> - R7 — инвертированный (0 = вкл).
> - Для огибающей установите амплитуду = `#10` и задайте R11–R13.
> - Используйте готовые плееры (PT3, AYFX) — не пишите с нуля.
