# 5.8 Текстовый режим 80×32

> **Навигация:** [← 5.7 Double buffering](07_double_buffer.md) | [Оглавление](../README.md) | [5.9 Примеры графики →](09_examples.md)

---

## Что это

«Текстовый режим» Sprinter — это **программная абстракция поверх графического режима**. Видеоконтроллер всегда выводит пиксели (собственно «текстового» режима в железе нет), а BIOS/DSS предоставляют набор функций, работающих с **виртуальной текстовой сеткой** 80×32 символа (шрифт 8×8 на базе графического режима 640×256).

Программе достаточно вызывать DSS-функции для вывода символов — растеризация и позиционирование курсора выполняются BIOS-ом внутри.

Это основной режим для работы таких приложений как:
- **Sprinter TextEditor** ([https://github.com/witchcraft2001/sprinter-texteditor](https://github.com/witchcraft2001/sprinter-texteditor))
- Командная оболочка DSS
- Утилиты SETUP, форматирования, отладки

> **Замечание:** FlexNavigator ([https://github.com/witchcraft2001/flexnavigator](https://github.com/witchcraft2001/flexnavigator)) — это пример приложения, использующего **графический** режим 640×256 4bpp (16 цветов) для собственной отрисовки окон и текста без DSS text API.

---

## Характеристики

| Параметр | Значение |
|---------|---------|
| Сетка | **80 столбцов × 32 строки** (2 560 ячеек) |
| Шрифт | 8 × 8 пикселей, 256 символов (псевдографика включена) |
| Базовое разрешение | 640 × 256 (16 столбцов × 32 строки × 8 = 640 × 256) |
| Цвет символа/фона | 4 бита на компонент (16 цветов) — ниббл атрибута |
| Буфер символов | поддерживается BIOS во внутренних структурах (не видим программе напрямую) |
| Прокрутка | через `Dss.Scroll #55` |
| Двойная буферизация | штатно — одна копия; сохранение/восстановление через `WinCopy` / `WinRest` |

---

## Включение текстового режима

Текстовый режим включается тем же `Dss.SetVMod`, что и графические. Точный байт режима зависит от версии BIOS и SDK, но наиболее практичный способ — **не переключать режим вручную**: при старте из DSS текстовый режим уже активен, и программа может сразу вызывать `PCHARS`/`PUTCHAR`.

```asm
DssRst          EQU #10
Dss.SetVMod     EQU #50
Dss.GetVMod     EQU #51

; Узнать текущий режим
        ld      c, Dss.GetVMod
        rst     DssRst
        ; A = код режима, B = видеостраница
```

Для приложений, которые сначала работают в графическом режиме, а потом возвращаются к текстовому, необходимо **сохранить** исходный режим через `GetVMod` и восстановить через `SetVMod`.

## API текстового режима (DSS)

Все функции — через `RST #10`, номер в регистре `C`:

| Код | Имя | Назначение |
|-----|-----|-----------|
| `#50` | `SetVMod` | Установить видеорежим (A = режим, B = видеостраница) |
| `#51` | `GetVMod` | Получить текущий режим (→ A, B) |
| `#52` | `Locate` | Установить позицию курсора (D = row, E = col) |
| `#53` | `Cursor` | Получить позицию курсора (→ D, E) |
| `#54` | `SelPage` | Выбрать активную видеостраницу (B) |
| `#55` | `Scroll` | Прокрутить окно (B = направление, D/E = поз., H/L = размер) |
| `#56` | `Clear` | Очистить окно (A = символ, B = атрибут, D/E = поз., H/L = размер) |
| `#57` | `RdChar` | Прочитать символ в позиции (→ A = символ, B = атрибут) |
| `#58` | `WrChar` | Записать символ с атрибутом (A = символ, B = атрибут, D/E = поз.) |
| `#59` | `WinCopy` | Скопировать окно в буфер (IX:B = адрес в памяти, D/E/H/L = геометрия) |
| `#5A` | `WinRest` | Восстановить окно из буфера |
| `#5B` | `PutChar` | Печать символа (с обработкой CR, LF, Tab и скроллом) |
| `#5C` | `PChars` | Печать ASCIIZ-строки (HL = адрес строки) |

## Атрибут символа

Байт атрибута кодирует цвет текста и фона в формате, совместимом с традиционным подходом `INK | PAPER << 4`:

| Биты | Поле | Диапазон |
|------|------|---------|
| [3:0] | INK (цвет символа) | 0–15 |
| [7:4] | PAPER (цвет фона) | 0–15 |

Цвета индексируются в 256-цветной палитре Sprinter. Первые 16 записей палитры при старте установлены в «ZX-подобные» цвета (см. [5.5 Палитра](05_palette.md)).

Пример атрибутов:
- `#0F` — белый текст на чёрном фоне
- `#17` — белый текст на синем
- `#F0` — чёрный текст на белом
- `#4E` — жёлтый текст на красном

## Базовые примеры

### Hello, World в текстовом режиме

```asm
DssRst          EQU #10
Dss.PChars      EQU #5C
Dss.Locate      EQU #52
Dss.WaitKey     EQU #30
Dss.Exit        EQU #41

        org     #8100 - 512
        db      "EXE", 0
        dw      #0200, #0200, 0, #8100, main, #BFFF
        ds      512 - 16

        org     #8100
main:
        ; Установить курсор в (row=5, col=10)
        ld      d, 5
        ld      e, 10
        ld      c, Dss.Locate
        rst     DssRst

        ; Напечатать ASCIIZ строку
        ld      hl, msg
        ld      c, Dss.PChars
        rst     DssRst

        ; Ждать клавишу
        ld      c, Dss.WaitKey
        rst     DssRst

        ld      c, Dss.Exit
        rst     DssRst

msg:    db      "Hello from text mode!", 13, 10, 0
```

### Цветной символ

```asm
; Записать символ '*' (0x2A) в позицию (row=10, col=20) с атрибутом yellow-on-red
        ld      a, '*'
        ld      b, #4E              ; PAPER=4 (red), INK=14 (yellow)
        ld      d, 10               ; row
        ld      e, 20               ; col
        ld      c, #58              ; WrChar
        rst     #10
```

### Очистка окна с цветом

```asm
; Очистить прямоугольник 30 колонок × 5 строк от (row=2, col=10) синим фоном
        ld      a, ' '              ; заполняющий символ
        ld      b, #17              ; PAPER=1 (blue), INK=7 (gray)
        ld      d, 2                ; start row
        ld      e, 10               ; start col
        ld      h, 5                ; height
        ld      l, 30               ; width
        ld      c, #56              ; Clear
        rst     #10
```

### Прокрутка окна вверх

```asm
; Прокрутить окно 80×24 вверх на 1 строку
        ld      b, 1                ; направление (1 = вверх)
        ld      d, 0                ; start row
        ld      e, 0                ; start col
        ld      h, 24               ; height
        ld      l, 80               ; width
        ld      c, #55              ; Scroll
        rst     #10
```

### Чтение символа и атрибута

```asm
; Прочитать символ из позиции (row=3, col=5)
        ld      d, 3
        ld      e, 5
        ld      c, #57              ; RdChar
        rst     #10
        ; A = символ, B = атрибут
        ld      (saved_char), a
        ld      (saved_attr), a     ; ошибочно, см. ниже

saved_char:     db 0
saved_attr:     db 0
```

### Сохранение и восстановление области экрана

```asm
; Сохранить область 40×10 в RAM-буфер (для «всплывающих» диалогов)
SaveArea:
        ld      d, 5                ; start row
        ld      e, 20               ; start col
        ld      h, 10               ; height
        ld      l, 40               ; width
        ld      ix, save_buffer
        ld      b, 0                ; старший байт адреса
        ld      c, #59              ; WinCopy
        rst     #10
        ret

; Восстановить
RestoreArea:
        ld      d, 5
        ld      e, 20
        ld      h, 10
        ld      l, 40
        ld      ix, save_buffer
        ld      b, 0
        ld      c, #5A              ; WinRest
        rst     #10
        ret

save_buffer:    ds 40 * 10 * 2      ; символ + атрибут × площадь
```

## Курсор

### Установка позиции

```asm
        ld      d, <row>            ; 0..31
        ld      e, <col>            ; 0..79
        ld      c, #52
        rst     #10
```

### Получение позиции

```asm
        ld      c, #53              ; Cursor
        rst     #10
        ; D = row, E = col
```

> **Замечание:** DSS не рисует мигающий курсор сам — его отображение на экране возлагается на приложение (обычно через `WrChar` с инвертированным атрибутом в текущей позиции и восстановление по таймеру).

## Спецсимволы и управление

Функции `PutChar` и `PChars` обрабатывают несколько управляющих кодов:

| Код | Действие |
|-----|---------|
| 13 (CR) | Возврат каретки (курсор в колонку 0) |
| 10 (LF) | Перевод строки (row++) |
| 9 (Tab) | Табуляция до следующей границы 8 |
| ≥ 32 | Обычный символ, печать с инкрементом колонки |

При переходе за правый край идёт автоматический перенос на следующую строку; при достижении нижнего края — автоматическая прокрутка.

## Псевдографика

Шрифт BIOS включает 256 символов с псевдографикой IBM CP866/CP437 (для рамок, таблиц, диалогов). Примеры:

| Код | Символ | Назначение |
|-----|--------|-----------|
| `#C9` | ╔ | Левый верхний угол |
| `#BB` | ╗ | Правый верхний угол |
| `#C8` | ╚ | Левый нижний |
| `#BC` | ╝ | Правый нижний |
| `#BA` | ║ | Вертикаль |
| `#CD` | ═ | Горизонталь |

Это позволяет строить рамки окон и таблицы без графических операций:

```asm
; Нарисовать одинарную рамку 40×10 от (row=5, col=10)
DrawFrame:
        ; Верхняя строка
        ld      d, 5
        ld      e, 10
        ld      c, #52 : rst #10     ; Locate

        ld      a, #C9               ; ╔
        ld      c, #5B : rst #10     ; PutChar

        ld      b, 38                ; 38 символов горизонтали
.top:
        ld      a, #CD               ; ═
        push    bc
        ld      c, #5B : rst #10
        pop     bc
        djnz    .top

        ld      a, #BB               ; ╗
        ld      c, #5B : rst #10

        ; Боковые стороны и нижняя — аналогично
        ret
```

## Когда текстовый режим — правильный выбор

Используйте текстовый режим, если:
- Приложение — **утилита, редактор, файловый менеджер, настройки, отладчик**.
- Нужно быстро выводить много текста (акселератор рендеринга уже встроен в BIOS).
- Хочется простого 80×32-интерфейса без возни с пикселями.
- Нужна прокрутка, окна, сохранение/восстановление — всё это «из коробки».

**Не** используйте текстовый режим, если:
- Нужна игровая графика со спрайтами — тогда 320×256 + акселератор.
- Нужна точная работа с палитрой на пиксельном уровне.
- Требуется double buffering — текстовый режим однокадровый.

## Ключевые моменты

> - Текстовый режим Sprinter — программная абстракция BIOS/DSS поверх графического режима 640×256.
> - Сетка 80×32 символа (шрифт 8×8), 16-цветный атрибут PAPER|INK.
> - API: DSS функции `#52`–`#5C` (Locate, WrChar, PutChar, PChars, Scroll, Clear, RdChar, WinCopy/Rest).
> - По умолчанию при старте из DSS программа уже находится в текстовом режиме — специально включать его не нужно.
> - Оптимально для редакторов, файловых менеджеров, утилит.
> - Есть псевдографика (CP866) для построения рамок и диалогов.
> - Двойной буферизации нет; для «всплывающих» диалогов используйте `WinCopy`/`WinRest`.
