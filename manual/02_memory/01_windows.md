## Архитектура банкинга

Z80 работает с 64 КБ линейного адресного пространства. Sprinter делит его на **четыре независимых окна по 16 КБ**, каждое из которых может указывать на произвольную **физическую страницу** RAM, VRAM или ROM.

```
Адресное пространство Z80:

   0xFFFF ┌───────────────────┐
          │      WIN3         │  ← управляется портом #E2
          │   0xC000–0xFFFF   │
   0xBFFF ├───────────────────┤
          │      WIN2         │  ← управляется портом #C2
          │   0x8000–0xBFFF   │
   0x7FFF ├───────────────────┤
          │      WIN1         │  ← управляется портом #A2
          │   0x4000–0x7FFF   │
   0x3FFF ├───────────────────┤
          │      WIN0         │  ← управляется портом #82
          │   0x0000–0x3FFF   │
   0x0000 └───────────────────┘
```

Каждый порт хранит **номер текущей страницы** (8 бит). Запись нового номера мгновенно переключает окно. Чтение возвращает текущий номер.

---

## Регистры управления окнами

| Порт | Имя | Окно | Z80-адрес |
|------|-----|------|-----------|
| `#82` | `EmmWin.P0` | WIN0 | `0x0000–0x3FFF` |
| `#A2` | `EmmWin.P1` | WIN1 | `0x4000–0x7FFF` |
| `#C2` | `EmmWin.P2` | WIN2 | `0x8000–0xBFFF` |
| `#E2` | `EmmWin.P3` | WIN3 | `0xC000–0xFFFF` |

```asm
EmmWin.P0   EQU     #82
EmmWin.P1   EQU     #A2
EmmWin.P2   EQU     #C2
EmmWin.P3   EQU     #E2
```

---

## Диапазоны номеров страниц

| Диапазон | Тип памяти | Физический размер |
|---------|-----------|------------------|
| `#00`–`#1F` | RAM (основная, 512 КБ) | 32 × 16 КБ |
| `#20`–`#2F` | RAM (расширенная, при наличии) | 16 × 16 КБ |
| `#40`–`#4F` | RAM системная (DSS) | 16 × 16 КБ |
| `#50`–`#5F` | VRAM | 16 × 16 КБ |
| `#5C`–`#5F` | VRAM (спрайты) | 4 × 16 КБ |
| `#80`–`#FF` | ROM / Flash | 128 × 16 КБ |

Полезные константы:

```asm
; VRAM
VPAGE_TILES     EQU     #50     ; плитки/тайлы видеорежима
VPAGE_SPRITES   EQU     #5C     ; страница спрайтов

; Системные
SYS_PAGE        EQU     #FE     ; системная страница DSS
```

## Базовые операции

### Переключить одно окно

```asm
; Переключить WIN3 на страницу 7 (RAM)
        ld      a, 7
        out     (EmmWin.P3), a

; Теперь Z80-адреса 0xC000–0xFFFF указывают на
; физические байты 7 * 16384 = 114688 ... 131071
```

### Прочитать текущую страницу окна

```asm
; Узнать, какая страница сейчас в WIN1
        in      a, (EmmWin.P1)
        ; A = номер текущей страницы
```

### Сохранить и восстановить все окна

Перед переключением обязательно сохраняйте текущие страницы:

```asm
EmmWin.P0   EQU     #82
EmmWin.P1   EQU     #A2
EmmWin.P2   EQU     #C2
EmmWin.P3   EQU     #E2

; Сохранить все 4 окна в буфер
SavePages:
        in      a, (EmmWin.P0)
        ld      (saved_page0), a
        in      a, (EmmWin.P1)
        ld      (saved_page1), a
        in      a, (EmmWin.P2)
        ld      (saved_page2), a
        in      a, (EmmWin.P3)
        ld      (saved_page3), a
        ret

; Восстановить все 4 окна из буфера
RestorePages:
        ld      a, (saved_page0)
        out     (EmmWin.P0), a
        ld      a, (saved_page1)
        out     (EmmWin.P1), a
        ld      a, (saved_page2)
        out     (EmmWin.P2), a
        ld      a, (saved_page3)
        out     (EmmWin.P3), a
        ret

saved_page0:    db  0
saved_page1:    db  0
saved_page2:    db  0
saved_page3:    db  0
```

## Типовые конфигурации памяти

### Конфигурация при запуске BIOS

```
WIN0 → ROM страница 0    (BIOS выполняется)
WIN1 → RAM страница 2
WIN2 → RAM страница 10
WIN3 → RAM страница 0
```

### Конфигурация DSS-программы

```
WIN0 → RAM стр. 0        (данные программы, стек)
WIN1 → RAM стр. 2        (данные/буферы)
WIN2 → RAM стр. 10       (код программы, ORG #8100)
WIN3 → RAM стр. переменная (VRAM / данные)
```

### Конфигурация с VRAM в WIN3

```asm
; Подключить VRAM (страница #50) в WIN3 для прямого доступа
        ld      a, VPAGE_TILES  ; #50
        out     (EmmWin.P3), a
; Теперь 0xC000–0xFFFF = VRAM (первые 16 КБ видеобуфера)
```

### ZX Spectrum режим памяти

При активном ZX-режиме маппинг фиксирован:

```
WIN0 → ROM (ZX BASIC ROM)
WIN1 → RAM страница 5    (ZX экран 0x4000–0x5AFF + атрибуты)
WIN2 → RAM страница 2
WIN3 → RAM страница, задаётся портом 0x7FFD
```

## Вычисление физического адреса

Зная номер страницы и смещение внутри 16 КБ окна:

```
Физический адрес = page_number × 16384 + offset_within_window
```

Где `offset_within_window = Z80_address & 0x3FFF`.

```asm
; Вычислить физический адрес байта по Z80-адресу 0x8200
; (при WIN2 = страница 10)
;
; offset = 0x8200 & 0x3FFF = 0x0200 = 512
; physical = 10 * 16384 + 512 = 164352 = 0x28200
```

## Многостраничные операции

### Копирование страницы через временное окно

Для копирования данных из одной физической страницы в другую нужно временно переключить одно из окон:

```asm
; Копировать 16 КБ из страницы src_page в страницу dst_page
; Используем WIN1 (0x4000) как источник, WIN3 (0xC000) как приёмник

CopyPage:
        ; src_page — номер страницы источника
        ; dst_page — номер страницы приёмника
        in      a, (EmmWin.P1)
        push    af              ; сохранить WIN1
        in      a, (EmmWin.P3)
        push    af              ; сохранить WIN3

        ld      a, (src_page)
        out     (EmmWin.P1), a  ; WIN1 = источник
        ld      a, (dst_page)
        out     (EmmWin.P3), a  ; WIN3 = приёмник

        ld      hl, #4000       ; источник: начало WIN1
        ld      de, #C000       ; приёмник: начало WIN3
        ld      bc, #4000       ; 16384 байт
        ldir

        pop     af
        out     (EmmWin.P3), a  ; восстановить WIN3
        pop     af
        out     (EmmWin.P1), a  ; восстановить WIN1
        ret

src_page:   db  0
dst_page:   db  0
```

### Доступ к BIOS при WIN0 = RAM

По умолчанию BIOS находится в WIN0 (ROM). После того, как DSS переключает WIN0 на RAM, прямой доступ к ROM невозможен. Для вызова BIOS используйте только `RST #08` / `RST #30` — они работают через аппаратный механизм BIOS-overlay.

## Ограничения и особенности

- **Нельзя** переключать страницы во время передачи данных IDE (`INIR`/`OTIR`) — нарушит транзакцию.
- WIN2 (`0x8000`–`0xBFFF`) содержит **исполняемый код** программы. Переключение WIN2 под собой равнозначно self-modification: следующая инструкция будет из новой страницы.
- Системные страницы DSS (`#FE`, `#FC`, `#40`–`#4F`) **зарезервированы**. Не переключайте на них окна без знания последствий.
- Спрайтовые страницы (`#5C`–`#5F`) являются частью VRAM и могут быть видны видеоконтроллером. Прямая запись в VRAM через банкинг корректна.

---

## Ключевые моменты

> - Четыре независимых окна: `#82`, `#A2`, `#C2`, `#E2` — по одному на каждые 16 КБ Z80-адресов.
> - Номер страницы в порт — мгновенное переключение.
> - Чтение из порта возвращает текущий номер страницы.
> - **Всегда** сохраняйте и восстанавливайте страницы при временных переключениях.
> - RAM: `#00`–`#1F`; VRAM: `#50`–`#5F`; ROM: `#80`–`#FF`.
