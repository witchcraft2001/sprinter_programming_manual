# Скрытые возможности и недокументированные функции

## VGA_SWITCH — переключение видеовыхода

### Назначение

`VGA_SWITCH` позволяет переключиться между двумя банками VRAM для вывода на второй монитор (VGA). При включённом `VGA_SWITCH` сигнал `V_CS1` направляет обращения к VRAM на альтернативный банк.

### Активация

```
Горячая клавиша: F12 + Shift
```

Реализация:
```vhdl
VGA_SWITCH = TFF((KEYS.kb_sh & !KEYS.kb_ctrl & !KEYS.kb_alt), KEYS.kb_F12,, /RESET)
SVIDEO.VGA_SW = VGA_SWITCH
-- В VIDEO2.TDF:
V_CS1 = VGA_SW
```

`TFF` — T-триггер: каждое нажатие F12+Shift инвертирует `VGA_SWITCH`.

### Применение

- Позволяет аппаратно разделить два видеобуфера без изменения кода
- Первый банк VRAM → первый монитор (`V_CS1=0`)
- Второй банк VRAM → второй монитор (`V_CS1=1`)
- Переключение происходит мгновенно (в следующем цикле CLK42) без мерцания

## ALT_ACC — альтернативная 2D-адресация акселератора

### Порт 0xC7

Регистр `ALT_ACC` управляет 2D-режимом адресации в акселераторе:

```vhdl
ALT_ACC[].clk = /IOWR
ALT_ACC[].ena = DFF((DCPP[] == 0xC7), CLK42)
ALT_ACC[]     = D[]
```

При `ALT_ACC=1` акселератор использует двумерную адресацию с переносом по строкам:
- `XCNT` — счётчик по X (горизонталь)
- `XAGR` — шаг по X (ширина спрайта)
- `AAGR` — шаг по Y (ширина экранной строки минус ширина спрайта)

Это позволяет вставлять прямоугольные спрайты без ручного перевычисления адресов.

### Порт 0xC5/0xCD — RGMOD

```vhdl
RGMOD[].clk = /IOWR
RGMOD[].ena = DFF((DCPP[] == 0b1100X101), CLK42)  -- 0xC5 или 0xCD
RGMOD[].d   = D[]
```

Регистр `RGMOD[7:0]` управляет видеорежимом:

| Бит | Назначение |
|-----|-----------|
| `0` | `RGMOD0` → `DIR_PORT[3]` → параметр адресации VRAM |
| `1` | `RGMOD1` → `DIR_PORT[4]` → Screen Off (если `SCREEN_OFF="USE"`) |

## FN_ACC — логические операции акселератора

Стандартный режим акселератора (`ACC_MODE[3:0]`) выполняет копирование/заливку. Но `FN_ACC` расширяет это логическими операциями над данными:

| `FN_ACC[1:0]` | Операция над данными MDO |
|---------------|-------------------------|
| `00` | `MDO = ACC_DATA` (прямая запись) |
| `01` | `MDO = ACC_DATA AND MDI` (AND с содержимым памяти) |
| `10` | `MDO = ACC_DATA OR MDI` (OR) |
| `11` | `MDO = ACC_DATA XOR MDI` (XOR) |

Где `MDI` — данные, считанные из VRAM перед записью (read-modify-write).

### Применение

```asm
; XOR-маска (инвертирование спрайта по маске):
LD A, 0b11 << 0   ; FN_ACC = XOR
OUT (0xCF), A

; Теперь LD r,r вместо копирования будет делать XOR с текущим содержимым VRAM
```

Это эквивалент `ROP = XOR` в оконных системах (эффект курсора, резиновая рамка).

## CORRECT_1F — патч декодирования `IN A,(C)`

### Проблема

В ZX Spectrum при выполнении `IN r,(C)` если ни одно устройство не отвечает, на шине данных появляется `0xFF`. Однако ряд программ ожидает значение `0x1F` (биты 5, 4, 3 установлены). Sprinter аппаратно эмулирует это поведение.

### Реализация (из ACCELER.TDF)

```vhdl
-- При IN A,(C) если D[7:5] == 0b111 и D[4:3] == 0b11:
-- Сбрасываем биты [4:3] в 0
CORRECT_1F: если результат IN = 0xFF → принудительно биты[4:3]=0 → 0x1F
```

Применяется только к инструкции `IN r,(C)` (`/IOM=0, /RD=0`). Прозрачно для программ.

## TEST_SWITCH — генератор тестовых паттернов клавиатуры

### Активация

```
Горячая клавиша: F12 + Ctrl  (только в режиме SPECTRUM)
```

```vhdl
DECODE.TEST_R = TFF(KEYS.kb_ctrl, KEYS.kb_f12,, /RESET)
```

При `TEST_R=1` логика опроса клавиатуры заменяется тестовым генератором, который последовательно выдаёт коды всех клавиш. Позволяет проверить матрицу клавиатуры без реального нажатия.

**Важно:** работает только в режиме совместимости с ZX Spectrum (не в DOS/Sprinter-mode).

## TURBO — режим 7 МГц

### Включение

Два источника турбо-режима:

1. **Программный** (аппаратный регистр):
   ```vhdl
   TB_SW: управляется через DECODE (порт в DCP)
   ```

2. **Аппаратный** (ручной переключатель):
   ```vhdl
   TURBO_HAND: внешний вход на FPGA
   ```

Результирующий сигнал:
```vhdl
TURBO = TB_SW or TURBO_HAND
```

### Генерация CLK_Z80

```vhdl
CLKZ1 = DECODE.CLK_Z80
-- В DCP.TDF:
CLK_Z80 = TURBO ? CLK42/6 (7 МГц) : CLK42/12 (3.5 МГц)
```

### Влияние на тайминги

В турбо-режиме:
- Частота CPU 7 МГц (удвоение)
- Все WAIT-состояния от ROM/периферии остаются (аппаратно контролируются `/WAIT_ALL`)
- DMA акселератора работает на той же частоте CLK42 — эффективность относительно выше
- Covox-частоты остаются прежними (зависят от CTH1, не от CLK_Z80)

## AY_CH_DIR — расширенное стерео AY

Стандартный AY-3-8910 выводит каналы A, B, C в фиксированной конфигурации. Sprinter добавляет управление стерео-панорамой через регистр адреса `0x10` AY:

```vhdl
-- Адрес 0x10 (выходит за пределы стандартных 0x00-0x0F):
AY_CH_DIR[7:0]  -- управление направлением каналов A/B/C
```

| Бит | Назначение |
|-----|-----------|
| `[1:0]` | Направление канала A (`00`=оба, `01`=левый, `10`=правый, `11`=нет) |
| `[3:2]` | Направление канала B |
| `[5:4]` | Направление канала C |

Это позволяет создавать стерео-эффекты в программах, использующих стандартный AY для вывода трекерной музыки.

## SOFT_RESET — программный сброс

### Через порт

Запись в порт `0x2E` вызывает программный сброс:

```asm
OUT (0x2E), A    ; любое значение — вызывает /RESET
```

### Через клавиатуру

Механизм двойного нажатия:
```vhdl
-- 2-tap counter: два последовательных нажатия RESET в течение короткого времени
-- вызывают аппаратный сброс
```

При однократном нажатии выполняется «мягкий» сброс (только CPU), при двойном — полный аппаратный сброс (очистка всего состояния FPGA).

## ALL_MODE — глобальный регистр режима

Порт `0xC3`:

```vhdl
ALL_MODE[7:0].clk = /IOWR
ALL_MODE[7:0].ena = DFF((DCPP[] == 0xC3), CLK42)
ALL_MODE[].prn    = /RESET   -- сброс в 0xFF
```

| Бит | Назначение |
|-----|-----------|
| `0` | `ALL_MODE0` — разрешить акселератор (`ACC.ACC_ENA = ALL_MODE0`) |
| `[7:1]` | Зарезервированы / зависят от конкретной прошивки |

**После сброса все биты = 1** (`.prn = /RESET` означает preset при активном `/RESET`).

Это значит, что акселератор включён по умолчанию сразу после включения питания.

## HOLD — регистр синхросигналов FDD/VGA

Порт `0xCB`:

```vhdl
HOLD[].clk = /IOWR
HOLD[].ena = DFF((DCPP[] == 0xCB), CLK42)
HOLD[].d   = D[]
-- После сброса: HOLD[7,6,5,2,1] = 1, HOLD[3,0] = 0
```

| Бит | Назначение |
|-----|-----------|
| `[3:0]` | `SINC_HOLD[3:0]` — горизонтальное смещение синхронизации |
| `[7:4]` | `SINC_HOLD[8:4]` — вертикальное смещение синхронизации |

Используется для точной подстройки позиции изображения на мониторе через `COPY_SINC_H` / `COPY_SINC_V`.

## Водяной знак / идентификация прошивки

В `DCP.MIF` ряд индексов содержит специальные значения:
- Индекс `0xEC` → PAGE=`0x0FF` → специальный флаг System ROM
- Эти значения можно использовать для идентификации версии прошивки через опрос регистров DECODE

Чтение версии:
```asm
IN A, (версия_порта)   ; зависит от конкретной прошивки
```
