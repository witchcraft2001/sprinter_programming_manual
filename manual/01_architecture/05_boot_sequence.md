# 1.5 Последовательность загрузки

> **Навигация:** [← 1.4 Карта портов](04_port_map.md) | [Оглавление](../README.md) | [2.1 Окна памяти →](../02_memory/01_windows.md)

---

## Обзор

Для программиста полезно разделять два разных процесса:

1. **холодный запуск BIOS** — от аппаратного сброса до начала чтения
   загрузочного сектора;
2. **дальнейшую загрузку DSS** — уже после того, как BIOS передал управление
   загрузчику.

В этом разделе подробно рассматривается только первая часть, потому что именно
она относится к архитектуре BIOS и начальному состоянию машины.

```
Сброс (RESET)
    │
    ▼
1. Аппаратный старт FPGA/CPLD
    │
    ▼
2. BIOS в ROM
    │  bootstrap, CMOS, POST, autodetect, выбор устройства
    ▼
3. Начало чтения boot sector / системного сектора
```

Полная цепочка DSS описана в разделе [4.1 Обзор DSS](../04_dss/01_dss_overview.md).

---

## Полная картина

Если смотреть на весь путь целиком, загрузка Sprinter проходит четыре фазы:

```
Сброс (RESET)
    │
    ▼
1. Аппаратный сброс FPGA (ACEX + MAX7000)
    │  минимальное стартовое состояние логики
    ▼
2. BIOS (ROM страницы 0–7, адрес 0x0000)
    │  bootstrap + инициализация, SETUP, автоопределение HDD
    ▼
3. Загрузчик с HDD / FDD / ROM
    │  Читает загрузочный сектор / EXE-файл
    ▼
4. DSS (Disk SubSystem)
    │  Загружается в защищённые страницы RAM
    ▼
   Запуск пользовательской программы (EXE)
```

---

## Фаза 1: Аппаратный сброс

При подаче питания или сигнале RESET:

1. **MAX7000** и ACEX выходят из reset в аппаратно заданное стартовое
   состояние.
2. Z80 получает минимальный стартовый маппинг, достаточный для начала
   выполнения BIOS из ROM.
3. Уже на этом этапе BIOS выполняет ранний bootstrap и программно догружает
   конфигурационные данные в аппаратную логику.

Подтверждённый пример есть в исходнике `EXP_DCP2.ASZ`: процедура `DCP_INIT`
берёт встроенный блок `ALTERA`, устанавливает `HL=ALTERA`, `DE=#C000` и
последовательно пишет байты по адресам конфигурационного окна. Это прямое
подтверждение того, что как минимум часть стартовой конфигурации загружается
**самим Z80**, а не автономно одной только CPLD.

4. После завершения раннего bootstrap активны:
   - DCP и стартовая маршрутизация портов;
   - видеологика;
   - контроллер памяти и стандартный набор системных окон.

5. Начальный маппинг памяти после сброса:

```
WIN0  0x0000–0x3FFF → ROM страница 0  (начало BIOS)
WIN1  0x4000–0x7FFF → RAM страница 2
WIN2  0x8000–0xBFFF → RAM страница 10
WIN3  0xC000–0xFFFF → RAM страница 0
```

6. Z80 начинает выполнение с адреса **0x0000** — это первая инструкция BIOS.

## Фаза 2: BIOS

### Карта Flash / ROM

Flash (512 КБ) содержит BIOS, SETUP и служебные конфигурационные данные.
Для программиста важно не точное физическое размещение каждого блока, а то,
что BIOS стартует из ROM-страниц, а конфигурационные таблицы доступны
прошивке как встроенные данные.

Осторожная реконструкция по доступным исходникам выглядит так:

```
Flash               ROM-страница    Содержимое
───────────────────────────────────────────────
0x00000 ...        ...             BIOS / служебные блоки
...                ...             ...
                   Стр. 0 (#80)    BIOS начало (0x0000)
                   Стр. 1–5        BIOS основной код
                   Стр. 6–7        BIOS дополнения
                   старшие стр.     SETUP и конфигурационные данные
```

Точное физическое размещение всех конфигурационных блоков зависит от версии
прошивки; как переносимый API это не фиксируется.

### Точки входа BIOS

BIOS расположен по адресу `0x0000` в WIN0. Ключевые адреса:

| Адрес | Назначение |
|-------|-----------|
| `0x0000` | Точка входа после сброса (`DI / HALT`) |
| `0x0008` | RST #08 — BIOS API (EMM функции) |
| `0x0010` | IPOINT — внутренний BIOS вызов |
| `0x0018` | RST #18 — внутренний вызов |
| `0x0038` | RST #38 / INT — обработчик IM1 |
| `0x0100` | Основной код BIOS |
| `0x0107` | Альтернативная точка входа |
| `0x1000` | SETUP утилита |
| `0x3FD0` | `BIOS` метка — точка входа в резидентные функции |
| `0x3FE0` | Сброс в режим ZX / выход в Basic |
| `0x3FE8` | Тёплый старт (warm boot) |
| `0x3FF8` | Холодный рестарт |

### Последовательность инициализации BIOS

```
START:
  DI                       ; запрет прерываний
  ; ранний bootstrap уже вывел машину в рабочий стартовый маппинг
  CALL KINIT               ; инициализация клавиатуры
  CALL OPENDOS             ; подготовка DSS окружения
  CALL ZXCLS               ; очистка экрана (ZX режим)
  CALL READING             ; чтение CMOS настроек
  CALL TCHEKSM             ; проверка контрольной суммы CMOS
  CALL NZ, SETDEFX         ; если сумма неверна — установить defaults
  ; Инициализация расширенных прерываний (#C124–#C127)
  CALL INSTALL             ; инициализация экрана и вывод логотипа
  CALL AUTODET             ; автоопределение IDE устройств
  CALL CTRLKEY             ; проверка Del = вход в SETUP
  CALL BOOTSYS             ; загрузка системного диска
  ; если неудача — BOOTALT (альтернативный диск)
  ; если неудача — перезагрузка или ZX Basic
```

### CMOS регистры (настройки BIOS)

Настройки хранятся в CMOS (RTC), доступны через порты `#1C`–`#1E`:

| Регистр | Назначение |
|---------|-----------|
| `#0E` | Флаги: тест памяти, safe RAM disks, задержка старта, язык, быстрый старт |
| `#0F` | Типематик: on/off, задержка, скорость повторения |
| `#10` | Системный диск (биты [2:0]) и альтернативный диск (биты [6:4]) |
| `#11` | FDD типы (720/1.44/нет) и IDE master/slave режимы |
| `#12`–`#15` | IDE Master: цилиндры, головки, секторы |
| `#16`–`#19` | IDE Slave: цилиндры, головки, секторы |
| `#1A` | Цветовая схема |
| `#1B` | Аппаратная конфигурация: turbo, тип компьютера (Sprinter/Spectrum/Pentagon) |
| `#1C` | Режим сброса (Basic 128/48, TR-DOS, расширение) |
| `#1D` | Сообщение перезагрузки, защита HDD от записи |
| `#1E` | TR-DOS маршрутизация дисков A–D |

Значения системного диска (CMOS `#10`, биты [2:0]):

| Код | Устройство |
|-----|-----------|
| 0 | FDD 1 |
| 1 | FDD 2 |
| 2 | IDE (HDD) 1 |
| 3 | IDE (HDD) 2 |
| 4 | RAM-диск |

### Вход в SETUP утилиту

Во время POST нажатие **Del** переводит в SETUP:

```
BIOS проверяет: CALL CTRLKEY → если Delete нажата → JR NZ, ABOOT
```

SETUP расположен по адресу `0x1000` в ROM и занимает страницы `0x8000`–`0xB3FF` при выполнении.

## Фаза 3: начало загрузки с устройства

### Порядок выбора загрузочного устройства

BIOS последовательно пробует:
1. **BOOTSYS** — системный диск из CMOS `#10` (биты [2:0])
2. **BOOTALT** — альтернативный диск из CMOS `#10` (биты [6:4])
3. Если оба провалились — перезагрузка или выход в ZX Basic

```
BOOTSYS: LD BC, #0710
BOOT000: CALL G_VALUE        ; прочитать тип диска из CMOS
         JP Z, FDSTART       ; FDD 1 или FDD 2
         JP Z, HDSTART       ; HDD 1 или HDD 2
         JP Z, RDSTART       ; RAM-диск
```

### Что именно делает BIOS на этом этапе

После `BOOTSYS`/`BOOTALT` BIOS:

1. читает CMOS `#10` и определяет тип стартового устройства;
2. переходит в `FDSTART`, `HDSTART` или `RDSTART`;
3. для FDD использует вызов `IPOINT` с `C=#51`, для IDE — путь через
   `AUTOIDE`/`HDRIVER6`;
4. подготавливает буфер и **начинает чтение первого сектора**.

Именно этот момент и является границей BIOS boot logic в узком смысле.

### Что не стоит смешивать с BIOS boot

Дальнейшие шаги уже относятся не к cold boot BIOS, а к коду загрузчика:

- разбор boot sector;
- поиск и чтение системных файлов;
- разворачивание DSS в защищённые страницы RAM;
- установка `RST #10`;
- запуск оболочки или пользовательской программы.

Эта часть важна, но это уже не "начало загрузки BIOS", а **инициализация DSS**.

## Фаза 4: DSS и запуск программ

Подробно см. [4.1 Обзор DSS](../04_dss/01_dss_overview.md) и
[4.5 Формат EXE](../04_dss/05_dss_exe.md).

### Системные страницы DSS

После загрузки DSS занимает несколько страниц RAM для резидентного кода:

| Страница | Назначение |
|---------|-----------|
| `#FE` | Системная страница DSS (резидент) |
| `#FC` | Дополнительная системная страница |
| `#40`–`#4F` | Зона системных страниц DSS |

Программа не должна использовать эти страницы напрямую. Для получения списка **доступных страниц** используйте DSS API `EMM_FN5` (#C5).

### Формат EXE-файла

DSS загружает программы в формате `.EXE`:

```asm
; Заголовок EXE (512 байт перед кодом)
        org     #8100-512       ; = 0x7F00

        db      "EXE", 0        ; сигнатура
        dw      #0200           ; версия формата
        dw      0, 0, 0, 0, 0   ; зарезервировано
        dw      begin           ; адрес точки входа
        dw      begin           ; адрес точки входа (дубль)
        dw      #BFFF           ; начальный стек
        ds      490             ; заполнение до 512 байт

begin:  ; реальный код начинается здесь
        org     #8100
```

Программный код загружается в WIN2 по адресу `0x8100`. Первые 512 байт — заголовок EXE.

### Карта памяти при запуске DSS-программы

```
Z80-адрес       Окно    Содержимое
──────────────────────────────────────────────────
0x0000–0x3FFF   WIN0    RAM — данные программы, стек
0x4000–0x7FFF   WIN1    RAM — данные/буферы
0x7F00–0x80FF   WIN1/2  EXE-заголовок (512 байт)
0x8000–0xBFFF   WIN2    Код и данные программы
0x8100           -      ORG программы (точка входа)
0xC000–0xFFFF   WIN3    RAM — VRAM или данные программы
```

### Вызовы после старта программы

После передачи управления на `0x8100`:
- DSS уже инициализирован, `RST #10` работает.
- BIOS API через `RST #08` и `RST #30` доступен.
- Прерывания выключены (`DI`). Программа сама настраивает IM2.
- Стек находится по адресу `0xBFFF`.

## Ключевые адреса системы

| Адрес | Назначение |
|-------|-----------|
| `#C020` | `SYSTEM_ID` — идентификатор системы |
| `#C0FE` | `SYS_SP` — дополнительный системный стек |
| `#C100` | `DISK_TYPE` — маршрутизация дисков |
| `#C104`–`#C107` | `COPY_PAGE0–3` — копии текущих страниц окон |
| `#C108` | `RAMD_VARS` — переменные RAM-дисков |
| `#C11A` | `SP_SAVE` — сохранённый адрес стека |
| `#C124` | `INTADDR` — адрес расширенного обработчика прерываний |
| `#C126` | `INTPAGE` — страница для расширенного прерывания |
| `#C127` | `INT_ID` — идентификатор (`#AA` = активен) |
| `#C1C0` | `IDE0` — дескриптор IDE устройства 0 |
| `#C1C8` | `IDE1` — дескриптор IDE устройства 1 |
| `#C1E0` | `FDD0` — дескриптор FDD 0 |
| `#C1E8` | `FDD1` — дескриптор FDD 1 |

## Расширенный обработчик прерываний BIOS

BIOS поддерживает механизм регистрации пользовательского обработчика прерываний через системные страницы:

```asm
; Регистрация обработчика прерываний через BIOS
; INTADDR (#C124) = адрес обработчика
; INTPAGE (#C126) = страница, содержащая обработчик
; INT_ID  (#C127) = #AA (признак активного обработчика)

        ld      c, PAGE3        ; #E2
        in      b, (c)          ; сохранить текущую страницу
        push    bc
        ld      a, SYSPAGE      ; #FE
        out     (c), a          ; переключить на системную страницу

        ld      hl, #C124
        ld      (hl), e         ; lo-байт адреса обработчика
        inc     hl
        ld      (hl), d         ; hi-байт адреса
        inc     hl
        ld      (hl), a         ; страница (в a уже SYSPAGE)
        inc     hl
        ld      (hl), #AA       ; активировать

        pop     bc
        out     (c), b          ; восстановить страницу
```

---

## Ключевые моменты

> - После RESET Z80 начинает с `0x0000` — это BIOS в ROM WIN0.
> - FPGA битстрим загружается MAX7000 из Flash **до** запуска Z80.
> - BIOS читает CMOS, определяет устройства, загружает DSS с диска.
> - DSS занимает системные страницы RAM и активирует `RST #10`.
> - Программы загружаются в окно 2 по адресу `0x8100` (ORG 0x8100).
> - Первые 512 байт файла EXE — заголовок, реальный код с `0x8100`.
> - Все системные переменные DSS хранятся в RAM `0xC000`–`0xC1FF` на системной странице `#FE`.
