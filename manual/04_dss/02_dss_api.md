> **Навигация:** [← 4.1 Обзор DSS](01_dss_overview.md) | [Оглавление](../README.md) | [4.3 Файловая система →](03_dss_filesystem.md)

## Соглашения вызова

- **Вектор:** `RST #10`
- **Номер функции:** в регистре `C`
- **Параметры:** `A`, `B`, `D`, `E`, `H`, `L`, `IX`, `IY`
- **Возврат:**
  - `CF = 0` — успех; результаты в регистрах
  - `CF = 1` — ошибка; `A` = код ошибки

```asm
DssRst  EQU #10
```

---

## Системные функции

### `#00` — VERSION: версия DSS

**Вход:** `C = #00`
**Выход:** `DE` = версия (например `#0200` = 2.0)

```asm
        ld      c, #00
        rst     #10
        ; DE = #0200
```

---

### `#01` — CHDISK: выбрать текущий диск

**Вход:** `C = #01`, `A` = номер диска (0 = A:, 1 = B:, …)
**Выход:** `CF = 0` → `A` = MaxDisk; `CF = 1` → неверный номер.

---

### `#02` — CURDISK: получить текущий диск

**Выход:** `A` = номер текущего диска (0 = A:).

---

### `#03` — DSKINFO: информация о диске

**Вход:** `A` = номер диска.
**Выход:** параметры FAT тома.

---

### `#09` — BOOTDSK: загрузочный диск

**Выход:** `A` = номер диска, с которого была загружена система.

---

## Файловые операции

### `#0A` — CREATE: создать файл (перезаписать)

**Вход:**
- `C = #0A`
- `HL` = адрес строки с именем файла (ASCIIZ)

**Выход:**
- `CF = 0` → `A` = номер файлового дескриптора (handle)
- `CF = 1` → `A` = код ошибки

---

### `#0B` — CREAT_N: создать файл (только если не существует)

Идентично `#0A`, но возвращает ошибку, если файл уже существует.

---

### `#0E` — DELETE: удалить файл

**Вход:** `HL` = имя файла
**Выход:** `CF = 0` успех

---

### `#10` — RENAME: переименовать файл

**Вход:**
- `HL` = старое имя
- `DE` = новое имя

---

### `#11` — OPEN: открыть файл

**Вход:**
- `C = #11`
- `HL` = имя файла
- `A` = режим (`FileMode.Read`, `FileMode.Write`, `FileMode.RW`)

**Выход:**
- `CF = 0` → `A` = дескриптор файла
- `CF = 1` → ошибка

```asm
        ld      hl, filename
        ld      a, #01              ; FileMode.Read
        ld      c, #11              ; OPEN
        rst     #10
        jr      c, .error
        ld      (handle), a

filename:   db "TEST.TXT", 0
handle:     db  0
```

---

### `#12` — CLOSE: закрыть файл

**Вход:** `A` = дескриптор
**Выход:** CF = 0 при успехе.

---

### `#13` — READ: чтение из файла

**Вход:**
- `C = #13`
- `A` = дескриптор
- `HL` = адрес буфера в памяти
- `DE` = количество байт для чтения (0 = 16384)

**Выход:**
- `CF = 0` → прочитано; реальное количество байт в `DE`
- `CF = 1` → ошибка

```asm
        ld      a, (handle)
        ld      hl, buffer
        ld      de, 256             ; 256 байт
        ld      c, #13
        rst     #10

buffer:     ds  256
```

---

### `#14` — WRITE: запись в файл

**Вход:**
- `A` = дескриптор
- `HL` = адрес буфера
- `DE` = количество байт

```asm
        ld      a, (handle)
        ld      hl, data
        ld      de, 100
        ld      c, #14
        rst     #10
```

---

### `#15` — MOVE_FP: перемещение файлового указателя (seek)

**Вход:**
- `C = #15`
- `A` = дескриптор
- `B` = режим:
  - `0` = от начала (SET)
  - `1` = от текущей позиции (CUR)
  - `2` = от конца (END)
- `HL:IX` = смещение (32 бита, HL = старшие, IX = младшие)

**Выход:** `HL:IX` = новая позиция в файле.

```asm
; Перейти в начало файла
        ld      a, (handle)
        ld      b, 0
        ld      hl, 0
        ld      ix, 0
        ld      c, #15
        rst     #10
```

---

### `#16` — ATTRIB: атрибуты файла

**Вход:** `HL` = имя, `A` = новые атрибуты или `#FF` для чтения.

---

### `#17` — GET_D_T: получить дату/время файла

### `#18` — PUT_D_T: установить дату/время файла

## Поиск файлов

### `#19` — F_FIRST: первый файл в каталоге

**Вход:**
- `C = #19`
- `DE` = адрес буфера результата (структура FAT-записи, 32 байта)
- `HL` = маска имени (например `"*.EXE"`)
- `A` = атрибуты (обычно `#FF` для всех)
- `B` = дополнительные флаги

**Выход:**
- `CF = 0` → буфер заполнен, в DE структура:
  - `[0..10]` — имя 8.3 (padded spaces)
  - `[11]` — атрибуты
  - `[22..23]` — время
  - `[24..25]` — дата
  - `[26..27]` — стартовый кластер
  - `[28..31]` — размер файла
- `CF = 1` → не найдено

```asm
        ld      de, find_buf
        ld      hl, mask
        ld      a, #FF
        ld      b, 0
        ld      c, #19
        rst     #10
        jr      c, .nothing

mask:       db  "*.EXE", 0
find_buf:   ds  32
```

---

### `#1A` — F_NEXT: следующий файл

**Вход:** `DE` = тот же буфер, что и у F_FIRST.
**Выход:** аналогично F_FIRST.

```asm
find_loop:
        ld      de, find_buf
        ld      c, #1A
        rst     #10
        jr      c, .done
        ; обработать найденный файл
        jr      find_loop
.done:
```

---

### `#1B` — MKDIR / `#1C` — RMDIR

**Вход:** `HL` = имя каталога.

---

### `#1D` — CHDIR: сменить каталог

**Вход:** `HL` = путь (например `"\DOCS"`, `".."`, `"C:\"`).

---

### `#1E` — CURDIR: текущий каталог

**Вход:** `HL` = буфер (128+ байт).
**Выход:** в буфере строка текущего пути.

```asm
        ld      hl, cwd_buf
        ld      c, #1E
        rst     #10

cwd_buf:    ds  128
```

## Время

### `#21` — SYSTIME: текущее время

**Выход:**
- `D` = день, `E` = месяц
- `IX` = год (4 цифры, например 2026)
- `H` = часы, `L` = минуты, `B` = секунды
- `C` = день недели (1=Пн, 7=Вс)

```asm
        ld      c, #21
        rst     #10
        ld      (day), de
        ld      (year), ix
        ld      a, h
        ld      (hour), a
```

---

### `#22` — SETTIME: установить время

Параметры как у SYSTIME.

## Клавиатура

### `#30` — WAITKEY: ждать клавишу

Блокирует до нажатия клавиши.
**Выход:** `A` = код нажатой клавиши (ASCII или расширенный).

```asm
        ld      c, #30
        rst     #10
        ; A = код клавиши
```

---

### `#31` — SCANKEY: неблокирующее чтение

**Выход:**
- `ZF = 1` (ZF_ON) — нет нажатия
- `ZF = 0` (ZF_OFF) — `A` = код клавиши

```asm
        ld      c, #31
        rst     #10
        jr      z, .no_key          ; ZF=1 → нет клавиши
        ; A = клавиша
.no_key:
```

---

### `#33` — CTRLKEY: состояние модификаторов

- bit 4 = Delete
- bit 0/1 = Shift L/R
- bit 3 = Ctrl
- bit 2 = Alt

## Управление памятью

### `#38` — SETWIN: установить страницу в окно

**Вход:**
- `C = #38`
- `A` = ID блока памяти
- `B` = логический индекс страницы внутри блока
- `H` = номер окна × 64 (0, 0x40, 0x80, 0xC0)

**Выход:** физическая страница подключена в указанное окно.

```asm
; Подключить 2-ю страницу блока ID=5 в WIN3 (H = 0xC0)
        ld      a, 5
        ld      b, 1                ; вторая страница (индекс 1)
        ld      h, #C0              ; WIN3
        ld      c, #38
        rst     #10
```

---

### `#39`/`#3A`/`#3B` — SETWIN1/2/3

Сокращённые варианты `SETWIN` для WIN1/WIN2/WIN3.

**Вход:** `A` = ID блока, `B` = логический индекс.

```asm
; То же что и выше, но короче
        ld      a, 5
        ld      b, 1
        ld      c, #3B              ; SETWIN3
        rst     #10
```

---

### `#3C` — INFOMEM: информация о памяти

Обёртка вокруг `Bios.Emm_Fn0`.

**Выход:** `HL` = всего RAM страниц, `BC` = свободных.

---

### `#3D` — GETMEM: выделить блок памяти

Обёртка вокруг `Bios.Emm_Fn2`.

**Вход:** `B` = количество страниц.
**Выход:** `A` = ID блока, CF=0.

```asm
        ld      b, 4                ; 4 страницы = 64 КБ
        ld      c, #3D              ; GETMEM
        rst     #10
        jr      c, .no_mem
        ld      (block_id), a
```

---

**Вход:** `A` = ID блока.

## Запуск программ

### `#40` — EXEC: запустить программу

**Вход:**
- `C = #40`
- `HL` = имя файла с аргументами (например `"PROG.EXE arg1 arg2"`)

**Выход:**
- `CF = 0` успех (выполнено и вернулось управление)
- `CF = 1` ошибка загрузки (`A` = код)

```asm
        ld      hl, cmdline
        ld      c, #40
        rst     #10
cmdline:    db  "TEST.EXE param1", 0
```

---

### `#41` — EXIT: выйти из программы

**Вход:** `A` = код возврата (опционально).

```asm
        xor     a
        ld      c, #41
        rst     #10
```

## Видео и текстовый вывод

### `#50` — SETVMOD: установить видеорежим

**Вход:** `A` = режим (0=ZX, 1=320×256, 2=640×256), `B` = видеостраница.

## Коды ошибок DSS

| Код | Описание |
|-----|----------|
| `#01` | Файл не найден / неверный параметр |
| `#02` | Неверный формат / режим |
| `#03` | Ошибка доступа |
| `#04` | Файл уже существует |
| `#05` | Недостаточно памяти |

---

## Ключевые моменты

> - Все функции DSS через `RST #10`, номер в `C`.
> - Файлы: Open/Read/Write/Close + Seek через MOVE_FP.
> - Каталоги: F_First + F_Next для обхода, MkDir/ChDir для навигации.
> - Память: GetMem (`#3D`) и SetWin (`#38`–`#3B`) для динамической RAM.
> - Запуск программ через Exec (`#40`), выход — только через Exit (`#41`).
> - Видеовывод: PutChar (`#5B`), PChars (`#5C`) для большинства случаев.
