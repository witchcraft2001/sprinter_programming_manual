## Соглашения вызова

- **Вектор:** `RST #10`
- **Номер функции:** в регистре `C`
- **Параметры:** `A`, `B`, `D`, `E`, `H`, `L`, `IX`, `IY`
- **Возврат:**
  - `CF = 0` — успех; результаты в регистрах
  - `CF = 1` — ошибка; `A` = код ошибки

```asm
DssRst  EQU #10
```

---

## Системные функции

### `#00` — VERSION: версия DSS

**Вход:** `C = #00`
**Выход:** `DE` = версия (например `#0200` = 2.0)

```asm
        ld      c, #00
        rst     #10
        ; DE = #0200
```

---

### `#01` — CHDISK: выбрать текущий диск

**Вход:** `C = #01`, `A` = номер диска (0 = A:, 1 = B:, …)
**Выход:** `CF = 0` → `A` = MaxDisk; `CF = 1` → неверный номер.

---

### `#02` — CURDISK: получить текущий диск

**Выход:** `A` = номер текущего диска (0 = A:).

---

### `#03` — DSKINFO: информация о диске

**Вход:** `A` = номер диска.
**Выход:** параметры FAT тома.

---

### `#09` — BOOTDSK: загрузочный диск

**Выход:** `A` = номер диска, с которого была загружена система.

---

## Файловые операции

### `#0A` — CREATE: создать файл (перезаписать)

**Вход:**
- `C = #0A`
- `HL` = адрес строки с именем файла (ASCIIZ)

**Выход:**
- `CF = 0` → `A` = номер файлового дескриптора (handle)
- `CF = 1` → `A` = код ошибки

---

### `#0B` — CREAT_N: создать файл (только если не существует)

Идентично `#0A`, но возвращает ошибку, если файл уже существует.

---

### `#0E` — DELETE: удалить файл

**Вход:** `HL` = имя файла
**Выход:** `CF = 0` успех

---

### `#10` — RENAME: переименовать файл

**Вход:**
- `HL` = старое имя
- `DE` = новое имя

---

### `#11` — OPEN: открыть файл

**Вход:**
- `C = #11`
- `HL` = имя файла
- `A` = режим (`FileMode.Read`, `FileMode.Write`, `FileMode.RW`)

**Выход:**
- `CF = 0` → `A` = дескриптор файла
- `CF = 1` → ошибка

```asm
        ld      hl, filename
        ld      a, #01              ; FileMode.Read
        ld      c, #11              ; OPEN
        rst     #10
        jr      c, .error
        ld      (handle), a

filename:   db "TEST.TXT", 0
handle:     db  0
```

---

### `#12` — CLOSE: закрыть файл

**Вход:** `A` = дескриптор
**Выход:** CF = 0 при успехе.

---

### `#13` — READ: чтение из файла

**Вход:**
- `C = #13`
- `A` = дескриптор
- `HL` = адрес буфера в памяти
- `DE` = количество байт для чтения (0 = 16384)

**Выход:**
- `CF = 0` → прочитано; реальное количество байт в `DE`
- `CF = 1` → ошибка

```asm
        ld      a, (handle)
        ld      hl, buffer
        ld      de, 256             ; 256 байт
        ld      c, #13
        rst     #10

buffer:     ds  256
```

---

### `#14` — WRITE: запись в файл

**Вход:**
- `A` = дескриптор
- `HL` = адрес буфера
- `DE` = количество байт

```asm
        ld      a, (handle)
        ld      hl, data
        ld      de, 100
        ld      c, #14
        rst     #10
```

---

### `#15` — MOVE_FP: перемещение файлового указателя (seek)

**Вход:**
- `C = #15`
- `A` = дескриптор
- `B` = режим:
  - `0` = от начала (SET)
  - `1` = от текущей позиции (CUR)
  - `2` = от конца (END)
- `HL:IX` = смещение (32 бита, HL = старшие, IX = младшие)

**Выход:** `HL:IX` = новая позиция в файле.

```asm
; Перейти в начало файла
        ld      a, (handle)
        ld      b, 0
        ld      hl, 0
        ld      ix, 0
        ld      c, #15
        rst     #10
```

---

### `#16` — ATTRIB: атрибуты файла

**Вход:** `HL` = имя, `A` = новые атрибуты или `#FF` для чтения.

## Поиск файлов

### `#19` — F_FIRST: первый файл в каталоге

**Вход:**
- `C = #19`
- `DE` = адрес буфера результата (структура FAT-записи, 32 байта)
- `HL` = маска имени (например `"*.EXE"`)
- `A` = атрибуты (обычно `#FF` для всех)
- `B` = дополнительные флаги

**Выход:**
- `CF = 0` → буфер заполнен, в DE структура:
  - `[0..10]` — имя 8.3 (padded spaces)
  - `[11]` — атрибуты
  - `[22..23]` — время
  - `[24..25]` — дата
  - `[26..27]` — стартовый кластер
  - `[28..31]` — размер файла
- `CF = 1` → не найдено

```asm
        ld      de, find_buf
        ld      hl, mask
        ld      a, #FF
        ld      b, 0
        ld      c, #19
        rst     #10
        jr      c, .nothing

mask:       db  "*.EXE", 0
find_buf:   ds  32
```

---

### `#1A` — F_NEXT: следующий файл

**Вход:** `DE` = тот же буфер, что и у F_FIRST.
**Выход:** аналогично F_FIRST.

```asm
find_loop:
        ld      de, find_buf
        ld      c, #1A
        rst     #10
        jr      c, .done
        ; обработать найденный файл
        jr      find_loop
.done:
```

---

### `#1B` — MKDIR / `#1C` — RMDIR

**Вход:** `HL` = имя каталога.

---

### `#1D` — CHDIR: сменить каталог

**Вход:** `HL` = путь (например `"\DOCS"`, `".."`, `"C:\"`).

---

### `#1E` — CURDIR: текущий каталог

**Вход:** `HL` = буфер (128+ байт).
**Выход:** в буфере строка текущего пути.

```asm
        ld      hl, cwd_buf
        ld      c, #1E
        rst     #10

cwd_buf:    ds  128
```

## Время

### `#21` — SYSTIME: текущее время

**Выход:**
- `D` = день, `E` = месяц
- `IX` = год (4 цифры, например 2026)
- `H` = часы, `L` = минуты, `B` = секунды
- `C` = день недели (1=Пн, 7=Вс)

```asm
        ld      c, #21
        rst     #10
        ld      (day), de
        ld      (year), ix
        ld      a, h
        ld      (hour), a
```

---

### `#22` — SETTIME: установить время

Параметры как у SYSTIME.

## Клавиатура

### `#30` — WAITKEY: ждать клавишу

Блокирует до нажатия клавиши.
**Выход:** `A` = код нажатой клавиши (ASCII или расширенный).

```asm
        ld      c, #30
        rst     #10
        ; A = код клавиши
```

---

### `#31` — SCANKEY: неблокирующее чтение

**Выход:**
- `ZF = 1` (ZF_ON) — нет нажатия
- `ZF = 0` (ZF_OFF) — `A` = код клавиши

```asm
        ld      c, #31
        rst     #10
        jr      z, .no_key          ; ZF=1 → нет клавиши
        ; A = клавиша
.no_key:
```

---

### `#32` — ECHOKEY: чтение с эхо

Блокирует выполнение до нажатия клавиши. Отображает мигающий курсор (`_`) и выводит нажатый символ на экран (эхо).

**Вход:** `C = #32`
**Выход:**
- `A` = ASCII-код символа
- `D` = позиционный код клавиши (`#00–#5A`)
- `E` = ASCII-код (= `A`)
- `B` = модификаторы (Shift/Ctrl/Alt)
- `C` = состояние блокировок (CapsLock, NumLock и т.д.)

```asm
        ld      c, #32
        rst     #10
        ; A = символ, D = скан-код, B = модификаторы
```

---

### `#33` — CTRLKEY: состояние модификаторов

**Выход:** `B` = биты состояния:
- bit 4 = Delete
- bit 0/1 = Shift L/R
- bit 3 = Ctrl
- bit 2 = Alt

---

### `#35` — K_CLEAR: очистить буфер и выполнить функцию

Сбрасывает буфер клавиатуры, затем вызывает указанную клавиатурную функцию.

**Вход:** `C = #35`, `B` = номер функции (`#30`–`#34`)
**Выход:** зависит от вызванной функции.
**Ошибка:** `CF = 1`, `A = 1` — неверный номер функции.

```asm
; Очистить буфер и ждать свежую клавишу
        ld      b, #30              ; WAITKEY
        ld      c, #35              ; K_CLEAR
        rst     #10
        ; A = код новой клавиши
```

---

### `#36` — K_SETUP: настройка клавиатуры

Конфигурация раскладки и звуковых сигналов. Подфункция в `B`:

```
B   Описание                   Вход A / Выход A
───────────────────────────────────────────────────────
0   Установить раскладку       A = номер (0–7):
                                 0–3 = основная (норм/shift/caps/caps+shift)
                                 4–7 = альтернативная (рус.)
1   Получить флаги звука       → A = текущие флаги
2   Установить флаги звука     A = флаги:
                                 бит 0 = звук при полном буфере
                                 бит 1 = звук при смене раскладки
```

**Вход:** `C = #36`, `B` = подфункция, `A` = параметр (для B=0,2)
**Ошибка:** `CF = 1` при `B ≥ 3`.

---

### `#37` — TESTKEY: проверить буфер клавиатуры

Неблокирующая проверка — **не извлекает** клавишу из буфера (в отличие от SCANKEY `#31`).

**Вход:** `C = #37`
**Выход:**
- `ZF = 1` — буфер пуст
- `ZF = 0` — клавиша доступна:
  - `A` = ASCII-код, `D` = позиционный код, `B` = модификаторы, `C` = блокировки

```asm
        ld      c, #37              ; TESTKEY
        rst     #10
        jr      z, .empty           ; ZF=1 → нет клавиш
        ; A = код, клавиша остаётся в буфере
```

## Управление памятью

### `#38` — SETWIN: установить страницу в окно

**Вход:**
- `C = #38`
- `A` = ID блока памяти
- `B` = логический индекс страницы внутри блока
- `H` = номер окна × 64 (0, 0x40, 0x80, 0xC0)

**Выход:** физическая страница подключена в указанное окно.

```asm
; Подключить 2-ю страницу блока ID=5 в WIN3 (H = 0xC0)
        ld      a, 5
        ld      b, 1                ; вторая страница (индекс 1)
        ld      h, #C0              ; WIN3
        ld      c, #38
        rst     #10
```

---

### `#39`/`#3A`/`#3B` — SETWIN1/2/3

Сокращённые варианты `SETWIN` для WIN1/WIN2/WIN3.

**Вход:** `A` = ID блока, `B` = логический индекс.

```asm
; То же что и выше, но короче
        ld      a, 5
        ld      b, 1
        ld      c, #3B              ; SETWIN3
        rst     #10
```

---

### `#3C` — INFOMEM: информация о памяти

Обёртка вокруг `Bios.Emm_Fn0`.

**Выход:** `HL` = всего RAM страниц, `BC` = свободных.

---

### `#3D` — GETMEM: выделить блок памяти

Обёртка вокруг `Bios.Emm_Fn2`.

**Вход:** `B` = количество страниц.
**Выход:** `A` = ID блока, CF=0.

```asm
        ld      b, 4                ; 4 страницы = 64 КБ
        ld      c, #3D              ; GETMEM
        rst     #10
        jr      c, .no_mem
        ld      (block_id), a
```

---

### `#3E` — FREEMEM: освободить блок

**Вход:** `A` = ID блока.

---

### `#3F` — SETMEM: изменить размер блока памяти

Изменяет размер ранее выделенного блока. Блок должен принадлежать текущей задаче.

**Вход:** `C = #3F`, `A` = ID блока, `B` = новый размер (в страницах)
**Выход:** `CF = 0` → успех; `CF = 1` → `A` = код ошибки (`#1F` = чужой блок, `#1E` = нехватка памяти).

```asm
        ld      a, (block_id)
        ld      b, 6                ; новый размер — 6 страниц
        ld      c, #3F              ; SETMEM
        rst     #10
        jr      c, .resize_error
```

## Запуск программ

### `#40` — EXEC: запустить программу

**Вход:**
- `C = #40`
- `HL` = имя файла с аргументами (например `"PROG.EXE arg1 arg2"`)

**Выход:**
- `CF = 0` успех (выполнено и вернулось управление)
- `CF = 1` ошибка загрузки (`A` = код)

```asm
        ld      hl, cmdline
        ld      c, #40
        rst     #10
cmdline:    db  "TEST.EXE param1", 0
```

---

### `#41` — EXIT: выйти из программы

**Вход:** `C = #41`, `B` = код возврата.

Код из `B` сохраняется в переменную `ERLEVEL`. Управление возвращается в родительскую программу.

```asm
        ld      b, 0                ; exit code 0
        ld      c, #41
        rst     #10
```

---

### `#42` — WAIT: получить код завершения

Возвращает код завершения последней программы (`ERLEVEL`). **Не является паузой** — возвращает результат немедленно.

**Вход:** `C = #42`
**Выход:** `A` = код завершения; `ZF = 1` если `A = 0`.

```asm
        ld      c, #42
        rst     #10
        or      a
        jr      nz, .child_failed   ; код возврата ≠ 0
```

---

### `#43` — GSWITCH: извлечь параметр из командной строки

Извлекает следующее слово (токен) из командной строки, разделённой пробелами.

**Вход:** `C = #43`, `HL` = указатель на текущую позицию в строке, `DE` = буфер для токена
**Выход:**
- `HL` = обновлённый указатель (на следующий токен)
- `CF = 0` → есть ещё параметры
- `CF = 1` → конец строки (последний токен)

Буфер `DE` заполняется ASCIIZ-строкой извлечённого токена.

```asm
; Разобрать "file.txt /v /o"
        ld      hl, cmdline
        ld      de, token
        ld      c, #43              ; GSWITCH
        rst     #10
        ; token = "file.txt", HL → " /v /o"
        jr      c, .last
        ld      c, #43
        rst     #10
        ; token = "/v", HL → " /o"

cmdline:    db  "file.txt /v /o", 0
token:      ds  64
```

### `#44` — DOSNAME: имя OS

**Выход:** `HL` = строка "DSS 1.60".

### `#45` — EX_PATH: разбор строки пути

Разбирает строку пути на компоненты. Подфункция в `B`:

```
B   Описание                  Буфер DE               Размер
──────────────────────────────────────────────────────────────
0   Разобрать строку          —                       —
1   Извлечь имя диска         "A:\0"                  13
2   Извлечь путь              "DIR1\DIR2\\0"          256
3   Извлечь имя файла         "FILE.EXT\0"            13
4   Извлечь расширение        "TXT\0"                 4
5   Извлечь всё               структура 4×DW          см. ниже
```

**Вход:** `C = #45`, `B` = подфункция, `HL` = строка пути, `DE` = буфер
**Выход (CF=0):** `A` = битовая маска (бит 7=wildcard, 3=диск, 2=путь, 1=расширение, 0=имя)

Подфункция 5: `DE` содержит 4 указателя — `DW buf_drv, buf_path, buf_name, buf_ext`.

Подробнее: [4.5 Формат EXE](05_dss_exe.md#разбор-пути--ex_path-45).

---

### `#46` — ENVIRON: переменные окружения

Управление переменными окружения. Подфункция в `B`:

```
B     Описание                    Вход
──────────────────────────────────────────────────
#FF   Инициализация (сброс)       —
#00   Чтение всего блока          DE = буфер
#01   Получить переменную         HL = "NAME\0", DE = буфер
#02   Установить/удалить          HL = "NAME=VALUE\0"
```

**Вход:** `C = #46`, `B` = подфункция
**GETENV (B=#01):** `A = #FF` → найдена; `A = 0` → не найдена.
**SETENV (B=#02):** `HL = "NAME=\0"` — удаляет переменную.

Имена автоматически приводятся к верхнему регистру. Максимум: имя 32, значение 255 символов.

Подробнее: [4.5 Формат EXE](05_dss_exe.md#переменные-окружения--environ-46).

---

### `#47` — APPINFO: информация о приложении

Возвращает информацию о текущей программе. Подфункция в `B`:

```
B   Описание                        Результат в буфере DE
─────────────────────────────────────────────────────────────
0   Параметры командной строки      "arg1 arg2\0"
1   Каталог приложения              "C:\PROGS\\0"
2   Полный путь к EXE               "C:\PROGS\APP.EXE\0"
```

**Вход:** `C = #47`, `B` = подфункция (0–2), `DE` = буфер
**Выход:** `CF = 0` → буфер заполнен (ASCIIZ); `CF = 1` → `B > 2`.

Подробнее: [4.5 Формат EXE](05_dss_exe.md#информация-о-приложении--appinfo-47).

### `#5B` — PUTCHAR: напечатать символ

**Вход:** `A` = символ (с обработкой CR, LF, Tab, скроллингом).

```asm
        ld      a, 'A'
        ld      c, #5B
        rst     #10
```

---

### `#5C` — PCHARS: напечатать строку ASCIIZ

**Вход:** `HL` = адрес строки, терминированной 0.

```asm
        ld      hl, message
        ld      c, #5C
        rst     #10
message:    db  "Hello, world!", 13, 10, 0
```

---

### `#5F` — PRINT: вывод символа на принтер (LPT)

Отправляет один символ на параллельный порт принтера. **Не является форматированной печатью** — это низкоуровневый вывод через LPT (`#1B`/`#1C`).

**Вход:** `C = #5F`, `A` = символ
**Выход:**
- `CF = 0` → символ отправлен
- `CF = 1` → принтер занят (busy)
- `B` = символ

```asm
; Напечатать символ 'A' на принтере
        ld      a, 'A'
        ld      c, #5F
        rst     #10
        jr      c, .printer_busy
```

## Коды ошибок DSS

| Код | Описание |
|-----|----------|
| `#01` | Файл не найден / неверный параметр |
| `#02` | Неверный формат / режим |
| `#03` | Ошибка доступа |
| `#04` | Файл уже существует |
| `#05` | Недостаточно памяти |

## Ключевые моменты

> - Все функции DSS через `RST #10`, номер в `C`.
> - Файлы: Open/Read/Write/Close + Seek через MOVE_FP.
> - Каталоги: F_First + F_Next для обхода, MkDir/ChDir для навигации.
> - Память: GetMem (`#3D`) и SetWin (`#38`–`#3B`) для динамической RAM.
> - Запуск программ через Exec (`#40`), выход — только через Exit (`#41`).
> - Видеовывод: PutChar (`#5B`), PChars (`#5C`) для большинства случаев.
